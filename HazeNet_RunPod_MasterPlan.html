HazeNet — RunPod Master Plan 🚀

เอา pipeline เวอร์ชันสมบูรณ์ + ข้อมูลใหญ่สุด → เทรนจริงบน RunPod → ปิดโปรเจกต์ให้ดีจริง

เอกสารชุดที่ 4 ต่อจาก Pipeline_Explained · vs_AFNO-StormCast · Architecture_DeepDive

ข้อมูล: 2016–2024 ฤดูเผาเต็ม โมเดล: CLNO + physics + quantile diffusion = ทางเลือกถ้าจำเป็น

เป้าหมายเอกสารนี้: เปลี่ยน pipeline ที่ตอนนี้รันบนเครื่องบ้าน (446 วัน, 7 channels, CLNO เดี่ยว) ให้กลายเป็น เวอร์ชันใหญ่ที่ reproducible + checkpoint ได้ + แพ็กเป็น Docker แล้วยกขึ้นไปเทรนบน RunPod GPU จริง โดยจ่ายเงินเฉพาะตรงที่คุ้ม และมี Gate ตัดสินชัดเจนว่าเมื่อไหร่เรียกว่า "เสร็จและดีจริง"

0.1 · เงิน RunPod ช่วยตรงไหน — อ่านก่อนจ่าย 💸

ความเข้าใจผิดที่แพงที่สุดคือ "เช่า GPU ใหญ่ = โมเดลดีขึ้น + เสร็จเร็วขึ้น" สำหรับโปรเจกต์นี้ จริงแค่บางส่วน:

❌ GPU แพง "ไม่" ช่วย ถ้า…

  • เทรน CLNO เดิม (0.4M params) บน 446 วัน — เราพิสูจน์แล้วว่า ลด params 24M→0.4M ผล test แทบไม่ขยับ → เน็ตไม่ใช่คอขวด
  • งานจริงคือ data engineering + ฟิสิกส์ ซึ่งรันบน CPU ก็ได้

✅ GPU คุ้ม เมื่อ…

  • ข้อมูลใหญ่ขึ้นมาก (9 ปี × ฤดูเต็ม × ~15 channels)
  • เพิ่มหัวที่หนักจริง — quantile และ (ทีหลัง) diffusion
  • รัน ablation หลายสิบรอบ ขนานกันเพื่อตัด feature ที่ไม่ช่วย
กลยุทธ์ประหยัด: เก็บ/จัดข้อมูล + EDA บนเครื่องบ้าน → จ่าย GPU เฉพาะตอนเทรนจริง → ใช้ spot/community + checkpoint resume กันpod หลุดแล้วเสียเงินฟรี

0.3 · ตัวจำกัดจริงของโปรเจกต์ = ป้ายเฉลย ไม่ใช่ compute 🏷️

เปรียบเทียบ
Features (ลม/ไฟ/BLH) เหมือน "โจทย์" มีถึงปี 1940 (ERA5) · แต่ PM2.5 ป้ายเฉลย เหมือน "เฉลย" — ถ้าไม่มีเฉลย โมเดลก็เรียนไม่ได้ ต่อให้ GPU แรงแค่ไหน เพราะนี่คือ supervised learning
  • FIRMS VIIRS (ไฟ) — ย้อนถึง 2012 · ERA5 (อุตุ) — ย้อนถึง 1940 → ไม่ใช่ตัวจำกัด
  • PM2.5 ground truth (OpenAQ + กรมควบคุมมลพิษไทย PCD) — หนาแน่นพอจริงราว ~2016+ → นี่คือเพดาน
  • ทางยืดข้อมูล (ออปชันอนาคต): ใช้ AOD ดาวเทียม เป็น label เสริม/auxiliary target — แต่เพิ่ม noise; เลื่อนไว้ก่อน
สิ่งที่ต้องทำใน Phase 1: รวบรวมสถานี PM2.5 ให้ได้มากสุด (OpenAQ + PCD + เพื่อนบ้านถ้ามี) เพราะ จำนวนสถานี × จำนวนวันที่มีเฉลย = ขนาดข้อมูลเทรนจริง

Phase 2Feature engineering (ฟิสิกส์/ภูมิศาสตร์) 🖥️ เครื่องบ้าน

เติม channel ที่มีรากฐานฟิสิกส์จริง — ป้อนเชื้อให้ met-encoder คำนวณ K และ b ฉลาดขึ้น

Featureฟิสิกส์ที่อยู่เบื้องหลังที่มา
Emission curve (FRP→E แบบ log/saturating)ไฟหนักมากๆ ฝุ่นไม่ได้เพิ่มเชิงเส้น — แก้ over-predictแปลงจาก FRP
BLH (ความสูงชั้นผสม)อากาศปิด = ฝุ่นสะสมในปริมาตรเล็กERA5
Ventilation coef = BLH × wind speedดัชนีระบายอากาศคลาสสิก — อธิบายปี 2023 โดยตรงERA5 (คำนวณ)
Inversion strength = T₈₅₀ − T_sfcชั้นอุ่นทับเย็น = ฝาปิดกดควันก้นหุบเขาERA5 multi-level
TPI / valley indexก้นหุบเขา (เชียงใหม่/เชียงราย) กักควันคำนวณจาก DEM
ENSO / ONIเอลนีโญ→แล้ง→ไฟเยอะ — "ตัวแปรหาย" ที่อธิบายความต่างข้ามปีNOAA ONI
ฝนสะสม N วันwet deposition ข้ามวัน + เชื้อเพลิงแห้งERA5 (คำนวณ)

Phase 3อัปเกรดโมเดล (ตรงนี้ GPU เริ่มคุ้ม) ⌨️ เขียนโค้ด

คงแกน CLNO (conditionally-linear — ให้ attribution ฟรี) แต่เพิ่ม 2 อย่าง:

  • Emission curve head — แปลง FRP→E ด้วยฟังก์ชัน saturating ที่เรียนรู้ได้ แทนใช้ FRP ตรงๆ
  • Quantile head (P10 / P50 / P90) — ทำนายช่วงความเชื่อมั่น แทน ensemble (ถูกกว่า) ทำให้บอก "วันนี้ฝุ่นน่าจะ 80–220" ได้
PM2.5s = Σg Ksg(met) · φ(FRPg) + bs(met) φ = emission curve (saturating) · ทำนาย 3 quantile พร้อมกันด้วย pinball loss
▶ เข้า
โค้ด model_operator.py เดิม
■ ออก
โมเดลเวอร์ชันใหญ่ + attribution ยังใช้ได้

diffusion head (จับพีค) ไม่อยู่ในรอบนี้ — รอดูว่า physics + curve จับพีคได้แค่ไหนก่อน ตามลำดับที่ตัดสินใจไว้

💰 RunPod จริง + ค่าใช้จ่าย (ตัวเลขคร่าวๆ)

เลือก GPU

  • โมเดลขนาดนี้ + ข้อมูล GB-scale → A100 80GB (~$1.5–2/ชม. community) เหลือเฟือ
  • ไม่ต้อง H100 — จ่ายแพงโดยไม่ได้ประโยชน์
  • dry-run/debug → ใช้ GPU ถูกสุด (RTX 4090 ~$0.4/ชม.)

งบประมาณรวม

  • เทรน CLNO+physics+quantile ต่อ run: หลักชั่วโมงเดียว
  • หลายสิบ run + ablation → ~$50–300 ทั้งโปรเจกต์
  • diffusion (ถ้าทำ) จะหนักกว่า — เผื่องบเพิ่ม
3 กฎประหยัด: (1) data/EDA บนเครื่องบ้าน จ่าย GPU เฉพาะเทรน · (2) checkpoint resume + spot/community · (3) dry-run บน GPU ถูกก่อนเสมอ
