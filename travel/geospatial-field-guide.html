01
Geospatial Foundation Models

โมเดลรากฐานเชิงพื้นที่ & Geo-AI

เทคโนโลยีที่ "นิยาม" ยุค 2024–2026 — โมเดลขนาดใหญ่ที่เทรนด้วย self-supervised learning บนภาพดาวเทียมจำนวนมหาศาลแบบไม่ต้องมี label แล้วได้ "embedding" ที่นำไปทำงานปลายทางได้สารพัด ด้วยข้อมูล label นิดเดียว — หลักการเดียวกับที่ GPT เทรนบนข้อความ

หัวใจของโมเดล: Masked Autoencoder (MAE) — เรียนรู้โลกด้วยการ "เดาส่วนที่ถูกปิด"

Build Roadmap

แนะนำแผนการสร้าง Product

เรียงตามลำดับที่ได้ผลคุ้มค่า/เร็วที่สุด — เริ่มจากฐานข้อมูล open + spatial index เพื่อให้มี data layer ก่อน แล้วค่อยเพิ่ม AI layer ที่ differentiate

Stage 1 · เดือน 0–3 · Foundation

Open Data + Spatial Index + Storage

ใช้ Overture Maps + Foursquare OS Places (GeoParquet, Apache 2.0) เป็น POI/base layer เชื่อม + de-duplicate ด้วย GERS ID / Placekey · standardize ทุกงานรวมกลุ่มบน H3 (neighborhood, geofence, "ใกล้เคียง") · ใช้ DuckDB + GeoParquet สำหรับ analytics และ PostGIS สำหรับ transactional

Stage 2 · เดือน 3–9 · AI Differentiation

Geospatial Foundation Models สำเร็จรูป

ดึง AlphaEarth Foundations embeddings จาก Google Earth Engine (CC-BY-4.0) เป็น feature สำเร็จรูปสำหรับงาน land-cover/ลักษณะสถานที่ ไม่ต้องเทรน EO model เอง · prototype GeoCLIP (MIT) สำหรับฟีเจอร์ "ค้นพบจากรูป" · ลองแค่ถ้า embedding สำเร็จรูปไม่พอ จึง fine-tune Prithvi/Clay ผ่าน TerraTorch

Stage 3 · เดือน 6–12 · Core Product Features

Routing + Trip Planner + 3D Preview

Self-host Valhalla (isochrone, multimodal) หรือ OSRM (ความเร็วสูงสุด) บน OSM · Hybrid LLM + optimizer trip planner: LLM ร่าง → ground ด้วยเวลาเปิดจริง + travel time → local-search solver ตรวจ hard constraint อย่าลอง pure-LLM เพราะ pass rate <5% · Google Photorealistic 3D Tiles ผ่าน Cesium สำหรับ 3D เมือง · 3D Gaussian Splatting สำหรับสถานที่เฉพาะที่ Google ยังไม่ครอบคลุม · frontend: MapLibre GL + deck.gl + PMTiles

Stage 4 · เดือน 12+ · AR + Forecasting

VPS / AR + Crowd Forecasting

Google ARCore Geospatial API สำหรับ AR wayfinding ใน Street View coverage (ไทยครอบคลุมดี) + Niantic VPS สำหรับ centimeter pose ในสถานที่ที่ scan แล้ว · ST-GNN (เริ่มจาก Graph WaveNet/STAEformer baseline) บน foot-traffic + trajectory data ของตัวเอง สำหรับ timed-entry และ alert แออัด

⚠ สิ่งที่ต้องระวัง — ตรวจสอบก่อนนำไปใช้
  • Niantic LGM: "Large Geospatial Model" เป็น vision และ program ที่กำลังพัฒนา ตัวเลข 50M networks / 150T params อธิบาย VPS ปัจจุบัน ไม่ใช่โมเดลเดียว generalist — การ generalize ข้ามสถานที่ยังเป็น research goal
  • Accuracy เป็น context-specific: VPS "ระดับเซนติเมตร" ใช้ได้ในที่ mapping ดีและแสงดี · indoor Wi-Fi RTT รองรับแค่ Android ~30% ไม่ขึ้น iOS · ETA improvement ของ Google (40–50%) เป็น city-specific ไม่ใช่ global
  • Foundation model limit: AlphaEarth / SatCLIP resolution ~10m เหมาะ landscape/เมือง ไม่เหมาะแยกคน/รถ · ตัวเลข "outperform 23.9%" มาจากผู้เขียน model เอง ต้องทดสอบบนข้อมูลของตัวเองก่อนเชื่อ
  • License ต้องตรวจ: OSM data = ODbL (share-alike) · Overture row ที่มาจาก OSM = ODbL · Foursquare = Apache 2.0 · AlphaEarth = CC-BY-4.0 (ต้องให้ credit) — รวมกันมีผลต่อสิ่งที่ ship เชิงพาณิชย์ได้
  • วงการเปลี่ยนเร็วมาก: PMTiles v3, MapLibre WebGPU, CARTO Agentic GIS, routing-in-Snowflake ล้วนออกหรืออยู่ระหว่าง dev ในปี 2025–2026 ตรวจสถานะปัจจุบันก่อน commit architecture
