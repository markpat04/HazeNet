Geospatial Data & GeoAI 2026

คู่มือฉบับสมบูรณ์: แหล่งข้อมูลเปิด (Open Data), การวิเคราะห์เชิงพื้นที่ (Spatial Analysis) และเทคโนโลยี AI ทางภูมิศาสตร์ที่ล้ำสมัยที่สุด (State-of-the-Art)

1. แหล่งข้อมูล Open Data (ใช้ฟรี & วิธีดึง)

Copernicus Data Space (ESA)

ภาพถ่ายดาวเทียม Sentinel 1, 2, 3, 5P

แหล่งรวมข้อมูลดาวเทียมยุโรป ฟรีและอัปเดตตลอดเวลา เหมาะสำหรับดูการใช้ประโยชน์ที่ดิน (Land cover) และมลพิษ (NO2, AOD)

วิธีดึง: ใช้ STAC API (SpatioTemporal Asset Catalog) ผ่านไลบรารี pystac-client ใน Python เพื่อดึงเฉพาะพื้นที่ที่ต้องการโดยไม่ต้องโหลดทั้งโลก

Google Earth Engine (GEE)

Cloud Computing Petabyte-scale

คลังข้อมูลและแพลตฟอร์มวิเคราะห์ที่ทรงพลังที่สุด ฟรีสำหรับการวิจัย รวบรวมทั้ง Landsat, Sentinel, สภาพอากาศ, และความสูงภูมิประเทศ (DEM)

วิธีดึง: ใช้ GEE Python API (import ee) เขียนโค้ดประมวลผลบน Cloud แล้วดึงแค่ผลลัพธ์ (เช่น ภาพที่ผ่านการคำนวณแล้ว) ลงมาที่เครื่อง

OpenStreetMap (OSM)

Vector Data ถนน/อาคาร/POI

แผนที่แบบ Vector ที่ฝูงชนร่วมกันสร้าง (Crowdsourced) แม่นยำเรื่องถนน ตึก และสถานที่สำคัญ (POI) ทั่วโลก

วิธีดึง: ใช้ไลบรารี osmnx ใน Python โหลดข้อมูลโครงข่ายถนนมาวิเคราะห์เส้นทางได้ทันที

Microsoft Planetary / AWS Open Data

Cloud Native Cloud-Optimized GeoTIFF

เซิร์ฟเวอร์คลาวด์ที่เก็บข้อมูลภูมิศาสตร์ในรูปแบบ COG (Cloud-Optimized GeoTIFF) ทำให้ดึงข้อมูลแค่บางพิกเซลได้โดยไม่ต้องโหลดไฟล์ใหญ่ๆ

วิธีดึง: ใช้ rasterio หรือ xarray อ่านข้อมูลตรงจาก URL ของ S3 Bucket




2. แผนการสอน: วิเคราะห์ข้อมูล Geo ตั้งแต่ต้นจนจบ

Roadmap สิ่งที่ต้องรู้และต้องทำเป็น

1. เข้าใจประเภทข้อมูลและพิกัด

ต้องแยกให้ออกระหว่าง Vector (ตึก, ถนน - ใช้ Geopandas) และ Raster (ภาพถ่ายดาวเทียม, อุณหภูมิ - ใช้ Rasterio/Xarray) และสิ่งสำคัญที่สุดคือ CRS (Coordinate Reference System) หากจับข้อมูล 2 ชุดที่ CRS ต่างกันมาวางทับกัน มันจะไปอยู่คนละซีกโลก

2. การทำ Data Preprocessing

ไฟล์ Geo มักจะมีขนาดใหญ่มาก (ระดับ GB) ต้องรู้วิธีทำ Bounding Box ตัดเอาเฉพาะพื้นที่ที่สนใจ (Clipping), การปรับขนาดกริดให้ตรงกัน (Resampling) และการจัดการกับก้อนเมฆที่บังภาพดาวเทียม (Cloud Masking)

3. การวิเคราะห์ (Spatial Join & Zonal Stats)

จุดตัดที่ทำให้ Geo Data ทรงพลังคือการหาความสัมพันธ์เชิงพื้นที่ เช่น การหาค่าเฉลี่ยฝุ่นละออง (Raster) ที่ตกกระทบในแต่ละเขตอำเภอ (Vector) - เทคนิคนี้เรียกว่า Zonal Statistics




3. State-of-the-Art GeoAI (อัปเดต 2026)

ยุค 2025-2026 คือยุคของ "Spatio-Temporal Foundation Models (GFMs)"
โมเดล AI ไม่ได้ถูกสร้างมาเพื่องานเดียวอีกต่อไป แต่เป็นโมเดลยักษ์ที่ถูก Pre-train มาแล้ว และสามารถทำ Zero-shot หรือ Fine-tune ได้ในหลายโดเมน

Earth Observation Foundation Models

Prithvi TerraMind Clay

ทำอะไร: โมเดล AI ยักษ์ที่อ่านข้อมูลจากหลายเซ็นเซอร์ (Multispectral, SAR, Elevation) พร้อมกัน เพื่อตีความพื้นผิวโลก

SOTA อย่างไร: แทนที่จะเทรนโมเดลจับตึก 1 ตัว จับป่า 1 ตัว ปัจจุบันเราใช้ Foundation Model อย่าง Prithvi (NASA/IBM) ที่เทรนบนข้อมูลภูมิศาสตร์มหาศาล แล้วค่อยนำมาทำ Fine-tune เล็กน้อยเพื่อทำงานได้ทุกอย่าง (Segmentation, Classification) ในระดับ Global scale

Agentic GeoAI & Geo-LLMs

LLMs + GIS AlphaEarth

ทำอะไร: การผสาน Large Language Models (LLMs) เข้ากับเครื่องมือ GIS ทำให้เราสามารถโต้ตอบกับแผนที่ด้วยภาษาพูด

SOTA อย่างไร: ไม่ต้องเขียนโค้ดเพื่อค้นหาข้อมูลอีกต่อไป ระบบ Agentic Workflow จะเปลี่ยนคำสั่งเช่น "หาพิกัดโรงงานที่เสี่ยงน้ำท่วมในอยุธยา" ให้เป็นโค้ด Python (SQL/GEE) ไปรันดึงข้อมูล ตัดภาพดาวเทียม และสรุปผลเป็นตารางกลับมาให้โดยอัตโนมัติ

Physics-Informed Spatio-Temporal Models

Neural ODEs HazeNet / GraphCast

ทำอะไร: AI สำหรับพยากรณ์ปรากฏการณ์ที่เปลี่ยนแปลงตามเวลาและสถานที่ (เช่น อากาศ, มลพิษ, น้ำท่วม)

SOTA อย่างไร: ก้าวข้าม Black-box AI แบบเดิมๆ สู่โมเดลที่ถูกจำกัดด้วยกฎทางฟิสิกส์ (Physics constraint) เพื่อไม่ให้ทายผลผิดธรรมชาติ โมเดลสภาพอากาศระดับโลกเปลี่ยนไปใช้ Graph Neural Networks หรือ Fourier Neural Operators ที่เร็วกว่า Supercomputer นับหมื่นเท่า (พยากรณ์ข้ามทวีปได้ในเสี้ยววินาที)

Human Dynamics & Mobility

PDFM Amazon Chronos-2

ทำอะไร: วิเคราะห์รูปแบบการเคลื่อนที่ของมนุษย์ การกระจายตัวของประชากร และการเกิดอุบัติเหตุ/อาชญากรรม

SOTA อย่างไร: การใช้ Population Dynamics Foundation Models (PDFM) ผสานข้อมูลสัญญาณมือถือ, POI, และภาพดาวเทียมเข้าด้วยกัน เพื่อทำนายการเติบโตของเมือง การจราจร และความหนาแน่นประชากรแบบ Real-time ซึ่งโมเดล Time-series ล่าสุดอย่าง Chronos-2 ได้เข้ามามีบทบาทในการทำ Zero-shot forecasting อย่างแม่นยำ
