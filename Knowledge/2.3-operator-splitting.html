2.3 — Operator Splitting (Strang)

แยกสมการก้อนใหญ่เป็นส่วน ๆ แล้วแก้ทีละส่วน — วิธีต่อ "ฟิสิกส์" เข้ากับ "นิวรอล"

ต่อจาก 2.2 (มี 𝒜 แล้ว) · บทนี้ตอบว่า "แล้วเอา 𝒜 มาต่อกับพจน์อื่นยังไง" · ผูกกับ HazeNet ตลอด

ปัญหาที่บทนี้แก้: สมการ ADR (1.2) มีหลายพจน์เกิดขึ้นพร้อมกัน — ลมหอบ (advection), กระจาย (diffusion), ปล่อยใหม่ (emission), หายไป (loss) · การแก้ทุกพจน์พร้อมกันในก้าวเดียวยากและยุ่ง · Operator splitting คือเทคนิคคลาสสิกของ CTM: "อย่าแก้ทุกอย่างพร้อมกัน — แก้ทีละพจน์ ต่อกันเป็นทอด ๆ ใน 1 timestep" · นี่คือกาวที่เชื่อม 𝒜 (ฟิสิกส์ จาก 2.2) เข้ากับส่วนที่นิวรอลเรียน (g, D) อย่างเป็นระเบียบ

1. ปัญหา — สมการมีหลายพจน์เกิดขึ้นพร้อมกัน

ในความจริง ภายในหนึ่งชั่วโมง ควันก้อนหนึ่งพร้อมกัน: ถูกลมหอบไป + กระจายออก + มีควันใหม่เติม + บางส่วนตกหาย · สมการ ADR เขียนรวมทุกอย่าง:

∂C/∂t = −∇·(uC) + ∇·(K∇C) + E − L·C
advection + diffusion + emission − loss (เกิดพร้อมกันหมด)

การออกแบบวิธีเชิงตัวเลขที่แก้ 4 พจน์นี้พร้อมกันในก้าวเดียว ทั้งซับซ้อนและไม่ยืดหยุ่น — แต่ละพจน์มีลักษณะคณิตต่างกัน (advection ชอบ semi-Lagrangian, loss เป็นการคูณง่าย ๆ) · อยากใช้วิธีที่เหมาะกับแต่ละพจน์

2. ไอเดียหลัก — แก้ทีละพจน์ ต่อกันเป็นทอด ๆ ★

Operator splitting บอกว่า: ไม่ต้องแก้พร้อมกัน · ภายใน 1 timestep ให้แก้ทีละพจน์ โดยเอาผลลัพธ์ของพจน์ก่อนเป็นอินพุตของพจน์ถัดไป:

= แต่ละพจน์กลายเป็น "ขั้นตอนย่อย" ที่แก้แยกได้ด้วยวิธีที่เหมาะที่สุดของมัน · advection ใช้ semi-Lagrangian (2.2), loss ใช้การคูณ ฯลฯ · พอต่อครบทุกพจน์ = ได้ Ct+1

3. อุปมา — ทำอาหารทีละขั้น

เหมือนทำผัดกะเพรา: คุณไม่โยนทุกอย่างลงกระทะพร้อมกันแล้วหวังให้ออกมาดี · คุณทำเป็นสเต็ป: เจียวกระเทียม → ใส่เนื้อ → ปรุงรส → ใส่ใบกะเพรา · แต่ละสเต็ปใช้เทคนิค/ไฟต่างกัน และผลของสเต็ปก่อนเป็นวัตถุดิบของสเต็ปถัดไป · operator splitting คือการ "ทำสมการเป็นสเต็ป" แบบเดียวกัน

ข้อดีคือความยืดหยุ่น: ถ้าวันหนึ่งอยากเปลี่ยนวิธีทำ diffusion ก็เปลี่ยนแค่สเต็ปนั้น ไม่ต้องรื้อทั้งสมการ

4. Strang splitting — แบบ "สมมาตร" ที่แม่นกว่า ★

การแก้ทีละพจน์ต่อกันตรง ๆ (เรียก Lie splitting) ใช้ได้ แต่มี error เล็กน้อยเพราะ "ลำดับ" ของพจน์มีผล (ทำ advection ก่อน loss ≠ ทำ loss ก่อน advection เป๊ะ ๆ)

Strang splitting แก้ปัญหานี้ด้วยการทำแบบสมมาตร: แบ่งพจน์แรกเป็นครึ่ง ทำหัว-ท้าย ประกบพจน์กลาง · เช่น:

ผลคือความแม่นดีขึ้น (จาก "อันดับ 1" เป็น "อันดับ 2" — error เล็กลงเร็วเมื่อ Δt เล็ก) โดยจ่ายแค่ทำ advection ครึ่ง step สองครั้ง · CTM ระดับโลกนิยมใช้ Strang

ไม่ต้องจำคณิตลึก: สิ่งที่ต้องรู้คือ — Strang = "ทำพจน์หัวเป็นครึ่ง ประกบท้ายอีกครึ่ง เพื่อความสมมาตร → แม่นขึ้น" · เป็นมาตรฐานที่เราเลือกตามตำรา CTM ไม่ใช่สิ่งที่เราคิดใหม่

5. Splitting ใน HazeNet — กาวเชื่อมฟิสิกส์ + นิวรอล ★★

นี่คือเหตุผลที่บทนี้สำคัญต่อโปรเจกต์ · จำสมการแกนได้ไหม:

Ct+1 = 𝒜(u) Ct + gθ·E − Dθ⊙Ct

นี่คือ operator splitting ที่แต่ละพจน์มาจาก "คนละแหล่ง":

ขั้นตอน (พจน์)ใครทำเป็นอะไร
𝒜(u) — advectionฟิสิกส์ (semi-Lagrangian, 2.2)ฝังตายตัว ไม่เรียน
gθ·E — emissionนิวรอล (AFNO) ทำนายสัมประสิทธิ์ gเรียนจากข้อมูล
Dθ⊙C — lossนิวรอล (AFNO) ทำนายอัตราหาย Dเรียนจากข้อมูล

= operator splitting คือโครงสร้างที่อนุญาตให้เราผสม "ฟิสิกส์ที่รู้แน่" กับ "นิวรอลที่เรียน" ในสมการเดียว · advection เราไม่เชื่อให้นิวรอลทำ (จาก 2.2) → ใส่ฟิสิกส์ · ส่วน emission/loss ซับซ้อนเกินจะเขียนสูตร → ให้นิวรอลเรียนสัมประสิทธิ์

นี่คือ "conditionally-linear operator" (หัวใจ 3.5): สังเกตว่าทุกพจน์เชิงเส้นใน C (𝒜 คูณ C, D คูณ C, E ไม่ขึ้นกับ C) · สัมประสิทธิ์ g, D ถูกทำนายจากอากาศ+static (ไม่ใช่จาก C) → พอกำหนดอากาศแล้ว สมการเป็นเชิงเส้นใน C · "conditionally" = เชิงเส้นเมื่อให้อากาศ · operator splitting คือสิ่งที่ทำให้โครงสร้างนี้สะอาดและคงความเชิงเส้นไว้ทุกพจน์ → superposition (2.5) ยังใช้ได้

6. ทำไม splitting ช่วยงานเรา — 3 ข้อ

  1. แยกฟิสิกส์ออกจากการเรียน: ใส่ความรู้ที่แน่นอน (advection, ไม่ติดลบ, อนุรักษ์มวล) เป็นพจน์ตายตัว → นิวรอลโฟกัสเรียนแค่ส่วนที่ยากจริง → เรียนง่ายขึ้น ใช้ข้อมูลน้อยลง (สำคัญมากเพราะเรามีแค่ 2 เดือน/1 GPU)
  2. คงความเชิงเส้นทุกพจน์: แต่ละ sub-step ออกแบบให้เชิงเส้นใน C → superposition รอด → attribution (C2) ทำได้
  3. ยืดหยุ่น/ดีบักง่าย: ทดสอบแต่ละพจน์แยกได้ (เช่นปิด diffusion ดูว่า advection ถูกไหม) → ตรงกับปรัชญา error analysis ของเรา (แยกว่า error มาจากพจน์ไหน)
เชื่อม oracle substitution (Module 5): เพราะสมการแยกเป็นพจน์ เราสามารถ "เปลี่ยนพจน์เป็นของจริง (oracle)" ทีละตัว เพื่อวัดว่า error มาจากพจน์ไหน — เช่น ใส่ลมจริงแทนลม ERA5 ดูว่า error ลดเท่าไร = ε_wind · operator splitting ทำให้การผ่า error แบบนี้เป็นไปได้

7. โค้ดโครงสร้าง 1 timestep

def hazenet_step(C, wind, met, static):
    """เดินหนึ่ง timestep ด้วย operator splitting"""
    # --- พจน์ที่นิวรอลทำนายสัมประสิทธิ์ (จากอากาศ ไม่ใช่จาก C) ---
    g = afno_emit(met, static)       # สัมประสิทธิ์ปล่อย g_theta
    D = afno_loss(met, static)       # อัตราหาย D_theta (0..1)
    E = emission_from_fire(met)      # ควันใหม่จากไฟ (FRP, 1.1)

    # --- ขั้นตอน splitting (ต่อกันเป็นทอด) ---
    C = advect(C, wind)              # 1) 𝒜: ฟิสิกส์ semi-Lagrangian (2.2)
    C = C + g * E                    # 2) emission
    C = C - D * C                    # 3) loss  (= (1-D)*C)
    return C                        # C^{t+1}

# ทุกบรรทัดเชิงเส้นใน C -> superposition รอด -> แยกแหล่งได้ (2.5)
# advect ฝังตายตัว · g,D มาจากนิวรอล · ทั้งหมด differentiable -> adjoint ฟรี

นี่คือสมการแกนในรูปโค้ด · สังเกตว่ามันคือ "สเต็ปทำอาหาร" ที่ผสมฟิสิกส์ (advect) กับนิวรอล (g, D) ลงตัว

8. Splitting error (ความซื่อสัตย์)

การแยกแก้ทีละพจน์ไม่ฟรี 100% · เพราะในความจริงทุกพจน์เกิดพร้อมกัน การทำทีละอันสร้าง error เล็กน้อยเรียก splitting error · ตัวอย่าง: ควันที่เพิ่งปล่อย (emission) ควรถูกลมหอบ (advection) ทันทีในชั่วโมงนั้น แต่ถ้าเราทำ advection ก่อน emission ทีหลัง ควันใหม่จะ "ตกรอบ" การหอบไปหนึ่ง step

  • splitting error เล็กลงเมื่อ Δt เล็ก และ Strang (สมมาตร) ทำให้เล็กกว่า Lie อีก
  • เป็นส่วนหนึ่งของ ε_structure (จาก 2.1) — error จากการออกแบบเชิงตัวเลข
ทำไมยอมรับได้: splitting error เล็กมากเทียบกับ error จากแหล่งอื่น (ลม ERA5 หยาบ, บัญชีการปล่อยไฟ) · และเราได้ของแลกที่คุ้มกว่ามาก: โครงสร้างที่เชิงเส้น+พิสูจน์คุณสมบัติได้+ผสมฟิสิกส์กับนิวรอลได้ · ในเปเปอร์เราจะระบุ splitting error เป็นข้อจำกัดที่รู้ตัว และวัดขนาดมันด้วย convergence test (ลด Δt แล้วดูว่า error ลดตามทฤษฎีไหม)

9. Cheat sheet + แบบฝึกหัด

สรุปจำ:
  • ปัญหา: ADR มีหลายพจน์เกิดพร้อมกัน แก้พร้อมกันยาก
  • Operator splitting: แก้ทีละพจน์ ต่อกันเป็นทอด ใน 1 timestep (เหมือนทำอาหารเป็นสเต็ป)
  • Lie = ต่อตรง ๆ (อันดับ 1) · Strang = สมมาตร ½+เต็ม+½ (อันดับ 2 แม่นกว่า)
  • ใน HazeNet: advection=ฟิสิกส์ตายตัว(2.2) · emission/loss=นิวรอลเรียนสัมประสิทธิ์ g,D
  • ★★ กาวเชื่อม: splitting คือโครงสร้างที่ผสม "ฟิสิกส์ที่รู้แน่" + "นิวรอลที่เรียน" ในสมการเดียว
  • คงความเชิงเส้นทุกพจน์ → superposition รอด → attribution ได้ (= conditionally-linear, 3.5)
  • 3 ประโยชน์: แยกฟิสิกส์ออกจากการเรียน · คงเชิงเส้น · ดีบัก/ผ่า error ได้ (oracle)
  • ข้อจำกัด: splitting error (ทำทีละอันแทนพร้อมกัน) = ส่วนของ ε_structure · เล็กลงเมื่อ Δt เล็ก

แบบฝึกหัด 1 — operator splitting ช่วยให้เรา "ผสมฟิสิกส์กับนิวรอล" ได้ยังไง? ยกตัวอย่างพจน์ที่เป็นฟิสิกส์ vs นิวรอลในงานเรา

เฉลย

splitting แยกสมการเป็นขั้นตอนย่อยที่แก้แยกได้ → เราเลือกวิธีต่างกันต่อแต่ละพจน์ · advection (𝒜) = ฟิสิกส์ฝังตายตัว (semi-Lagrangian) เพราะต้องรักษาความเชิงเส้น/ไม่ติดลบ · emission (g) และ loss (D) = นิวรอล AFNO เรียนสัมประสิทธิ์จากอากาศ เพราะซับซ้อนเกินเขียนสูตร · ทั้งสองมาต่อกันใน 1 timestep ได้เพราะ splitting

แบบฝึกหัด 2 — Strang ต่างจาก Lie splitting ยังไง และทำไมถึงแม่นกว่า?

เฉลย

Lie: แก้พจน์ต่อกันตรง ๆ (A เต็ม → B เต็ม) อันดับ 1 · Strang: ทำพจน์หัวเป็นครึ่ง ประกบท้ายอีกครึ่ง (A½ → B เต็ม → A½) แบบสมมาตร · ความสมมาตรทำให้ error จาก "ลำดับพจน์มีผล" หักล้างกันมากขึ้น → อันดับ 2 (error เล็กลงเร็วกว่าเมื่อ Δt เล็ก) โดยจ่ายแค่ทำ A ครึ่ง step สองครั้ง

แบบฝึกหัด 3 — splitting error คืออะไร และทำไมเรายอมรับมันได้?

เฉลย

splitting error = error จากการแก้พจน์ทีละอันแทนที่จะพร้อมกัน (เช่น ควันที่เพิ่งปล่อยตกรอบการหอบไป 1 step) · ยอมรับได้เพราะ (1) มันเล็กมากเทียบกับ error จากลม ERA5 หยาบ/บัญชีไฟ, (2) เล็กลงเมื่อ Δt เล็ก + Strang ช่วย, (3) เราได้โครงสร้างที่เชิงเส้น+พิสูจน์ได้+ผสมฟิสิกส์/นิวรอลแลกมา ซึ่งคุ้มกว่ามาก · จัดเป็นส่วนของ ε_structure ที่เรารายงานอย่างซื่อสัตย์

จบ 2.3 แล้วคุณควร: เข้าใจว่า operator splitting = แก้สมการหลายพจน์ทีละพจน์ต่อกันเป็นทอด (เหมือนทำอาหารเป็นสเต็ป), รู้ว่า Strang = แบบสมมาตรที่แม่นกว่า, และที่สำคัญสุด — เห็นว่า splitting คือกาวที่เชื่อม 𝒜 (ฟิสิกส์ 2.2) เข้ากับ g,D (นิวรอล) ในสมการแกนเดียว โดยคงความเชิงเส้นทุกพจน์ → ปูตรงสู่ conditionally-linear operator (3.5)
