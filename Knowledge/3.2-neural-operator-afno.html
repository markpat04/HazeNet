3.2 — Neural Operator + AFNO / FourCastNet

นิวรอลที่เรียน "ตัวดำเนินการ" ในโดเมนความถี่ — backbone ℱθ ตัวทำนาย g, D ของเรา

ต่อจาก 3.1 (Fourier) โดยตรง · เอา FFT มาประกอบเป็นนิวรอลจริง · ผูกกับ HazeNet ตลอด

บทนี้ประกอบทุกอย่างเข้าด้วยกัน: 3.1 ให้เครื่องมือ Fourier · บทนี้เอามาสร้าง AFNO = นิวรอลที่เป็น "สมอง" ของ HazeNet · หน้าที่ของมันไม่ใช่พยากรณ์ฝุ่นตรง ๆ แต่ดูสนามอากาศแล้วทำนายสัมประสิทธิ์ g (ปล่อย) และ D (สูญเสีย) ที่ป้อนเข้าสมการแกน (จาก 2.3/2.5) · เราจะเข้าใจว่า "neural operator" ต่างจากนิวรอลธรรมดายังไง ทำไม AFNO เหมาะกับเรา และมันต่อกับฟิสิกส์ของเราอย่างไรโดยไม่ทำลาย superposition

1. "Operator" คืออะไร — ต่างจากฟังก์ชันธรรมดา ★

นี่คือแนวคิดใหม่ที่ต้องจับให้ได้ก่อน:

รับเข้า → ออกตัวอย่าง
Function (ฟังก์ชัน)ตัวเลข → ตัวเลขf(3) = 9
Operator (ตัวดำเนินการ)ฟังก์ชันทั้งอัน → ฟังก์ชันทั้งอัน"หาอนุพันธ์" รับเส้นโค้ง → คืนเส้นโค้งความชัน

operator คือ "เครื่องที่กินทั้งสนาม คืนทั้งสนาม" · เช่น "เอาสนามอุณหภูมิทั้งแผนที่ → คืนสนามลมทั้งแผนที่" · สมการ ADR (1.2) ก็เป็น operator: รับสนามควันตอนนี้ → คืนสนามควันตอนหน้า

เชื่อมงานเรา: สิ่งที่เราอยากให้นิวรอลเรียนคือ "operator" — รับสนามอากาศทั้งแผนที่ (ลม, ความชื้น, PBL...) → คืนสนามสัมประสิทธิ์ g, D ทั้งแผนที่ · ไม่ใช่ "ฟังก์ชัน" ที่รับเลขเดียวคืนเลขเดียว · นี่คือเหตุผลที่เราต้อง "neural operator" ไม่ใช่ MLP ธรรมดา

2. Neural Operator คืออะไร ★

Neural Operator = นิวรอลที่ออกแบบมาเรียน "operator" (สนาม→สนาม) · จุดเด่นสำคัญคือ resolution-invariant (ไม่ผูกกับความละเอียดกริด) — เทรนที่กริดหนึ่ง ใช้ที่อีกกริดได้ เพราะมันเรียน "กฎฟิสิกส์ที่อยู่เบื้องหลัง" ไม่ใช่ "ค่าต่อพิกเซล"

ทำไมสำคัญกับเรา: CNN ธรรมดา (0.5) ผูกกับกริดที่เทรน + receptive field เล็ก · Neural operator เรียน "ตัวดำเนินการเชิงฟิสิกส์" → เหมาะกับการ emulate CTM (1.5) ที่เป็น operator แก้สมการ ADR · เราอยากให้นิวรอลเรียน "พฤติกรรมฟิสิกส์" ไม่ใช่จำภาพ

Neural operator มีหลายแบบ — แบบที่ดังและเหมาะกับสนามอากาศที่สุดคือตระกูล Fourier (FNO → AFNO) เพราะใช้ประโยชน์จาก Convolution Theorem (3.1)

3. FNO — Fourier Neural Operator ★★

FNO คือ neural operator ที่ใช้ FFT เป็นหัวใจ · หนึ่ง "FNO block" ทำ 3 ขั้น (ตรงกับรูปใน 3.1):

  1. FFT สนามอินพุตไปโดเมนความถี่ (3.1)
  2. คูณด้วยน้ำหนัก R ที่เทรนได้ ต่อแต่ละความถี่ — นี่คือ "convolution global ที่เรียนได้" (Convolution Theorem) · จุดสำคัญ: เก็บแค่ความถี่ต่ำ (modes น้อย ๆ) → ตัดรายละเอียดจุกจิก + ประหยัดพารามิเตอร์
  3. iFFT กลับ + บวก "เส้นทางท้องถิ่น" (linear W ต่อจุด) + ผ่าน activation

ซ้อนหลาย block = FNO · แต่ละ block เห็นทั้งแผนที่ (global) เพราะ Fourier

"modes" คืออะไร (จำคำนี้): FNO ไม่เก็บทุกความถี่ — เก็บแค่ความถี่ต่ำสุด K ตัวแรก (เรียก "Fourier modes") · เหตุผล: ความถี่ต่ำ = โครงสร้างใหญ่ที่สำคัญ (3.1), ความถี่สูง = noise จุกจิก · เก็บน้อย = พารามิเตอร์น้อย = เทรนเร็ว ประหยัด GPU (สำคัญกับข้อจำกัดเรา)

4. AFNO ต่างจาก FNO ยังไง

AFNO (Adaptive Fourier Neural Operator) = FNO เวอร์ชันปรับปรุงให้ขยายขนาด (scale) ได้ดีและประหยัดหน่วยความจำ สำหรับสนามความละเอียดสูง · ปรับ 2 อย่างหลัก:

  • แชร์น้ำหนักแบบ block-diagonal MLP ในโดเมนความถี่: แทนที่จะมีน้ำหนัก R เต็มก้อน (พารามิเตอร์เยอะ) ใช้ MLP เล็ก ๆ ที่แชร์กัน → พารามิเตอร์น้อยลงมาก
  • Soft-thresholding (sparsity): ตัดความถี่ที่ไม่สำคัญทิ้ง → เร็วขึ้น ทนต่อ overfit

5. FourCastNet — หลักฐานว่า AFNO เวิร์กกับอากาศ

FourCastNet (NVIDIA) คือโมเดลพยากรณ์อากาศโลกที่ใช้ AFNO เป็น backbone · มันพยากรณ์ตัวแปรอากาศหลายตัวทั่วโลกที่ 0.25° (เท่า ERA5, จาก 1.3) ได้เร็วกว่าโมเดลฟิสิกส์ดั้งเดิมหลายหมื่นเท่า โดยความแม่นแข่งได้

ทำไมเรื่องนี้สำคัญกับเรา: FourCastNet พิสูจน์แล้วว่า "AFNO เรียน operator ของพลวัตบรรยากาศได้จริง และเร็วมาก" · งานเราคือแนวคิดเดียวกันแต่ขยับไปอีกชั้น: FourCastNet emulate โมเดลอากาศ · HazeNet emulate โมเดลเคมี-ขนส่ง (CTM) และเพิ่มโครงสร้างฟิสิกส์ + attribution · เราต่อยอดจากของที่พิสูจน์แล้ว ไม่ได้เริ่มจากศูนย์
หมายเหตุอ้างอิง: ผมระบุชื่องาน (FourCastNet/AFNO/FNO) จากความรู้ · ก่อนเขียนลงเปเปอร์จริง ให้ verify arXiv ID + ปี + ตัวเลขความเร็วให้ตรงเป๊ะอีกครั้ง (ผมเคยแฟลกไว้แล้วในไฟล์ overview)

6. AFNO ใน HazeNet — ทำนาย g, D ★★

นี่คือจุดที่ทุกอย่างต่อกัน · AFNO ไม่ได้พยากรณ์ฝุ่น C โดยตรง · มันทำหน้าที่เดียว: ดูสนามอากาศ+static → ออกสนามสัมประสิทธิ์

แล้วสมการแกน (ฟิสิกส์ + tracer, จาก 2.3/2.5) เอา g, D ไปคูณกับ C แบบเชิงเส้น · นี่คือการแบ่งงานที่สำคัญ:

นี่คือหัวใจที่ปูสู่ 3.5 (conditionally-linear): ความไม่เชิงเส้นทั้งหมดถูกกักไว้ใน AFNO ที่ทำงานกับอากาศ · พอ AFNO ออก g, D มาแล้ว (เป็นแค่ "ตัวเลขสัมประสิทธิ์") การประกอบกับ C เป็นเชิงเส้นล้วน → superposition (2.5) รอด → attribution ได้ · ถ้าเผลอเอา AFNO ไปแตะ C โดยตรง (เช่น FFT บน C) ทุกอย่างพัง · กฎทองจาก 2.5/3.1 คือเส้นแบ่งที่ AFNO ห้ามข้าม

7. ทำไมเลือก AFNO — 3 เหตุผล

  1. Global receptive field: Fourier เห็นทั้งแผนที่พร้อมกัน (3.1) → จับการพัดพาควันระยะไกลข้ามพรมแดน (1.4) ได้โดยตรง ไม่ต้องซ้อนชั้นเยอะแบบ CNN
  2. ประหยัด GPU + scale ได้: AFNO ออกแบบมาเพื่อสนามความละเอียดสูงโดยพารามิเตอร์ไม่ระเบิด → เหมาะกับ 1 GPU/2 เดือน
  3. พิสูจน์แล้วกับอากาศ: FourCastNet แสดงว่า AFNO emulate พลวัตบรรยากาศได้จริง → ความเสี่ยงต่ำ เราต่อยอดของที่เวิร์ก
ทางเลือกอื่นที่พิจารณา: U-Net (3.3) — receptive field ดีแต่ยังท้องถิ่นกว่า · GNN — ยืดหยุ่นแต่ตั้ง graph ยุ่ง · Transformer/ViT — global แต่กิน GPU มหาศาล (attention ∝ N²) · AFNO ได้ global เหมือน transformer แต่ถูกกว่ามาก (Fourier ∝ N·logN) = จุดที่ลงตัวที่สุดสำหรับข้อจำกัดเรา · (เราอาจผสม U-Net เป็น decoder — ดู 3.3)

8. โค้ดโครงสร้าง AFNO block (อ่านพอเข้าใจ)

import torch, torch.nn as nn

class AFNOBlock(nn.Module):
    def __init__(self, dim, modes):
        super().__init__()
        self.modes = modes               # เก็บกี่ความถี่ (ต่ำ ๆ)
        self.mlp = nn.Sequential(        # MLP เล็กแชร์กัน (หัวใจ "A" ของ AFNO)
            nn.Linear(dim, dim), nn.GELU(), nn.Linear(dim, dim))
        self.w = nn.Conv2d(dim, dim, 1)  # เส้นทางท้องถิ่น

    def forward(self, x):                 # x = สนามอากาศ (B, dim, H, W)
        z = torch.fft.rfft2(x)           # 1) ไปโดเมนความถี่ (3.1)
        z = z[..., :self.modes, :self.modes]   # เก็บแค่ความถี่ต่ำ
        z = apply_mlp_complex(z, self.mlp)     # 2) แปลงต่อความถี่ (เรียนได้)
        z = soft_threshold(z)            #    ตัดความถี่ไม่สำคัญ (sparsity)
        y = torch.fft.irfft2(z, s=x.shape[-2:])  # 3) กลับเป็นสนาม
        return nn.GELU()(y + self.w(x))  # + ท้องถิ่น + activation

# ซ้อนหลาย block = backbone ℱθ
# หัวสุดท้าย: Conv -> softplus -> g ;  Conv -> sigmoid -> D  (จาก 2.4)
เชื่อม 2.4 (hard constraint): หัวสุดท้ายของ AFNO ปิดท้ายด้วย softplus (ให้ g ≥ 0) และ sigmoid (ให้ 0 ≤ D ≤ 1) · ไม่ว่า AFNO ข้างในจะออกอะไร สัมประสิทธิ์ที่ส่งเข้าสมการแกนถูกบีบให้อยู่ในกรอบฟิสิกส์เสมอ → non-negativity การันตีได้ (2.4)

9. Cheat sheet + แบบฝึกหัด

สรุปจำ:
  • Operator: รับ "สนามทั้งอัน" → คืน "สนามทั้งอัน" (≠ ฟังก์ชันที่รับเลขคืนเลข)
  • Neural Operator: นิวรอลเรียน operator · resolution-invariant · เรียนกฎฟิสิกส์ ไม่ใช่จำพิกเซล
  • FNO: FFT → คูณน้ำหนักต่อความถี่ (เก็บแค่ low modes) → iFFT + เส้นทางท้องถิ่น
  • modes: จำนวนความถี่ต่ำที่เก็บ · น้อย = เบา/เร็ว/กัน overfit
  • AFNO: FNO ที่เบา+scale ได้ (MLP แชร์ + sparsity) · backbone ของ FourCastNet
  • FourCastNet: หลักฐานว่า AFNO emulate อากาศได้จริง เร็วมาก → เราต่อยอดสู่ CTM
  • ★★ ใน HazeNet: AFNO ดูอากาศ → ออก g,D · ไม่แตะ C → superposition รอด
  • 3 เหตุผลเลือก: global RF · ประหยัด GPU/scale · พิสูจน์แล้วกับอากาศ
  • หัวปิดท้าย: softplus(g≥0), sigmoid(D∈0..1) → hard constraint (2.4)

แบบฝึกหัด 1 — "operator" ต่างจาก "function" ยังไง และทำไมงานเราต้องการ neural operator ไม่ใช่ MLP ธรรมดา?

เฉลย

function รับเลข→คืนเลข (f(3)=9) · operator รับสนามทั้งอัน→คืนสนามทั้งอัน (เช่น สนามอากาศ→สนามลม) · งานเราต้องรับสนามอากาศทั้งแผนที่ (ลม/ความชื้น/PBL ทุกจุด) แล้วออกสนาม g,D ทั้งแผนที่ พร้อมจับความสัมพันธ์เชิงพื้นที่ (ควันจุดหนึ่งกระทบอีกจุด) → เป็นงาน operator · MLP ธรรมดาจัดการ "ทั้งสนามพร้อมความสัมพันธ์เชิงพื้นที่+resolution invariance" ไม่ได้ดี

แบบฝึกหัด 2 — ทำไม FNO เก็บแค่ "low Fourier modes" และนี่ช่วยข้อจำกัด 1 GPU/2 เดือนของเรายังไง?

เฉลย

ความถี่ต่ำ = โครงสร้างใหญ่ที่สำคัญ (ม่านควันทั่วภาค) · ความถี่สูง = รายละเอียดจุกจิก/noise · เก็บแค่ low modes → พารามิเตอร์น้อยลงมาก → เทรนเร็ว ใช้ GPU/หน่วยความจำน้อย และกัน overfit (ไม่ไปจำ noise) · ตรงกับข้อจำกัดเรา (1 GPU, 2 เดือน, ข้อมูลจำกัด) พอดี

แบบฝึกหัด 3 — AFNO เป็นนิวรอลที่ "ไม่เชิงเส้น" แต่สมการเราต้อง "เชิงเส้นใน C" เพื่อ superposition · สองอย่างนี้อยู่ด้วยกันได้ยังไง?

เฉลย

เพราะ AFNOไม่แตะ C เลย · มันรับแต่อากาศ (ไม่เชิงเส้นได้เต็มที่) แล้วออก g, D เป็น "ตัวเลขสัมประสิทธิ์" · จากนั้นสมการแกนค่อยเอา g, D ไปคูณ/บวกกับ C แบบเชิงเส้นล้วน · ความไม่เชิงเส้นทั้งหมดถูกกักไว้ฝั่งอากาศ → เมื่อกำหนดอากาศแล้ว สมการเป็นเชิงเส้นใน C → superposition รอด · นี่คือ "conditionally-linear" (หัวใจ 3.5) · ห้าม AFNO แตะ C โดยตรง (เช่น FFT บน C) เด็ดขาด

จบ 3.2 แล้วคุณควร: เข้าใจว่า operator = สนาม→สนาม, neural operator เรียนกฎฟิสิกส์ (resolution-invariant), FNO ใช้ FFT (เก็บ low modes), AFNO = เวอร์ชันเบา/scale ได้ (backbone FourCastNet), และที่สำคัญสุด — AFNO ใน HazeNet ดูอากาศออก g,D โดยไม่แตะ C → กักความไม่เชิงเส้นไว้ฝั่งอากาศ → superposition รอด · นี่คือสะพานตรงสู่หัวใจ 3.5
