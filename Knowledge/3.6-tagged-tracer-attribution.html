1. ปัญหาที่ต้องแก้ — "ควันที่เชียงใหม่มาจากไหน?"

ลองนึกภาพสถานการณ์จริง: วันที่ 22 มีนาคม 2023 สถานีวัดคุณภาพอากาศที่เชียงใหม่รายงาน PM2.5 = 210 μg/m³ — ระดับอันตรายมาก

คำถามที่ทุกคนอยากรู้

  • ควันมาจากไฟป่าในพม่าเหนือพรมแดน หรือไฟป่าในภาคเหนือของเราเอง?
  • ถ้าบังคับดับไฟในพม่าทั้งหมด PM2.5 จะลดลงเท่าไหร่?
  • ควันจากลาวและกัมพูชามีส่วนแค่ไหน?
  • ถ้าจะจัดสรรเงินดับไฟ ควรเน้นประเทศไหนก่อน?

แบบจำลองทั่วไปบอกแค่ค่ารวม — คำนวณ C_total(x, t) ทั้ง domain แต่ไม่สามารถบอกว่าแต่ละ μg/m³ มาจากแหล่งไหน

ปัญหาหลัก: เราเห็นผลลัพธ์รวม แต่ไม่รู้ส่วนแบ่ง — เหมือนเห็นน้ำในอ่างแต่ไม่รู้ว่ามาจากก๊อกไหนเท่าไหร่

Tagged-tracer แก้ปัญหานี้โดยตรง — ไม่ใช่การเดา แต่เป็นการคำนวณที่มีหลักการทางฟิสิกส์รองรับ

2. Tagged-Tracer คืออะไร — อุปมาและหลักการ

2.1 อุปมา: ย้อมน้ำต้นน้ำ

จินตนาการแม่น้ำที่มีสาขามาจาก 4 ทิศ ก่อนที่น้ำจะมารวมกัน เราเทสี:
สาขาเหนือ → สีน้ำเงิน  ·  สาขาตะวันออก → สีส้ม  ·  สาขาใต้ → สีเขียว  ·  สาขาตะวันตก → สีแดง

เมื่อน้ำมาถึงปลายน้ำ ดูสีแล้วรู้ทันที: "ปากแม่น้ำนี้น้ำ 45% มาจากเหนือ, 30% ตะวันออก, 15% ใต้, 10% ตะวันตก"

Tagged-tracer คือการทำสิ่งเดียวกันกับอนุภาคฝุ่น — ติดป้ายชื่อที่แหล่งกำเนิด แล้วให้ป้ายลอยตามฝุ่นไปตลอดทาง

2.2 วิธีการทำงาน

แทนที่จะมีความเข้มข้นฝุ่นรวม C(x,t) เพียงตัวเดียว เราสร้าง N concentration fields แยกกัน:

สัญลักษณ์ความหมายตัวอย่าง SEA
C(1)(x,t)ความเข้มข้นจากแหล่ง 1 เท่านั้นไฟป่าพม่าเหนือ
C(2)(x,t)ความเข้มข้นจากแหล่ง 2 เท่านั้นไฟป่าภาคเหนือไทย
C(3)(x,t)ความเข้มข้นจากแหล่ง 3 เท่านั้นไฟป่าลาว
………
C(N)(x,t)ความเข้มข้นจากแหล่ง Nฝุ่นพื้นหลัง (background)

แต่ละ C(s) วิ่งผ่านฟิสิกส์เดียวกัน (ลม, การแพร่กระจาย, decay) — เพียงแต่เริ่มจากแหล่งต่างกัน

กฎหลัก: C_total = C(1) + C(2) + C(3) + … + C(N)  ·  ที่ทุกจุด (x,t)
Attribution ณ จุดใดๆ = C(s)(x,t) / C_total(x,t) คือสัดส่วนจากแหล่ง s

3. ทำไมถึงทำได้ — คณิตศาสตร์ (Superposition อีกครั้ง)

3.1 ทบทวน 2.5 Superposition

จาก 2.5 เราพิสูจน์แล้วว่า สมการ ADR เชิงเส้นใน C:

-- สมการ ADR (1.2) --
∂C/∂t + u·∇C = D∇²C - λC + Q(x,t)

-- ถ้า C = C⁽¹⁾ + C⁽²⁾ แต่ละตัวสอดคล้อง --
∂C⁽¹⁾/∂t + u·∇C⁽¹⁾ = D∇²C⁽¹⁾ - λC⁽¹⁾ + Q⁽¹⁾(x,t)
∂C⁽²⁾/∂t + u·∇C⁽²⁾ = D∇²C⁽²⁾ - λC⁽²⁾ + Q⁽²⁾(x,t)

-- บวกกัน → C_total ก็สอดคล้องสมการกับ Q_total = Q⁽¹⁾ + Q⁽²⁾ --
∂(C⁽¹⁾+C⁽²⁾)/∂t + ... = ... + (Q⁽¹⁾ + Q⁽²⁾)(x,t)  ✓

กุญแจ: สมการฟิสิกส์เชิงเส้น → ผลรวมของ solutions ก็เป็น solution → แต่ละ tagged tracer วิ่งอิสระกันได้โดยไม่ต้องคุยกัน

3.2 พิสูจน์ว่า Attribution บวกได้ครบ 100%

สมมติมี S แหล่ง: C_total(x,t) = Σs=1..S C(s)(x,t)

Attribution ของแหล่ง s ที่จุด (x,t):

α⁽ˢ⁾(x,t) = C⁽ˢ⁾(x,t) / C_total(x,t)

ตรวจสอบว่ารวมได้ 100%:

Σ α⁽ˢ⁾ = Σ C⁽ˢ⁾ / C_total = C_total / C_total = 1  ✓

→ เสมอ 100% ที่ทุกจุด ทุกเวลา โดยไม่ต้อง normalize เพิ่ม

3.3 Counterfactual — ถ้าดับไฟแหล่ง s

# ถ้าดับไฟแหล่ง s ทั้งหมด → ลบ C⁽ˢ⁾ออก
C_without_s(x,t) = C_total(x,t) - C⁽ˢ⁾(x,t)

# ที่สถานีเชียงใหม่ (lat=18.8, lon=98.9)
reduction = C⁽ˢ⁾[chiang_mai, t]  # μg/m³ ที่ลดลงได้
pct = 100 * C⁽ˢ⁾[chiang_mai, t] / C_total[chiang_mai, t]
ทำไมสำคัญมาก: "ถ้าพม่าดับไฟ PM2.5 ที่เชียงใหม่ลดลง 38%" คือข้อมูลเชิงนโยบายที่ตรงมาก — ไม่ใช่การคาดเดา แต่คำนวณได้จากฟิสิกส์

4. Smoke Fingerprint — ลายนิ้วมือควัน

4.1 ทำไมแต่ละแหล่งมี "ลายนิ้วมือ" ต่างกัน

C(s)(x,t) ที่เราวัดได้ตามสถานีต่างๆ สร้างรูปแบบเชิงพื้นที่-เวลาที่ เฉพาะตัวสำหรับแต่ละแหล่ง เพราะ:

ปัจจัยทำให้ fingerprint ต่างกันอย่างไร
ระยะทางแหล่งไกล → ฝุ่นกระจายมากขึ้น → เข้มข้นต่ำกว่า
ทิศลมลมพัดไปทิศเหนือ → แหล่งทางใต้มีผลสูง แหล่งทางเหนือไม่มีผล
ค่า PBLบรรยากาศผสมดี → กระจายตัวในแนวตั้ง → เจือจาง
decay rate λฝุ่นสลายตัวตามเวลา → แหล่งไกลหรือช้า → เจือจางมากกว่า
ภูมิประเทศเทือกเขากั้น → แหล่งฝั่งตะวันตกแตกต่างกับฝั่งตะวันออก

4.2 Fingerprint ในทางปฏิบัติ

สมมติสถานีวัด 10 แห่งทั่วภาคเหนือ เวลา 48 ชั่วโมง fingerprint ของแหล่ง s คือ:

# shape: (10 สถานี × 48 ชั่วโมง) = vector 480 มิติ
fingerprint_s = C⁽ˢ⁾[stations, 0:48]  # μg/m³ ที่แต่ละสถานีแต่ละชั่วโมง

สองแหล่งที่อยู่ใกล้กันมาก fingerprint อาจคล้ายกัน → attribution แยกได้ยาก (ill-conditioned) — เป็น caveat ที่จะพูดถึงใน Section 9

5. เชื่อมกับ HazeNet (3.5) — ทำไมต้องเป็น Conditionally-Linear

5.1 ปัญหากับนิวรอลทั่วไป

ถ้าโมเดลไม่เชิงเส้นใน C:
F(C⁽¹⁾ + C⁽²⁾ | met) ≠ F(C⁽¹⁾ | met) + F(C⁽²⁾ | met)
→ ไม่สามารถ propagate C⁽ˢ⁾ แยกกันได้
→ Tagged-tracer พัง — attribution ไม่ถูกต้อง

นี่คือเหตุผลที่ HazeNet ต้องเป็น conditionally-linear (3.5) — ไม่ใช่แค่ดีกว่า แต่ จำเป็น สำหรับ attribution

5.2 วิธีที่ HazeNet ทำ Tagged-Tracer

สมการ HazeNet (3.5):

F_θ(C | met) = Φ_θ(C_total, met) · C

เมื่อ Φ_θ คือ "operator coefficients" ที่นิวรอลเรียนมา (depends on C_total และ met)

วิธีใช้กับ tagged tracers:

-- Step 1: คำนวณ Φ จาก C_total (physics ต้องเห็น total field) --
C_total = Σ C⁽ˢ⁾                     # รวมทุกแหล่ง
Φ = Φ_θ(C_total, met)                # operator จากทั้ง domain

-- Step 2: apply Φ ให้แต่ละ tracer (linear → อิสระจากกัน) --
C⁽ˢ⁾_next = Φ · C⁽ˢ⁾   สำหรับแต่ละ s

-- ตรวจสอบ: Σ C⁽ˢ⁾_next = Φ · (Σ C⁽ˢ⁾) = Φ · C_total ✓ --
ทำไม Φ ต้องคำนวณจาก C_total (ไม่ใช่ C⁽ˢ⁾)?
เพราะ Φ คือ "สภาพแวดล้อมบรรยากาศ" — ลมจริง, การผสม, decay จริง — ต้องเห็น ฝุ่นทั้งหมด จึงจะคำนวณ physics ถูกต้อง
ถ้าคำนวณ Φ จาก C⁽ˢ⁾ เดี่ยวๆ = บอกว่า "ฝุ่นของพม่าอยู่คนเดียวในอากาศว่าง" → ผิดฟิสิกส์

5.3 ภาพรวมความสัมพันธ์

2.5 Superposition → พื้นฐานคณิตศาสตร์ที่ทำให้ C = Σ C⁽ˢ⁾ ใช้ได้
      ↓
3.5 Conditionally-Linear → ทำให้ HazeNet เชิงเส้นใน C (F(ΣC⁽ˢ⁾) = ΣF(C⁽ˢ⁾))
      ↓
3.6 Tagged-Tracer → ใช้ทั้งสองข้างบนติดป้ายและตามฝุ่นแต่ละแหล่ง
      ↓
3.7 Adjoint → ย้อนกลับจากฝุ่นที่วัดได้ → หาว่าแหล่งไหนปล่อยเท่าไหร่ (ต่อไป)

6. Source Regions ในเอเชียตะวันออกเฉียงใต้

6.1 ทำไมต้องจัดกลุ่มแหล่ง

ในความเป็นจริง มีไฟหลายหมื่นจุดทั่ว SEA ในช่วง Haze Season (มีนาคม–เมษายน) เราไม่สามารถติดป้ายแยกทุกจุดไฟได้เพราะ:

ข้อจำกัดจริง:
· หากมี 50,000 จุดไฟ → ต้องการ 50,000 concentration fields → RAM หมดทันที (A100 มีแค่ 40 GB)
· จุดไฟข้างเคียงกันมาก fingerprint แทบเหมือนกัน → แยก attribution ไม่ออก (ill-conditioned)

6.2 การแบ่ง Source Regions แบบ HazeNet

แบ่งตาม ลักษณะภูมิประเทศ + ระบอบลม + พรมแดน ได้ประมาณ 8–12 zones:

IDRegionตัวอย่างแหล่งที่มาBadge
S1พม่าเหนือ (Sagaing/Mandalay)ไฟป่าทีค, deforestationS1-MYN
S2ภาคเหนือไทย (เชียงใหม่/เชียงราย)เผาไร่ข้าวโพด, ป่าS2-NTH
S3ลาวเหนือ (Luang Prabang)ไฟเผาไร่หมุนเวียนS3-LAO
S4กัมพูชา/เวียดนามใต้เผาตอซัง, ฤดูเก็บเกี่ยวS4-CAM
S5Background (อินเดีย/มหาสมุทร)ฝุ่นทะเลทราย, พื้นหลังS5-BKG
…ภาคกลาง, ภาคตะวันออกเฉียงเหนือ ฯลฯ……

ทั้งหมด N = 8–12 regions ให้ RAM-friendly บน A100 (~16× H×W fields)

6.3 วิธีสร้าง Source Mask

import torch
import numpy as np

# สมมติ grid 128×128 ครอบ SEA (90°E–110°E, 5°N–25°N)
H, W = 128, 128
n_sources = 8

# สร้าง mask แต่ละ region (True = เป็น pixel ของ region นั้น)
source_masks = torch.zeros(n_sources, H, W)  # bool map

# S1: Myanmar เหนือ — pixel ที่ lat > 20°N, lon < 100°E
lat_grid, lon_grid = torch.meshgrid(
    torch.linspace(5, 25, H),
    torch.linspace(90, 110, W), indexing='ij'
)
source_masks[0] = (lat_grid > 20) & (lon_grid < 100)  # S1-MYN
source_masks[1] = (lat_grid > 17) & (lat_grid <= 20) & (lon_grid > 97) & (lon_grid < 102)  # S2-NTH
# ... ต่อ S3–S7
source_masks[7] = ~source_masks[:7].any(0)  # S8-BKG = ทุกที่ที่เหลือ

8. โค้ด Implementation ใน HazeNet

8.1 โครงสร้างหลัก

import torch
import torch.nn as nn

class HazeNetWithAttribution(nn.Module):
    def __init__(self, n_sources=8):
        super().__init__()
        self.n_sources = n_sources
        self.phi_net = PhiNetwork()    # เรียน operator Φ (3.5)
        self.advect = SemiLagrangian() # ฟิสิกส์จาก 2.2

    def forward_with_attribution(self, C_tagged, met):
        """
        C_tagged: (n_sources, H, W) — tagged fields
        met:      (n_met, H, W)    — อุตุนิยม (ลม, PBL ฯลฯ)
        returns:  C_tagged_next (n_sources, H, W)
        """
        n_s, H, W = C_tagged.shape

        # Step 1: รวมเป็น total → ใช้คำนวณ Φ
        C_total = C_tagged.sum(0)              # (H, W)

        # Step 2: คำนวณ operator coefficients จาก total
        Phi = self.phi_net(C_total, met)       # (H, W, n_coeff)

        # Step 3: apply Φ ให้แต่ละ tagged tracer
        C_tagged_next = torch.zeros_like(C_tagged)
        for s in range(n_s):
            C_tagged_next[s] = self.apply_phi(Phi, C_tagged[s], met)

        # ตรวจว่า sum ยังตรงกัน (sanity check)
        # assert torch.allclose(C_tagged_next.sum(0), apply_phi(Phi, C_total, met))

        return C_tagged_next

    def get_attribution(self, C_tagged):
        # α⁽ˢ⁾(x,t) = C⁽ˢ⁾(x,t) / C_total(x,t)
        C_total = C_tagged.sum(0, keepdim=True)  # (1, H, W)
        attribution = C_tagged / (C_total + 1e-8)   # (n_s, H, W) — avoid /0
        return attribution  # sums to 1 along dim=0

8.2 Initialization — ติดป้ายจากข้อมูลไฟจริง

def init_tagged_tracers(frp_grid, source_masks, n_sources, H, W):
    """
    frp_grid:    (H, W)           — Fire Radiative Power จาก FIRMS
    source_masks:(n_sources, H,W) — boolean mask แต่ละ region
    returns:     C_tagged (n_sources, H, W)
    """
    C_tagged = torch.zeros(n_sources, H, W)
    for s in range(n_sources):
        # emission จากแหล่ง s = FRP × mask × emission factor
        C_tagged[s] = frp_grid * source_masks[s] * 0.01  # kg/m² → μg/m³
    return C_tagged

# ใช้งาน
C_tagged = init_tagged_tracers(frp_day1, source_masks, n_sources, H, W)
model = HazeNetWithAttribution(n_sources=n_sources)

# Rollout 48 ชั่วโมง
history = [C_tagged]
for t in range(48):
    C_tagged = model.forward_with_attribution(C_tagged, met[t])
    history.append(C_tagged)  # เก็บ attribution ทุก step

attribution_48h = torch.stack(history)  # (48, n_sources, H, W)
print(f"Attribution tensor: {attribution_48h.shape}")
# → torch.Size([48, 8, 128, 128])

9. ข้อจำกัดที่ต้องซื่อสัตย์

9.1 เคมีไม่เชิงเส้น (Non-linear Chemistry)

ปัญหา: สมมติฐาน superposition ถือว่าเคมีในบรรยากาศ เชิงเส้นใน C

จริงๆ แล้ว:
  • SOA (Secondary Organic Aerosol): PM2.5 บางส่วนเกิดจากปฏิกิริยาเคมีระหว่างสาร → ถ้า C⁽¹⁾ และ C⁽²⁾ ทำปฏิกิริยาร่วมกัน attribution ไม่แม่น
  • Hygroscopic growth: ฝุ่นดูดซับน้ำตามความชื้น → rate depends on total C → non-linear
  • Scavenging: ฝนล้างฝุ่น — rate อาจ non-linear
HazeNet ตอบอย่างไร: สำหรับ primary PM2.5 จากไฟ (ซึ่งเป็น focus ของงานเรา) linearity ถือดีพอ — error จาก non-linear chemistry อยู่ใน ~5–15% ซึ่งเล็กกว่า uncertainty อื่น

9.2 Φ คำนวณจาก C_total — Small-Source Problem

ปัญหา: Φ_θ(C_total, met) เห็น total field แต่ถ้าแหล่ง s มีขนาดเล็กมาก (เช่น 0.1% ของ total) C⁽ˢ⁾ จะถูก transport ด้วย Φ ที่ "ไม่ได้ออกแบบมาสำหรับมัน"

ในทางปฏิบัติ: แหล่งใหญ่ใน SEA (พม่า/ไทยเหนือ) อยู่ใน 10–40% ของ total → error น้อย · แหล่งเล็กมาก (< 1%) attribution อาจมี noise สูง

9.3 Attribution Resolution ≠ Fire Resolution

ปัญหา: เราแบ่ง 8 regions ไม่ใช่ 50,000 จุดไฟ → attribution บอกได้แค่ระดับ "ประเทศ" หรือ "ภาค"

ไม่สามารถบอกว่า "ไฟจุดที่ GPS (20.5°N, 99.2°E) มีผลเท่าไหร่ที่เชียงใหม่" → ต้องใช้ adjoint (3.7) แทน

9.4 ไม่บอกแหล่งที่ "ซ่อนอยู่"

ปัญหา: Tagged-tracer ต้องรู้แหล่งทุกแหล่ง ก่อน จึงจะ run ได้

ถ้าไฟเกิดขึ้นในพื้นที่ที่เราไม่ได้ assign region → ถูกนับเป็น "background" → attribution ผิด
ต้องมี monitoring ไฟที่ดี (FIRMS near-realtime) เพื่อ assign source regions ให้ทันเวลา

11. แบบฝึกหัด — ทดสอบความเข้าใจ

Q1: Memory Budget

ถ้า n_sources = 12 regions, rollout 72 ชั่วโมง, grid H×W = 256×256, เก็บ attribution_48h เป็น float32 — tensor มีขนาดกี่ MB?

ดูเฉลย

12 × 72 × 256 × 256 × 4 bytes = 12 × 72 × 262,144 × 4 = 905,969,664 bytes ≈ 864 MB
เกือบ 1 GB เฉพาะ attribution — ถ้า A100 มี 40 GB ยังพอ แต่ต้องระวังถ้า batch หลาย sample

Q2: Superposition Check

หลังรัน forward_with_attribution() แล้ว มีวิธีตรวจสอบว่า attribution ถูกต้องแบบง่ายที่สุดคืออะไร?

ดูเฉลย

assert torch.allclose(C_tagged.sum(0), C_total_from_single_run, atol=1e-5)
คือผลรวม C⁽ˢ⁾ ทุก s ต้องเท่ากับผลที่ได้จากรันด้วย C_total ตัวเดียว — ถ้าไม่เท่า = bug ใน apply_phi

Q3: Policy Simulation

สถานีเชียงใหม่วัด PM2.5 = 150 μg/m³ · attribution ระบุ S1-MYN = 42%, S2-NTH = 35%, S3-LAO = 23%
ถ้าพม่าลดไฟได้ 60% (ไม่ใช่ 100%) PM2.5 ที่เชียงใหม่ควรลดลงไปเหลือเท่าไหร่?

ดูเฉลย

C⁽¹⁾ = 42% × 150 = 63 μg/m³
ลดไฟ 60% → C⁽¹⁾ ลดลง 60% → ลดลง 0.6 × 63 = 37.8 μg/m³
PM2.5 ใหม่ = 150 − 37.8 = 112.2 μg/m³
(สมมติ linearity ในแหล่ง — valid เพราะ conditionally-linear)

Q4: ทำไม Φ ต้องมาจาก C_total

ถ้าเราคำนวณ Φ จาก C⁽¹⁾ แยก (ไม่ใช่ C_total) แล้วใช้กับ C⁽¹⁾ ผลจะผิดอย่างไร?

ดูเฉลย

Φ_θ(C⁽¹⁾, met) จะ "เห็น" atmosphere ที่มีแต่ฝุ่นจากพม่า — ราวกับแหล่งอื่นไม่มีอยู่
ฟิสิกส์บางอย่างขึ้นกับ C_total จริง (เช่น optical depth ที่กระทบ PBL, aerosol feedback) → coefficient Φ จะ underestimate
ผลลัพธ์: C⁽¹⁾_next ผิด → Σ C⁽ˢ⁾_next ≠ C_total_next → superposition พัง

Q5: Tagged-tracer vs Source Inversion

ถ้ามีไฟเกิดขึ้นในพื้นที่ที่เราไม่รู้ก่อน (unknown location) — Tagged-tracer ช่วยได้ไหม? ถ้าไม่ได้ ต้องใช้เครื่องมือไหน?

ดูเฉลย

Tagged-tracer ช่วยไม่ได้ — เราต้องรู้แหล่งก่อน จึงจะ assign ป้ายและ initialize C⁽ˢ⁾ ได้
ถ้าไฟอยู่ใน "background region" มันถูกนับเป็น background ทั้งหมด → attribution ผิด
สำหรับ unknown source → ต้องใช้ Source Inversion (3.7) ซึ่งย้อนกลับจากค่าที่วัดได้ไปหาแหล่งที่น่าจะเป็น
