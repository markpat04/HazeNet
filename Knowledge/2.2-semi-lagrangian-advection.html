2.2 — Semi-Lagrangian Advection ⭐ CORE

วิธีเลื่อนควันตามลม — คือ "ส่วน 𝒜 (ฟิสิกส์)" ในหัวใจของ HazeNet

ต่อจาก 2.1 (CFL) โดยตรง · นี่คือกลไกที่ทำให้ advection ของเราเร็ว เสถียร และไม่ติดลบ · ผูกกับ HazeNet ตลอด

ทำไมบทนี้เป็น CORE: ในสมการแกน Ct+1 = 𝒜(u)Ct + g·E − D⊙Ct ตัว 𝒜(u) คือ "ตัวเลื่อนควันตามลม" และมันคือฟิสิกส์ที่เราฝังตายตัว ไม่ให้นิวรอลเรียนเอง · Semi-Lagrangian คือวิธีสร้าง 𝒜 ที่ (1) เลี่ยงข้อจำกัด CFL (จาก 2.1), (2) ไม่ทำให้ควันติดลบ, (3) เป็นเชิงเส้นใน C → รองรับ superposition (2.5) · ทั้งสามอย่างนี้คือเหตุผลที่ HazeNet "พิสูจน์คุณสมบัติได้" และ "แยกแหล่งได้เป๊ะ"

1. Advection คืออะไร (ทบทวนจาก 1.2)

Advection = การที่ควันถูกลมหอบไป (พจน์ −∇·(uC) ในสมการ ADR) · เหมือนใบไม้ลอยตามน้ำ — ตัวมันไม่เคลื่อนเอง แต่ "ตัวกลาง" (ลม) พาไป

คำถามทางคอมพิวเตอร์คือ: "ฉันมีค่าควันในทุกช่องกริดตอนนี้ (Ct) · ลมพัดไป 1 timestep แล้ว ค่าควันในทุกช่องตอน t+1 เป็นเท่าไร?" · มี 2 วิธีคิด (ข้อ 2)

2. สองมุมมอง: Eulerian vs Lagrangian ★

มุมมองเปรียบเทียบคำถามที่ถาม
Eulerian
(อยู่กับที่)		ยืนบนสะพานมองน้ำไหลผ่าน"ช่องนี้ มีควันไหลเข้า-ออกเท่าไร?"
Lagrangian
(เคลื่อนตาม)		นั่งเรือลอยไปกับน้ำ"ก้อนควันก้อนนี้ ลอยไปไหน?"

Eulerian (finite difference จาก 2.1) = จับตาช่องคงที่ ดูควันไหลผ่าน → ปัญหา: ติดข้อจำกัด CFL (ถ้าลมเร็วควันกระโดดข้ามช่อง → ระเบิด)

Lagrangian = ตามก้อนควันไป → เสถียรดี แต่ปัญหา: ก้อนควันลอยไปอยู่ "ระหว่างช่อง" ตำแหน่งมั่ว ๆ ไม่ตรงกริด → จัดการยาก

Semi-Lagrangian = ลูกผสมที่เอาข้อดีทั้งสอง: "semi" (กึ่ง) เพราะมันใช้มุมมอง Lagrangian (ตามรอย) แต่บังคับให้คำตอบลงที่กริดคงที่แบบ Eulerian เสมอ · ได้ทั้งความเสถียร (Lagrangian) และความเป็นระเบียบบนกริด (Eulerian) · นี่คือวิธีที่ CTM ระดับโลกและ HazeNet ใช้

3. ไอเดียหลัก — "ตามรอยย้อนกลับ" (backward trajectory) ★★ หัวใจ

นี่คือไอเดียเดียวที่ต้องเข้าใจ · แทนที่จะถามว่า "ควันจะไปไหน" (ยาก เพราะปลายทางไม่ตรงกริด) เราถามกลับว่า:

"ควันที่จะมาอยู่ในช่องนี้ตอน t+1 — เมื่อ 1 step ก่อน มันมาจากที่ไหน?"

คำตอบ: มันมาจากจุดที่อยู่ "ทวนลม" (upwind) ห่างออกไปเท่ากับระยะที่ลมพัดได้ใน 1 step · เราแค่ย้อนรอยกลับไปหาจุดนั้น แล้วเอาค่าควันที่จุดนั้นมาใส่ช่องเป้าหมาย

"จุดต้นทาง (departure point)" = ตำแหน่งช่องเป้าหมาย ลบ ระยะที่ลมพัด:

xต้นทาง = xเป้าหมาย − u · Δt
อุปมาง่าย ๆ: อยากรู้ว่ากลิ่นที่จมูกคุณตอนนี้มาจากไหน → เดินทวนลมไปตามระยะที่ลมพัดในเวลาที่ผ่านมา → เจอต้นตอ · semi-Lagrangian ทำแบบนี้ทุกช่อง ทุก step

4. สามขั้นตอนของ semi-Lagrangian (ต่อหนึ่งช่อง)

  1. หาจุดต้นทาง: จากช่องเป้าหมาย ย้อนรอยทวนลม x − u·Δt → ได้พิกัดจุดต้นทาง
  2. Interpolate: จุดต้นทางมักไม่ตรงช่องกริดพอดี → ประมาณค่าควันที่จุดนั้นจากช่องข้างเคียง (ข้อ 5)
  3. กำหนดค่า: เอาค่าที่ได้ มาเป็นค่าควันของช่องเป้าหมายตอน t+1

ทำซ้ำทุกช่องในกริด = ได้สนามควันทั้งผืนตอน t+1 ที่ "ถูกลมหอบไปแล้ว"

5. Interpolation — เมื่อจุดต้นทางไม่ตรงช่อง

จุดต้นทาง (สีเหลืองในรูปข้อ 3) แทบไม่เคยตกตรงกลางช่องพอดี — มันตกอยู่ "ระหว่าง" 4 ช่อง · เราต้องเดาค่าควันที่จุดนั้นจาก 4 ช่องรอบ ๆ โดยใช้ การถ่วงน้ำหนักตามระยะ (bilinear interpolation)

ยิ่งจุดต้นทางใกล้มุมไหน → ค่ามุมนั้นยิ่งมีน้ำหนักมาก · ผลคือค่าประมาณที่ "นุ่มนวล" ไม่กระโดด

จุดสำคัญที่ต้องจำ: bilinear interpolation เป็นการถ่วงน้ำหนักเชิงเส้น (น้ำหนักทั้งหมด ≥ 0 และรวมกันได้ 1) · คุณสมบัติ "น้ำหนัก ≥ 0" นี้แหละที่ทำให้ ควันไม่มีทางติดลบ (ข้อ 7) และ "เชิงเส้น" ที่ทำให้รองรับ superposition · interpolation ไม่ใช่แค่ดีเทล แต่เป็นเหตุผลที่ semi-Lagrangian มีคุณสมบัติดี ๆ

6. ทำไมมันเสถียร — ไม่ติดข้อจำกัด CFL ★

จำจาก 2.1: finite difference (Eulerian) ระเบิดถ้า CFL = u·Δt/Δx > 1 (ควันกระโดดเกิน 1 ช่อง) · semi-Lagrangian ไม่มีปัญหานี้ เพราะ:

ต่อให้ลมแรงจนจุดต้นทางอยู่ห่างไปหลายช่อง เราก็แค่ "ย้อนรอยไปไกลขึ้น" แล้ว interpolate ที่นั่น · ไม่มีการ "กระโดดทีละช่องจนพัง" — เราไปถึงจุดต้นทางโดยตรงเสมอ

เชื่อม HazeNet (สำคัญมาก): เพราะไม่ติด CFL เราใช้ Δt ใหญ่ (รายชั่วโมง) ได้ → เดิน step น้อยกว่า CTM มาก → เร็วกว่ามาก · นี่คือหนึ่งในเหตุผลรากฐานที่ HazeNet รันเป็นวินาทีขณะที่ WRF-Chem (1.5) ใช้ชั่วโมง — เราเลือกวิธีเชิงตัวเลขที่ปล่อยให้ก้าวเวลายาวได้

7. ทำไมมันเป็น "เชิงเส้นใน C" → กลายเป็นเมทริกซ์ 𝒜 ★★

นี่คือจุดที่เชื่อมตรงเข้าหัวใจของทั้งโปรเจกต์ · สังเกตว่าทุกขั้นตอนของ semi-Lagrangian (ย้อนรอย + interpolate) ทำกับ "ตำแหน่ง" และ "น้ำหนัก" ซึ่งขึ้นกับลมเท่านั้น ไม่ขึ้นกับค่าควัน C

ค่าควันใหม่ของแต่ละช่อง = ผลรวมถ่วงน้ำหนักของค่าควันเก่าในช่องข้างเคียง:

Ct+1i = w1Cta + w2Ctb + w3Ctc + w4Ctd
น้ำหนัก w ขึ้นกับลม (u) · ค่าควัน C เป็นแค่ตัวถูกคูณ

นี่คือการคูณเมทริกซ์เป๊ะ ๆ: Ct+1 = 𝒜(u) · Ct โดย 𝒜(u) คือเมทริกซ์ที่บรรจุน้ำหนัก w ทั้งหมด (ขึ้นกับลม) · นี่คือ "ตัว 𝒜" ในสมการแกนของเรา!

คุณสมบัติทองคำ 3 ข้อของ 𝒜 (ที่ semi-Lagrangian ให้ฟรี):
  • เชิงเส้นใน C → 𝒜(Cพม่า+Cลาว) = 𝒜Cพม่า+𝒜Cลาว = superposition (2.5) → แยกแหล่งได้เป๊ะ
  • น้ำหนัก ≥ 0 → ควันออกมาไม่มีทางติดลบ = non-negativity (2.4) โดยโครงสร้าง
  • น้ำหนักรวม = 1 → ไม่สร้าง/ทำลายมวลจากการเลื่อน = ใกล้ mass conservation (2.4)
นี่คือหัวใจของ "physics-structured": เราไม่ปล่อยให้นิวรอลเรียน advection เอง (มันจะละเมิดคุณสมบัติพวกนี้ → แยกแหล่งไม่ได้ + ควันติดลบได้) · เราฝัง 𝒜 จาก semi-Lagrangian ลงไปตายตัว แล้วให้นิวรอล (AFNO) เรียนแค่ "ค่าสัมประสิทธิ์" ของพจน์อื่น (g, D) · เพราะแบบนี้เราถึงเคลม "พิสูจน์คุณสมบัติได้" ในเปเปอร์ A* ได้ — มันเป็นจริงโดยโครงสร้าง ไม่ใช่ความหวังว่านิวรอลจะเรียนถูก

8. โค้ดจำลอง (1 มิติ เพื่อให้เห็นภาพ)

import torch
import torch.nn.functional as F

def semi_lagrangian_1d(C, u, dx, dt):
    """เลื่อนควัน C ตามลม u หนึ่ง timestep (1 มิติ)"""
    N = C.shape[0]
    idx = torch.arange(N, dtype=torch.float)
    # 1) หาจุดต้นทาง: ย้อนรอยทวนลม
    departure = idx - u * dt / dx          # x - u*dt (หน่วยช่อง)
    # 2) interpolate ค่าที่จุดต้นทาง (เชิงเส้น)
    i0 = torch.floor(departure).long()
    frac = departure - i0.float()          # น้ำหนัก (0..1)
    i0 = i0.clamp(0, N-2)
    C_new = (1-frac) * C[i0] + frac * C[i0+1]   # ถ่วงน้ำหนัก >= 0
    return C_new

# สังเกต: C_new เป็น "ผลรวมเชิงเส้นของ C" -> นี่คือ 𝒜·C นั่นเอง
# น้ำหนัก (1-frac), frac ทั้งคู่ >= 0 -> ควันไม่ติดลบ
เชื่อม PyTorch (0.1, 0.2): ในของจริง 2 มิติ เราใช้ F.grid_sample() ของ PyTorch ทำ bilinear interpolation บน GPU ทีเดียวทั้งกริด (ขนานกับ batch ของ tracer ทุกแหล่ง — จาก 1.5) · และเพราะทุกขั้นตอนเป็น operation ของ PyTorch → autograd หาอนุพันธ์ผ่าน 𝒜 ได้ → ส่วนหนึ่งของ "adjoint ฟรี" (0.2) ที่ทำให้ counterfactual/inversion เป็นจริง

9. ข้อจำกัด — numerical diffusion (ความซื่อสัตย์)

semi-Lagrangian ไม่ได้สมบูรณ์แบบ · ทุกครั้งที่ interpolate (เฉลี่ยค่า) มันทำให้สนามควัน "เบลอลงนิดนึง" — ยอดควันที่เคยแหลมคมจะค่อย ๆ ราบลงเมื่อเดินหลาย step · เรียก numerical diffusion (การกระจายเทียมจากตัวเลข ไม่ใช่ฟิสิกส์จริง)

  • ยิ่ง interpolate แบบ low-order (เชิงเส้น) ยิ่งเบลอ → ใช้ interpolation อันดับสูงขึ้น (cubic) ช่วยลดได้ แต่แลกความเร็ว
  • เบลอเกินจริง = ยอด PM2.5 พีคถูกกดต่ำ → เป็นปัญหากับ "วันวิกฤต" ที่เราสนใจ
เชื่อม error analysis (Module 5): numerical diffusion เป็นส่วนหนึ่งของ ε_structure · เราจะวัดมันแยก: รัน advection อย่างเดียวกับสนามที่รู้คำตอบ (analytic test) เทียบว่าเบลอไปเท่าไร · การรู้ว่า error มาจาก "ตัวเลขเบลอ" ไม่ใช่ "นิวรอลพลาด" คือหัวใจของการแยกแหล่ง error ระดับ A*

10. Cheat sheet + แบบฝึกหัด

สรุปจำ:
  • Advection = ลมหอบควัน (พจน์ −∇·(uC)) · 𝒜 คือตัวเลื่อนนี้ในโมเดล
  • Eulerian (อยู่กับที่, finite diff) = ติด CFL · Lagrangian (ตามก้อน) = หลุดกริด
  • Semi-Lagrangian = ลูกผสม: ตามรอยย้อนกลับ แต่ลงกริดคงที่
  • หัวใจ: "ช่องนี้ตอน t+1 ควันมาจากไหน?" → ย้อนรอย x − u·Δt → interpolate
  • 3 ขั้น: หาจุดต้นทาง → interpolate (bilinear) → ใส่ค่าช่องเป้าหมาย
  • เสถียร ไม่ติด CFL → ใช้ Δt ใหญ่ → เร็วกว่า CTM
  • เชิงเส้นใน C → Ct+1 = 𝒜(u)·Ct = เมทริกซ์ ★★
  • 3 คุณสมบัติฟรี: เชิงเส้น→superposition(2.5) · w≥0→ไม่ติดลบ(2.4) · Σw=1→อนุรักษ์มวล(2.4)
  • ข้อจำกัด: numerical diffusion (เบลอ) = ส่วนของ ε_structure

แบบฝึกหัด 1 — semi-Lagrangian ถามคำถามต่างจาก finite difference อย่างไร? (ตอบเป็นคำถามที่แต่ละวิธีถาม)

เฉลย

finite difference (Eulerian): "ช่องคงที่นี้ มีควันไหลเข้า-ออกสุทธิเท่าไรใน 1 step?" (จับตาช่อง) · semi-Lagrangian: "ควันที่จะมาอยู่ช่องนี้ตอน t+1 เมื่อ step ก่อนมาจากที่ไหน?" (ย้อนรอยทวนลม) · การถามแบบย้อนรอยทำให้ไปถึงจุดต้นทางโดยตรงเสมอ → ไม่ระเบิดแม้ลมแรง

แบบฝึกหัด 2 — ทำไม "น้ำหนัก interpolation ≥ 0" ถึงสำคัญกับการเคลม "ควันไม่ติดลบ"?

เฉลย

ค่าควันใหม่ = ผลรวมถ่วงน้ำหนักของค่าควันเก่า · ถ้าค่าควันเก่าทุกตัว ≥ 0 (ซึ่งจริง — ความเข้มข้นเป็นลบไม่ได้) และน้ำหนักทุกตัว ≥ 0 → ผลรวมก็ต้อง ≥ 0 เสมอ · ดังนั้นควันออกมาไม่มีทางติดลบโดยโครงสร้าง = non-negativity (2.4) ที่เรา "พิสูจน์ได้" ไม่ใช่หวังให้นิวรอลเรียนถูก

แบบฝึกหัด 3 — ทำไมเราถึงฝัง 𝒜 (semi-Lagrangian) ตายตัว แทนที่จะให้นิวรอลเรียน advection เอง?

เฉลย

ถ้าให้นิวรอลเรียน advection เอง มันอาจ (1) ละเมิดความเชิงเส้น → superposition พัง → แยกแหล่งไม่ได้ (ทำลาย contribution C2), (2) ทำควันติดลบ, (3) ไม่อนุรักษ์มวล · การฝัง 𝒜 จากฟิสิกส์ทำให้คุณสมบัติพวกนี้เป็นจริงโดยโครงสร้าง พิสูจน์ได้ และให้นิวรอลโฟกัสเรียนแค่ส่วนที่ยากจริง (สัมประสิทธิ์ g, D) = "physics-structured" ที่เป็นจุดขายระดับ A*

จบ 2.2 แล้วคุณควร: เข้าใจว่า advection ทำด้วย semi-Lagrangian (ย้อนรอยทวนลม + interpolate), รู้ว่ามันเสถียร (ไม่ติด CFL) จึงเร็ว, และที่สำคัญสุด — เข้าใจว่ามันเป็นเชิงเส้นใน C → กลายเป็นเมทริกซ์ 𝒜(u) ที่ให้คุณสมบัติทองคำ 3 ข้อ (superposition / ไม่ติดลบ / อนุรักษ์มวล) ฟรี · นี่คือรากฐานของ "physics-structured" และเหตุผลที่เรา "พิสูจน์คุณสมบัติได้"

ขั้นต่อไปแนะนำ: 2.3 Operator splitting (Strang) — วิธี "แยกสมการเป็นส่วน ๆ แล้วแก้ทีละส่วน" ที่ทำให้เราเอา 𝒜 (ฟิสิกส์ บทนี้) มาต่อกับส่วนที่นิวรอลเรียน (g, D) ได้อย่างเป็นระบบ · หรือกระโดดไป 2.4 (mass conservation/non-negativity) ที่บทนี้เพิ่งเกริ่นคุณสมบัติไว้ · บอกเลขถัดไปได้เลยครับ
