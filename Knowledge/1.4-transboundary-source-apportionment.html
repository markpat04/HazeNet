1.4 — ควันข้ามพรมแดน & Source Apportionment

"ฝุ่นนี้มาจากไหน?" — คำถามที่ทั้งโปรเจกต์ของเราตั้งใจจะตอบ

ต่อจาก 1.3 (ลมพาควันข้ามแดน) · บทนี้คือ "หัวใจของ pain point" และ "จุดขายหลัก" (C2 attribution) · ผูกกับ HazeNet ตลอด

ทำไมบทนี้สำคัญมาก: สามบทก่อนสอนว่าฝุ่นคืออะไร (1.1), ฟิสิกส์การเคลื่อนของควัน (1.2), และอากาศที่ขับมัน (1.3) · บทนี้รวมทุกอย่างเข้าเป็นคำถามเชิงนโยบายจริง: "PM2.5 ที่เชียงใหม่วันนี้ กี่เปอร์เซ็นต์ มาจากในประเทศ vs ข้ามพรมแดน?" · ถ้าตอบไม่ได้ → แก้ปัญหาผิดที่ · Source Apportionment = ศาสตร์ของการตอบคำถามนี้ และ HazeNet ตั้งใจตอบมันด้วยวิธีใหม่

1. ควันข้ามพรมแดน (Transboundary Smoke) คืออะไร

ทุกฤดูแล้ง (กุมภาพันธ์–เมษายน) เกษตรกรในภูมิภาคลุ่มน้ำโขง — เมียนมา, ลาว, ไทยตอนบน, กัมพูชา — เผาเศษวัสดุการเกษตร (ข้าวโพดเลี้ยงสัตว์, ข้าวไร่) และมีไฟป่าจำนวนมาก · ควันที่เกิดขึ้น ไม่หยุดที่เส้นพรมแดน — ลมตะวันตก/ตะวันตกเฉียงใต้ (จาก 1.3) หอบมันข้ามภูเขาเข้าสู่ภาคเหนือของไทย

คำว่า "receptor" (ผู้รับ) คือจุดที่เราวัดฝุ่น เช่น เชียงใหม่ · คำว่า "source" (แหล่ง) คือจุดที่ควันเกิด เช่น ไร่ในเมียนมา · ปัญหาคือ ที่ receptor เราเห็นแค่ "ฝุ่นรวม" — มองไม่ออกว่าก้อนไหนมาจากแหล่งไหน

เชื่อม 3 บทก่อน: ไฟ (FRP จาก 1.1) = แหล่ง · ลม (จาก 1.3) = ตัวพา · PBL ต่ำ (จาก 1.3) = ตัวกักให้พุ่ง · ทั้งหมดอธิบายได้ด้วย advection ในสมการ ADR (จาก 1.2) · บทนี้แค่ตั้งคำถามว่า "แล้วเราจะ ย้อนกลับ ไปบอกสัดส่วนแหล่งได้ยังไง"

2. ทำไมมันเป็น pain point จริง ★

นี่ไม่ใช่ปัญหาวิชาการลอย ๆ — มันคือทางตันเชิงนโยบาย:

  • ถ้าฝุ่นส่วนใหญ่มาจากในประเทศ → มาตรการที่ถูกคือ ห้ามเผา/จัดการเชื้อเพลิงในไทย
  • ถ้าส่วนใหญ่ข้ามพรมแดนมา → ห้ามเผาในไทยอย่างเดียวไม่พอ ต้องเจรจาระหว่างประเทศ (เช่น ASEAN Haze Agreement)

การเลือกผิด = เสียทรัพยากร + โทษผิดคน + ฝุ่นไม่ลด · และที่แย่กว่านั้น: สัดส่วนเปลี่ยนไปทุกวัน ตามทิศลม — บางวันลมพัดจากเมียนมา บางวันนิ่งและฝุ่นเป็นของในพื้นที่เอง

ช่องว่างที่แท้จริง: เครื่องมือที่ให้คำตอบ "วันนี้กี่ %" แบบเร็วพอจะใช้ตัดสินใจ แทบไม่มี · งานวิจัยที่ตอบได้ (WRF-Chem tagged-tracer) ใช้เวลารันหลายชั่วโมง–วันต่อเหตุการณ์ และต้องผู้เชี่ยวชาญ · ผลจึงออกมาเป็นรายงาน "ย้อนหลังรายฤดู" ไม่ใช่ "พยากรณ์รายวัน" · HazeNet ตั้งใจอุดช่องนี้

3. Source Apportionment คืออะไร (แบ่งสัดส่วนแหล่งที่มา)

Source Apportionment (SA) = การแยกว่า "ฝุ่นรวม" ที่จุดวัด ประกอบด้วยสัดส่วนจากแต่ละแหล่งเท่าไร · เป้าหมายคือสมการง่าย ๆ:

Cรวม = C(เมียนมา) + C(ลาว) + C(ในประเทศ) + C(อื่น ๆ)

ถ้าวัด Cรวม = 120 µg/m³ แล้วแยกได้ว่า 54 มาจากเมียนมา (45%), 36 ในประเทศ (30%)… นั่นคือ source apportionment ที่สมบูรณ์

มีสองตระกูลวิธีใหญ่ — เข้าใจทั้งสองจะเห็นว่า HazeNet ยืนตรงไหน:

ตระกูลแนวคิดทิศทาง
Receptor model
(เช่น PMF)		เริ่มจาก "ฝุ่นที่วัดได้" แล้วใช้สถิติแยกว่ามีกี่ "ลายเซ็นเคมี"ปลายทาง → ต้นทาง
Source-oriented model
(เช่น WRF-Chem tagged tracer)		เริ่มจาก "ไฟแต่ละจุด" แล้วจำลองฟิสิกส์ว่าควันไปถึงไหนต้นทาง → ปลายทาง
HazeNet อยู่ตระกูลไหน: ตระกูล source-oriented (เหมือน WRF-Chem) — เราเริ่มจากไฟ ติดป้ายควัน แล้วจำลองการเคลื่อน · แต่เราทำด้วยนิวรอลที่เร็วและหาอนุพันธ์ได้ ซึ่งเปลี่ยนเกม (ดูข้อ 7)

4. วิธีที่มีอยู่ #1 — Receptor Model / PMF

PMF (Positive Matrix Factorization) = วิธีสถิติยอดนิยม · เก็บตัวอย่างฝุ่นจริงมาวิเคราะห์องค์ประกอบเคมี (เช่น โพแทสเซียม = ตัวบ่งชี้การเผาชีวมวล, วาเนเดียม = น้ำมันเตา) แล้วใช้คณิตศาสตร์แยกว่ามีกี่ "แหล่ง" ที่มีลายเซ็นเคมีต่างกัน

อุปมา: เหมือนชิมแกงแล้วบอกว่า "มีกะทิ มีพริก มีปลาร้า" จากรสชาติ — โดยไม่ต้องเห็นตอนทำ · PMF "ชิม" องค์ประกอบเคมีของฝุ่นแล้วเดาแหล่ง

ข้อดี / ข้อจำกัด

  • ✅ อิงข้อมูลวัดจริง น่าเชื่อถือเชิงเคมี
  • ❌ บอกได้แค่ "ประเภทแหล่ง" (เช่น "การเผาชีวมวล") ไม่บอกพิกัดประเทศ — เผาในไทยกับเผาในเมียนมามีลายเซ็นเคมีเหมือนกัน!
  • ❌ ต้องเก็บตัวอย่าง+วิเคราะห์ในแล็บ → ช้า รายเดือน/รายฤดู ไม่ใช่รายวัน
จุดอ่อนสำคัญต่อปัญหาเรา: PMF แยก "ชีวมวล vs น้ำมัน" ได้ แต่แยก "ชีวมวลเมียนมา vs ชีวมวลไทย" ไม่ได้ เพราะเคมีเหมือนกัน · ส่วน "ข้ามพรมแดนกี่ %" เป็นเรื่องภูมิศาสตร์+ลม ซึ่งต้องใช้โมเดลฟิสิกส์ (ข้อ 5) → นี่คือเหตุผลที่เราเลือกตระกูล source-oriented

5. วิธีที่มีอยู่ #2 — Tagged Tracer (WRF-Chem / CMAQ-ISAM)

นี่คือ "มาตรฐานทอง" ของการแยกแหล่งเชิงภูมิศาสตร์ · แนวคิดคือ "ติดป้ายสีให้ควัน":

  1. แบ่งพื้นที่ต้นทางเป็นโซน (เช่น เมียนมา=แดง, ลาว=น้ำเงิน, ไทย=เขียว)
  2. ปล่อยควันออกจากไฟ พร้อม "ติดป้ายสี" ตามโซนต้นทาง
  3. รันโมเดลฟิสิกส์ (advection+diffusion) ให้ควันแต่ละสีเคลื่อนไปพร้อมกัน
  4. ที่เชียงใหม่ นับว่ามีควันสีไหนกี่ % → นั่นคือ source apportionment

ข้อดี / ข้อจำกัด

  • ✅ แยกเชิงภูมิศาสตร์ได้จริง (ตอบ "กี่ % จากเมียนมา" ได้)
  • ✅ อิงฟิสิกส์ → น่าเชื่อถือ เป็น ground truth ให้เรา validate (จาก research plan: WRF-Chem ให้เชียงราย 45%)
  • ❌ ช้ามาก — รันหนึ่งเหตุการณ์ใช้ชั่วโมง–วัน บนคลัสเตอร์ใหญ่
  • ❌ ต้องเพิ่ม "ป้ายสี" ทุกโซนที่อยากแยก → ยิ่งหลายแหล่งยิ่งช้าทวีคูณ
  • ❌ หาอนุพันธ์ไม่ได้ง่าย → ตอบ "ถ้าลดไฟเมียนมา 50% ฝุ่นจะลดเท่าไร" (counterfactual) ต้องรันใหม่ทั้งดุ้น

ซื่อสัตย์: ตัวเลขเหล่านี้เป็นรายฤดู/เฉลี่ย ไม่ใช่รายวัน และความละเอียดต่างจากเรา (20 กม. vs 0.1°) และคนละปี · ดังนั้นใช้ validate "ระดับสัดส่วนโดยรวม" ได้ แต่ "รายวันเป๊ะ ๆ" ยังต้องระวัง (รายละเอียดข้อ 9)

7. HazeNet ทำต่างยังไง — Contribution C2 ★★ จุดขายหลัก

HazeNet เลือกตระกูล source-oriented (เหมือน tagged tracer) แต่แก้จุดอ่อน 3 ข้อของ WRF-Chem พร้อมกัน:

มิติWRF-Chem taggedHazeNet (เรา)
ความเร็วชั่วโมง–วัน/เหตุการณ์วินาที (นิวรอลรันบน GPU)
แยกแหล่งต้องเพิ่มป้ายสี รันใหม่ติดป้ายในตัว (tagged tracer) ออกมาในการรันเดียว
Counterfactual / inversionรันใหม่ทั้งดุ้นหาอนุพันธ์ได้ → "adjoint ฟรี" (จาก 0.2)
พยากรณ์ + แยกแหล่งคนละระบบโมเดลเดียวทำทั้งคู่

หัวใจคือ: เราเอา "การติดป้ายสี" ของ tagged tracer มาใส่ในนิวรอลที่ออกแบบให้เป็นเชิงเส้นใน C (conditionally-linear operator) · เพราะมันเชิงเส้น → ควันแต่ละสีบวกกันได้ (superposition) → แยกแหล่งได้เป๊ะตามนิยาม ไม่ใช่ประมาณ

ชื่อเล่นที่เราใช้: "Smoke Fingerprint" — ลายนิ้วมือควัน · โมเดลบอกได้ว่าฝุ่นที่เชียงใหม่ตอนนี้ "ลายนิ้วมือ" ชี้ไปแหล่งไหนกี่ % แบบเรียลไทม์
เชื่อม 0.2 (autograd): จำ "adjoint ฟรี" ได้ไหม? เพราะ HazeNet หาอนุพันธ์ได้ → ∂(ฝุ่นเชียงใหม่)/∂(ไฟเมียนมา) คำนวณได้ทันทีด้วย .backward() · นี่คือสิ่งที่ WRF-Chem ทำไม่ได้ และเป็นเหตุผลที่ counterfactual ("ลดไฟ 50% → ฝุ่นลดเท่าไร") กับ source inversion ทำได้ในโมเดลเดียว

8. ทำไม "superposition" ทำให้แยกแหล่งได้ (สะพานสู่ heart 2/3)

นี่คือเหตุผลคณิตศาสตร์ที่ทุกอย่างทำงาน — และเป็นหัวข้อ 2.5 (หัวใจที่ 2) ที่จะสอนลึกต่อไป · ปูพื้นไว้ตรงนี้:

จาก 1.2 เราเห็นว่า 4 พจน์แรกของ ADR (advection, diffusion, emission, loss) เป็นเชิงเส้นใน C · คุณสมบัติเชิงเส้นแปลว่า:

ถ้า E = E(พม่า) + E(ลาว)  แล้ว  C = C(พม่า) + C(ลาว)
รันควันแต่ละแหล่งแยกกัน แล้วผลรวม = รันรวมทีเดียว — เป๊ะ ไม่ใช่ประมาณ

= เราปล่อยควัน "สีพม่า" กับ "สีลาว" พร้อมกันในการรันเดียว แล้วผลลัพธ์ของแต่ละสีบวกกันได้พอดีเป็นฝุ่นรวม · ดังนั้นที่ปลายทางเราแค่ดูว่าแต่ละสีมีเท่าไร = source apportionment ฟรีในตัว

นี่คือเหตุผลที่เราปฏิเสธ diffusion downscaling: ฟังก์ชัน diffusion ไม่เชิงเส้น → Diffusion(Cพม่า+Cลาว) ≠ Diffusion(Cพม่า)+Diffusion(Cลาว) → ทำลาย superposition → แยกแหล่งไม่ได้ · การรักษาความเชิงเส้นคือสิ่งศักดิ์สิทธิ์ของโปรเจกต์นี้

9. caveat & การ validate (ความซื่อสัตย์เชิงวิทยาศาสตร์)

  • PM2.5 ไม่อนุรักษ์ 100% — มี Rchem (จาก 1.2) ทำให้เกิดฝุ่นทุติยภูมิ ที่ superposition ไม่ครอบคลุม → เราโฟกัส "ควันปฐมภูมิ (primary smoke)" และระบุข้อจำกัดนี้ชัดเจนในเปเปอร์
  • Ground truth เป็นรายฤดู — เทียบสัดส่วนรวมได้ แต่ counterfactual รายเหตุการณ์ต้องหา "natural experiment" (เช่น ช่วงฝนตกดับไฟ → ดูฝุ่นลดจริงไหม, เชื่อม 1.3) 2–3 เคสมายืนยัน
  • โซนแหล่งหยาบ — เราแบ่งเป็นประเทศ/ภูมิภาค ไม่ใช่รายแปลง → ตอบ "เมียนมากี่ %" ได้ แต่ "อำเภอไหนในเมียนมา" ไม่ได้
ทำไม caveat คือจุดแข็ง ไม่ใช่จุดอ่อน: งานระดับ A* ต่างจากงานทั่วไปตรงที่รู้ขอบเขตตัวเอง · การบอกชัดว่า "เราแยก primary smoke เชิงภูมิศาสตร์ได้เป๊ะ แต่ secondary ยังเป็นข้อจำกัด" ทำให้ reviewer เชื่อถือมากกว่าการอ้างเกินจริง

10. Cheat sheet + แบบฝึกหัด

สรุปจำ:
  • Transboundary smoke = ควันข้ามพรมแดน (พม่า/ลาว → เหนือไทย) พาด้วยลม
  • pain point = "กี่ % ข้ามแดน vs ในประเทศ?" ตอบไม่ได้ → แก้ผิดที่ + เปลี่ยนทุกวันตามลม
  • Source Apportionment = แยก Cรวม = ΣC(แหล่ง)
  • PMF (receptor): ชิมเคมี → บอกประเภท · แต่แยกเชิงภูมิศาสตร์ไม่ได้
  • Tagged tracer (WRF-Chem): ติดป้ายสี → แยกเชิงภูมิศาสตร์ได้ · แต่ช้า + หาอนุพันธ์ไม่ได้
  • HazeNet (C2): tagged tracer แบบนิวรอล → เร็ว + แยกในตัว + adjoint ฟรี
  • กุญแจคณิต: เชิงเส้นใน C → superposition → แยกแหล่งเป๊ะ (= heart 2.5)
  • ground truth ไทย: เชียงราย ~45% ข้ามแดน, เฉลี่ย ~30% (ใช้ validate C2)

แบบฝึกหัด 1 — PMF บอกว่า "ฝุ่นเชียงใหม่ 51% มาจากการเผาชีวมวล" · ข้อมูลนี้ตอบคำถามนโยบาย "ควรห้ามเผาในไทย หรือต้องเจรจาเมียนมา" ได้ไหม? เพราะอะไร?

เฉลย

ตอบไม่ได้ · PMF บอกแค่ "ประเภทแหล่ง" = การเผาชีวมวล แต่แยกไม่ออกว่าเผาในไทยหรือในเมียนมา เพราะลายเซ็นเคมีเหมือนกัน · คำถามนโยบายต้องการการแยกเชิงภูมิศาสตร์ ซึ่งต้องใช้โมเดลฟิสิกส์ (tagged tracer / HazeNet) ไม่ใช่ receptor model

แบบฝึกหัด 2 — ทำไม HazeNet ต้อง "เชิงเส้นใน C" ถึงจะแยกแหล่งได้เป๊ะ? (ตอบด้วยคำว่า superposition)

เฉลย

เพราะเชิงเส้น → superposition: ถ้าปล่อยควันจากหลายแหล่งพร้อมกัน ผลรวม = ผลของแต่ละแหล่งบวกกันพอดี · ดังนั้นเราติดป้ายควันแต่ละแหล่ง รันครั้งเดียว แล้วผลของแต่ละสีคือสัดส่วนแหล่งนั้นเป๊ะตามนิยาม (ไม่ใช่ประมาณ) · ถ้าโมเดลไม่เชิงเส้น (เช่นใส่ diffusion) → ผลรวมเพี้ยน → แยกไม่ได้

แบบฝึกหัด 3 — WRF-Chem แยกแหล่งได้แม่นเหมือนกัน แล้วทำไมเรายังต้องสร้าง HazeNet?

เฉลย

3 เหตุผล: (1) เร็ว — WRF-Chem ใช้ชั่วโมง–วัน/เหตุการณ์ ใช้พยากรณ์รายวันไม่ทัน; HazeNet วินาที · (2) หาอนุพันธ์ได้ — counterfactual ("ลดไฟ 50%→?") และ source inversion ทำได้ด้วย .backward() โดยไม่รันใหม่ · (3) โมเดลเดียวทำทั้งพยากรณ์+แยกแหล่ง+counterfactual · WRF-Chem เป็น ground truth ที่ดี แต่ใช้จริงรายวันไม่ได้

จบ 1.4 แล้วคุณควร: เข้าใจว่า "ควันข้ามพรมแดน" คือ pain point จริง, รู้สองตระกูลวิธีแยกแหล่ง (PMF receptor vs tagged tracer source-oriented), เห็นว่า HazeNet = tagged tracer แบบนิวรอลที่เร็ว+แยกในตัว+adjoint ฟรี, และเข้าใจว่า "ความเชิงเส้น → superposition" คือกุญแจที่ทำให้แยกแหล่งได้เป๊ะ
