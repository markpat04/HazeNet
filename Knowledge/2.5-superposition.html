2.5 — Superposition (หลักการบวกซ้อน) ⭐ CORE ❤ หัวใจ 2/3

ทำไม C = Σ C⁽ˢ⁾ ถึงเป็นจริง — กุญแจที่ทำให้ attribution + counterfactual เป็นไปได้

ไคลแมกซ์ของ Module 2 · ทุกบท (2.1–2.4) ปูทางมาสู่บทนี้ · นี่คือเหตุผลทางคณิตศาสตร์ที่ HazeNet "แยกแหล่งได้เป๊ะ"

ทำไมบทนี้คือ "หัวใจที่ 2": ทั้งโปรเจกต์ขายของอยู่ 3 อย่าง — พยากรณ์, แยกแหล่ง (attribution, C2), และ counterfactual · สองอย่างหลังเป็นไปไม่ได้เลยถ้าไม่มี superposition · superposition คือหลักการว่า "ถ้าระบบเป็นเชิงเส้น ผลของหลายสาเหตุรวมกัน = ผลของแต่ละสาเหตุบวกกัน" · เพราะ Module 2 ทั้งหมดออกแบบให้สมการเชิงเส้นใน C (2.2 ทำ 𝒜 เชิงเส้น, 2.3 splitting คงเชิงเส้น, 2.4 บังคับฟิสิกส์โดยไม่ทำลายเชิงเส้น) → superposition จึงใช้ได้ → C = Σ C⁽ˢ⁾ เป็นจริง · บทนี้คือที่ที่ทุกอย่างมาบรรจบ

1. "เชิงเส้น (linear)" คืออะไร — รากฐาน ★

ระบบ f เป็น "เชิงเส้น" ถ้ามันมี 2 คุณสมบัติ:

① f(a + b) = f(a) + f(b)   (บวกก่อนหรือหลังก็เท่ากัน)

② f(k·a) = k·f(a)   (คูณค่าคงที่ก่อนหรือหลังก็เท่ากัน)

ตัวอย่างเชิงเส้น: "คูณ 3" → f(x)=3x · f(2+5)=3(7)=21 = f(2)+f(5)=6+15=21 ✓ · ตัวอย่างไม่เชิงเส้น: "ยกกำลังสอง" → f(x)=x² · f(2+5)=49 ≠ f(2)+f(5)=4+25=29 ✗

หัวใจคือ: ระบบเชิงเส้น "แยกส่วน" ได้ — คุณแยกอินพุตเป็นชิ้น ๆ ประมวลแต่ละชิ้น แล้วเอาผลมาบวกกัน ได้คำตอบเดียวกับประมวลทั้งก้อน · ระบบไม่เชิงเส้นทำแบบนี้ไม่ได้ (ชิ้นส่วนมัน "กวน" กันเอง)

2. Superposition คืออะไร ★★

Superposition principle = ผลโดยตรงของความเชิงเส้น · ถ้าระบบเชิงเส้น และมีหลาย "สาเหตุ" (แหล่งควัน) ป้อนเข้าพร้อมกัน → ผลรวม = ผลของแต่ละสาเหตุบวกกัน

ถ้า E = E⁽¹⁾ + E⁽²⁾ + …   แล้ว   C = C⁽¹⁾ + C⁽²⁾ + … = Σ C⁽ˢ⁾

"รันควันแต่ละแหล่งแยกกันแล้วบวก" ให้ผลเท่ากันเป๊ะกับ "รันทุกแหล่งรวมทีเดียว" · นี่คือ superposition · และมันจริงก็ต่อเมื่อระบบเชิงเส้น

3. พิสูจน์ง่าย ๆ ว่าทำไมมันจริงในสมการเรา

จำสมการแกน (เขียนรวมพจน์ที่เชิงเส้นใน C เป็น operator ℒ):

Ct+1 = 𝒜·Ct + g·E − D⊙Ct  =  ℒ(Ct) + g·E

โดย ℒ(C) = 𝒜C − D⊙C เป็นเชิงเส้นใน C (จาก 2.2, 2.4) · ทีนี้ปล่อย 2 แหล่ง: ให้ C⁽¹⁾ แก้สมการที่มีแค่ E⁽¹⁾ และ C⁽²⁾ แก้ที่มีแค่ E⁽²⁾ · บวกกัน:

C⁽¹⁾ᵗ⁺¹ = ℒ(C⁽¹⁾ᵗ) + g·E⁽¹⁾
C⁽²⁾ᵗ⁺¹ = ℒ(C⁽²⁾ᵗ) + g·E⁽²⁾
────────────────────────── บวกกัน
C⁽¹⁾ᵗ⁺¹+C⁽²⁾ᵗ⁺¹ = ℒ(C⁽¹⁾ᵗ)+ℒ(C⁽²⁾ᵗ) + g·(E⁽¹⁾+E⁽²⁾)
              # ℒ เชิงเส้น => ℒ(a)+ℒ(b)=ℒ(a+b)
              = ℒ(C⁽¹⁾ᵗ+C⁽²⁾ᵗ) + g·E_รวม

บรรทัดสุดท้ายคือสมการเดียวกันเป๊ะกับการรัน C รวมที่มี E รวม · ดังนั้น C⁽¹⁾+C⁽²⁾ = C_รวม · จบ — superposition เป็นจริงเพราะ ℒ เชิงเส้น (ขั้น "ℒ(a)+ℒ(b)=ℒ(a+b)") · ถ้า ℒ ไม่เชิงเส้น ขั้นนี้พังทันที

นี่คือเหตุผลที่ Module 2 ทุกบทย้ำ "เชิงเส้น": ทุกการตัดสินใจ (semi-Lagrangian แทน CNN เรียน advection, g/D ทำนายจากอากาศไม่ใช่จาก C, hard constraint แทน soft) ล้วนเพื่อรักษาความเชิงเส้นใน C · เพราะมันคือเงื่อนไขเดียวที่ทำให้บรรทัดพิสูจน์ข้างบนผ่าน → ทำให้ attribution เป็นจริง

4. แปลงเป็น "ติดป้ายควัน" (tagged tracer) ★★

superposition แปลว่าเราติดตามควันแต่ละแหล่งเป็น "ช่อง" แยกได้ พร้อมกันในการรันเดียว · เพิ่มมิติ tracer ให้ tensor (จาก 0.1):

C.shape = (source, time, lat, lon)
#          ↑ มิติแหล่ง: 0=พม่า, 1=ลาว, 2=ไทย, ...

E.shape = (source, time, lat, lon)
# E[0] = ไฟเฉพาะในพม่า, E[1] = ไฟเฉพาะลาว, ...

เดินสมการแกน "พร้อมกันทุก source" (operator เดียวกัน 𝒜, g, D ใช้กับทุก tracer เพราะอากาศชุดเดียวกัน):

for t in rollout:
    C = advect(C, wind)        # 𝒜 ทำกับทุก source พร้อมกัน (broadcast)
    C = C + g * E[:, t]        # แต่ละ source เติมไฟของตัวเอง
    C = C * (1 - D)            # loss เดียวกันทุก source

C_total = C.sum(0)            # รวมทุก source = PM2.5 รวม (superposition)
กุญแจที่สวยงาม: เพราะ superposition C.sum(0) (รวมทุกแหล่ง) = PM2.5 ที่พยากรณ์รวม เป๊ะ · เราจึงเทรนโมเดลให้ C.sum(0) ตรงกับสถานีจริง แล้วได้การแยกแหล่ง C[s] มาฟรีโดยไม่ต้องมี label แยกแหล่ง! (ซึ่งเราไม่มีอยู่แล้ว — จาก 1.4)

5. ทำไมนี่ = attribution ฟรี (Contribution C2)

หลังรันเสร็จ เรามี C[s] = ควันจากแหล่ง s ที่ทุกจุด ทุกเวลา · ถามอะไรก็ตอบได้ทันที:

# ฝุ่นที่เชียงใหม่ตอนนี้ มาจากแต่ละแหล่งกี่ %
share = C[:, t, cm_lat, cm_lon] / C[:, t, cm_lat, cm_lon].sum()
# share = [0.45, 0.30, 0.20, 0.05]  -> พม่า45% ลาว30% ไทย20% อื่น5%
เทียบ ground truth (จาก 1.4/1.5): เราเอา share นี้ไปเทียบกับ WRF-Chem tagged tracer (เชียงราย ~45% ข้ามแดน) · ถ้าใกล้ = พิสูจน์ว่า attribution ของเราตรงฟิสิกส์มาตรฐานทอง ไม่ใช่เลขมั่ว = contribution C2 มีของจริง · และเราทำได้ในวินาที ขณะที่ WRF-Chem ใช้ชั่วโมง (จุดขายความเร็ว)
ทำไมงานอื่นทำไม่ได้: โมเดล AI ทั่วไป (PM2.5-GNN, ConvLSTM) พยากรณ์ "ฝุ่นรวม" เป็นเลขก้อนเดียว — แยกแหล่งไม่ได้เพราะไม่เชิงเส้น/ไม่มีโครงสร้าง tracer · HazeNet แยกได้เพราะ superposition ที่ build จากความเชิงเส้น · นี่คือสิ่งที่ทำให้งานเราเป็นงานใหม่ ไม่ใช่ "อีกโมเดลพยากรณ์ฝุ่น"

6. ทำไม = counterfactual ได้

Counterfactual = "ถ้า...จะเกิดอะไร" · superposition ทำให้ตอบได้ทันทีโดยไม่ต้องรันใหม่:

  • "ถ้าไม่มีไฟพม่าเลย ฝุ่นเชียงใหม่จะเป็นเท่าไร?" → คำตอบ = C_total − C[พม่า] (ลบส่วนพม่าออก) · ตรง ๆ เพราะแต่ละแหล่งบวกกันเชิงเส้น
  • "ถ้าลดไฟพม่าลง 50%?" → C_total − 0.5·C[พม่า] · เพราะ f(0.5·E) = 0.5·f(E) (คุณสมบัติเชิงเส้นข้อ ②)
นี่ทรงพลังมาก: WRF-Chem ตอบคำถามนี้ต้อง "รันใหม่ทั้งดุ้น" (อีกหลายชั่วโมง) · HazeNet ตอบด้วยการลบ/คูณ tensor ที่มีอยู่แล้ว = ทันที · และเพราะโมเดล differentiable (0.2) เรายังหา "ลดไฟตรงไหนช่วยเชียงใหม่ที่สุด" ด้วย gradient ได้ (= source inversion, adjoint ฟรี)
ความซื่อสัตย์ระดับ counterfactual: การลบ tracer ใช้ได้กับ primary smoke เชิงเส้น · ถ้าไฟลดมากจนเคมีทุติยภูมิเปลี่ยน behavior แบบไม่เชิงเส้น การประมาณนี้จะคลาดเคลื่อน → เราจึง validate counterfactual กับ "natural experiment" (เช่นฝนดับไฟจริง, จาก 1.3) 2–3 เคส เพื่อยืนยันขอบเขตที่เชื่อถือได้

7. เส้นตายของ superposition — สิ่งที่ห้ามทำ

superposition เปราะบาง · operation ไม่เชิงเส้นแม้แต่จุดเดียวในเส้นทาง C ทำลายมันทั้งหมด · ตัวอย่างสิ่งที่ห้ามใส่ลงบน C โดยตรง:

operationเชิงเส้น?ผลต่อ superposition
𝒜·C, D⊙C, +E (สมการแกน)✅ปลอดภัย
C² , √C , log C❌พัง
ReLU/sigmoid บน C❌พัง
Diffusion downscaling บน C❌พัง (เหตุผลที่เราปฏิเสธ!)
sigmoid/softplus บน g,D (สัมประสิทธิ์)—✅ ปลอดภัย (ไม่ใช่ฟังก์ชันของ C)
กฎทอง: ฟังก์ชันไม่เชิงเส้นทำได้เฉพาะกับสัมประสิทธิ์ (g, D ที่มาจากอากาศ) ห้ามทำกับตัว C เอง · นี่คือนิยามเป๊ะ ๆ ของ "conditionally-linear" (3.5): ไม่เชิงเส้นในอากาศได้เต็มที่ แต่ต้องเชิงเส้นใน C · จำกฎนี้ไว้ = จำหัวใจการออกแบบทั้งหมด

8. โค้ด tagged tracer (เต็มขึ้นนิด)

def rollout_tagged(C0, E, wind, met, static, steps):
    """C0,E shape = (S, lat, lon) ; S = จำนวนแหล่ง"""
    C = C0
    history = []
    for t in range(steps):
        g = F.softplus(afno_emit(met[t], static))   # สัมประสิทธิ์ (ไม่ขึ้นกับ C)
        D = torch.sigmoid(afno_loss(met[t], static))
        # --- ทุกบรรทัดเชิงเส้นใน C, broadcast ข้ามมิติ source ---
        C = advect(C, wind[t])         # 𝒜 เดียวกันทุก source
        C = C + g * E[t]               # แต่ละ source เติมไฟตัวเอง
        C = C * (1 - D)
        history.append(C)
    C = torch.stack(history)           # (steps, S, lat, lon)
    C_total = C.sum(1)               # รวมแหล่ง = พยากรณ์รวม (เทรนกับสถานีจริง)
    return C_total, C                  # รวม + แยกแหล่ง (ฟรี!)
เชื่อม 1.5 (ทำไมเร็วกว่า WRF-Chem): มิติ source เป็นแค่ batch ของ tensor บน GPU → เพิ่มแหล่งจาก 4 เป็น 10 แทบไม่เพิ่มเวลา (ขนานกันหมด) · ใน WRF-Chem การเพิ่ม tracer = เพิ่มงานคำนวณเชิงตัวเลขทวีคูณ · นี่คือข้อได้เปรียบที่จับต้องได้จาก superposition + GPU

9. caveat — R_chem ทำลาย superposition (ซื่อสัตย์)

ย้ำจาก 1.2/2.4: พจน์เคมี Rchem (ฝุ่นทุติยภูมิ) ไม่เชิงเส้น — มันขึ้นกับการทำปฏิกิริยาระหว่างสารหลายตัว · ถ้าใส่ R_chem แบบเต็มลงบน C โดยตรง → superposition พัง → attribution ผิด

ทางเลือกที่เราใช้ (ซื่อสัตย์และทำได้ใน 2 เดือน):

  • โฟกัส primary smoke: หมอกควันชีวมวลภาคเหนือส่วนใหญ่เป็น primary (PMF ~51% ชีวมวล, 1.4) → superposition ใช้ได้ดีกับส่วนหลัก
  • secondary เป็นพจน์แยก: ใส่เป็น residual/correction ที่ทำ "หลัง" การแยกแหล่ง หรือระบุเป็นข้อจำกัดในเปเปอร์ — ไม่ปนกับ tracer ที่ต้องเชิงเส้น
  • เคลมตามจริง: "attribution เป๊ะสำหรับ primary transport" ไม่ใช่ "PM2.5 ทุกโมเลกุล"
นี่ทำให้งานแข็งแรงขึ้น ไม่ใช่อ่อนลง: reviewer A* เคารพงานที่รู้ว่าหลักการตัวเองใช้ได้ถึงไหน · การบอกชัดว่า "superposition การันตี primary, secondary เป็น residual" ทำให้ contribution C2 น่าเชื่อถือกว่าการอ้างคลุมว่าแยกได้ทุกอย่าง

10. Cheat sheet + แบบฝึกหัด

สรุปจำ:
  • เชิงเส้น: f(a+b)=f(a)+f(b) และ f(k·a)=k·f(a) → "แยกส่วนได้"
  • Superposition: ระบบเชิงเส้น → ผลรวม = Σ ผลของแต่ละสาเหตุ → C = Σ C⁽ˢ⁾
  • พิสูจน์: ขึ้นกับขั้น ℒ(a)+ℒ(b)=ℒ(a+b) ซึ่งจริงเพราะ ℒ เชิงเส้น (ผลของ Module 2)
  • tagged tracer: เพิ่มมิติ source ใน tensor → รันทุกแหล่งพร้อมกัน → C.sum(0)=รวม
  • ★ attribution ฟรี: เทรนให้ผลรวมตรงสถานีจริง → ได้ C[s] แยกแหล่งฟรี (ไม่ต้องมี label แยก)
  • counterfactual: "ไม่มีพม่า"=C_total−C[พม่า] · "ลด 50%"=C_total−0.5C[พม่า] · ทันที ไม่รันใหม่
  • กฎทอง: ฟังก์ชันไม่เชิงเส้นทำได้เฉพาะกับ สัมประสิทธิ์ g,D ห้ามทำกับ C = "conditionally-linear" (3.5)
  • ทำไมปฏิเสธ diffusion downscaling: มันไม่เชิงเส้นบน C → พัง superposition
  • caveat: R_chem (secondary) ไม่เชิงเส้น → โฟกัส primary, secondary เป็นพจน์แยก

เฉลย

เพราะ superposition การันตีว่า C.sum(0) (รวมทุก tracer) = PM2.5 รวมเป๊ะ · เราจึงเทรนแค่ให้ผลรวมตรงกับสถานีจริง (ซึ่งเรามี — Air4Thai) · เมื่อผลรวมถูก และโครงสร้าง tracer ถูกฟิสิกส์ (แต่ละ tracer เริ่มจากไฟของแหล่งตัวเอง + ขนส่งเชิงเส้น) → การแยก C[s] แต่ละแหล่งโผล่มาเองอย่างถูกต้อง โดยไม่ต้องมี label ว่า "ฝุ่นก้อนนี้มาจากพม่า" (ซึ่งวัดตรงไม่ได้)

ReLU เป็นฟังก์ชันไม่เชิงเส้นบน C → ReLU(C⁽¹⁾+C⁽²⁾) ≠ ReLU(C⁽¹⁾)+ReLU(C⁽²⁾) → ทำลาย superposition → แยกแหล่งผิด · แทนที่จะกันติดลบด้วย ReLU บน C เราออกแบบให้ไม่มีทางติดลบตั้งแต่แรก (2.4): 𝒜 น้ำหนัก≥0, g=softplus≥0, loss=C·(1−D) กับ D=sigmoid · บังคับฟิสิกส์ผ่านสัมประสิทธิ์ ไม่ใช่ผ่านฟังก์ชันบน C → ทั้งไม่ติดลบและคงเชิงเส้น

แบบฝึกหัด 3 — อธิบายด้วย superposition ว่าทำไม "ถ้าลดไฟพม่า 50% ฝุ่นเชียงใหม่จะเป็นเท่าไร" ตอบได้ทันทีโดยไม่รันโมเดลใหม่

เฉลย

เพราะระบบเชิงเส้น คุณสมบัติ ② บอกว่า f(0.5·E_พม่า) = 0.5·f(E_พม่า) = 0.5·C[พม่า] · และ superposition บอกว่าฝุ่นรวม = ผลแต่ละแหล่งบวกกัน · ดังนั้นฝุ่นใหม่ = C_total − C[พม่า] + 0.5·C[พม่า] = C_total − 0.5·C[พม่า] · เราแค่ลบครึ่งหนึ่งของ tracer พม่าที่คำนวณไว้แล้วออก = ดำเนินการ tensor ทันที ไม่ต้องรัน rollout ใหม่ (ต่างจาก WRF-Chem ที่ต้องรันใหม่ทั้งดุ้น)

🎉 จบ 2.5 = จบหัวใจที่ 2/3 + จบ Module 2! คุณเข้าใจกุญแจคณิตของทั้งโปรเจกต์แล้ว: เพราะ Module 2 ออกแบบให้สมการเชิงเส้นใน C → superposition เป็นจริง → C = Σ C⁽ˢ⁾ → ติดป้ายควัน (tagged tracer) → ได้ attribution ฟรี (C2) + counterfactual ทันที · และคุณรู้กฎทอง: ฟังก์ชันไม่เชิงเส้นทำได้เฉพาะกับสัมประสิทธิ์ ห้ามทำกับ C
