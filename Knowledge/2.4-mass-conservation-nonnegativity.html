2.4 — Mass Conservation · Non-negativity · Monotone ⭐ CORE

"มวลไม่รั่ว / ควันไม่ติดลบ / ไม่สร้างยอดปลอม" — คุณสมบัติที่เราเคลมว่า "พิสูจน์ได้"

ต่อจาก 2.2–2.3 (เกริ่นคุณสมบัติไว้) · บทนี้รวบ 3 คุณสมบัติให้ชัดว่าคืออะไร ทำไมต้องมี และได้มายังไง · ผูกกับ HazeNet ตลอด

ทำไมบทนี้เป็น CORE: จุดขายใหญ่ของ HazeNet คือคำว่า "physics-structured" — เราไม่ได้แค่หวังว่านิวรอลจะเรียนฟิสิกส์ถูก แต่บังคับให้ฟิสิกส์เป็นจริงโดยโครงสร้าง · 3 คุณสมบัติในบทนี้คือสิ่งที่เรา "พิสูจน์ได้" ในเปเปอร์ A*: (1) มวลไม่รั่ว (mass conservation), (2) ควันไม่ติดลบ (non-negativity), (3) ไม่สร้างยอด/หลุมปลอม (monotone) · นิวรอลทั่วไป (เช่น CNN/GNN ดิบ ๆ) ละเมิดทั้งสามได้ง่าย — การการันตีมันคือสิ่งที่ทำให้งานเราต่างและน่าเชื่อถือ

1. ทำไมต้องสนใจคุณสมบัติพวกนี้

ลองนึกว่าโมเดลพยากรณ์ฝุ่นออกมาแบบนี้: ช่องหนึ่งมี PM2.5 = −15 µg/m³ (ติดลบ!) · อีกช่องควันรวมทั้งโดเมน "เพิ่มขึ้นเอง" ทั้งที่ไม่มีไฟใหม่ · ผลพยากรณ์แบบนี้ผิดฟิสิกส์ชัด ๆ และทำลายความเชื่อถือทันที

คุณสมบัติ 3 ข้อนี้คือ "กฎธรรมชาติ" ที่ผลลัพธ์ต้องเคารพ · ถ้าโมเดลละเมิด = ผิดแน่นอน · ถ้าการันตีได้ = ตัดความผิดประเภทนี้ทิ้งทั้งหมดตั้งแต่ออกแบบ

ทำไมสำคัญต่อ A*: reviewer จะถามว่า "โมเดล AI ของคุณเคารพฟิสิกส์ไหม หรือแค่ fit ข้อมูล?" · ถ้าตอบได้ว่า "เราพิสูจน์ได้ว่ามวลไม่รั่ว ควันไม่ติดลบ โดยโครงสร้าง" = แต้มต่อมหาศาล · นี่คือความต่างจาก "นิวรอลกล่องดำ" ทั่วไป

2. Mass Conservation — มวลไม่รั่ว ★

กฎ: ควันไม่หายไปเฉย ๆ และไม่โผล่มาเอง · มวลควันรวมทั้งโดเมนจะเปลี่ยนได้เฉพาะจาก: (+) ปล่อยใหม่จากไฟ, (−) ตกหาย/ฝนชะ, (±) ไหลเข้า-ออกขอบโดเมน · ส่วน "การเลื่อนตามลม (advection)" ห้ามเปลี่ยนมวลรวม — แค่ย้ายที่

HazeNet ได้มาจากไหน: semi-Lagrangian (2.2) ที่ออกแบบให้ "น้ำหนัก interpolation รวม = 1" → การเลื่อนไม่สร้าง/ทำลายมวล · เป็นคุณสมบัติของ 𝒜 โดยตรง · (หมายเหตุ: semi-Lagrangian พื้นฐานอนุรักษ์มวลโดยประมาณดีมาก; ถ้าต้องการเป๊ะ 100% มีเวอร์ชัน "conservative semi-Lagrangian" ให้เลือก — เราจะระบุว่าใช้แบบไหนในเปเปอร์)

3. Non-negativity — ความเข้มข้นไม่ติดลบ ★

กฎที่ชัดที่สุด: ความเข้มข้นฝุ่นเป็นลบไม่ได้ · มี PM2.5 = 0 (ไม่มีฝุ่น) ได้ แต่ −5 µg/m³ ไม่มีความหมายในโลกจริง · โมเดลที่พ่นค่าติดลบ = ผิดทันที

HazeNet ได้มาจากไหน (2 ชั้น): (1) advection: น้ำหนัก interpolation ≥ 0 + ค่าเก่า ≥ 0 → ผลรวม ≥ 0 (พิสูจน์แล้วใน 2.2) · (2) loss: เขียนเป็น C·(1−D) โดยบังคับ 0 ≤ D ≤ 1 (ใช้ sigmoid) → คูณค่าบวกด้วยเลข 0..1 ยังบวก · (3) emission: บวกค่าบวก (g·E ≥ 0) · ทุกพจน์รักษา ≥ 0 → ผลลัพธ์ ≥ 0 เสมอ โดยโครงสร้าง

4. Monotone — ไม่สร้างยอด/หลุมปลอม ★

กฎละเอียดขึ้น: การเลื่อนควันไม่ควรสร้างค่าสูงสุดใหม่ที่ไม่มีมาก่อน (overshoot) หรือค่าต่ำกว่าที่เคยมี (undershoot) · ถ้าควันเข้มสุดตอน t คือ 100 การหอบมันไปไม่ควรทำให้เกิดจุด 130 ขึ้นมาเอง

ทำไมสำคัญ: scheme ที่ไม่ monotone จะสร้าง "การแกว่ง" รอบ ๆ ขอบควันคม ๆ → เกิดค่าติดลบปลอมและยอดปลอม → ทั้งผิดฟิสิกส์และทำให้ค่า PM2.5 พีคเพี้ยน (สำคัญมากกับวันวิกฤต)

HazeNet ได้มาจากไหน: semi-Lagrangian แบบ monotone interpolation (เช่น จำกัด slope / ใช้ linear ที่ไม่ overshoot) → การันตีว่าค่าใหม่อยู่ในช่วง min–max ของช่องเพื่อนบ้านเสมอ ไม่ทะลุออกนอกช่วง · นี่คือ "monotone scheme" ในชื่อบท

5. นิวรอลดิบ ๆ ละเมิดคุณสมบัติพวกนี้ยังไง

ทำไมเรื่องนี้ถึงเป็นจุดขาย? เพราะนิวรอลทั่วไป (ConvLSTM, GNN, transformer ดิบ) ไม่รู้จักฟิสิกส์ — มันแค่ปรับน้ำหนักให้ output ใกล้ target:

คุณสมบัตินิวรอลดิบทำพลาดยังไง
Non-negativityregression ทั่วไปพ่นค่าติดลบได้สบาย (โดยเฉพาะที่ค่าจริงใกล้ 0)
Mass conservationไม่มีอะไรบังคับให้ผลรวมคงที่ → "สร้าง/ทำลาย" ควันมั่ว
Monotoneoutput แกว่งรอบขอบคม → ยอด/หลุมปลอม

วิธีแก้แบบ "ลวก ๆ" คือใส่ penalty ใน loss (soft constraint) ให้โมเดล "พยายาม" เคารพฟิสิกส์ · แต่นั่นแค่ลงโทษเมื่อผิด ไม่การันตีว่าไม่ผิด — ยังละเมิดได้

ความต่างชี้ขาด: งานอื่นที่ใช้ "soft physics loss" ทำได้แค่ "ส่วนใหญ่เคารพฟิสิกส์" · HazeNet ทำ "เคารพฟิสิกส์เสมอ พิสูจน์ได้" เพราะฝังเป็นโครงสร้าง (hard constraint) ไม่ใช่ penalty · นี่คือประโยคที่จะเขียนในเปเปอร์และเป็นจุดต่างที่ reviewer จับได้ทันที

6. HazeNet การันตีได้ยังไง — สรุปรวม ★★

รวบทุกอย่าง: คุณสมบัติทั้ง 3 มาจากการออกแบบสมการแกน ไม่ใช่จากการเทรน:

Ct+1 = 𝒜(u) Ct + g·E − D⊙Ct
คุณสมบัติมาจาก
Non-negativity𝒜 น้ำหนัก≥0 · g·E≥0 · loss เขียนเป็น C(1−D), 0≤D≤1 (sigmoid)
Mass conservation (ของ advection)𝒜 น้ำหนักรวม=1 (semi-Lagrangian)
Monotone𝒜 ใช้ monotone interpolation (ค่าใหม่อยู่ใน min–max เพื่อนบ้าน)
เชิงเส้นใน C (โบนัส → 2.5)ทุกพจน์คูณ/บวก C เชิงเส้น · g,D ทำนายจากอากาศ ไม่ใช่ C
นี่คือความหมายของ "physics-structured": นิวรอล (AFNO) ทำนายแค่ "สัมประสิทธิ์" g, D โดยถูกบีบให้อยู่ในกรอบที่ปลอดภัย (เช่น D ผ่าน sigmoid → 0..1 เสมอ) · ไม่ว่านิวรอลจะทำนายอะไร ผลลัพธ์ไม่มีทางละเมิดคุณสมบัติฟิสิกส์ · เราจึง "พิสูจน์ได้" ด้วยพีชคณิตง่าย ๆ ไม่ต้องอาศัยว่าเทรนสำเร็จ

7. แต่เดี๋ยว — PM2.5 จริงไม่อนุรักษ์! (ความซื่อสัตย์ที่สำคัญสุด)

นี่คือ caveat ที่ต้องพูดให้ชัดที่สุดในทั้งบท · จาก 1.2: สมการ ADR มีพจน์ที่ 5 คือ Rchem (เคมีในบรรยากาศ) ที่สร้างฝุ่นทุติยภูมิ (secondary aerosol) ขึ้นมาใหม่ · พจน์นี้ไม่อนุรักษ์มวลและไม่เชิงเส้น

เราจึงไม่เคลมเกินจริง: เราไม่ได้บอกว่า "PM2.5 ทั้งหมดอนุรักษ์มวล" (ไม่จริง) · เราเคลมเฉพาะ "ควันปฐมภูมิ (primary smoke) ที่ขนส่งด้วย advection/diffusion/loss นั้นอนุรักษ์มวล+ไม่ติดลบ+เชิงเส้น พิสูจน์ได้" · ส่วน secondary เราจัดการแยก (เช่น เป็นพจน์ residual ที่นิวรอลเรียน หรือระบุเป็นข้อจำกัด) · การขีดเส้นขอบเขตที่เคลมได้ให้ชัด คือสิ่งที่แยกงาน A* ออกจากงานที่อ้างเกินจริง
วิธีพูดในเปเปอร์: "คุณสมบัติเชิงโครงสร้าง (mass-conservation, non-negativity, linearity) ใช้กับ tracers การขนส่งปฐมภูมิ ซึ่งเป็นองค์ประกอบหลักของหมอกควันชีวมวลในภาคเหนือ (สอดคล้อง PMF ~51% ชีวมวล จาก 1.4) · กระบวนการทุติยภูมิถูกจัดการเป็นพจน์แยกและระบุเป็นข้อจำกัด"

8. โค้ด + วิธีพิสูจน์/ทดสอบ

# บังคับคุณสมบัติด้วยโครงสร้าง ไม่ใช่ penalty
D = torch.sigmoid(afno_loss(met, static))   # 0 <= D <= 1  -> non-neg ของ loss
g = F.softplus(afno_emit(met, static))      # g >= 0       -> emission ไม่ลบ

C = advect_monotone(C, wind)                # 𝒜: น้ำหนัก>=0, รวม=1, monotone
C = C + g * E                               # เติม (>=0)
C = C * (1 - D)                          # หาย (คูณ 0..1)  -> ยัง >=0
assert (C >= 0).all()                       # จริงเสมอโดยโครงสร้าง

วิธี "ทดสอบ" คุณสมบัติ (สำหรับเปเปอร์)

  • Non-negativity: รัน rollout ยาว ๆ แล้ว assert C.min() >= 0 ทุก step (จะผ่านเสมอถ้าโครงสร้างถูก)
  • Mass conservation: ปิด emission+loss เหลือ advection ล้วน → เช็คว่า C.sum() คงที่ทุก step (ภายใน error ตัวเลขเล็ก ๆ)
  • Monotone: ให้สนามเริ่มเป็น "กล่องสี่เหลี่ยม" (มีขอบคม) หอบไป → เช็คว่าไม่เกิดค่าเกิน max เดิม/ต่ำกว่า min เดิม
เชื่อม error analysis (Module 5): การทดสอบเหล่านี้คือ "unit test ฟิสิกส์" · ถ้าผ่าน = ตัดความเป็นไปได้ที่ error มาจาก "ละเมิดฟิสิกส์" ทิ้ง → error ที่เหลือมาจากแหล่งอื่น (ลม, การปล่อย, การเรียน) · ช่วยให้แยกแหล่ง errorได้สะอาด

9. Cheat sheet + แบบฝึกหัด

สรุปจำ:
  • Mass conservation: ควันไม่หาย/ไม่โผล่เอง · advection แค่ย้ายที่ ไม่เปลี่ยนมวลรวม
  • Non-negativity: ความเข้มข้น ≥ 0 เสมอ (ติดลบ = ผิดทันที)
  • Monotone: การเลื่อนไม่สร้างยอด/หลุมปลอม (ไม่แกว่งรอบขอบคม)
  • นิวรอลดิบ ละเมิดทั้งสามได้ · soft physics loss = "ลงโทษเมื่อผิด" ไม่การันตี
  • HazeNet = hard constraint: ฝังในโครงสร้างสมการ → พิสูจน์ได้ ไม่ใช่หวังให้เทรนถูก
  • กลไก: 𝒜 (น้ำหนัก≥0,รวม=1,monotone) · g=softplus(≥0) · D=sigmoid(0..1), loss=C(1−D)
  • ★ caveat: PM2.5 จริงไม่อนุรักษ์ (R_chem ทุติยภูมิ) → เคลมเฉพาะ primary smoke เท่านั้น
  • ทดสอบ: assert C≥0 · sum คงที่ตอน advection ล้วน · ทดสอบกล่องคม monotone

แบบฝึกหัด 1 — ทำไม "soft physics loss" (ใส่ penalty ใน loss) ถึงอ่อนกว่า "hard constraint" (ฝังโครงสร้าง) ในการการันตี non-negativity?

เฉลย

soft loss แค่ลงโทษเมื่อโมเดลพ่นค่าติดลบ → โมเดล "พยายาม" หลีกเลี่ยง แต่ยังพ่นค่าติดลบได้ (โดยเฉพาะกรณีที่ไม่เคยเห็นตอนเทรน) → การันตีไม่ได้ · hard constraint ฝังให้ผลลัพธ์เป็นบวกโดยพีชคณิต (เช่น คูณค่าบวกด้วย sigmoid 0..1) → ไม่ว่านิวรอลทำนายอะไร ผลก็ ≥ 0 เสมอ → พิสูจน์ได้ ไม่ขึ้นกับว่าเทรนสำเร็จไหม

แบบฝึกหัด 2 — loss term เขียนเป็น C·(1−D) โดย D = sigmoid(...) · อธิบายว่าทำไมรูปนี้การันตีทั้ง non-negativity และ "ควันลดลงเท่านั้น ไม่เพิ่ม"

เฉลย

sigmoid บีบ D ให้อยู่ 0 ≤ D ≤ 1 เสมอ · ดังนั้น (1−D) อยู่ใน 0..1 · เอา C (ที่ ≥ 0) คูณเลข 0..1 → ผลยัง ≥ 0 (non-negativity ✓) และ ≤ C เดิม (ลดลงเท่านั้น ไม่มีทางทำให้ควันเพิ่มจากพจน์ loss ✓) · ฟิสิกส์ถูก: "การสูญเสีย" ต้องลดควัน ไม่ใช่เพิ่ม

แบบฝึกหัด 3 — เพื่อนเขียนในเปเปอร์ว่า "โมเดลเราอนุรักษ์มวล PM2.5" · ทำไมประโยคนี้อันตราย และควรแก้เป็นอะไร?

เฉลย

อันตรายเพราะ PM2.5 จริงไม่อนุรักษ์มวล — มีฝุ่นทุติยภูมิจาก R_chem ที่สร้างขึ้นใหม่ในบรรยากาศ · reviewer ที่รู้เคมีบรรยากาศจะจับได้ทันทีว่าเคลมเกินจริง → เสียความน่าเชื่อถือ · ควรแก้เป็น: "เราการันตีคุณสมบัติเชิงโครงสร้าง (อนุรักษ์มวล/ไม่ติดลบ/เชิงเส้น) สำหรับ tracers การขนส่งปฐมภูมิ ส่วนกระบวนการทุติยภูมิจัดการแยกและระบุเป็นข้อจำกัด" — ขีดขอบเขตที่เคลมได้ให้ชัด

จบ 2.4 แล้วคุณควร: เข้าใจ 3 คุณสมบัติฟิสิกส์ (มวลไม่รั่ว / ไม่ติดลบ / ไม่สร้างยอดปลอม), รู้ว่านิวรอลดิบละเมิดได้แต่ HazeNet การันตีด้วยโครงสร้าง (hard constraint) จึง "พิสูจน์ได้" — ผ่าน 𝒜 (น้ำหนัก≥0,รวม=1) + g=softplus + D=sigmoid · และที่สำคัญสุด เข้าใจ caveat ว่าเราเคลมเฉพาะ primary smoke ไม่ใช่ PM2.5 ทั้งหมด
