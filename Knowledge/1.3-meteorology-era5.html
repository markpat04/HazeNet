1.3 — ตัวแปรอุตุนิยม & ERA5

"สภาพอากาศ" คือตัวขับการเคลื่อนของควัน — และเป็นอินพุตหลักที่ป้อนเข้าโมเดล

ต่อจาก 1.2 โดยตรง · จุดสำคัญ: แมปแต่ละตัวแปรกลับเข้า "พจน์" ในสมการ ADR · ผูกกับ HazeNet ตลอด

ทำไมบทนี้ต่อจาก 1.2 พอดี: สมการ ADR (1.2) มี "ตัวขับ" ซ่อนอยู่ในทุกพจน์ — ลมขับ advection, ฝน/ความชื้นขับการสูญเสีย, ความสูงชั้นผสม (PBL) กำหนดว่าควันจะกระจุกหรือเจือจาง · ตัวขับพวกนี้ทั้งหมดคือ "ตัวแปรอุตุนิยม" ที่เราดึงมาจาก ERA5 · เข้าใจบทนี้ = เข้าใจว่าอะไรทำให้ฝุ่นวันหนึ่งวิกฤตอีกวันหนึ่งสบาย

1. ERA5 คืออะไร — "ประวัติศาสตร์อากาศที่ดีที่สุด"

ERA5 คือชุดข้อมูล reanalysis ของสภาพอากาศโลก ผลิตโดย ECMWF (ยุโรป) แจกฟรีผ่าน Copernicus CDS

"reanalysis" คืออะไร (สำคัญ)

เราไม่มีเครื่องวัดทุกจุดบนโลกทุกชั่วโมง · reanalysis คือการเอา การสังเกตจริง (สถานี, บอลลูน, ดาวเทียม) มาผสมกับแบบจำลองอากาศ (data assimilation) → ได้ "ค่าประมาณที่ดีที่สุด" ของอากาศในอดีต เต็มทุกจุดบนกริด ทุกชั่วโมง

คุณสมบัติค่า
ความละเอียดเชิงพื้นที่~0.25° (~31 กม.)
ความถี่รายชั่วโมง
ช่วงเวลา1940 – ปัจจุบัน
เข้าถึงฟรี ผ่าน Copernicus CDS (รายละเอียดการดึงอยู่ใน 4.1)
บทบาทใน HazeNet: ERA5 คือแหล่ง "สภาพอากาศ" ทั้งหมดของเรา — ลม, ความชื้น, ฝน, PBL · เราดึงมา regrid ให้เข้ากริด 0.1° ของเรา (หัวข้อ 4.3) แล้วใช้เป็นอินพุต channel (จาก 1.1)

2. อุตุนิยม = "ตัวขับ" ทุกพจน์ในสมการ ADR ★

จำสมการ ADR จาก 1.2 ได้ไหม? ตัวแปรอุตุนิยมแต่ละตัวคือ "ตัวขับ" ของพจน์ต่าง ๆ:

∂C/∂t = −∇·(uC) + ∇·(K∇C) + E − L·C + Rchem
u = ลม · K = กระจาย (ขึ้นกับเสถียรภาพ/PBL) · L = สูญเสีย (ขึ้นกับฝน/ความชื้น)

= อุตุนิยมไม่ใช่ "ข้อมูลเสริม" แต่เป็น สิ่งที่กำหนดว่าฟิสิกส์จะทำงานยังไง · บทนี้จะไล่ทีละตัวแปรว่าขับพจน์ไหน

3. ลม (u, v) — ตัวขับ advection ★ สำคัญสุด

ลมในคอมพิวเตอร์เก็บเป็น 2 องค์ประกอบ: u (ตะวันออก-ตะวันตก) และ v (เหนือ-ใต้) · รวมกันได้ทั้งทิศทางและความเร็ว

นี่คือ u ตัวเดียวกับในพจน์ −∇·(uC) เป๊ะ · ลมแรง+ทิศไหน → ควันถูกหอบไปทางนั้นเร็วแค่นั้น

เชื่อมกับงานเรา: ลม ERA5 คือ อินพุตที่ป้อนตรงเข้าตัว advection (semi-Lagrangian, 2.2) — มันบอกว่าจะลากก้อนควันไปทางไหน · นี่คือสาเหตุที่ควันเมียนมามาถึงเชียงใหม่ได้: ลมตะวันตก/ตะวันตกเฉียงใต้ในฤดูแล้งหอบควันข้ามแดนมา · ลมคือตัวแปรอุตุนิยมที่สำคัญที่สุดของเรา

4. PBL Height — "ฝาครอบ" ที่กักควัน ★ ทำไมฝุ่นพุ่ง

PBL (Planetary Boundary Layer) = ชั้นอากาศใกล้พื้นที่มีการคลุกเคล้า · "ความสูง PBL" = ความสูงของ "กล่อง" ที่ควันถูกผสมอยู่ — เหมือนฝาครอบ

กุญแจ: ไฟปล่อยควันเท่าเดิม แต่ถ้า "ฝา" ต่ำ ควันกระจุกในปริมาตรเล็ก → ความเข้มข้นตรงพื้น (PM2.5) สูงพุ่ง · กลางคืน/เช้ามืด PBL ต่ำ (inversion) → ฝุ่นมักแย่สุดช่วงเช้า

ทำไมเชียงใหม่หนักเป็นพิเศษ: เป็นแอ่งกระทะล้อมภูเขา — อากาศเย็นนิ่งก้นแอ่ง + inversion กด "ฝา" ต่ำลง → ควันออกไม่ได้ สะสมหลายวัน · นี่คือเหตุผลที่ PBL + ภูมิประเทศ (DEM จาก 1.1/4.1) สำคัญต่อการพยากรณ์จุดวิกฤต
เชื่อมกับงานเรา: PBL ไม่ได้อยู่ในพจน์ ADR ตรง ๆ แต่กำหนด "ปริมาตรที่ควันกระจาย" → มีผลต่อความเข้มข้นที่พื้น · เราป้อน PBL เป็นอินพุตให้เน็ตเรียนรู้ว่า "ฝาต่ำ = ความเข้มข้นพุ่ง" (ส่วนหนึ่งของสัมประสิทธิ์ใน 3.5)

5. ความชื้น (Humidity) — บวมน้ำ + ช่วยชะ

ความชื้นสัมพัทธ์ (คำนวณจากอุณหภูมิ + จุดน้ำค้าง) มีผล 2 ทาง:

  • Hygroscopic growth: อนุภาคฝุ่นดูดน้ำบวมขึ้นเมื่อชื้น → กระเจิงแสงมากขึ้น = AOD สูงขึ้น (โดย PM2.5 มวลแห้งอาจเท่าเดิม) → นี่คือเหตุผลหนึ่งที่ AOD ≠ PM2.5 จาก 1.1
  • ช่วยการสูญเสีย: ความชื้นสูง/มีเมฆ มักมาคู่กับการชะล้าง (พจน์ L)
เชื่อมกับงานเรา: ความชื้นเป็นอินพุตที่ช่วยให้เน็ตแก้ความสัมพันธ์ AOD↔PM2.5 และปรับอัตราการสูญเสีย L ได้ถูกต้องขึ้น

6. ฝน (Precipitation) — ตัวขับการสูญเสีย L

ฝนคือ "เครื่องล้างอากาศ" — หยดน้ำจับฝุ่นแล้วพาลงพื้น (wet deposition / washout) · เป็นตัวขับหลักของพจน์ −L·C

★ จุดเชื่อมที่สำคัญ — natural experiment: ถ้ามีฝนตกผิดฤดูดับไฟในบางพื้นที่ หรือชะควันหายไปกะทันหัน → นี่คือ "การทดลองธรรมชาติ" ที่เราใช้ตรวจสอบ counterfactual (V4 ในแผน master / หัวข้อ 5.5) ว่าโมเดลทำนายการลดของฝุ่นถูกไหม · ฝนจึงไม่ใช่แค่อินพุต แต่เป็น "เครื่องมือ validate" ด้วย

7. อุณหภูมิ & เสถียรภาพ (Stability)

  • อุณหภูมิ 2 เมตร + การไล่ระดับอุณหภูมิตามความสูง → กำหนด เสถียรภาพของอากาศ
  • Temperature inversion (อุณหภูมิกลับด้าน: อากาศอุ่นทับอากาศเย็น) = อากาศนิ่งมาก → กด PBL ต่ำ → กักควัน (เชื่อมข้อ 4)
  • อุณหภูมิยังเร่ง/ชะลอปฏิกิริยาเคมี (พจน์ R_chem)

อุณหภูมิจึงเป็นตัวขับทางอ้อมของทั้ง PBL (กระจุก) และ R_chem (เคมี)

8. มิติแนวตั้ง — Surface vs Pressure Levels (เกร็ด)

บรรยากาศเป็น 3 มิติ · ERA5 มีข้อมูล 2 แบบ:

  • Surface / single level: ค่าที่ผิว/ใกล้ผิว (ลม 10ม., อุณหภูมิ 2ม., PBL, ฝน) — ใช้เป็นหลักในงานเรา
  • Pressure levels: ค่าที่ระดับความสูงต่าง ๆ (เช่น ลมที่ 850 hPa) — บอก pattern ลมระดับบนที่หอบควันข้ามแดน
ทางเลือกออกแบบ: เราอาจเริ่มจากโมเดล 2 มิติ (ผิว/คอลัมน์) เพื่อความเรียบง่าย+ประหยัด GPU แล้วค่อยพิจารณาเพิ่มบางระดับความสูงถ้าจำเป็น · การลดจาก 3D เป็น 2D เป็น "การ simplify ฟิสิกส์" ที่ต้องบันทึกว่าเป็นแหล่ง error หนึ่ง (ε_structure ใน 5.5)

10. เชื่อม HazeNet + ข้อควรระวัง

ตัวแปรอุตุนิยมเหล่านี้กลายเป็น input channels (จาก 0.5/1.1):

x.shape = (batch, channels, 85, 105)
# channels = [PM2.5, wind_u, wind_v, precip, humidity, PBL, temp, AOD, ...]

ลม → ขับ advection โดยตรง · ที่เหลือ → เน็ตใช้ทำนายสัมประสิทธิ์ (g, D, K) ใน 3.5

ข้อควรระวัง (ซื่อสัตย์ — เชื่อม error analysis):
  • ERA5 หยาบ (~31 กม.) — ลมในภูมิประเทศภูเขาซับซ้อนของภาคเหนืออาจคลาดเคลื่อน → เป็นแหล่ง ε_wind ใน error budget (5.5) · นี่คือเหตุผลที่เราจะทำ oracle wind substitution (แทนลมด้วยความละเอียดสูงกว่า ดูว่า error ลดเท่าไร)
  • reanalysis ไม่ใช่ความจริง 100% — เป็นแบบจำลอง+สังเกตผสมกัน มี bias ของตัวเอง โดยเฉพาะแถบภูเขา/ชายแดน
  • 2D simplification — ถ้าเราละมิติแนวตั้ง ต้องบันทึกเป็นข้อจำกัด

▶ เฉลย
  1. การสังเกตเป็นจุด ๆ ไม่ครบ · reanalysis เอาการสังเกตผสมแบบจำลอง → ได้ค่าเต็มกริดทุก ชม.

  3. PBL ต่ำ (inversion/กลางคืน/แอ่งเขา) → ควันกระจุกในปริมาตรเล็ก → ความเข้มข้นพื้นพุ่ง
  4. ความชื้นทำอนุภาคบวมน้ำ → กระเจิงแสงมาก AOD สูง ทั้งที่มวลแห้ง PM2.5 อาจเท่าเดิม

จบ 1.3 แล้วคุณควร: รู้ว่า ERA5 คือแหล่งสภาพอากาศ (reanalysis) · เข้าใจว่าตัวแปรอุตุนิยมแต่ละตัวขับพจน์ไหนใน ADR · เข้าใจว่าทำไม PBL ทำให้ฝุ่นพุ่ง และทำไม ERA5 หยาบเป็นแหล่ง error
