1.1 — PM2.5 / AOD / FRP คืออะไร

3 คำที่เป็นวัตถุดิบและเป้าหมายของทั้งโปรเจกต์ — เริ่มโมดูล 1 (โดเมนปัญหา)

สอนจากศูนย์สำหรับคนไม่มีพื้นภูมิศาสตร์ · เน้นว่าแต่ละตัวต่างกันยังไง + เป็น input/output ตรงไหน · ผูกกับ HazeNet ตลอด

ภาพเดียวที่ต้องจำ — นี่คือ "ห่วงโซ่สาเหตุ":
ไฟ (วัด FRP) → ปล่อยควัน → ลมพัดพา → เรา "เห็น" ควันได้ 2 แบบ: จากอวกาศ (AOD) และ ที่เราหายใจตรงพื้น (PM2.5)
ในโปรเจกต์เรา: FRP = อินพุตต้นทาง (สาเหตุ) · AOD = อินพุตสังเกตการณ์ (ควันที่เห็นจากบน) · PM2.5 = เป้าหมายที่เราพยากรณ์ (สิ่งที่คนแคร์)

1. PM2.5 — สิ่งที่เราพยากรณ์ (เป้าหมาย) ★

PM2.5 = Particulate Matter ที่มีเส้นผ่านศูนย์กลาง ≤ 2.5 ไมโครเมตร (ฝุ่นละอองจิ๋ว) · เล็กแค่ไหน? เทียบกับเส้นผมคน:

เพราะเล็กมาก มันจึง เข้าลึกถึงถุงลมปอดและซึมเข้ากระแสเลือดได้ → อันตรายต่อสุขภาพมากกว่าฝุ่นใหญ่ (ก่อโรคหัวใจ/ปอด/มะเร็ง)

หน่วยและเกณฑ์

เรื่องค่า
หน่วยวัดµg/m³ (ไมโครกรัมต่อลูกบาศก์เมตร) = มวลฝุ่นในอากาศ 1 ลบ.ม.

มาจากไหน + วัดยังไง

  • แหล่ง: การเผาไหม้ — การเผาชีวมวล (ไฟป่า/เผาไร่) ซึ่งเป็นโฟกัสเรา, จราจร, อุตสาหกรรม
  • วัด: สถานีภาคพื้น (Air4Thai ของกรมควบคุมมลพิษ / OpenAQ) — แม่นยำแต่เป็น จุด ๆ กระจายห่าง (ไม่ครอบคลุมทุกพื้นที่)
บทบาทใน HazeNet: PM2.5 คือ เป้าหมาย (target / label) — สิ่งที่โมเดลเราพยากรณ์ และใช้เทียบว่าแม่นไหม · แต่เพราะสถานี "เป็นจุดห่าง ๆ" เราจึงต้องการสัญญาณอื่นมาเติมช่องว่างเชิงพื้นที่ (นั่นคือที่ AOD เข้ามา)

2. FRP — ไฟต้นทาง (อินพุตสาเหตุ)

FRP = Fire Radiative Power = พลังความร้อนที่ไฟแผ่ออกมา วัดเป็น เมกะวัตต์ (MW) · ไฟใหญ่/ร้อนกว่า = FRP สูงกว่า = ปล่อยควันมากกว่า

  • วัด: ดาวเทียมตรวจจับ "จุดความร้อนผิดปกติ" (hotspot) — สินค้าหลักคือ NASA FIRMS (เซนเซอร์ VIIRS ละเอียด 375 ม., MODIS)
  • ใช้ทำอะไร: เป็น ตัวแทนของ "อัตราการปล่อยควัน" (emission) — มี FRP มาก ≈ ปล่อยฝุ่นมาก · แปลง FRP → emission ด้วยสูตร (เช่น FINN/FEER)
ข้อจำกัดที่ต้องรู้ (สำคัญต่อ error analysis ของเรา):
  • ดาวเทียมผ่านเป็นช่วง ๆ — เห็นไฟแค่ตอนบินผ่าน (ไม่กี่ครั้ง/วัน) → ไฟที่เกิด-ดับระหว่างนั้นพลาด
  • เมฆบังตรวจไม่เห็น → ไฟใต้เมฆหายไปจากข้อมูล
  • ไฟเล็ก/ไฟคุ (smoldering) ตรวจจับยาก
นี่คือที่มาของ ε_emission ในการแยก error (หัวข้อ 5.5) — ความผิดของอินพุตไฟส่วนหนึ่งมาจากตรงนี้ · และเป็นเหตุผลที่ไอเดีย source inversion (เดาไฟที่พลาดจากควันปลายทาง) มีค่า

3. AOD — ควันที่เห็นจากอวกาศ (อินพุตสังเกตการณ์) ★

AOD = Aerosol Optical Depth = ปริมาณที่ละอองลอย (รวมควัน) บัง/กระเจิงแสง ตลอด ทั้งคอลัมน์อากาศจากพื้นถึงอวกาศ · เป็น ตัวเลขไร้หน่วย (0 = ฟ้าใส, >1 = ขมุกขมัวมาก)

  • วัด: ดาวเทียมมองลงมา — Himawari (P-Tree ของ JAXA) ให้ รายชั่วโมง, MODIS · ครอบคลุมพื้นที่กว้างเป็นแผนที่ต่อเนื่อง
  • ทำไมมีค่า: เติมช่องว่างที่สถานี PM2.5 ไปไม่ถึง — สถานีเป็นจุด แต่ AOD เป็นแผนที่เต็ม → เห็น "ลิ้นควัน" ลอยข้ามแดนได้จากบน
บทบาทใน HazeNet: AOD เป็น อินพุตสังเกตการณ์ ที่บอก "ตอนนี้ควันหนาตรงไหนบ้าง" แบบเต็มพื้นที่ · ช่วยให้โมเดลเห็นภาพควันที่สถานีจุด ๆ มองไม่เห็น

4. ★ ความต่างสำคัญ: AOD ≠ PM2.5 (อย่าสับสน)

มือใหม่มักคิดว่า AOD กับ PM2.5 เป็นสิ่งเดียวกัน — ไม่ใช่ และความต่างนี้สำคัญมากต่อ design เรา

ทำไมแปลง AOD → PM2.5 ตรง ๆ ไม่ได้: ควันอาจลอยอยู่บนสูง (AOD สูง แต่พื้นอากาศดี) หรือกระจุกตรงพื้น (AOD กับ PM2.5 สูงทั้งคู่) · ความสัมพันธ์ขึ้นกับ ความสูงชั้นผสม (PBL), ความชื้น, ชนิดละออง → ไม่ใช่ 1:1 · ดังนั้น AOD เป็นแค่ "สัญญาณช่วย" ไม่ใช่คำตอบ เรายังต้องใช้ PM2.5 สถานีเป็น ground-truth จริง · นี่คือเหตุผลที่ในแผน validation เรา validate AOD แยกจาก PM2.5

6. เชื่อมกับ HazeNet — ทั้ง 3 ตัวอยู่ในสมการเรายังไง

จำสมการแกนจาก 3.5 ได้ไหม — ตอนนี้คุณรู้แล้วว่าตัวแปรมาจากไหน:

Ct+1 = advect(Ct, ลม) + g·E(จาก FRP) − D·Ct C = PM2.5 (เป้าหมาย) · E จาก FRP (ไฟ) · ลมจาก ERA5 · AOD ใช้เป็นสัญญาณช่วย/ตรวจสอบ
สรุปการเชื่อม: FRP ป้อนเป็นแหล่ง E → ลม พัดพา (advect) → ออกมาเป็น PM2.5 ที่เราพยากรณ์ · AOD คอยช่วยให้โมเดลเห็นควันเต็มพื้นที่และใช้ตรวจสอบ · 3 ตัวที่เรียนวันนี้คือ 3 เสาของข้อมูลทั้งโปรเจกต์

แบบฝึกหัด

  1. PM2.5 อันตรายกว่าฝุ่นใหญ่เพราะอะไร?
  2. FRP วัดอะไร หน่วยอะไร เราเอาไปใช้เป็นอะไรในโมเดล?
  3. AOD กับ PM2.5 ต่างกันตรงไหนสำคัญที่สุด?
  4. ทำไมเราใช้ AOD ทั้งที่มี PM2.5 สถานีอยู่แล้ว?
  5. ทำไมข้อมูล FRP ถึงมี "ช่องโหว่" และมันเชื่อมกับไอเดีย source inversion ยังไง?
▶ เฉลย
  1. เล็กมาก (≤2.5µm) เข้าลึกถึงถุงลมปอด+ซึมเข้ากระแสเลือด
  2. วัดพลังความร้อนของไฟ หน่วย MW · ใช้เป็นตัวแทน emission (แหล่งปล่อยควัน E)
  3. AOD นับทั้งคอลัมน์อากาศ (รวมควันบนสูง) · PM2.5 นับเฉพาะตรงพื้นที่เราหายใจ → แปลงตรง ๆ ไม่ได้
  4. สถานี PM2.5 เป็นจุดห่าง ๆ · AOD เป็นแผนที่เต็มพื้นที่รายชั่วโมง → เติมช่องว่างเชิงพื้นที่ + เห็นควันข้ามแดน
  5. ดาวเทียมผ่านเป็นช่วง + เมฆบัง → ไฟบางส่วนพลาด · source inversion ใช้ "ควันปลายทางที่สังเกตได้" แก้ย้อนหาไฟที่หายไป
จบ 1.1 แล้วคุณควร: แยกออกชัดว่า FRP (สาเหตุ) / AOD (ควันจากบน) / PM2.5 (ที่เราหายใจ) ต่างกันยังไง · รู้ว่าแต่ละตัวเป็น input/output ตรงไหนในสมการเรา · และเข้าใจกับดัก "AOD ≠ PM2.5"

ขั้นต่อไปแนะนำ: 1.2 สมการ Advection-Diffusion-Reaction (ADR) — แกนฟิสิกส์ของทั้งโปรเจกต์ และเป็น 1 ใน "3 หัวใจ" · ตอนนี้คุณรู้แล้วว่า C, E, ลม คืออะไร พร้อมเข้าใจสมการที่ร้อยมันเข้าด้วยกัน
