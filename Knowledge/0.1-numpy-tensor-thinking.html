1. tensor คืออะไร — "บันไดมิติ"

tensor (หรือ NumPy เรียกว่า ndarray = N-dimensional array) คือ ก้อนตัวเลขที่จัดเรียงเป็นตาราง จะกี่มิติก็ได้ เราไต่บันไดจากง่ายไปยาก:

มิติ (ndim)ชื่อตัวอย่างในชีวิตตัวอย่างใน HazeNet
0scalar (จำนวนเดี่ยว)อายุ = 25ค่าฝุ่น 1 จุด 1 เวลา = 37
1vector (แถว)คะแนนสอบ 5 วิชาฝุ่นที่เชียงใหม่ราย ชม. ทั้งวัน [24 ค่า]
2matrix (ตาราง)ภาพขาวดำแผนที่ฝุ่น 1 เวลา [lat, lon]
33D arrayวิดีโอขาวดำ (เฟรม×สูง×กว้าง)แผนที่ฝุ่นหลายเวลา [time, lat, lon]
4+ND arrayวิดีโอสี (เฟรม×สี×สูง×กว้าง)[time, ตัวแปร, lat, lon]
เชื่อมกับงานเรา: "แผนที่ฝุ่น" ที่โมเดลพยากรณ์ออกมา ก็คือ matrix 2 มิติ ตัวหนึ่ง — แต่ละช่องคือค่าฝุ่นที่พิกัดนั้น พอมีหลายเวลาก็ซ้อนกันเป็น 3 มิติ พอมีหลายตัวแปร (ฝุ่น+ลม+อุณหภูมิ) ก็เป็น 4 มิติ เข้าใจบันไดนี้ = เข้าใจว่าข้อมูลทั้งโปรเจกต์หน้าตาเป็นยังไง

2. คำศัพท์ 3 คำที่ต้องติดตัว: shape, axis, dtype

นี่คือ 3 คำที่คุณจะพูดทั้งวันเวลาเขียนโค้ด

shape = "รูปร่าง" (ขนาดแต่ละมิติ)

import numpy as np

dust_map = np.array([[12, 37, 88],
                     [55, 41, 19]])   # ตาราง 2 แถว 3 คอลัมน์

print(dust_map.shape)   # → (2, 3)   # 2 แถว, 3 คอลัมน์
print(dust_map.ndim)    # → 2        # กี่มิติ
print(dust_map.size)    # → 6        # มีกี่ตัวเลขทั้งหมด (2×3)

shape เป็น tuple ที่บอกขนาดของแต่ละมิติ — (2, 3) อ่านว่า "มิติแรกขนาด 2, มิติที่สองขนาด 3" · นี่คือสิ่งที่คุณจะ print(x.shape) เช็กตลอดเวลาเพื่อ debug

axis = "แกน" (เลขประจำแต่ละมิติ เริ่มที่ 0)

แต่ละมิติมีหมายเลขเรียกว่า axis นับจาก 0 — สำคัญมากเวลาเราจะ "บวกตามแกนใดแกนหนึ่ง" (ดูส่วน 7)

axis 0 = ไล่ลงตามแถว · axis 1 = ไล่ไปตามคอลัมน์ · (ใน 3D: axis 0=time, axis 1=lat, axis 2=lon)

dtype = "ชนิดข้อมูล"

print(dust_map.dtype)   # → int64     # เลขจำนวนเต็ม

x = np.array([1.5, 2.0])
print(x.dtype)          # → float64   # เลขทศนิยม
ทำไม dtype สำคัญกับเรา: ค่าฝุ่นเป็นทศนิยม (float32) · GPU ชอบ float32 มากกว่า float64 (เร็วกว่า ใช้แรมครึ่งเดียว) — เวลาเทรนบน A100 ที่แรมจำกัด เราจะใช้ float32 เกือบทุกที่ ถ้าเผลอใช้ float64 แรมหมดไว

3. สร้าง array

# จาก list
a = np.array([1, 2, 3])

# ก้อนศูนย์/หนึ่ง — ใช้จอง "ที่ว่าง" ขนาดที่ต้องการ
grid = np.zeros((85, 105))      # แผนที่ว่างขนาดกริดของเรา (lat=85, lon=105)
ones = np.ones((3, 4))

# ลำดับเลข
np.arange(0, 24)                # [0,1,2,...,23]  เช่น 24 ชั่วโมง
np.linspace(0, 1, 5)           # [0, 0.25, 0.5, 0.75, 1.0]  แบ่งเท่า ๆ กัน 5 จุด

# สุ่ม — ใช้ตอน init โมเดล หรือทำข้อมูลทดสอบ
np.random.rand(2, 3)           # สุ่ม 0..1 ขนาด 2×3
เชื่อมกับงานเรา: np.zeros((85, 105)) คือวิธีสร้าง "แผนที่ฝุ่นว่าง ๆ" ขนาดกริดภาคเหนือของเรา แล้วค่อยเติมค่าทีหลัง — คุณจะทำแบบนี้บ่อยตอนสร้าง datacube

4. Indexing & Slicing — หยิบข้อมูลบางส่วน

นี่คือทักษะที่ใช้บ่อยที่สุด — หยิบค่า/หยิบแถว/หยิบพื้นที่ออกมาจากก้อนใหญ่

m = np.array([[12, 37, 88],
              [55, 41, 19],
              [ 7, 63, 24]])

m[0, 1]        # → 37     # แถว 0 คอลัมน์ 1 (นับจาก 0!)
m[1]           # → [55 41 19]   # ทั้งแถว 1
m[:, 0]        # → [12 55 7]    # ทั้งคอลัมน์ 0  ( : = "เอาหมด")
m[0:2, 1:3]    # → [[37 88]
               #     [41 19]]    # พื้นที่สี่เหลี่ยม แถว0-1 คอลัมน์1-2

กฎ slicing: start:stop (ไม่รวม stop) · : เดี่ยว ๆ = "เอาทั้งแกนนี้" · เลขติดลบนับจากท้าย (m[-1] = แถวสุดท้าย)

ตัวอย่าง HazeNet: ถ้า dust มี shape (time, lat, lon) = (720, 85, 105) (1 เดือน รายชั่วโมง):
  • dust[0] = แผนที่ฝุ่นชั่วโมงแรก (shape (85,105))
  • dust[:, 40, 60] = ค่าฝุ่นที่จุด (lat=40, lon=60) ทุกชั่วโมง (เช่น สถานีเชียงใหม่ ตลอดเดือน)
  • dust[0:24] = ฝุ่น 24 ชั่วโมงแรก

5. Element-wise & ทำไม "ห้ามใช้ for loop"

การคำนวณบน array ทำกับ ทุกช่องพร้อมกัน — เรียกว่า vectorization นี่คือหัวใจที่ทำให้ NumPy เร็ว

a = np.array([1, 2, 3])
b = np.array([10, 20, 30])

a + b          # → [11 22 33]   # บวกตรงตำแหน่งกัน
a * 2          # → [2 4 6]      # คูณทุกช่องด้วย 2
a ** 2         # → [1 4 9]      # ยกกำลังทุกช่อง
np.sqrt(b)     # → [3.16 4.47 5.48]
กฎเหล็กของวงการ: ถ้าคุณเขียน for วนทีละช่องของ array ใหญ่ ๆ แสดงว่าทำผิดทาง — มันช้ากว่าหลายร้อยเท่า เขียนเป็น operation บนทั้ง array แทนเสมอ

# ❌ ช้า — อย่าทำ
for i in range(len(a)):
    out[i] = a[i] * 2

# ✅ เร็ว — ทำแบบนี้
out = a * 2
เชื่อมกับงานเรา: การคำนวณ error (พยากรณ์ − จริง) ทำทั้งแผนที่พร้อมกัน: diff = pred - true — pred กับ true เป็น array ขนาด (85,105) ลบกันทีเดียวได้ผลทั้งแผนที่ ไม่ต้อง loop ทีละจุด

6. Broadcasting — "ยืดอัตโนมัติ" (เวทมนตร์ที่ต้องเข้าใจ)

เกิดอะไรขึ้นถ้า array คนละขนาด มาคำนวณกัน? NumPy จะ "ยืด" ตัวเล็กให้เข้ากับตัวใหญ่อัตโนมัติ ถ้าขนาดเข้ากันได้ — เรียก broadcasting

grid = np.array([[1, 2, 3],
                 [4, 5, 6]])        # shape (2, 3)

grid + 100      # scalar ยืดไปทุกช่อง → [[101 102 103]
                #                              [104 105 106]]

row = np.array([10, 20, 30])        # shape (3,)
grid + row      # row ยืดลงทุกแถว → [[11 22 33]
                #                            [14 25 36]]

กฎ broadcasting (จำง่าย ๆ)

เทียบ shape จาก ขวาไปซ้าย แต่ละมิติต้อง (ก) เท่ากัน หรือ (ข) ฝั่งใดฝั่งหนึ่งเป็น 1 ถึงจะยืดได้

ABยืดได้?
(85, 105)(105,)✅ B ยืดลงทุกแถว
(85, 105)(85, 1)✅ B ยืดไปทุกคอลัมน์
(85, 105)(85,)❌ ไม่ได้ (มิติขวาสุด 105≠85)
เชื่อมกับงานเรา: สมมติเราอยากปรับค่าฝุ่นทุกจุดด้วย "ค่าตามละติจูด" (เช่น ปัจจัยภูมิประเทศต่อแถว) — เก็บปัจจัยเป็น array (85, 1) แล้ว dust * factor มันจะยืดคูณทุกคอลัมน์ให้เอง · broadcasting ทำให้โค้ดสั้นและเร็ว แต่ ถ้าไม่เข้าใจมันจะ bug เงียบ ๆ (รูปร่างผิดแต่ไม่ error) — เลยต้อง print(x.shape) เช็กเสมอ

7. Reduction ตาม axis — "ยุบมิติ" ★ สำคัญสุดสำหรับงานเรา

reduction = ยุบหลายค่าให้เหลือค่าเดียว เช่น sum, mean, max · ความลับอยู่ที่ เลือก axis ว่าจะยุบแกนไหน

m = np.array([[1, 2, 3],
              [4, 5, 6]])          # shape (2, 3)

m.sum()              # → 21          # ยุบหมด เหลือค่าเดียว
m.sum(axis=0)        # → [5 7 9]     # ยุบแกน 0 (แถว) → เหลือ 3 คอลัมน์
m.sum(axis=1)        # → [6 15]      # ยุบแกน 1 (คอลัมน์) → เหลือ 2 แถว
m.mean(axis=0)       # → [2.5 3.5 4.5]
วิธีจำที่ไม่มีวันลืม: axis=k แปลว่า "แกน k หายไป" · m.shape=(2,3) → sum(axis=0) ทำให้ 2 หาย เหลือ shape (3,) · sum(axis=1) ทำให้ 3 หาย เหลือ shape (2,)
★ จุดที่เชื่อมกับ "หัวใจโปรเจกต์" (หัวข้อ 2.5 Superposition):
จำสมการ C = Σ_s C⁽ˢ⁾ ได้ไหม? — "ฝุ่นรวม = ผลบวกของฝุ่นจากทุกแหล่ง" ในโค้ด ถ้าเราเก็บ tracer ทุกแหล่งเป็น array C_tracers shape (source, lat, lon) เช่น (4, 85, 105) (4 แหล่ง): 
C_total = C_tracers.sum(axis=0)    # ยุบแกน source → ได้ฝุ่นรวม (85,105)
สมการทั้งหมดของ attribution กลายเป็นโค้ดบรรทัดเดียว! นี่คือเหตุผลว่าทำไม "คิดเป็น tensor" ถึงสำคัญ — ฟิสิกส์ที่ดูซับซ้อนกลายเป็น .sum(axis=0)
★ จุดที่เชื่อมกับ error metric (หัวข้อ 5.1 RMSE): 
diff = pred - true              # (85,105) ลบกันทั้งแผนที่
rmse = np.sqrt((diff**2).mean())  # ยกกำลังสอง → เฉลี่ยทุกช่อง → ราก
RMSE ทั้งสูตร = ยกกำลังสอง (element-wise) + เฉลี่ย (reduction) + ราก · คุณเพิ่งเข้าใจสูตรประเมินผลหลักของงานไปแล้ว

เกร็ด: keepdims=True ทำให้แกนที่ยุบไม่หายแต่เหลือขนาด 1 (เช่น (2,3)→(2,1)) — มีประโยชน์มากเวลาจะเอาผลไป broadcasting ต่อ เช่น normalize: m / m.sum(axis=1, keepdims=True)

8. Reshape & จัดแกนใหม่

a = np.arange(12)              # [0,1,...,11]  shape (12,)
a.reshape(3, 4)               # จัดใหม่เป็น 3×4 (จำนวนช่องต้องเท่าเดิม: 3×4=12)
a.reshape(2, -1)             # -1 = "คำนวณให้เอง" → (2, 6)

img = np.zeros((85, 105))     # (lat, lon)
img[np.newaxis, :, :].shape  # → (1, 85, 105)   # เพิ่มมิติหน้า (เช่น เพิ่มแกน time/batch)

x = np.zeros((85, 105, 3))    # (lat, lon, channel)
x.transpose(2, 0, 1).shape   # → (3, 85, 105)   # สลับให้ channel มาก่อน
เชื่อมกับงานเรา: PyTorch (หัวข้อ 0.2) คาดหวังข้อมูลภาพในรูป (batch, channel, height, width) แต่ข้อมูล geospatial มักมาเป็น (lat, lon, channel) — คุณจะต้อง transpose สลับแกนบ่อยมาก เพื่อให้ shape ตรงกับที่โมเดลต้องการ · 90% ของ bug ตอนต่อท่อข้อมูลคือ "shape ไม่ตรง"

9. ภาพรวม: ข้อมูล HazeNet ทั้งหมดเป็น tensor หน้าตายังไง

รวมทุกอย่างที่เรียนมา — นี่คือ "หน้าตาจริง" ของข้อมูลในโปรเจกต์:

ข้อมูลshape (ตัวอย่าง)ความหมายแต่ละแกน
แผนที่ฝุ่น 1 เวลา(85, 105)(lat, lon)
ฝุ่นทั้งเดือน(720, 85, 105)(time, lat, lon)
อินพุตหลายตัวแปร(720, 6, 85, 105)(time, ตัวแปร[ฝุ่น,u,v,temp,...], lat, lon)
tracer แยกแหล่ง(720, 4, 85, 105)(time, source[เมียนมา,ลาว,ไทย,..], lat, lon)
batch ตอนเทรน(16, 6, 85, 105)(batch, channel, lat, lon)

มินิตัวอย่างที่ร้อยทุกอย่างเข้าด้วยกัน

import numpy as np

# สมมติ tracer ฝุ่นจาก 4 แหล่ง บนกริด 85×105 (เลขสุ่มแทนของจริง)
C = np.random.rand(4, 85, 105).astype(np.float32)   # (source, lat, lon)

# 1) ฝุ่นรวม = บวกทุกแหล่ง (superposition!)
C_total = C.sum(axis=0)                # shape (85, 105)

# 2) สัดส่วนแต่ละแหล่ง (attribution!) — broadcasting
share = C / C_total[np.newaxis, :, :]  # (4,85,105) / (1,85,105) → (4,85,105)

# 3) ที่จุดเชียงใหม่ (lat=40, lon=60) แหล่งไหนเด่นสุด?
cnx = share[:, 40, 60]                 # เวกเตอร์ 4 ค่า รวมได้ ~1.0
print(cnx, cnx.sum())

# 4) error เทียบค่าจริง (RMSE)
true = np.random.rand(85, 105).astype(np.float32)
rmse = np.sqrt(((C_total - true)**2).mean())
print("RMSE =", rmse)

โค้ดนี้ใช้: สร้าง array (3), reduction axis (7), broadcasting (6), indexing (4), element-wise (5) — ทุกอย่างในบทนี้อยู่ใน 10 บรรทัด และมันคือแก่นของ attribution + การวัดผลจริง ๆ ของ HazeNet

10. สะพานไป PyTorch (หัวข้อ 0.2)

PyTorch tensor เหมือน NumPy array เกือบทุกอย่าง — ที่เพิ่มมาคือ (ก) รันบน GPU ได้ (ข) คำนวณ gradient อัตโนมัติ (autograd)

import torch

x_np = np.zeros((85, 105), dtype=np.float32)
x_t  = torch.from_numpy(x_np)      # NumPy → Torch (แทบจะของชิ้นเดียวกัน)
x_t  = x_t.cuda()                  # ส่งขึ้น GPU

# syntax เกือบเหมือนกันเป๊ะ:
x_t.sum(dim=0)     # torch ใช้คำว่า dim แทน axis (ความหมายเดียวกัน)
x_t.shape          # เหมือนกัน
back = x_t.cpu().numpy()           # Torch → NumPy กลับ
สรุปสะพาน: ถ้าคุณคิด NumPy เป็น = คุณคิด PyTorch เป็นทันที 90% · ต่างแค่ axis→dim และมันวิ่งบน GPU + จำ gradient ได้ · นั่นคือเหตุผลที่ 0.1 ต้องมาก่อน 0.2

  1. x = np.zeros((720, 6, 85, 105)) — x.ndim เท่าไร? x[0,0] shape อะไร?
  2. มี C shape (4, 85, 105) อยากได้ฝุ่นรวมทุกแหล่ง เขียนยังไง? ได้ shape อะไร?
  3. pred และ true ต่างเป็น (85,105) เขียน RMSE ยังไง?

▶ เฉลย
  1. ndim = 4 · x[0,0] หยิบ time=0, ตัวแปร=0 → shape (85, 105)
  2. C.sum(axis=0) → shape (85, 105)
  3. np.sqrt(((pred - true)**2).mean())
  4. (105,) ✅ ได้ (ยืดลงทุกแถว) · (85,) ❌ ไม่ได้ (มิติขวาสุด 105≠85 ต้องทำเป็น (85,1) ก่อน)
