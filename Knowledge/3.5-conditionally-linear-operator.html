3.5 — Conditionally-Linear Operator ⭐ CORE ❤ หัวใจ 3/3 สุดท้าย

บทไคลแมกซ์ — ที่ที่ฟิสิกส์ (1.2) + superposition (2.5) + AFNO (3.2) มาบรรจบเป็น "งานใหม่"

ทุกบทใน 1.x–3.4 ปูทางมาที่นี่ · บทนี้คือ สมการเดียวที่นิยามทั้ง HazeNet และเหตุผลว่าทำไมเปเปอร์เราเป็นงานใหม่ระดับ A*

คำสัญญาของบทนี้: เมื่อจบบทนี้คุณจะมองสมการแกนหนึ่งบรรทัด แล้วเห็นทั้งโปรเจกต์ในนั้น — เห็นฟิสิกส์ (ADR), เห็นนิวรอล (AFNO), เห็น attribution (superposition), เห็นว่าทำไมความเร็ว/แม่นยำ/พิสูจน์ได้อยู่ในที่เดียวกัน · "conditionally-linear" คือชื่อสองคำที่สรุปงานวิจัย C1 (Contribution 1: conditionally-linear neural transport emulator) ของเปเปอร์ · ก่อนหน้านี้เป็นจิ๊กซอว์ — บทนี้คือภาพรวม

1. ตั้งโจทย์ — ความตึงเครียดที่ต้องคลี่

เรามีความต้องการขัดแย้งกัน 2 ฝั่ง:

ฝั่ง ฟิสิกส์ / Attributionฝั่ง AI / Expressivity
ต้องเชิงเส้นใน C เพื่อ superposition (2.5) → attribution + counterfactualต้องไม่เชิงเส้นมากพอจะจับพลวัตซับซ้อนของบรรยากาศ (เคมี อากาศ ภูมิประเทศ)
ต้องการ hard constraint (มวล/ไม่ลบ) จาก 2.4ต้องการความยืดหยุ่นของนิวรอลที่เรียนจากข้อมูล
ต้องเร็วพอใช้รายวันต้องแม่นพอแข่ง CTM ได้

ถ้าเลือกฝั่งเดียวจะได้อย่างเสียอย่าง: เชิงเส้นล้วน → ไม่สามารถจับฟิสิกส์ซับซ้อน · ไม่เชิงเส้นล้วน → พัง superposition · บทนี้คือทางออกที่เลือกได้ทั้งคู่ผ่านการ "แยกบทบาท"

2. ทางออก — แยกบทบาท ★★

กุญแจคือสังเกตว่า: ใน CTM จริง สิ่งที่ทำให้สมการ ADR ซับซ้อนคือ "สัมประสิทธิ์" (ลม K การปล่อย อัตราหาย ฯลฯ) ซึ่งขึ้นกับอากาศและภูมิประเทศ · แต่ตัวสมการเอง (operator) เป็นเชิงเส้นใน C · ความซับซ้อนอยู่ที่ "สัมประสิทธิ์เป็นอะไร" ไม่ใช่ "C ถูก operate ยังไง"

ความซับซ้อนทั้งหมด ⇒ สัมประสิทธิ์  |  โครงสร้างที่กระทำต่อ C ⇒ เชิงเส้น

ดังนั้นเราแยกบทบาท: ให้นิวรอลรับผิดชอบ "สัมประสิทธิ์" (จัดการความไม่เชิงเส้น) · ให้ฟิสิกส์รับผิดชอบ "กระทำต่อ C" (รักษาความเชิงเส้น)

3. นิยาม "Conditionally-Linear" ★★

นี่คือคำสองคำที่ต้องเข้าใจให้เป๊ะ:

conditionally-linear = "เชิงเส้น เมื่อให้ สิ่งใดสิ่งหนึ่ง"

ในงานเรา: operator ของเราไม่เชิงเส้นในอินพุตรวม (อากาศ × C) แต่เชิงเส้นใน C เมื่อกำหนดอากาศแล้ว · แปลตรง ๆ:

  • ถ้าคุณคงอากาศไว้ (ลม, ความชื้น, PBL... เหมือนเดิม) แล้วเปลี่ยนแค่ C → ผลตอบสนองเป็นเชิงเส้น
  • ถ้าคุณเปลี่ยนอากาศ → สัมประสิทธิ์ g, D เปลี่ยน → operator ก็เปลี่ยน (ไม่เชิงเส้นในอากาศ)
ℒmet(C) = 𝒜(u) C + gθ(met,static)·E − Dθ(met,static)⊙C
สัมประสิทธิ์ 𝒜(u), g(met), D(met) = ฟังก์ชันของ "met" (ไม่เชิงเส้นได้) · แต่ไม่ใช่ฟังก์ชันของ C
operator ℒmet เชิงเส้นใน C (ทุกพจน์คูณ/บวก C เชิงเส้น)
อุปมา: เหมือนเครื่อง printing — "เปลี่ยน plate (อากาศ)" ทำได้ทุกแบบสร้างสรรค์ไม่มีข้อจำกัด · แต่เมื่อ plate ถูกตั้งแล้ว "การกดกระดาษ (C)" เป็นการกระทำเชิงเส้นเสมอ — กระดาษ 2 แผ่นกดผ่าน plate เดียวกัน = ผลของแต่ละแผ่นบวกกัน (superposition)

4. สมการแกนเต็ม — อ่านทีละสัญลักษณ์ ★★

Ct+1(s)  =  𝒜mono(ut)·Ct(s)  +  gθ(mett,static)·Et(s)  −  Dθ(mett,static)⊙Ct(s)

อ่านทีละชิ้น (เชื่อมทุกบทเก่า):

สัญลักษณ์คือมาจากบท

𝒜mono(u)operator เลื่อนตามลม monotone semi-Lagrangian (เชิงเส้นใน C, w≥0, Σw=1)2.2, 2.4
gθ(met,static)สัมประสิทธิ์ปล่อย ทำนายโดย AFNO · ผ่าน softplus → ≥03.2, 2.4
Et(s)ฟลักซ์การปล่อยจากไฟในโซน s (FIRMS/FRP)1.1, 1.4
Dθ(met,static)อัตราสูญเสีย ทำนายโดย AFNO · ผ่าน sigmoid → 0..13.2, 2.4

สังเกตรูปแบบ: ทุกพจน์คูณหรือบวก C เชิงเส้น · สัมประสิทธิ์ทั้งหมดมาจากนิวรอลที่กิน "met, static" — ไม่กิน C · นี่คือ conditionally-linear ที่เป็นจริงในโค้ดทุกบรรทัด

5. ทำไมต้อง "เมื่อให้อากาศ" (ไม่ใช่เชิงเส้นในอากาศ)

ทำไมเรายอมให้ไม่เชิงเส้นในอากาศได้? เพราะเราไม่ต้องทำ superposition กับอากาศ · ปัญหาที่เราถาม:

  • "ฝุ่นนี้มาจากแหล่งไหน?" → ทำ superposition กับ E (แหล่ง) → ต้องเชิงเส้นใน C ที่มาจาก E
  • "ถ้าลดไฟ 50%?" → scale E → ต้องเชิงเส้นใน C

เราไม่เคยถาม: "ฝุ่นนี้กี่ % มาจากลม 5 m/s vs ลม 8 m/s" — เพราะอากาศไม่ใช่ "แหล่ง" ที่ทำ attribution ได้ · ดังนั้นไม่จำเป็นต้องเชิงเส้นในอากาศ → ปลดล็อกให้ AFNO ใช้ความยืดหยุ่นเต็มที่ในฝั่งอากาศ

นี่คือทำไม "conditionally" ถึงทรงพลัง: เรารักษาความเชิงเส้นเฉพาะที่จำเป็น (ใน C เพื่อ attribution) ไม่ทั้งหมด · ได้ทั้งสองโลก — physics-grounded ที่จำเป็น + AI flexibility ที่อยากได้ · ไม่ใช่การประนีประนอม แต่เป็นการออกแบบให้ตรงกับธรรมชาติของปัญหา

6. คุณสมบัติ 5 ข้อที่ได้พร้อมกัน ★★

สมการแกนเดียวให้คุณสมบัติทั้ง 5 พร้อมกัน — ทั้งหมดมาฟรีจากการออกแบบ:

#คุณสมบัติมาจากกลไกทำให้ได้
1เชิงเส้นใน Cทุกพจน์คูณ/บวก C ไม่มี nonlinearity แตะ Csuperposition (2.5) → attribution
2Non-negativityg=softplus, D=sigmoid, 𝒜 w≥0C ≥ 0 พิสูจน์ได้ (2.4)
3Mass conservation𝒜 Σw=1มวลไม่รั่วจาก advection (2.4)
4Differentiableทุกบรรทัดเป็น op ของ PyTorchadjoint ฟรี → counterfactual + inversion (3.7)
5เร็ว (GPU-friendly)tensor ops + Fourier (N·logN)วินาที vs ชม. ของ WRF-Chem (1.5)
นี่คือคำตอบ "ทำไมงานใหม่": ไม่มีงานก่อนหน้าที่ให้ทั้ง 5 ข้อในสมการเดียวกัน · CTM ดั้งเดิม (WRF-Chem) ได้ 1+2+3 แต่ขาด 4+5 · นิวรอลดิบ (PM2.5-GNN, ConvLSTM) ได้ 5 แต่ขาด 1+2+3 · PINN ผสมฟิสิกส์แต่ใช้ soft constraint (ไม่ได้ 1+2+3 แบบพิสูจน์) · HazeNet = สถาปัตยกรรมแรก (สำหรับ PM2.5 ภูมิภาค) ที่ได้ครบ 5 ข้อ ผ่านการออกแบบ conditionally-linear

ความแตกต่างหลักจาก PINN: PINN ใส่ฟิสิกส์เป็นpenalty ใน loss (soft) — ละเมิดได้ ไม่พิสูจน์ · HazeNet ใส่ฟิสิกส์เป็นstructure ของ operator (hard) — ละเมิดไม่ได้โดยพีชคณิต · ความแตกต่างหลักจาก FNO ดิบ: FNO ทำ FFT บน C โดยตรง (ไม่เชิงเส้นใน C) · HazeNet ทำ FFT บน met เพื่อออกสัมประสิทธิ์ → คงเชิงเส้นใน C

8. โค้ดรวบของจริง (สมการนี้ในภาษา PyTorch)

class HazeNetStep(nn.Module):
    """หนึ่ง timestep ของ conditionally-linear operator"""
    def __init__(self):
        super().__init__()
        self.afno_g = AFNO(...)           # สำหรับ g (emission coeff)
        self.afno_D = AFNO(...)           # สำหรับ D (loss rate)

    def forward(self, C, wind, met, static, E):
        # shape: C, E = (S, lat, lon)  ; S=แหล่ง · met,static,wind = (..., lat, lon)
        # ---- ฝั่งอากาศ (nonlinear OK) — ออกสัมประสิทธิ์ ----
        g = F.softplus(self.afno_g(met, static))    # g ≥ 0    (2.4)
        D = torch.sigmoid(self.afno_D(met, static)) # 0≤D≤1   (2.4)
        #  ↑ AFNO ไม่เห็น C เลย ← นี่คือ "conditionally"

        # ---- ฝั่ง C (เชิงเส้นล้วน) — สมการแกน ----
        C = advect_monotone(C, wind)        # 𝒜·C  (2.2) — เชิงเส้น
        C = C + g.unsqueeze(0) * E        # +g·E         — เชิงเส้น (broadcast S)
        C = C * (1 - D).unsqueeze(0)     # −D⊙C         — เชิงเส้น
        return C                        # C^{t+1}_(s) ทุกแหล่งพร้อมกัน

# การันตี: ทุก op ในฝั่ง C เป็น linear → ℒ_met(C^(1)+C^(2)) = ℒ_met(C^(1))+ℒ_met(C^(2))
#         → superposition (2.5) เป็นจริงโดยพีชคณิต ไม่ใช่หวังว่าจะจริง
อ่านโค้ดให้ออกข้อความนี้: "AFNO ไม่เห็น C" คือบรรทัดที่ทำให้สมการเป็น conditionally-linear · ถ้าวันหนึ่งคุณเผลอเปลี่ยนเป็น afno_g(met, static, C) ทุกอย่างพังทันที — superposition พัง, attribution ผิด, ทฤษฎีบทใช้ไม่ได้ · บรรทัดนี้คือสัญญาทางคณิตศาสตร์ของทั้งโปรเจกต์

9. ขีดเส้นที่ห้ามข้าม (ซื่อสัตย์)

conditionally-linear ทรงพลังแต่มีขอบเขต · เพื่อให้เคลมตรงในเปเปอร์ A* ต้องชัดว่า:

  • ไม่ครอบคลุมเคมีทุติยภูมิ (R_chem): ฝุ่นทุติยภูมิ ไม่เชิงเส้น → ใส่ลงในโครงนี้ตรง ๆ ไม่ได้ · เราโฟกัส primary smoke (สอดคล้องสัดส่วน PMF ~51% ชีวมวล ในภาคเหนือ จาก 1.4) · secondary จัดการเป็น residual หรือระบุเป็นข้อจำกัด
  • "เมื่อให้อากาศ" = พึ่งคุณภาพอากาศ: ถ้าลม ERA5 หยาบ/ผิด → สัมประสิทธิ์ผิด → C เพี้ยน · นี่คือ ε_wind ในการแยก error (master plan)
  • Tracer หยาบ: เราติดป้ายระดับประเทศ/ภูมิภาค ไม่ใช่รายแปลง → ตอบ "พม่ากี่%" ได้ แต่ "อำเภอไหนในพม่า" ไม่ได้
  • เชิงเส้นไม่ได้แปลว่าง่าย: 𝒜 และสัมประสิทธิ์ที่นิวรอลเรียนยังซับซ้อนมาก · เราแค่ "เชิงเส้นใน C" ไม่ใช่ "เชิงเส้นง่าย ๆ"
ทำไม caveat คือจุดแข็ง: ระดับ A* ไม่ตัดสินจาก "อ้างได้กว้างแค่ไหน" แต่จาก "ขีดขอบเขตชัดแค่ไหน + พิสูจน์ในขอบเขตนั้นเข้มแค่ไหน" · การยอมรับขอบเขต = น่าเชื่อถือมากขึ้น ไม่ใช่น้อยลง

10. นี่คือ "งานใหม่" ในประโยคเดียว ★★

"HazeNet เป็น conditionally-linear neural transport emulator สำหรับ PM2.5 ภูมิภาค
ที่รักษาความเชิงเส้นใน C โดยโครงสร้าง (linearity-by-design)
จึงให้ attribution + counterfactual + adjoint ในโมเดลเดียวกับ การพยากรณ์"

นี่คือประโยคที่จะวางในบทคัดย่อ · ทุกคำมีน้ำหนัก:

  • "conditionally-linear" = ตอบทั้ง physics และ AI flexibility (ข้อ 2-3)
  • "neural transport emulator" = แทน CTM (1.5) ด้วยนิวรอล (= contribution C1)
  • "linearity-by-design" = ไม่ใช่ soft penalty ไม่ใช่หวังให้เรียนถูก = ต่างจาก PINN/นิวรอลดิบ
  • "attribution + counterfactual + adjoint ในโมเดลเดียวกับการพยากรณ์" = ความเป็น "unified framework" ที่ของเก่าไม่มี (= contribution C2, C3 ตามลำดับ)
นี่คือเหตุผลที่ตีพิมพ์ A* ได้: ไม่ใช่ "อีกหนึ่งโมเดลพยากรณ์ฝุ่น" · เป็นกรอบใหม่ที่ unify หลายงาน (forecast, attribution, source inversion, counterfactual) ด้วยการออกแบบ operator เดียว · กรอบนี้ทั่วไปกว่า PM2.5 ด้วย — ใช้กับ tracer transport อื่นได้ (เกสร อัลเลอร์เจน รังสี) เป็น "extension" ที่ระบุในเปเปอร์

  • ทางออก: แยกบทบาท — นิวรอลออก "สัมประสิทธิ์" · ฟิสิกส์ "กระทำต่อ C" เชิงเส้น

  • สมการแกน: C^{t+1}_(s) = 𝒜(u)·C + g_θ(met)·E − D_θ(met)⊙C
  • กฎเหล็ก: AFNO ไม่กิน C · กิน met,static เท่านั้น → ออก g,D
  • 5 คุณสมบัติพร้อมกัน: เชิงเส้น+nonneg+mass+diff+เร็ว → ของเก่าไม่มีครบ
  • ต่างจาก: WRF-Chem (ขาดเร็ว/adjoint), นิวรอลดิบ (ขาดฟิสิกส์), PINN (soft ไม่ hard), FNO ดิบ (FFT บน C)
  • ขอบเขต: primary smoke · ขึ้นกับคุณภาพ met (=ε_wind) · tracer ระดับประเทศ
  • ประโยคเดียว: "neural transport emulator ที่ linearity-by-design → unified forecast+attribution+counterfactual+adjoint"

แบบฝึกหัด 1 — อธิบายให้เพื่อนใน 2 ประโยค: "conditionally-linear" คืออะไร และทำไมงานเราจึงได้ทั้ง physics rigor และ AI flexibility

เฉลย

(1) Conditionally-linear = สมการของเราเชิงเส้นใน C เมื่อกำหนดอากาศแล้ว โดยให้นิวรอล (AFNO) รับผิดชอบ "สัมประสิทธิ์" g, D ที่ขึ้นกับอากาศเท่านั้น (ไม่กิน C) แล้วสมการแกนเอา g, D ไปประกอบกับ C เชิงเส้น · (2) เพราะนิวรอลทำงานในฝั่งอากาศได้ไม่จำกัด (AI flexibility) ในขณะที่ฝั่ง C ยังเชิงเส้นโดยโครงสร้าง (physics rigor → superposition, mass, non-neg) — ได้ทั้งคู่ผ่านการแยกบทบาท ไม่ใช่ประนีประนอม

แบบฝึกหัด 2 — เปรียบเทียบ HazeNet กับ PINN: ทั้งคู่อ้างว่าเป็น "physics-informed" · ต่างกันอย่างไรในเชิงคุณภาพการเคลม?

เฉลย

PINN: ใส่ฟิสิกส์เป็นpenalty ใน loss (soft constraint) — โมเดล "พยายาม" เคารพ PDE ระหว่างเทรน แต่ละเมิดได้ตอน inference โดยเฉพาะนอก distribution → เคลม "เคารพฟิสิกส์" อ่อน · HazeNet: ใส่ฟิสิกส์เป็นstructure ของ operator (hard constraint) — superposition/non-neg/massเป็นจริงโดยพีชคณิตไม่ว่านิวรอลทำนายอะไร → เคลม "พิสูจน์ได้" · ต่างกันที่ "เกือบเสมอ" vs "เสมอ พิสูจน์ได้" ซึ่งเป็นความต่างเชิงคุณภาพที่ reviewer A* จับได้ทันที

แบบฝึกหัด 3 — สมมติเพื่อนเสนอ "ให้ AFNO กิน C เพิ่มเข้าไปด้วย จะได้แม่นขึ้น" · ทำไมข้อเสนอนี้พังทั้งโปรเจกต์ และคำตอบของคุณคืออะไร?

เฉลย

ถ้า AFNO กิน C → g, D กลายเป็นฟังก์ชันของ C → สมการเป็น ℒ_met,C(C) ไม่ใช่ ℒ_met(C) อีกต่อไป → ไม่เชิงเส้นใน C → ℒ(C^(1)+C^(2)) ≠ ℒ(C^(1))+ℒ(C^(2)) → superposition พัง → C ≠ ΣC^(s) → attribution ผิด, counterfactual โดยการลบ tracer ใช้ไม่ได้, ทฤษฎีบท error bound ใช้ไม่ได้ → ทั้ง contribution C2 และ C3 ของเปเปอร์ล่ม · คำตอบ: "ความแม่นเพิ่มเล็กน้อยไม่คุ้มกับการสูญเสีย attribution ซึ่งเป็นจุดขายหลัก" · ถ้าอยากเพิ่มความแม่น ทำผ่านการขยาย AFNO ในฝั่ง met หรือเพิ่มตัวแปร met (ไม่ใช่ C)

🎉🎉 จบ 3.5 = จบ "3 หัวใจ" ครบแล้ว!
