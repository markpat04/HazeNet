3.1 — Fourier Transform / FFT

แปลงสัญญาณเป็น "ส่วนผสมของคลื่น" — ฐานคณิตของ AFNO (backbone นิวรอลเรา)

เริ่ม Module 3 (สถาปัตยกรรมนิวรอล) · บทนี้เป็นเครื่องมือคณิตที่ AFNO (3.2) ใช้ · ผูกกับ HazeNet ตลอด

ทำไมต้องเรียน Fourier ก่อน AFNO: backbone นิวรอลของเราชื่อ AFNO (Adaptive Fourier Neural Operator) — คำว่า "Fourier" อยู่ในชื่อเลย · มันทำงานโดยแปลงแผนที่ควันไปเป็น "โลกของคลื่นความถี่" ทำงานที่นั่น แล้วแปลงกลับ · ถ้าไม่เข้าใจว่า Fourier transform ทำอะไร จะไม่มีทางเข้าใจว่าทำไม AFNO "เห็นทั้งแผนที่พร้อมกัน" (global receptive field) ซึ่งเป็นเหตุผลที่มันเหมาะกับการพัดพาควันระยะไกลข้ามพรมแดน · บทนี้ปูเครื่องมือ ส่วน 3.2 จะเอาไปประกอบเป็นนิวรอล

1. ไอเดียหลัก — ทุกสัญญาณคือผลรวมของคลื่น ★

หัวใจของ Fourier คือข้อความเดียว: สัญญาณอะไรก็ตาม สามารถเขียนเป็นผลรวมของคลื่นไซน์/โคไซน์หลายความถี่ได้

อุปมาที่ดีที่สุด — คอร์ดเปียโน: เมื่อคุณกดคอร์ดเดียว เสียงที่ออกมาคือ "เสียงรวม" ที่ซับซ้อน · แต่หูที่ฝึกมา (หรือ Fourier) แยกออกได้ว่า "นี่คือโน้ต โด + มี + ซอล ผสมกัน" · Fourier transform = เครื่องแยกคอร์ดออกเป็นโน้ตเดี่ยว ๆ — บอกว่าสัญญาณมีคลื่นความถี่ไหนบ้าง แต่ละความถี่แรงแค่ไหน

2. โดเมนเวลา/พื้นที่ vs โดเมนความถี่ ★

ข้อมูลเดียวกัน มองได้ 2 มุม · Fourier transform คือ "ประตู" ที่สลับไปมาระหว่างสองมุมนี้ (ไม่เสียข้อมูล — แปลงกลับได้):

โดเมนแกนนอนคือบอกอะไร
เวลา/พื้นที่ (signal เดิม)เวลา หรือ ตำแหน่ง"ค่าเท่าไรที่จุดไหน"
ความถี่ (spectrum)ความถี่"มีคลื่นความถี่ไหนบ้าง แรงแค่ไหน"

สัญญาณคลื่นไซน์เดียวในโดเมนเวลา = แท่งเดียวในโดเมนความถี่ (ความถี่นั้น แรงเท่านั้น) · สัญญาณซับซ้อน = หลายแท่ง

3. อ่านสเปกตรัม — ความถี่ต่ำ vs สูง

  • ความถี่ต่ำ (low frequency): คลื่นช้า กว้าง = "ภาพรวมใหญ่ ๆ" ที่เปลี่ยนช้า เช่น แนวโน้มฝุ่นทั่วทั้งภาค
  • ความถี่สูง (high frequency): คลื่นถี่ = "รายละเอียดเล็ก ขอบคม" ที่เปลี่ยนเร็ว เช่น กลุ่มควันเฉพาะจุด
เชื่อมกับฝุ่น: สนาม PM2.5 ทั่วภาคเหนือ = ความถี่ต่ำคือ "ม่านควันใหญ่คลุมทั้งภาค", ความถี่สูงคือ "หย่อมควันเข้มเฉพาะจุดใกล้ไฟ" · Fourier ให้เราจัดการสองสเกลนี้แยกกันได้

4. FFT คืออะไร — ทำไมถึงเร็ว

DFT (Discrete Fourier Transform) = สูตรคำนวณ Fourier สำหรับข้อมูลไม่ต่อเนื่อง (ตัวเลขในกริด, จาก 2.1) · แต่คำนวณตรง ๆ ช้ามาก (∝ N²)

FFT (Fast Fourier Transform) = อัลกอริทึมอัจฉริยะที่คำนวณ DFT ตัวเดียวกัน แต่เร็วขึ้นมหาศาล (∝ N·log N) ด้วยการแบ่งปัญหาเป็นครึ่ง ๆ ซ้ำ ๆ · นี่คือเหตุผลที่ Fourier ใช้งานจริงได้

วิธีความเร็วN = 1 ล้านจุด
DFT ตรง ๆ∝ N²~10¹² operations (ช้ามาก)
FFT∝ N·log N~2×10⁷ (เร็วกว่าหลายหมื่นเท่า)
ไม่ต้องรู้ข้างในของ FFT: คุณไม่ต้อง implement FFT เอง — PyTorch/NumPy มี fft() ให้ · แค่รู้ว่ามัน "แปลงไปโดเมนความถี่ได้เร็ว" และ "แปลงกลับได้ด้วย ifft()" ก็พอใช้งาน AFNO

5. 2D FFT — สำหรับภาพและแผนที่ ★

FFT ขยายเป็น 2 มิติได้ตรง ๆ · แผนที่ฝุ่น (lat × lon, จาก 2.1) เป็น "ภาพ" → 2D FFT แยกมันเป็นคลื่นใน 2 ทิศ (แนวนอน+แนวตั้ง)

ในสเปกตรัม 2D: ตรงกลาง = ความถี่ต่ำ (โครงสร้างใหญ่), ออกไปขอบ = ความถี่สูง (รายละเอียดเล็ก) · นี่คือรูปแบบที่ AFNO จะเข้าไป "แก้ไข" (3.2)

6. คุณสมบัติวิเศษ — convolution กลายเป็นการคูณ ★★

นี่คือเหตุผลที่แท้จริงว่าทำไม Fourier มีค่าต่อ deep learning · มีทฤษฎีบทชื่อ Convolution Theorem:

convolution ในโดเมนพื้นที่  =  การคูณธรรมดา ในโดเมนความถี่
f ∗ g  ⟷  F̂ · Ĝ  (คูณทีละความถี่)

จำ convolution จาก 0.5 (CNN) ได้ไหม? มันคือ "เลื่อน filter ทับทุกจุด" — แพงเมื่อ filter ใหญ่ · แต่ในโดเมนความถี่ มันกลายเป็นแค่การคูณ (เร็วมาก) · ดังนั้น:

ทำ convolution ใหญ่ ๆ = FFT → คูณ → iFFT  (เร็วกว่า convolve ตรง ๆ มาก)
ทำไมนี่สำคัญกับควันข้ามแดน: CNN ธรรมดา (0.5) มี receptive field เล็ก — เห็นแค่บริเวณรอบ ๆ → ตามควันที่พัดไกลข้ามจังหวัด/ประเทศไม่ทัน (ต้องซ้อนหลายชั้น) · Fourier ทำ "convolution ขนาดเท่าทั้งภาพ" ได้ในก้าวเดียว → มองเห็นความสัมพันธ์ระยะไกลทั้งแผนที่พร้อมกัน (global receptive field) · เหมาะกับฟิสิกส์การพัดพาควันระยะไกลอย่างยิ่ง

7. ทำไมเชื่อมกับ HazeNet/AFNO ★★

AFNO (3.2) ใช้ Fourier เป็นหัวใจ · loop การทำงานคือ:

  1. เอาแผนที่อากาศ/ควัน → FFT ไปโดเมนความถี่
  2. นิวรอลเรียน "ตัวคูณ" ต่อแต่ละความถี่ (นี่คือ convolution ที่เรียนได้ + global)
  3. iFFT กลับมาเป็นแผนที่
บทบาทใน HazeNet: AFNO เป็น backbone ที่ทำนายสัมประสิทธิ์ g, D (จาก 2.3/2.5) จากสนามอากาศ · เราเลือก AFNO เพราะ (1) global receptive field เหมาะกับการพัดพาระยะไกล, (2) เร็ว/ประหยัด GPU (สำคัญกับข้อจำกัด 1 GPU), (3) เป็น backbone ของ FourCastNet ที่พิสูจน์แล้วว่าเก่งเรื่องสนามอากาศ · สำคัญ: AFNO ทำงานกับ "อากาศ" เพื่อออกสัมประสิทธิ์ — ไม่ได้ทำ FFT บนตัว C โดยตรง (ถ้าทำจะพัง superposition เพราะ FFT+คูณ+nonlinearity บน C ไม่เชิงเส้น) · กฎทองจาก 2.5 ยังคุมอยู่

8. โค้ด torch.fft (ลองเล่นได้)

import torch

# สัญญาณ 1D: คลื่นช้า + คลื่นเร็ว
t = torch.linspace(0, 1, 128)
signal = torch.sin(2*torch.pi*3*t) + 0.5*torch.sin(2*torch.pi*20*t)

spectrum = torch.fft.fft(signal)        # ไปโดเมนความถี่
power = spectrum.abs()                   # ความแรงแต่ละความถี่
# power จะมียอดที่ index 3 และ 20 (= ความถี่ของสองคลื่น) ✓

back = torch.fft.ifft(spectrum).real    # แปลงกลับ = signal เดิม
print(torch.allclose(signal, back, atol=1e-5))   # True (ไม่เสียข้อมูล)

# 2D สำหรับแผนที่
fmap = torch.fft.fft2(dust_map)         # (lat, lon) -> สเปกตรัม 2D
dust_back = torch.fft.ifft2(fmap).real  # กลับ
ลองเอง: รันโค้ดนี้แล้วดู power — จะเห็นยอดพุ่งที่ตำแหน่ง 3 กับ 20 พอดี = Fourier "แยกคอร์ดออกเป็นโน้ต" ได้จริง · นี่คือ intuition ทั้งหมดของบทนี้ในโค้ด 5 บรรทัด

9. Cheat sheet + แบบฝึกหัด

สรุปจำ:
  • Fourier transform: แยกสัญญาณเป็น "ผลรวมของคลื่นความถี่" (= แยกคอร์ดเป็นโน้ต)
  • 2 โดเมน: เวลา/พื้นที่ (ค่าที่จุด) ↔ ความถี่ (มีคลื่นไหนแรงแค่ไหน) · แปลงกลับได้ ไม่เสียข้อมูล
  • ความถี่ต่ำ=ภาพรวมใหญ่ · ความถี่สูง=รายละเอียดเล็ก/ขอบคม
  • FFT = อัลกอริทึมคำนวณ Fourier เร็ว (N·logN แทน N²) · ใช้ torch.fft
  • 2D FFT = สำหรับภาพ/แผนที่ (lat×lon)
  • ★★ Convolution Theorem: convolution พื้นที่ = การคูณในความถี่ → ทำ conv ใหญ่ได้เร็ว + global
  • AFNO: FFT → เรียนตัวคูณต่อความถี่ → iFFT · ให้ global receptive field (เหมาะการพัดพาไกล)
  • กฎทอง (2.5): AFNO ทำกับ "อากาศ" เพื่อออกสัมประสิทธิ์ g,D · ไม่ทำ FFT บน C เอง (จะพัง superposition)

แบบฝึกหัด 1 — อธิบายด้วยอุปมา "คอร์ดเปียโน" ว่า Fourier transform ทำอะไร

เฉลย

คอร์ดเปียโน = เสียงรวมที่ซับซ้อน · Fourier transform = เครื่องที่ฟังคอร์ดแล้วแยกออกว่าเป็นโน้ตอะไรบ้าง (โด+มี+ซอล) แต่ละโน้ตดังแค่ไหน · กับสัญญาณก็เหมือนกัน: รับสัญญาณซับซ้อน → บอกว่าประกอบด้วยคลื่นความถี่ไหนบ้าง แต่ละความถี่แรงเท่าไร · และแปลงกลับ (โน้ต→คอร์ด) ได้ด้วย ifft โดยไม่เสียข้อมูล

แบบฝึกหัด 2 — ทำไม "convolution = การคูณในโดเมนความถี่" ถึงเป็นข้อได้เปรียบของ AFNO เหนือ CNN ธรรมดาในงานควันข้ามแดน?

เฉลย

CNN ธรรมดามี receptive field เล็ก (เห็นแค่รอบ ๆ) → ต้องซ้อนหลายชั้นกว่าจะ "เห็นไกล" และยังจำกัด · ส่วน convolution ขนาดเท่าทั้งภาพแพงมากถ้าทำตรง ๆ · แต่ผ่าน Convolution Theorem มันกลายเป็นการคูณในโดเมนความถี่ (เร็ว) → AFNO ทำ "convolution ทั่วทั้งแผนที่" ในก้าวเดียว = global receptive field → จับความสัมพันธ์ระยะไกล (ควันพม่า→เชียงใหม่) ได้โดยตรง เหมาะกับฟิสิกส์การพัดพาไกล

แบบฝึกหัด 3 — ทำไม AFNO ถึงทำ FFT กับ "สนามอากาศเพื่อออกสัมประสิทธิ์ g,D" แทนที่จะทำ FFT กับ "ตัวความเข้มข้นควัน C" โดยตรง?

เฉลย

เพราะ FFT + เรียนตัวคูณ + nonlinearity เป็นกระบวนการไม่เชิงเส้น · ถ้าทำกับ C โดยตรง → AFNO(C⁽¹⁾+C⁽²⁾) ≠ AFNO(C⁽¹⁾)+AFNO(C⁽²⁾) → ทำลาย superposition (2.5) → แยกแหล่งไม่ได้ · ตามกฎทอง: ฟังก์ชันไม่เชิงเส้นทำได้เฉพาะกับสัมประสิทธิ์ (g,D ที่มาจากอากาศ) · AFNO จึงรับ "อากาศ" (ไม่เชิงเส้นได้เต็มที่) ออก g,D แล้วเอา g,D ไปคูณ C แบบเชิงเส้น = conditionally-linear (3.5)

จบ 3.1 แล้วคุณควร: เข้าใจว่า Fourier transform = แยกสัญญาณเป็นคลื่นความถี่ (แยกคอร์ดเป็นโน้ต), รู้จักโดเมนความถี่ (ต่ำ=ใหญ่/สูง=ละเอียด), FFT (เร็ว N·logN), 2D FFT สำหรับแผนที่, และที่สำคัญสุด — Convolution Theorem ที่ทำให้ทำ convolution global ได้เร็ว = รากฐานของ AFNO และเหตุผลที่มันเหมาะกับการพัดพาควันระยะไกล
