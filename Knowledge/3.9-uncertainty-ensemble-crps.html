1.2 ประเภทของ Uncertainty

Aleatoric Uncertainty

ความไม่แน่นอนจาก ธรรมชาติของข้อมูล — ลดไม่ได้ไม่ว่าจะมีข้อมูลมากแค่ไหน

ตัวอย่าง: ลม chaotic ที่ไม่มีทางทำนายได้ 100% · noise ในการวัด PM2.5

Epistemic Uncertainty

ความไม่แน่นอนจาก โมเดลที่ไม่รู้พอ — ลดได้ถ้ามีข้อมูลเพิ่ม

ตัวอย่าง: โมเดลไม่เคยเห็น scenario ไฟป่าขนาดใหญ่ใน Laos → ทำนายไม่มั่นใจ

ทำไมแยก 2 ประเภท: ถ้า uncertainty สูงเพราะ aleatoric → เก็บข้อมูลเพิ่มไม่ช่วย · ถ้าสูงเพราะ epistemic → เพิ่ม training data จาก scenario นั้นช่วยได้ · แยกแล้วรู้ว่าต้องแก้อะไร

2. Calibration — Uncertainty ที่ "ซื่อสัตย์"

2.1 Calibration คืออะไร

Calibrated = ถ้าบอกว่า "90% prediction interval" แล้วค่าจริงต้องอยู่ในนั้น 90% ของเวลา

บอก 90-CI → จริงอยู่ใน 90%

✗ Overconfident (แย่)
บอก 90-CI แคบ → จริงอยู่ใน 60%

⚠ Underconfident (สูญเปล่า)

2.2 วัด Calibration ยังไง — Reliability Diagram

def reliability_diagram(y_true, quantile_preds, quantiles):
    """
    y_true:        (N,) ค่าจริง
    quantile_preds:(N, Q) ทำนาย quantile แต่ละตัว
    quantiles:     [0.1, 0.25, 0.5, 0.75, 0.9]
    คำนวณ: สัดส่วนที่ y_true < quantile_pred ควรเท่ากับ quantile นั้น
    """
    coverage = []
    for i, q in enumerate(quantiles):
        frac = (y_true < quantile_preds[:, i]).float().mean()
        coverage.append(frac.item())
        print(f"Q{q*100:.0f}: ทำนาย {q:.2f} | จริง {frac:.2f}")
    # Calibrated = ทุกบรรทัด ทำนาย ≈ จริง
    return coverage

3. วิธี ① — Ensemble

①
Ensemble เร็วปานกลาง

แนวคิด: รัน N forward passes แต่ละรอบได้ค่าต่างกันนิดหน่อย → spread ของ N predictions คือ uncertainty

3.1 MC Dropout — ฟรีจาก Dropout ที่มีอยู่แล้ว

ปกติ dropout ปิดตอน inference (model.eval()) → ผลเดิมทุกรอบ
MC Dropout: เปิด dropout ตอน inference (model.train()) → แต่ละรอบ node ถูกปิดต่างกัน → ผลต่างกัน

import torch

def mc_dropout_predict(model, C, met, n_samples=50):
    """
    model: HazeNet ที่มี Dropout layer อยู่แล้ว
    n_samples: จำนวน forward passes
    """
    model.train()   # เปิด dropout ตอน inference!

    preds = []
    with torch.no_grad():
        for _ in range(n_samples):
            C_pred = model(C, met)   # แต่ละรอบ dropout ต่างกัน
            preds.append(C_pred)

    preds = torch.stack(preds)  # (n_samples, H, W)

    mean   = preds.mean(0)     # (H, W) prediction หลัก
    std    = preds.std(0)      # (H, W) uncertainty
    q10    = preds.quantile(0.1, dim=0)
    q90    = preds.quantile(0.9, dim=0)

    model.eval()  # reset กลับ
    return mean, std, q10, q90

# ตัวอย่างผลลัพธ์
mean, std, q10, q90 = mc_dropout_predict(hazenet, C_now, met_forecast)
print(f"PM2.5 at CM: {mean[48,35]:.1f} ± {std[48,35]:.1f} μg/m³")
# → PM2.5 at CM: 145.3 ± 18.7 μg/m³
print(f"90-CI: [{q10[48,35]:.0f}, {q90[48,35]:.0f}] μg/m³")
# → 90-CI: [113, 178] μg/m³

ข้อดี / ข้อจำกัด MC Dropout:

รายละเอียด
✓ ฟรีไม่ต้องเปลี่ยนโมเดลถ้ามี dropout อยู่แล้ว
✓ เร็วn_samples=50 → 50× forward pass (วินาทีบน A100)
⚠ Calibrationไม่รับประกัน calibrated — อาจ overconfident
⚠ ต้องมี Dropoutถ้า architecture ไม่มี dropout ต้องเพิ่มก่อน

3.2 Deep Ensemble — แม่นและ calibrated ที่สุด

Train โมเดล N ตัวอิสระจากกัน (random init ต่างกัน) → รวม prediction

def deep_ensemble_predict(models, C, met):
    """
    models: list ของ HazeNet N ตัว (trained independently)
    """
    preds = []
    with torch.no_grad():
        for model in models:
            model.eval()
            preds.append(model(C, met))

    preds  = torch.stack(preds)  # (N, H, W)
    mean   = preds.mean(0)
    var    = preds.var(0)        # epistemic uncertainty
    return mean, var

# ตัวอย่าง: 5 models → สร้าง 5 สมาชิก ensemble
ensemble_models = [HazeNet().to(device) for _ in range(5)]
# ... train แต่ละตัวแยก ...
mean_pred, var_pred = deep_ensemble_predict(ensemble_models, C_now, met_forecast)
ข้อจำกัด Deep Ensemble: 5 models = 5× RAM + 5× training time · บน A100 ที่จำกัด: ใช้ 3–5 models สูงสุด · อาจไม่ feasible ถ้าโมเดลใหญ่มาก

3.3 Physical Ensemble — เปลี่ยน met forcing

# วิธีที่ 3: เปลี่ยน meteorology input เล็กน้อย → ได้ physics-based spread
def physical_ensemble(model, C, met_base, n_members=10, met_noise=0.05):
    preds = []
    with torch.no_grad():
        for _ in range(n_members):
            met_perturbed = met_base * (1 + met_noise * torch.randn_like(met_base))
            preds.append(model(C, met_perturbed))
    return torch.stack(preds)
# spread สะท้อน sensitivity ต่อความไม่แน่นอนของ ERA5

4. วิธี ② — Quantile Regression

②
Quantile Regression เร็ว (1 pass)

แนวคิด: แทนที่จะทำนาย mean ตัวเดียว → ทำนาย Q10, Q25, Q50, Q75, Q90 พร้อมกัน
ได้ทั้ง forecast หลักและ uncertainty interval ใน forward pass เดียว

4.1 Quantile คืออะไร

Qτ(Y) = ค่าที่ "มี τ×100% โอกาส Y จะต่ำกว่านี้"

Quantileความหมายสำหรับ PM2.5ใช้ทำ
Q10 = 95 μg/m³10% โอกาส PM2.5 < 95 → มั่นใจ 90% ว่าสูงกว่า 95Lower bound ระวัง
Q50 = 145 μg/m³50% โอกาส PM2.5 < 145 (ค่า median)Forecast หลัก
Q90 = 185 μg/m³90% โอกาส PM2.5 < 185 → 10% โอกาสเกินUpper bound เสี่ยง

→ "PM2.5 พรุ่งนี้จะเป็น 145 [95–185]" บอกข้อมูลมากกว่า "145" เยอะมาก

4.2 สถาปัตยกรรม Quantile HazeNet

import torch.nn as nn

class QuantileHazeNet(nn.Module):
    """HazeNet ที่ output หลาย quantiles พร้อมกัน"""
    def __init__(self, base_hazenet, quantiles=(0.1, 0.25, 0.5, 0.75, 0.9)):
        super().__init__()
        self.base   = base_hazenet
        self.quantiles = quantiles
        self.n_q    = len(quantiles)

        # เพิ่ม head สำหรับ quantile output
        self.q_head = nn.Conv2d(
            in_channels=1,          # C_pred 1 channel
            out_channels=self.n_q,   # n_q quantiles
            kernel_size=1
        )

    def forward(self, C, met):
        C_mean = self.base(C, met)           # (H, W) — mean prediction
        C_mean_4d = C_mean.unsqueeze(0).unsqueeze(0)  # (1, 1, H, W)
        q_preds = self.q_head(C_mean_4d)     # (1, n_q, H, W)
        return q_preds.squeeze(0)            # (n_q, H, W)
        # q_preds[0] = Q10, [1] = Q25, [2] = Q50, [3] = Q75, [4] = Q90

# inference: 1 forward pass ได้ทุก quantile
model_q = QuantileHazeNet(hazenet)
q_preds = model_q(C_now, met_forecast)  # (5, 128, 128)
print(f"Q10={q_preds[0,48,35]:.1f}, Q50={q_preds[2,48,35]:.1f}, Q90={q_preds[4,48,35]:.1f}")
# → Q10=113.2, Q50=145.3, Q90=177.8

5. Pinball Loss — วิธีเทรน Quantile Regression

5.1 ทำไมไม่ใช้ MSE

ปัญหา: ถ้า train ด้วย MSE → โมเดลเรียนแค่ mean → ทุก quantile ออก mean เหมือนกัน → ไม่มีข้อมูล uncertainty
ต้องใช้ loss ที่บังคับให้แต่ละ quantile ประมาณ quantile ที่ถูกต้อง

5.2 Pinball Loss (Quantile Loss)

สำหรับ quantile τ ∈ (0,1) และ error e = y_true − ŷ_q:

-- Pinball Loss สำหรับ quantile τ --
L_τ(y, ŷ) = τ · max(y − ŷ, 0)  +  (1−τ) · max(ŷ − y, 0)
              ↑                       ↑
        โทษเมื่อ y > ŷ           โทษเมื่อ ŷ > y
        (ทำนายต่ำเกิน)            (ทำนายสูงเกิน)

เมื่อ τ = 0.9: โทษสูงเมื่อ under-predict (y > ŷ) → Q90 จะสูงขึ้น
เมื่อ τ = 0.1: โทษสูงเมื่อ over-predict (ŷ > y) → Q10 จะต่ำลง

ตัวอย่างตัวเลข: y_true = 150, ŷ = 130

Quantile τe = y−ŷ = +20L_τผลลัพธ์
τ = 0.10max(+20,0)=20 · 0.1 + max(−20,0)=0 · 0.92.0โทษน้อย (Q10 ที่ 130 ก็ OK)
τ = 0.5020 · 0.5 + 0 · 0.510.0โทษกลาง (Q50 ควรใกล้ median)
τ = 0.9020 · 0.9 + 0 · 0.118.0โทษมาก → Q90 จะถูกดันสูงขึ้น

→ gradient จาก L_0.9 ดัน ŷ_Q90 สูงขึ้น → หลังเทรนแล้ว Q90 จะมีค่าสูงกว่า Q50 → ถูกต้อง

def pinball_loss(pred_q, target, quantiles):
    """
    pred_q:    (n_q, H, W) — prediction แต่ละ quantile
    target:    (H, W)       — ค่าจริง PM2.5
    quantiles: [0.1, 0.25, 0.5, 0.75, 0.9]
    """
    target_q = target.unsqueeze(0).expand_as(pred_q)  # (n_q, H, W)
    tau = torch.tensor(quantiles).view(-1, 1, 1).to(pred_q.device)

    err = target_q - pred_q
    loss = torch.max(tau * err, (tau - 1) * err)  # element-wise
    return loss.mean()

# Training loop
optimizer = torch.optim.Adam(model_q.parameters(), lr=1e-3)
for batch_C, batch_y in dataloader:
    q_preds = model_q(batch_C, batch_met)  # (n_q, H, W)
    loss = pinball_loss(q_preds, batch_y, quantiles=[0.1, 0.25, 0.5, 0.75, 0.9])
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

6. วิธี ③ — CRPS (Continuous Ranked Probability Score)

③
CRPS Metric (ไม่ใช่ method)

สำคัญ: CRPS ไม่ใช่ วิธีสร้าง uncertainty — แต่คือ metric วัดว่า probabilistic forecast ดีแค่ไหน
ใช้ใน evaluation (Module 5) เพื่อเปรียบเทียบโมเดลต่างๆ

6.1 CRPS คืออะไร — ความหมายง่ายๆ

CRPS = วัด "ระยะห่าง" ระหว่าง forecast distribution F กับ ค่าจริง y

สำหรับ deterministic forecast (ไม่มี uncertainty):

CRPS(point, y) = |point − y| = MAE

CRPS ลด degeneracy เป็น MAE ถ้า forecast deterministic → ทำให้เทียบ probabilistic กับ deterministic ได้ตรงๆ

สำหรับ probabilistic forecast (ensemble หรือ quantile):

CRPS(F, y) = E[|X − y|] − ½·E[|X − X'|]
              ↑              ↑
         ความแม่น        spread ของ ensemble

term แรก = accuracy (ใกล้ truth ได้โบนัส) · term สอง = sharpness (ensemble spread ที่เหมาะสม)

6.2 คุณสมบัติที่ทำให้ CRPS ดี

คุณสมบัติความหมาย
Proper scoring rule"โกง" ไม่ได้ — ถ้าแกล้งบอก interval กว้างๆ CRPS จะแย่ลง ไม่ใช่ดีขึ้น
ลด degeneracy เป็น MAEเทียบ probabilistic กับ deterministic forecast ได้ในหน่วยเดียวกัน
Sensitive to calibrationoverconfident หรือ underconfident → CRPS สูงขึ้น
หน่วยเดียวกับ targetCRPS หน่วย μg/m³ → แปลความหมายได้ง่าย

6.3 การคำนวณ CRPS จาก Ensemble

def crps_ensemble(obs, ensemble):
    """
    obs:      (H, W) — ค่าที่วัดได้จริง
    ensemble: (N, H, W) — N ensemble members
    returns:  (H, W) — CRPS แต่ละ pixel (lower = better)
    """
    N = ensemble.shape[0]

    # Term 1: E[|X − y|] = mean absolute error vs obs
    accuracy = (ensemble - obs.unsqueeze(0)).abs().mean(0)  # (H, W)

    # Term 2: ½·E[|X − X'|] = mean spread ระหว่างสมาชิก ensemble
    spread = (ensemble.unsqueeze(0) - ensemble.unsqueeze(1)).abs()
    spread = spread.mean((0, 1)) / 2  # (H, W)

    crps = accuracy - spread  # (H, W)
    return crps

# ตัวอย่าง
crps_map = crps_ensemble(obs=pm25_truth, ensemble=hazenet_ensemble)
print(f"Mean CRPS: {crps_map.mean():.2f} μg/m³")
# → Mean CRPS: 12.43 μg/m³  (เทียบกับ MAE ~15 μg/m³)
# CRPS < MAE แปลว่า probabilistic forecast ให้ข้อมูลมากกว่า point forecast

6.4 CRPS จาก Quantile (ไม่ต้องมี ensemble)

def crps_from_quantiles(obs, q_preds, quantiles):
    """
    CRPS ≈ 2 × mean pinball loss (ทุก quantile)
    obs:        (H, W)
    q_preds:    (n_q, H, W)
    quantiles:  [0.1, 0.25, ...]
    """
    obs_exp = obs.unsqueeze(0).expand_as(q_preds)
    tau      = torch.tensor(quantiles).view(-1,1,1)

    pinball = torch.max(tau * (obs_exp - q_preds),
                         (tau-1) * (obs_exp - q_preds))

    crps_approx = 2 * pinball.mean(0)  # (H, W)
    return crps_approx

8. HazeNet ใช้อะไร — คำแนะนำและโค้ดเต็ม

8.1 คำแนะนำสำหรับ HazeNet

แนะนำ Primary: Quantile Regression
เหตุผล: 1 forward pass ได้ทั้ง Q10/Q50/Q90 → เหมาะกับ operational system · calibration ดี · asymmetric CI จับ "wildfire spike" ที่ upper tail หนักกว่าได้

แนะนำ Secondary: MC Dropout
เหตุผล: ฟรีถ้ามี Dropout อยู่แล้ว · ใช้เป็น epistemic uncertainty checker

Evaluation: CRPS
ใช้เป็น metric หลักแทน MAE เพื่อรายงาน probabilistic forecast quality

8.2 โค้ดเต็ม — Training + Inference + Evaluation

import torch
import torch.nn as nn

QUANTILES = [0.1, 0.25, 0.5, 0.75, 0.9]

# ─── Loss ────────────────────────────────────────
def pinball_loss(q_preds, target, quantiles=QUANTILES):
    target_exp = target.unsqueeze(0).expand_as(q_preds)
    tau = torch.tensor(quantiles, device=q_preds.device).view(-1,1,1)
    err = target_exp - q_preds
    return torch.max(tau * err, (tau - 1) * err).mean()

# ─── Training ─────────────────────────────────────
def train_quantile_hazenet(model_q, dataloader, n_epochs=50):
    opt = torch.optim.Adam(model_q.parameters(), lr=1e-3)
    for epoch in range(n_epochs):
        total_loss = 0
        for C, met, y_true in dataloader:
            q_preds = model_q(C, met)              # (n_q, H, W)
            q_preds_sorted, _ = q_preds.sort(0)   # fix crossing
            loss = pinball_loss(q_preds_sorted, y_true)
            opt.zero_grad(); loss.backward(); opt.step()
            total_loss += loss.item()
        if epoch % 10 == 0:
            print(f"Epoch {epoch}: pinball_loss={total_loss/len(dataloader):.4f}")

# ─── Inference ────────────────────────────────────
def forecast_with_uncertainty(model_q, C_now, met_forecast):
    model_q.eval()
    with torch.no_grad():
        q_preds = model_q(C_now, met_forecast)
        q_preds_sorted, _ = q_preds.sort(0)   # (n_q, H, W)

    q10, q25, q50, q75, q90 = q_preds_sorted

    # สำหรับสถานีเชียงใหม่
    cm = (48, 35)
    print(f"PM2.5 Forecast ที่เชียงใหม่:")
    print(f"  Q50 (median) : {q50[cm]:.1f} μg/m³")
    print(f"  80-CI (Q10-Q90): [{q10[cm]:.1f}, {q90[cm]:.1f}]")
    print(f"  50-CI (Q25-Q75): [{q25[cm]:.1f}, {q75[cm]:.1f}]")
    return q_preds_sorted

# ─── Evaluation ───────────────────────────────────
def evaluate_crps(model_q, test_loader):
    crps_total, n = 0.0, 0
    with torch.no_grad():
        for C, met, y_true in test_loader:
            q_preds = model_q(C, met)
            q_preds_sorted, _ = q_preds.sort(0)
            crps = crps_from_quantiles(y_true, q_preds_sorted, QUANTILES)
            crps_total += crps.mean().item()
            n += 1
    print(f"CRPS = {crps_total/n:.3f} μg/m³")
    return crps_total / n

9.2 Uncertainty เป็น "ความมั่นใจของ Fire Alert"

# Fire alert probability at station
def fire_alert_probability(q_preds, threshold=150.0):
    """ประมาณ P(PM2.5 > threshold) จาก quantile predictions"""
    q10, q25, q50, q75, q90 = q_preds
    # linear interpolation ระหว่าง quantiles
    # ถ้า Q90 < threshold → P ≈ 0 · ถ้า Q10 > threshold → P ≈ 1
    above_q90 = (q90 > threshold).float() * 0.95
    above_q75 = (q75 > threshold).float() * 0.75
    above_q50 = (q50 > threshold).float() * 0.50
    return (above_q90 + above_q75 + above_q50) / (0.95+0.75+0.5)

prob_alert = fire_alert_probability(q_preds_sorted)
print(f"P(PM2.5 > 150 μg/m³) ที่เชียงใหม่ = {prob_alert[48,35]:.1%}")
# → P(PM2.5 > 150 μg/m³) ที่เชียงใหม่ = 72.3%

Q1: Calibration Check

ทดสอบ 1000 วัน: โมเดลบอก "90% prediction interval" ปรากฏว่าค่าจริงอยู่ในนั้น 650 วัน จาก 1000 วัน
โมเดลนี้ over- หรือ under-confident? และควรแก้อย่างไร?

ดูเฉลย

Overconfident — บอก 90% CI แต่ครอบคลุมแค่ 65% → interval แคบเกินไป
วิธีแก้: ① ใช้ temperature scaling (ขยาย interval ด้วย factor) ② เพิ่ม regularization ระหว่าง train ③ ใช้ conformal prediction เป็น post-processing calibration ④ ตรวจ pinball loss ว่า train ด้วย quantile ถูกต้องหรือไม่

Q2: Pinball Loss Calculation

y_true = 180 μg/m³, ŷ_Q90 = 160 μg/m³
คำนวณ Pinball Loss สำหรับ τ = 0.9
แล้วตอบว่า gradient จะดัน Q90 ขึ้นหรือลง?

ดูเฉลย

e = y_true − ŷ = 180 − 160 = +20 (under-predict)
L_0.9 = max(0.9 × 20, (0.9−1) × 20) = max(18, −2) = 18
∂L/∂ŷ = −τ = −0.9 เมื่อ y > ŷ → gradient เป็นลบ → optimizer update: ŷ += lr × 0.9
→ Q90 ถูกดันสูงขึ้น → ครั้งหน้าจะ predict สูงกว่า 160 เพื่อ cover y=180 ได้

Q3: CRPS vs MAE

Deterministic model: MAE = 20 μg/m³
Quantile model: CRPS = 14 μg/m³
แปลความหมายการเปรียบเทียบนี้ได้ว่าอะไร?

ดูเฉลย

CRPS ลด degeneracy เป็น MAE สำหรับ deterministic forecast — ดังนั้นเทียบหน่วยเดียวกันได้ตรงๆ
CRPS 14 < MAE 20: probabilistic forecast ให้ข้อมูลมีคุณค่ากว่า point forecast
เหตุผล: ถึงแม้ Q50 อาจมี error ใกล้เคียง MAE แต่การรู้ว่า "จะสูงถึง Q90 หรือเปล่า" ช่วยลด penalty ใน CRPS
ใน report: "probabilistic forecast achieves CRPS of 14 μg/m³ vs deterministic MAE of 20 μg/m³ (30% improvement)"

Q4: Asymmetric Uncertainty

ทำไม wildfire PM2.5 forecast ถึงมักมี upper tail หนักกว่า lower tail? (กล่าวคือ Q90−Q50 > Q50−Q10)
และ quantile regression จับ asymmetry นี้ได้ไหม? MC Dropout?

ดูเฉลย

เหตุผล asymmetry: PM2.5 มีขอบเขตล่าง = 0 (ไม่ติดลบ) แต่ขอบเขตบนไม่จำกัด · ไฟป่าขนาดใหญ่หรือลมพัดตรงมาอาจทำให้ PM2.5 พุ่งสูงผิดคาดอย่างมาก → distribution บิดเบ้ขวา (right-skewed)
Quantile regression: จับได้ — เรียนจากข้อมูลจริงที่ asymmetric โดยตรง → Q90−Q50 > Q50−Q10 ได้อัตโนมัติ
MC Dropout: จับได้บางส่วน — spread มักสมมาตรกว่า เพราะ dropout เป็น symmetric perturbation → asymmetry น้อยกว่าความเป็นจริง

Q5: Proper Scoring Rule

โมเดล A บอก "90-CI: [100, 400]" (กว้างมาก) เพื่อให้ครอบคลุมค่าจริงเกือบทุกครั้ง
โมเดล B บอก "90-CI: [130, 170]" (แคบแต่ calibrated จริง)
CRPS ให้คะแนนโมเดลไหนดีกว่า และทำไม?

ดูเฉลย

โมเดล B ได้ CRPS ดีกว่า
เหตุผล: CRPS เป็น proper scoring rule — term spread (½E[|X−X'|]) ใน CRPS ลงโทษ ensemble ที่กว้างเกินไปโดยตรง
โมเดล A: accuracy term ต่ำ (ครอบ truth เสมอ) แต่ spread term สูงมาก (range 300 μg/m³) → CRPS รวมสูง
โมเดล B: accuracy ดี + spread ต่ำ → CRPS ต่ำกว่า
นี่คือเหตุผลที่ CRPS ดีกว่า "interval score" ธรรมดา — ไม่สามารถโกงด้วยการขยาย interval
