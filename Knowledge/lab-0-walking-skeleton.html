Lab 0 — Walking Skeleton 🟢 ลงมือจริง

ทำทุกอย่าง "เล็กจิ๋วแต่ครบวงจร" ให้ pipeline วิ่งทะลุต้นจนจบก่อน — แล้วค่อยขยาย

ภาคปฏิบัติของโมดูล 4–5 · เป้าหมาย: พิสูจน์ว่า "ท่อทั้งเส้นไม่รั่ว" ไม่ใช่ได้ผลลัพธ์สวย · รันบน CPU/T4 ฟรีเป็นหลัก (แทบไม่แตะ A100)

ปรัชญาเดียวที่ต้องยึด: "ให้มันวิ่งครบวงจรก่อน แล้วค่อยทำให้ใหญ่/ดี"

อย่าทำส่วนใดส่วนหนึ่งให้สมบูรณ์แบบก่อน · จงต่อ "ชิ้นบาง ๆ" ของทุกขั้นตอน (ดึงข้อมูล→datacube→โมเดล→loss→metric) เข้าด้วยกันให้รันไม่ error ภายในวันเดียว · บั๊กการเชื่อมต่อ (integration bug) คือสิ่งที่กินเวลามากที่สุดในโปรเจกต์จริง — skeleton นี้ทำให้เจอมันเร็วและถูก ก่อนเปลือง A100

  • นิยาม master grid จิ๋ว (lat 19.5→18, lon 98→100, step 0.1)

6
โมเดลจิ๋ว + Overfit Test 4.5/0.2/0.3
~half day · จุดพิสูจน์ใหญ่

โมเดล CNN จิ๋ว 2–3 ชั้น (ยังไม่ต้องเป็น 3.5) → เทรนให้ overfit 1 batch ให้ loss ลงเข้าใกล้ 0

ทำไม overfit test สำคัญที่สุด: ถ้าโมเดลจำ 1 batch ไม่ได้ (loss ไม่ลง) = มีบั๊กแน่ (gradient ไม่ไหล / target ผิด / loss ผิด) · นี่คือเทสต์ที่ถูกและเร็วที่สุดที่บอกว่า "วงจรเทรนทำงาน" · ผ่านอันนี้ก่อน ค่อยคิดเรื่องอื่น
เสร็จเมื่อ: เทรน 1 batch ซ้ำ ๆ แล้ว loss ลงเข้าใกล้ 0 · loss.backward() + opt.step() ทำงาน · เปิด bf16 ได้ (ถ้าใช้ GPU)
7
train สั้น + metric + 1 baseline 5.1/5.3
~half day
  • เทรนจริงสั้น ๆ (5–10 epoch) บน train จิ๋ว → ทำนาย test
  • คำนวณ MAE/RMSE (masked เฉพาะจุดสถานี, 5.1)
  • เทียบ persistence (ฟรี): pred = ค่าล่าสุด → คิด skill score
เสร็จเมื่อ: มีตัวเลข RMSE ของโมเดล vs persistence ออกมา (ผลจะยังไม่สวย — ไม่เป็นไร! เป้าคือ "ตัวเลขออกได้ครบวงจร")

ลองเครื่องมือ rigor 1 ชิ้น 5.4/5.6
~2–3 ชม.

พิสูจน์ว่า "ท่อประเมิน" ก็ทำงาน — ทำอย่างใดอย่างหนึ่ง:

  • bootstrap CI ของ RMSE (ฟรี, ไม่เทรนใหม่) — รายงาน "RMSE [95% CI]"
  • หรือ ablation จิ๋ว: ตัด emission channel ออก → เทรนซ้ำ → ดู delta
เสร็จเมื่อ: ได้ CI หรือ delta ออกมา 1 ตัว = ครบทั้ง "ท่อทำ + ท่อพิสูจน์"

หัวใจ: ขยายทีละแกนเดียว แล้วเช็คว่ายังเขียวอยู่ ก่อนขยายแกนถัดไป · ถ้าพังจะรู้ทันทีว่าเพราะแกนไหน

ลำดับขยายอะไรจาก → เป็นเริ่มแตะ A100?
1เวลา1 สัปดาห์ → 1 ฤดูเต็มยัง (T4)
2channel4 → +AOD +PBL +precip +RHยัง

โยง 4.5: สังเกตว่า A100 เริ่มแตะแค่ช่วงปลายบันได · ทั้ง Lab 0 + ขยาย 2 ขั้นแรกทำบนของฟรีได้หมด → ประหยัดงบ A100 ไว้ให้ run จริงตอนทุกอย่างนิ่งแล้ว (ตรงตามแผนงบ 15 ชม.)

10. กับดัก + caveat ของเฟสนี้

อย่าหลงทาง:
  • อย่าตัดสินวิทยาศาสตร์จาก skeleton — โมเดลจิ๋ว/ข้อมูล 1 สัปดาห์ จะให้ผลห่วย นั่นปกติ · เป้าคือ "ท่อไม่รั่ว" ไม่ใช่ "แม่น"
  • อย่าทำ stage ใด stage หนึ่งให้สมบูรณ์แบบ — เช่น เสีย 3 วันจูน regrid ให้เป๊ะ ทั้งที่ยังไม่รู้ว่าปลายท่อต่อได้ไหม · "พอใช้ได้" แล้วไปต่อ
  • อย่าเพิ่งแตะ A100 — ทั้ง Lab 0 อยู่บนของฟรี (โยง 4.5: เดบั๊กบนของถูก)
  • commit/เซฟทุก stage — skeleton ที่เขียวแล้วคือสมบัติล้ำค่า อย่าทำหาย
ถ้าติดขัด: stage ไหนพัง ให้แยกทดสอบเฉพาะ stage นั้นด้วยข้อมูลปลอม (เช่น np.random แทน datacube จริง) เพื่อแยกว่าเป็นบั๊กของ "ข้อมูล" หรือ "โค้ด" — เทคนิคเดียวกับ oracle/ablation แต่ใช้ debug

✅ Definition of Done — Lab 0 ทั้งหมด

  • ☐ Stage 0–8 เขียวครบ · รันสคริปต์เดียวจบ end-to-end
  • ☐ overfit test ผ่าน (loss → ~0 บน 1 batch)
  • ☐ มีตัวเลข RMSE โมเดล vs persistence
  • ☐ มี CI หรือ ablation delta อย่างน้อย 1 ตัว
  • ☐ ทำทั้งหมดบน CPU/T4 (ยังไม่แตะ A100)
  • ☐ เก็บ config + commit โค้ดที่เขียวแล้ว
