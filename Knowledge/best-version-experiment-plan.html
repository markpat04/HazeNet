1. ช่องว่างวิจัย + วิทยานิพนธ์ความใหม่

วางภูมิทัศน์ปัจจุบันก่อน เพื่อเห็นว่าเราเสียบช่องไหน:

ตระกูลตัวอย่าง SOTAเก่งอะไรทำ "attribution + inversion" ไม่ได้เพราะ
ML weatherGraphCast, Pangu, FourCastNet, GenCastพยากรณ์อากาศแม่นมาก/เร็วมากเป็น black box · ไม่ผูกกับแหล่งปล่อย

วิทยานิพนธ์ความใหม่ (3 เสา):
  1. Attribution-native: ฝัง superposition/linearity-in-emissions (หัวใจ 2.5→3.5) ไว้ในสถาปัตยกรรม → อ่านสัดส่วน "ควันจากแต่ละแหล่ง" ออกมาได้โดยตรงจาก forward model ไม่ใช่ XAI ติดทีหลัง
  2. Differentiable inversion: เพราะทั้งโมเดล differentiable → แก้ปัญหาผกผัน (3.7) ประเมิน emission จาก observation ได้ (แทน 4D-Var/adjoint ของ CTM ที่แพง)
  3. Data-driven แทน inventory ที่แน่นอน: เรียน source-receptor relationship จากข้อมูลจริง + ฟิสิกส์ ในภูมิภาคที่สถานี sparse (SEA transboundary) — โจทย์ที่ CTM ทำแพงและ ML ทั่วไปทำไม่ได้
Headline claim: "HazeNet เป็นโมเดลแรกที่รวม พยากรณ์ + source attribution + emission inversion ของ transboundary haze ไว้ในนิวรัลออเปอเรเตอร์เดียวที่ differentiable + physics-structured + probabilistic — ให้ skill พยากรณ์เทียบเท่า SOTA AQ ขณะที่เปิดความสามารถ attribution/inversion ที่โมเดล ML เดิมทำไม่ได้ และพิสูจน์ความถูกต้องของ attribution ด้วย controlled CTM digital-twin"

2. เทคโนโลยีใหม่ — Attribution-Native Differentiable Transport Operator

กลไกหลัก: operator ที่ เคอร์เนลการขนส่ง (transport kernel) เป็นฟังก์ชันของสภาพอากาศ (ลม/PBL/ฝน) แต่ เชิงเส้นเทียบกับ emission · ผลคือ "การมีส่วนร่วมของแต่ละแหล่ง" บวกกันเป็นผลรวมพอดี (superposition) → 3 ความสามารถจากโมเดลเดียว:

  • Forward = พยากรณ์ PM2.5 รวม (เทียบ Aurora-Air/CAMS)
  • อ่านแต่ละสาขา = source attribution (เทียบ CTM tagged-tracer)
  • ย้อน gradient ผ่านโมเดล = emission inversion (เทียบ adjoint/4D-Var)
ทำไมนี่ถึง "ใหม่": รวมแนวคิด Neural Operator (FNO/DeepONet — เรียน operator ระหว่างฟังก์ชัน) + differentiable physics (NeuralGCM) + source-receptor linearity (จาก CTM tagged-tracer/Green's function) เข้าเป็นสถาปัตยกรรมเดียวที่ attribution + inversion เป็น first-class citizen ไม่ใช่ของแถม

6. โปรโตคอลการประเมิน (หัวใจความน่าเชื่อถือ)

6.1 Deterministic + Probabilistic

  • Deterministic: MAE/RMSE/Bias/ACC · event: POD/FAR/CSI · onset-lead time · skill-vs-leadtime (5.1)
  • Probabilistic: CRPS, spread-skill ratio, rank histogram, reliability diagram, Brier/CRPSS เทียบ climatology (3.9)

6.2 การพิสูจน์ attribution — หัวใจที่ทำให้ "น่าเชื่อถือ"

ปัญหา: โลกจริงไม่มี "ความจริงของแหล่ง" (ไม่มีใครรู้แน่ว่า PM2.5 ที่เชียงใหม่มาจากพม่ากี่ %)
ทางออก — CTM Digital-Twin: รัน CMAQ/WRF-Chem ที่ "รู้คำตอบ" (เพราะเราตั้ง tagged-tracer เอง) → ใช้เป็น ground truth สังเคราะห์ → วัดว่า HazeNet กู้สัดส่วนได้ตรงไหม · เสริมด้วย FLEXPART back-trajectory + เคสเหตุการณ์จริงที่มีหลักฐานทิศลม

6.3 Cross-validation + Significance + Decomposition

  • Spatiotemporal CV + embargo + forward-chaining · leave-stations-out · leave-region-out (ทดสอบ transboundary generalization) (5.2)
  • Significance: block-bootstrap CI + paired test + multiple seeds (5.6)
  • Oracle/error decomposition: แยก error จาก met-forecast vs โมเดล (5.5) · attribution error bound (6.1)
  • Calibration ของ uncertainty + ของ inversion

7. Ablation + การวิเคราะห์ (เข้าใจว่าทำไมมันเวิร์ก)

ตัดอะไรทดสอบสมมุติฐาน
conditionally-linear → nonlinear ทั่วไปโครงสร้างเชิงเส้นต่อ emission จำเป็นต่อ attribution (3.5)
mass-aware constraint ออกการอนุรักษ์มวลช่วย attribution ไม่รั่ว (2.4)
physics conditioning (met-kernel) ออกการคุมเคอร์เนลด้วยลมเพิ่ม skill
diffusion head → deterministicgenerative ensemble ดีกว่าในการปรับเทียบ (3.9)
pretraining ออกfoundation transfer ช่วยตอน sparse (WS6)
channels ทีละกลุ่ม (AOD/TROPOMI/DEM)คุณค่าของแต่ละแหล่งข้อมูล

+ การวิเคราะห์: ดู learned kernel ตรงกับฟิสิกส์ลมไหม · scaling laws (skill vs ข้อมูล/พารามิเตอร์) · failure cases

8. ข้ออ้าง SOTA ที่หักล้างได้ (เขียนเป็นข้อ ๆ)

C1 (Forecast): skill พยากรณ์ PM2.5 หลายวันของ HazeNet ≥ Aurora-Air/CAMS ในภูมิภาค SEA (วัดด้วย CRPSS/RMSE, significance ผ่าน)
C2 (Attribution): attribution error เทียบ CTM digital-twin ต่ำกว่า baseline (XAI บน black-box, FLEXPART) อย่างมีนัยสำคัญ
C3 (Inversion): emission inversion แม่นใกล้ adjoint/4D-Var ของ CTM แต่เร็วกว่า ≥10–100×
C4 (Causal): intervention ใน twin ยืนยันว่า attribution เป็นเชิงเหตุผล ไม่ใช่แค่สหสัมพันธ์ (6.2)
C5 (Resource): เร็ว/ถูกกว่า CTM หลายระดับ → ใช้เตือนภัย real-time ได้

ทุกข้อต้องมี metric + baseline + การทดสอบนัยสำคัญ · ถ้าข้อไหนไม่ผ่าน ให้รายงานตรง ๆ (null result ก็มีคุณค่า 5.4)

10. ความเสี่ยง + ขีดจำกัดที่ซื่อสัตย์ (compute ทุบไม่ได้)

ต้องเขียนตรง ๆ ในเปเปอร์:
  • ไม่มี ground truth ของแหล่งในโลกจริง → attribution validate ผ่าน CTM twin (ซึ่งเองก็มี error ของมัน) + เคสจริงเชิงคุณภาพ · เป็นเพดานเชิงโครงสร้าง ไม่ใช่เรื่อง compute
  • CTM twin สืบทอด bias ของ CMAQ/WRF-Chem → "ตรงกับ twin" ≠ "ตรงกับธรรมชาติ 100%"
  • predictability limit ของ haze มีเพดานทางทฤษฎี (chaos) — โมเดลเทพแค่ไหนก็พยากรณ์ไกลไม่จำกัดไม่ได้
  • emission inventory ไม่แน่นอน → inversion ดีได้เท่าสมมุติฐาน prior/regularization (6.3)
  • identifiability ของ inversion — หลาย emission อาจให้ observation เดียวกัน (ill-posed)
  • การเปลี่ยน scope สู่เปเปอร์ = งานหลายเดือน–ปี ไม่ใช่ 2 เดือน · ต้องวางเป็นโปรแกรมระยะยาว
หลักความน่าเชื่อถือ: งานระดับโลกที่ดีไม่ใช่งานที่อ้างเยอะสุด แต่คืองานที่ ข้ออ้างทุกข้อหักล้างได้ + รู้ขอบเขตตัวเองชัด + ทำซ้ำได้ · ความซื่อสัตย์เรื่องขีดจำกัดคือสิ่งที่ reviewer ระดับ top venue ให้น้ำหนักสูง
