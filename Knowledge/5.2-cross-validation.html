ต่อจาก 5.1 (มี metric แล้ว) · ขยายจาก 4.4 (เคยแบ่งเวลาคร่าว ๆ) · โยง 0.4 (leakage) · 4.5 (แต่ละ fold = งบ A100) · 5.6 (CI ข้าม fold) · 1.4 (ต้นทางข้ามแดนไม่มีสถานี)

หัวใจ: ข้อมูลของเรามี "ความสัมพันธ์" 2 แกนที่ทำให้รั่วง่าย — เวลา (ชั่วโมงติดกันคล้ายกัน) และ พื้นที่ (สถานีใกล้กันคล้ายกัน) · CV ที่ถูกต้องต้อง "ตัด" ทั้งสองแกน

1. ทำไม k-fold แบบสุ่มถึง "โกหก" ⭐

CV มาตรฐานในตำรา ML คือ k-fold: สับข้อมูลสุ่มเป็น k กอง สลับกันเป็น test · ใช้ได้ดีกับข้อมูล "อิสระต่อกัน" (เช่น รูปแมว/หมา) · แต่ข้อมูลเรา ไม่อิสระ — จุดข้างเคียง (ทั้งเวลาและพื้นที่) คล้ายกันมาก

ทำไมรั่วแล้วผลปลอม: ถ้า test point มี "เพื่อนบ้านเกือบเหมือนกัน" อยู่ใน train → โมเดลแทบจะ "จำคำตอบ" ได้ → คะแนน CV สวยเวอร์ · แต่พอใช้จริงกับ "เวลา/สถานที่ที่ไม่มีเพื่อนบ้านใน train เลย" จะแป้ก · CV ที่รั่วให้ความมั่นใจปลอม ๆ ซึ่งอันตรายกว่าไม่มี CV เลย

2. 2 แกนของการรั่ว — เวลา และ พื้นที่

แกนความสัมพันธ์รั่วยังไงตัดด้วย
เวลา (autocorrelation)14:00 กับ 15:00 ฝุ่นเกือบเท่ากันชั่วโมงติดกันกระจายไป train+testtemporal blocking + gap
พื้นที่ (spatial autocorrelation)2 สถานีห่าง 5 กม. ค่าคล้ายกันสถานีใกล้กันอยู่คนละชุดspatial blocking / leave-stations-out
หลักของ Tobler: "ทุกอย่างสัมพันธ์กับทุกอย่าง แต่สิ่งที่อยู่ใกล้สัมพันธ์กันมากกว่า" · นี่คือกฎพื้นฐานของข้อมูลภูมิสารสนเทศ และเป็นเหตุผลที่ CV ของงาน GeoAI ต้อง "บล็อก" ทั้งเวลาและพื้นที่ ไม่ใช่สุ่ม

3. Temporal Blocking + Embargo ⭐

ขยายจากการแบ่งเวลาใน 4.4 ให้เป็นระบบ · 2 รูปแบบ:

① Blocked / leave-one-season-out

แบ่งเป็นก้อนฤดูต่อเนื่อง สลับกันเป็น test · เช่น 5 ฤดู → 5 fold (ฝึก 4 ฤดู ทดสอบ 1 ฤดูที่กันไว้)

② Forward-chaining (เคารพลูกศรเวลา)

เทรนอดีต→ทดสอบอนาคตเท่านั้น (ไม่เอาอนาคตมาทำนายอดีต) สมจริงกับการใช้งานจริง

Embargo (gap) คืออะไร — สำคัญ: เว้น "ช่องว่างเวลา" ระหว่าง train กับ test (เช่น 1–2 วัน) · เพราะถ้า test เริ่มต้น "ติดกับ" train ปลายสุด ความสัมพันธ์เชิงเวลาจะทำให้รั่วที่ขอบ · gap ตัดสายสัมพันธ์นั้น · ยิ่ง autocorrelation ยาว ยิ่งต้อง gap กว้าง

4. Spatial Blocking — กันสถานีข้างเคียงรั่ว

แบ่งพื้นที่เป็นบล็อก (เช่น ตารางหยาบ หรือรายภูมิภาค) แล้วให้ทั้งบล็อกอยู่ชุดเดียวกัน · ป้องกันไม่ให้ "สถานีในเมืองเดียวกัน" กระจายไป train และ test พร้อมกัน

5. Leave-Stations-Out — บททดสอบหัวใจของ HazeNet ⭐⭐

นี่คือ CV ที่สำคัญที่สุดของโปรเจกต์ · วิธี: กันสถานีบางส่วนออกจากการเทรนทั้งหมด (ไม่ให้โมเดลเห็นค่าจาก loss เลย) แล้ววัดว่าทำนายที่สถานีเหล่านั้นได้แม่นแค่ไหน

ทำไมนี่คือบททดสอบที่ตรงเป้าที่สุด: เป้าหมายทั้งโปรเจกต์คือผลิต แผนที่ PM2.5 เต็มกริด จากสถานีไม่กี่จุด · leave-stations-out วัด "ความสามารถเติมค่าตรงที่ไม่มีสถานี" โดยตรง — ซึ่งคือสิ่งที่ทำให้ output ของเรามีค่าจริง · ถ้าผ่าน LSO = พิสูจน์ว่าโมเดล "เรียนฟิสิกส์เชิงพื้นที่" ไม่ใช่แค่ "จำค่าที่สถานี"
from sklearn.model_selection import GroupKFold
# group = station_id → ทั้งสถานีไปอยู่ fold เดียว ไม่กระจาย
gkf = GroupKFold(n_splits=4)
for tr, te in gkf.split(X, y, groups=station_id):

6. 3 คำถาม generalize → 3 แบบ CV

แต่ละคำถามว่า "โมเดลทั่วไปได้แค่ไหน" ต้องใช้ CV คนละแบบตอบ:

คำถามCV ที่ตอบความยาก
ทำนาย อนาคต ที่ไม่เคยเห็นได้ไหม?temporal blocking (leave-season-out)กลาง
ทำนาย ที่ที่ไม่มีสถานี ได้ไหม?leave-stations-out / spatial blockingยาก
ทำนาย เวลาใหม่ + ที่ใหม่ พร้อมกัน?spatiotemporal blocking (กันทั้งคู่)ยากสุด = honest สุด
กลยุทธ์รายงาน: รายงานทั้ง 3 ระดับ → กรรมการเห็นว่าโมเดล "เก่งด้านไหน อ่อนด้านไหน" · คะแนนจะลดหลั่นจาก temporal (ง่ายสุด) → spatiotemporal (ยากสุด) · การที่คะแนน spatiotemporal ยังพอใช้ได้ = หลักฐานแกร่งสุดว่าโมเดลเรียนฟิสิกส์จริง

7. ปัญหาเฉพาะของเรา — ต้นทางข้ามแดนไม่มีสถานี

ข้อจำกัดเชิงโครงสร้างที่ต้องเขียนในรายงาน: เมียนมา/ลาว (ต้นทางควันหลัก, 1.4 + 4.1) แทบไม่มีสถานีสาธารณะ · นี่คือ "leave-region-out" แบบสุดขั้วที่เรา ไม่มีทางทดสอบโดยตรง เพราะไม่มี ground truth ในพื้นที่นั้นเลย

วิธีรับมืออย่างซื่อสัตย์: (1) ใช้ AOD ดาวเทียม (Himawari) เป็นตัวตรวจทานทางอ้อมในพื้นที่ต้นทาง · (2) ทำ leave-stations-out กับสถานีไทย "ชายแดน/ห่างไกล" เป็น proxy ของความสามารถ extrapolate · (3) ระบุชัดว่าความมั่นใจในพื้นที่ต้นทางต่ำกว่าในไทย — อย่าเคลมเกินหลักฐาน

8. งบ A100 vs จำนวน fold — ความจริงที่ตำราไม่บอก ⭐

ตำรา ML บอก "ทำ 10-fold CV" · แต่ แต่ละ fold = เทรนโมเดลใหม่ 1 รอบ = กิน A100-ชม. · จาก 4.5 เรามี ~15 ชม. → ทำ 10-fold (เทรน 10 รอบ) เป็นไปไม่ได้

ทางออกเชิงปฏิบัติสำหรับโปรเจกต์ 2 เดือน:
  • หลัก 1 split: temporal split หลัก (train อดีต / val / test ฤดูล่าสุด) — รันเต็ม 1 รอบ คือผลหลัก
  • + 1 LSO experiment: leave-stations-out 1 ชุด — พิสูจน์ความสามารถเชิงพื้นที่ (สำคัญสุด)
  • fold เพิ่มเฉพาะถ้าเหลือเวลา: เช่น leave-2-seasons-out เพื่อดู variance
  • เดบั๊ก CV pipeline บนโมเดลจิ๋ว/T4 ก่อน (4.5) — อย่าเปลือง A100 กับการลองวิธีแบ่ง
ระบุในรายงานว่า "จำกัด fold เพราะงบ compute" — นี่คือ caveat ที่กรรมการเข้าใจ ไม่ใช่จุดอ่อนที่ต้องซ่อน
เคล็ดประหยัด: ถ้าออกแบบดี ๆ การประเมิน temporal เกือบฟรี — เทรน 1 รอบแล้ววัดบนทุก test season ที่กันไว้ได้เลย · ส่วน leave-stations-out จำเป็นต้องเทรนใหม่ (เพราะต้องตัดสถานีออกจาก loss) → จัดสรรงบเผื่อ LSO โดยเฉพาะ

9. รายงานผล CV + Nested CV

รายงานเป็น "เฉลี่ย ± ส่วนเบี่ยงเบน" ข้าม fold

อย่ารายงานเลขเดียว · รายงาน mean ± std ข้าม fold → บอกทั้งความแม่นและความ "เสถียร" (โมเดลที่ดีในบางฤดูแต่พังในบางฤดู = ไม่น่าเชื่อถือ) · การกระจายนี้ต่อยอดเป็น confidence interval ใน 5.6

Nested CV — อย่าจูน hyperparameter บน test

leakage ที่เนียนสุด: ถ้าลอง learning rate 10 แบบแล้วเลือกตัวที่ "ได้คะแนน test ดีสุด" → คุณเพิ่งใช้ test ในการเลือกโมเดล = test ปนเปื้อน · ต้องจูนบน validation (หรือ inner CV loop) แล้วแตะ test ครั้งเดียวตอนจบ เท่านั้น · test คือ "ข้อสอบจริง" เปิดดูได้ครั้งเดียว

10. Cheat sheet + แบบฝึกหัด

Cheat sheet

  • ข้อมูลเรา ไม่อิสระ → k-fold สุ่ม = รั่ว = ผลปลอม
  • 2 แกนรั่ว: เวลา (autocorr) + พื้นที่ (Tobler) — ตัดทั้งคู่
  • temporal: blocking + embargo gap + forward-chaining
  • spatial/LSO: GroupKFold by station — กันสถานีออกทั้งตัว
  • leave-stations-out = บททดสอบหัวใจ (เติมจุดไม่มีสถานี)
  • 3 คำถาม → temporal / spatial / spatiotemporal (ยากสุด=honest สุด)
  • แต่ละ fold = งบ A100 → เลือก fold อย่างฉลาด · รายงาน mean±std
  • จูนบน val เท่านั้น · test เปิดครั้งเดียว

❓ แบบฝึกหัด

  1. ทำไม k-fold สุ่มถึงให้ผลดีปลอมกับข้อมูล spatiotemporal?
  2. embargo gap คืออะไร แก้การรั่วแบบไหน?
  3. leave-stations-out พิสูจน์ความสามารถอะไรที่ตรงกับเป้าโปรเจกต์?
  4. ทำไมทำ 10-fold ไม่ได้ในโปรเจกต์นี้ และแก้ยังไง?
  5. การจูน hyperparameter บน test ผิดยังไง เรียกว่าอะไร?
เฉลย
  1. จุดข้างเคียง (เวลา/พื้นที่) คล้ายกันมาก → test มี "เพื่อนบ้าน" ใน train → โมเดลเกือบจำคำตอบได้ → คะแนนสวยปลอม ใช้จริงแป้ก
  2. ช่องว่างเวลาระหว่าง train/test → ตัด autocorrelation ที่ขอบ (กันรั่วเชิงเวลาตรงรอยต่อ)
  3. ความสามารถ "เติม PM2.5 ตรงที่ไม่มีสถานี" = หัวใจของการผลิต field เต็มกริดจากสถานีไม่กี่จุด
  4. แต่ละ fold = เทรนใหม่ = งบ A100 ที่มีแค่ ~15 ชม. · แก้: 1 temporal split หลัก + 1 LSO + เพิ่มถ้าเหลือเวลา
  5. ใช้ test เลือกโมเดล = test ปนเปื้อน (leakage) · ต้องจูนบน val แล้วแตะ test ครั้งเดียว

✅ เช็กลิสต์ก่อนไป 5.3

  • ☐ ออกแบบ temporal split (มี embargo) + forward-chaining
  • ☐ ตั้ง leave-stations-out (GroupKFold by station) อย่างน้อย 1 ชุด
  • ☐ วางแผนจำนวน fold ให้พอดีงบ A100
  • ☐ จูน hyperparameter บน val เท่านั้น (test ครั้งเดียว)
  • ☐ รายงาน metric (5.1) เป็น mean±std ข้าม fold
  • ☐ เขียน caveat พื้นที่ต้นทางข้ามแดน (ไม่มี ground truth)
