🌍 NeuralGCM — วิเคราะห์เชิงลึก กระบวนทัศน์ physics+ML ของ HazeNet

"Neural General Circulation Models for Weather and Climate" — การประกอบร่าง physics+ML ระดับมาสเตอร์ ที่ตอบคำถาม "rollout ยาวจะเสถียรไหม"

ต้นฉบับ: Kochkov, Yuval, Langmore et al. (+ Battaglia, Hoyer) · Nature 2024 · Google · Nature · arXiv 2311.07222 · โค้ด (Dinosaur)

ทำไมอ่านตัวนี้เป็นตัวที่ 3: สองงานก่อน (FastCTM = forward, adjoint emulator = inverse) ทิ้งคำถามเดียวกันค้างไว้ — "พอ rollout ยาว ๆ จะเสถียรไหม? gradient จะระเบิดไหม?"

NeuralGCM คือคำตอบโดยตรง: มันคือ hybrid physics+ML ที่ rollout ได้เสถียรถึงระดับหลายสิบปี และเป็น ML ตัวแรกที่ทำ ensemble ได้ CRPS ดีกว่าโมเดลฟิสิกส์. ที่สำคัญ มันใช้ "differentiable programming" แบบเดียวกับที่ให้ HazeNet ทำ source inversion (3.7) — แต่ใช้เพื่อ "เทรนผ่าน solver". อ่านจบจะเห็นว่า HazeNet = NeuralGCM ที่เปลี่ยนโดเมนจาก "อากาศ" เป็น "ฝุ่น+แหล่งกำเนิด".

1. ภาพใหญ่ — GCM คืออะไร และ NeuralGCM ต่างจากเดิมยังไง

GCM (General Circulation Model) = แบบจำลองหมุนเวียนบรรยากาศทั้งโลก — เครื่องมือหลักของการพยากรณ์อากาศ/ภูมิอากาศมา 50 ปี. มันแก้สมการฟิสิกส์ของอากาศบนกริดทั่วโลก แล้วเดินเวลาไปข้างหน้า.

แนวทางดีเสีย
GCM ฟิสิกส์ล้วน (ดั้งเดิม)เคารพฟิสิกส์, เสถียร, รันยาวได้ช้ามาก, ส่วน "parameterization" ต้องจูนมือ → อคติ (bias) เยอะ
ML ล้วน (GraphCast/Pangu)เร็วมาก, แม่นช่วงสั้นrollout ยาวมักไม่เสถียร, ทำ climate ไม่ได้, ไม่เคารพฟิสิกส์
NeuralGCM (hybrid)เร็ว + เสถียรยาว + ฟิสิกส์คุม + ensemble ดียัง coarse, extrapolate ไป climate ใหม่ ๆ ไม่ได้
หัวใจของ NeuralGCM ในประโยคเดียว: "ให้ฟิสิกส์ทำสิ่งที่ฟิสิกส์เก่ง (การไหลขนาดใหญ่) และให้ ML ทำสิ่งที่ฟิสิกส์ห่วย (กระบวนการเล็ก ๆ ที่ต้องจูนมือ เช่น เมฆ)" แล้วเชื่อมทั้งสองให้ หาอนุพันธ์ได้ทั้งระบบ เพื่อเทรนร่วมกันแบบ end-to-end.
🎯 นี่คือปรัชญาเดียวกับ FastCTM และ HazeNet
จำ FastCTM ได้ไหม? เขาใช้ สูตรฟิกซ์ (upwind) สำหรับ transport และ UNet++ เรียนรู้ K/k/d ที่ไม่รู้สูตร. นั่นคือหลักการเดียวกับ NeuralGCM เป๊ะ — แค่ FastCTM ทำกับ "เคมี-ขนส่ง" ส่วน NeuralGCM ทำกับ "พลศาสตร์ทั้งโลก". HazeNet ก็ยืนบนหลักการนี้: physics สำหรับสิ่งที่รู้, ML สำหรับสิ่งที่ไม่รู้, เชื่อมแบบ differentiable.

2. ทฤษฎีรากฐาน — "Dynamical Core" vs "Physics"

หัวใจของ GCM ทุกตัวคือการแบ่งงานเป็น 2 ส่วน. ต้องเข้าใจตรงนี้ก่อน ทุกอย่างจะตามมาเอง:

ทำไมต้องแยก: เมฆก้อนหนึ่งเล็กกว่ากริด 140 กม. มาก → "มองไม่เห็น" บนกริด แต่มีผลต่ออากาศมหาศาล. นักวิทยาศาสตร์เลยต้อง "ประมาณผลรวม" ของมันด้วยสูตร เรียก parameterization. ปัญหาคือสูตรพวกนี้ไม่แม่น ต้องจูนมือ และเป็นแหล่งอคติ (bias) ที่ใหญ่ที่สุดของ climate model ทั้งหลาย.

ไอเดียทอง: NeuralGCM เก็บ dynamical core (ส่วนที่ฟิสิกส์เชื่อถือได้) ไว้เป็นฟิสิกส์จริง แต่ แทน parameterization ที่ห่วยทั้งหมดด้วย neural network เดียว ที่เรียนจากข้อมูลจริง (ERA5). → ได้ความแม่นของ ML + ความเสถียรของฟิสิกส์.

3. สถาปัตยกรรม NeuralGCM — 3 ส่วน

3.1 ส่วนที่ ① — Differentiable Dynamical Core ("Dinosaur")

  • แก้ primitive equations (สมการพื้นฐานการเคลื่อนที่ของอากาศขนาดใหญ่) บน sigma coordinates (แกนตั้งตามภูมิประเทศ)
  • แทนสนามแนวนอนด้วย spherical harmonics (วิธี spectral) + time integration แบบ semi-implicit
  • เขียนด้วย JAX → หาอนุพันธ์ได้ทั้ง forward และ backward (นี่คือกุญแจ — ดู §5, §8)

3.2 ส่วนที่ ② — Learned Physics Module (NN)

  • NN ทำนายผลของ "กระบวนการที่ resolve ไม่ได้": เมฆ, รังสี, ฝน, พลศาสตร์ย่อย
  • เป็น fully-connected NN + residual connections (ResNet — เหมือนเปเปอร์ adjoint!), ใช้น้ำหนักชุดเดียวกันทุก column ทั่วโลก
  • Input: ตัวแปร prognostic ในจุดนั้น, เงื่อนไขขอบ, รังสีดวงอาทิตย์ตกกระทบ, spatial embedding ที่เรียนรู้, gradient แนวนอน  Output: "tendencies" (อัตราการเปลี่ยน)

3.3 ส่วนที่ ③ — Encoder / Decoder

ตัวแปลงระหว่างข้อมูล ERA5 (พิกัดความดัน) กับโลกภายในของ core (sigma coordinates) — ใช้ interpolation + การแก้ที่เรียนรู้ด้วย NN แบบเดียวกับ learned physics.

3.4 การเชื่อมต่อแต่ละก้าวเวลา

state ใหม่ = DynamicalCore(state) + LearnedPhysics(state, แสงแดด, ...)

นี่คือ operator splitting อีกครั้ง (เหมือน FastCTM!) — ฟิสิกส์ที่รู้ + ML ที่เรียน รวมกันเป็น tendency แล้วเดินเวลา. วนซ้ำ.

4. ทฤษฎีที่ใช้ (ลงลึก)

4.1 Spectral Method / Spherical Harmonics

ปัญหา: จะแทน "สนามอากาศทั้งลูกโลก" ในคอมพิวเตอร์ยังไงให้คำนวณอนุพันธ์ได้แม่น? คำตอบ: เขียนเป็น "ผลรวมของลวดลายคลื่นบนทรงกลม" — เหมือน Fourier แต่บนลูกโลก. ลวดลายพื้นฐานเหล่านี้เรียก spherical harmonics.

ทำไม spectral ดี: ในรูปแบบคลื่นนี้ การคำนวณอนุพันธ์ (∇) ทำได้แม่นและง่าย (แค่คูณตัวเลข) — เหมาะกับการแก้ PDE บนทรงกลมมาก. นี่คือเหตุผลที่ GCM ระดับโลกชอบใช้ spectral. (เชื่อมกับ FNO/FourCastNet ที่ก็ทำงานใน Fourier space ด้วยเหตุผลคล้ายกัน)

4.2 Primitive Equations + Sigma Coordinates

Primitive equations = สมการการเคลื่อนที่ของอากาศแบบประมาณ (อนุรักษ์โมเมนตัม + มวล + พลังงาน + สมดุล hydrostatic + ก๊าซอุดมคติ) สำหรับการไหลขนาดใหญ่. Sigma coordinates = แกนตั้งที่ "ตามพื้นผิว" (σ=1 ที่พื้น, σ=0 ที่ยอดบรรยากาศ) → จัดการภูเขาได้ดี.

4.3 Parameterization — จุดที่ ML เข้ามาช่วย

กระบวนการเล็กกว่ากริด (เมฆ, การพาความร้อน) ต้องประมาณด้วยสูตร. ดั้งเดิมสูตรพวกนี้มาจากการ "จูนมือ" ของผู้เชี่ยวชาญ → เป็นศิลปะมากกว่าวิทยาศาสตร์ และเป็นแหล่ง bias หลัก. NeuralGCM แทนทั้งหมดด้วย NN เดียวที่เรียนจากข้อมูล → ลด bias ได้มาก.

5. ⭐ การเทรน end-to-end + Rollout Curriculum (หัวใจที่เราต้องการ)

5.1 เทรน "ผ่าน" solver — differentiable programming ตัวจริง

เพราะทั้ง dynamical core (JAX) และ learned physics (NN) หาอนุพันธ์ได้ → เทรน NN โดยให้ gradient ไหลทะลุผ่าน solver ฟิสิกส์ได้เลย. นี่คือ "differentiable programming" / "online training".

เทรน: ปรับ NN เพื่อให้ [Core+NN เดินเวลา หลายก้าว] ออกมาใกล้ ERA5 → loss.backward() ทะลุทุกก้าว
🎯 นี่คือเครื่องมือเดียวกับ source inversion ของเรา
NeuralGCM ใช้ "gradient ทะลุ solver" เพื่อ เทรน. HazeNet ใช้ "gradient ทะลุ solver" ตัวเดียวกันเพื่อ 2 อย่าง: (1) เทรน และ (2) source inversion ตอน inference (กลับทิศหา emission — บท 3.7). เครื่องมือเดียว ใช้กว้างกว่า. NeuralGCM พิสูจน์ว่าการ backprop ทะลุ solver ฟิสิกส์ "ทำได้จริงและเสถียร" ในสเกลใหญ่.

นี่คือเทคนิคที่มีค่าที่สุดสำหรับ HazeNet จากเปเปอร์นี้. ปัญหาที่ผมเตือนใน 2 ไฟล์ก่อน — "rollout ยาวแล้วระเบิด" — NeuralGCM แก้ด้วยวิธีง่ายแต่ทรงพลัง:

ทำไมได้ผล: ถ้าบังคับเทรน rollout 5 วันตั้งแต่แรก gradient ต้องทะลุหลายร้อยก้าว → ระเบิด/ฟุ้ง เทรนไม่ลง. แต่ถ้าเริ่ม 6 ชม. ให้โมเดล "เดินมั่นคง" ก่อน แล้วค่อยยืด → มันเรียนที่จะไม่สะสม error. เหมือนสอนเด็กเดินก่อนวิ่ง.

5.3 Ensemble จากการฉีด noise + เทรนด้วย CRPS

NeuralGCM-ENS สร้างความไม่แน่นอนโดย ฉีด noise (Gaussian random fields ที่มี correlation เชิงพื้นที่/เวลาที่เรียนรู้) เข้าไปใน encoder + learned physics → ได้สมาชิก ensemble หลายตัว. แล้ว เทรนโดยตรงเพื่อ minimize CRPS (ไม่ใช่ MSE).

🎯 เชื่อมกับบท 3.9
นี่คือ uncertainty อีกแนวทาง (เสริม quantile regression ที่เราเรียน): "ฉีด noise + เทรนด้วย CRPS". CRPS ที่เราเรียนในบท 3.9 คือ metric เดียวกับที่ NeuralGCM ใช้เป็น loss เลย. → HazeNet เลือกได้ระหว่าง quantile regression (เบา, 1 pass) หรือ stochastic+CRPS แบบ NeuralGCM (หนักกว่าแต่ ensemble แท้).

6. ผลลัพธ์

6.1 ความเร็ว (ตัวเลขที่ทำให้คนตะลึง)

NeuralGCM 1.4° เร็วกว่า X-SHiELD (cloud-resolving model) มากกว่า 3,500 เท่า
จำลองบรรยากาศ 1 ปี: X-SHiELD ใช้ 20 วัน · NeuralGCM ใช้ 8 นาที

6.2 ความแม่น (deterministic + ensemble)

  • Deterministic: สูสีกับ ML ชั้นนำ (GraphCast/Pangu) ที่ 1–10 วัน; เทียบเท่า ECMWF ensemble ที่ 1–15 วัน
  • Ensemble (NeuralGCM-ENS 1.4°): error ต่ำกว่า ECMWF-ENS เกือบทุกตัวแปร/lead time/ระดับความสูง ทั้ง ensemble-mean RMSE, spread-skill และ CRPS

→ เป็น ML ตัวแรกที่ทำ ensemble ได้ CRPS ดีกว่าโมเดลฟิสิกส์ระดับโลก

6.3 Climate ระยะยาว + พายุหมุนเขตร้อน

  • กำหนด SST แล้วรันได้เสถียร หลายสิบปี — ติดตาม global mean temperature + AMIP ได้สมจริง
  • ที่ 140 กม. เกิด พายุหมุนเขตร้อนที่สมจริง (ความถี่ + เส้นทาง); ปี 2020 ทำนายจำนวน/ความรุนแรงพายุตรงกับที่สังเกตจริง
  • โมเดลฝน: ทั้ง 37 initial conditions เสถียรครบ 20 ปี

8. ⭐ NeuralGCM ↔ HazeNet — เทียบทุกจุด

8.1 สิ่งที่ NeuralGCM "ยืนยัน/ให้แนวทาง" กับเรา

NeuralGCM ทำHazeNet หยิบไปใช้ยังไง
split: dynamical core (ฟิสิกส์) + learned physics (NN)= split ของเรา: upwind transport (ฟิสิกส์) + UNet++/FiLM (NN) — กระบวนทัศน์เดียวกัน พิสูจน์แล้วระดับสูงสุด
เทรน end-to-end ทะลุ solver (JAX autodiff)= กลไก source inversion ของเรา (3.7) — เครื่องมือเดียวกัน
rollout curriculum (6ชม.→5วัน)🟢 คำตอบของความเสี่ยงเสถียรภาพ ที่ค้างจาก 2 ไฟล์ก่อน — เอามาใช้ตรง ๆ
ensemble = ฉีด noise + เทรนด้วย CRPSทางเลือก uncertainty (3.9) เสริม quantile regression
operator splitting (core + physics รวม tendency)ยืนยันโครง modular ของเรา (เหมือน FastCTM)
ResNet ใน learned physicsยืนยันบล็อกพื้นฐาน (เหมือนเปเปอร์ adjoint)

8.2 สิ่งที่ HazeNet "ต่าง / ไปไกลกว่า"

  • โดเมน: NeuralGCM = อากาศ/ภูมิอากาศ. HazeNet = ฝุ่น PM2.5 + เคมีแอโรซอล (มี emission, chemistry, deposition ที่ NeuralGCM ไม่มี)
  • ใช้ differentiability กว้างกว่า: NeuralGCM ใช้เพื่อ เทรน เท่านั้น. HazeNet ใช้เพื่อเทรน และ source inversion ตอน inference (กลับทิศหาแหล่ง)
  • Attribution: HazeNet เพิ่ม tagged-tracer (3.6) แยกแหล่งเชิงพื้นที่ — NeuralGCM ไม่มีแนวคิดนี้
  • สเกล/งบ: NeuralGCM = Google. HazeNet = โดเมนเล็ก (SEA) + งบ ~15 A100-hr → เราเอาแนวคิดมาทำในสเกลที่เอื้อม

8.3 หมายเหตุปฏิบัติ: JAX vs PyTorch

เรื่องที่ต้องตัดสินใจ: NeuralGCM ใช้ JAX (เหมาะกับ differentiable physics solver มาก — diff ทะลุ solver ได้ลื่น). HazeNet น่าจะใช้ PyTorch (เข้าถึงง่ายกว่า, ระบบนิเวศใหญ่กว่า, คุณคุ้นจากบท 0.2 autograd). ทั้งคู่ทำ autodiff ได้ — เลือก PyTorch ได้ แต่ต้องเขียน solver ให้ differentiable เอง (ใช้ operation ที่ PyTorch track ได้, เลี่ยง in-place ที่ตัด graph). รู้ว่า JAX เป็นทางเลือกถ้า solver ซับซ้อนมาก.

8.4 สามงานรวมกัน = HazeNet

9. สรุป + เช็กลิสต์

NeuralGCM ในหนึ่งประโยค: "เก็บ dynamical core ที่ฟิสิกส์เชื่อถือได้ (เขียนใน JAX ให้ diff ได้) + แทน parameterization ที่ห่วยด้วย neural network เดียว → เทรน end-to-end ทะลุ solver โดยค่อย ๆ ยืด rollout → ได้โมเดลที่เร็ว 3,500× เสถียรหลายสิบปี และ ensemble CRPS ดีกว่า ECMWF."

✅ เช็กลิสต์ดึงมาใส่ HazeNet:

  • ☐ ยึดกระบวนทัศน์ "physics สำหรับสิ่งที่รู้ + ML สำหรับสิ่งที่ไม่รู้ + differentiable ทั้งระบบ"
  • ☐ ใช้ rollout curriculum (เริ่มสั้น→ยืดยาว) เพื่อเสถียรภาพ — แก้ความเสี่ยง #1 ของเรา
  • ☐ พิจารณา เทรนด้วย CRPS + ฉีด noise เป็นทางเลือก uncertainty (เทียบกับ quantile regression)
  • ☐ ตัดสินใจ JAX vs PyTorch ตั้งแต่ต้น (PyTorch ได้ แต่ต้องระวัง operation ที่ตัด autodiff graph)
  • ☐ ใช้ความ differentiable เพื่อ ทั้งเทรนและ source inversion (กว้างกว่า NeuralGCM)
  • ☐ เขียน caveat ให้ตรง: เราก็ extrapolate ไป regime ใหม่ไม่ได้ (เหมือน NeuralGCM)
🎯 ประโยคพูดกับกรรมการ
"HazeNet ใช้กระบวนทัศน์ hybrid differentiable เดียวกับที่ NeuralGCM (Nature 2024) พิสูจน์ว่าให้ทั้งความเร็ว ความเสถียรระยะยาว และ ensemble ที่ CRPS ดีกว่าโมเดลฟิสิกส์ — โดยเรานำเทคนิคสำคัญอย่าง rollout curriculum และ CRPS training มาใช้เพื่อความเสถียร. ความต่างคือเราขยายความ differentiable จาก 'การเทรน' ไปสู่ 'source inversion ตอน inference' และเพิ่มการ attribute แหล่งเชิงพื้นที่ สำหรับปัญหาหมอกควันข้ามแดนอาเซียน ในสเกลที่ทำได้จริงด้วยงบจำกัด"
