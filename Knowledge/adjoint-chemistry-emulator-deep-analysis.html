🔄 Forward + Adjoint Chemistry Emulator — วิเคราะห์เชิงลึก ครึ่งหลังของ HazeNet

"Deep Learning Emulator Towards Both Forward and Adjoint Modes of Atmospheric Gas-Phase Chemical Process" — เปเปอร์ที่พิสูจน์ว่า source inversion ของเราทำได้จริง

ต้นฉบับ: Atmosphere (MDPI) 2025, vol.16(9), 1109 · 21 ก.ย. 2025 · DOI 10.3390/atmos16091109

ทำไมเลือกอ่านตัวนี้ต่อจาก FastCTM: FastCTM สอน "ครึ่งหน้า" ของ HazeNet = ตัว forward model (พยากรณ์ฝุ่นไปข้างหน้า). แต่จุดขายจริงของเราคือ "ครึ่งหลัง" = source inversion (กลับทิศหาว่าฝุ่นมาจากแหล่งไหน) — ซึ่ง FastCTM แทบไม่แตะ.

เปเปอร์นี้คือ ครึ่งหลังที่ขาดไป: มันพิสูจน์ด้วยตัวเลขจริงว่า "เอา NN เคมีมาทำ adjoint (กลับทิศ) ได้ฟรีผ่าน autodiff" — แม่นถึง R²=0.995. นี่คือหลักฐานชิ้นเอกของบท 3.7 ที่คุณเรียนมา.

1. ภาพใหญ่ — เปเปอร์นี้แก้ปัญหาอะไร

ในแบบจำลองคุณภาพอากาศ (CMAQ) มี 2 "โหมด" การใช้งาน:

โหมดทำอะไรใช้ทำอะไร
Forward (ไปข้างหน้า)emission → ความเข้มข้นฝุ่นพยากรณ์ปกติ ("พรุ่งนี้ฝุ่นเท่าไหร่")
Adjoint (กลับทิศ)ความไว ∂(ฝุ่น)/∂(emission) — ฝุ่นที่จุดนี้ "ไว" ต่อแหล่งไหนsource inversion, data assimilation, "ลดที่ไหนได้ผลสุด"

ปัญหา: ทั้งสองโหมดช้ามาก และตัวการหลักคือ "เคมีก๊าซ" (gas-phase chemistry) — เปเปอร์ระบุชัดว่าเป็น "คอขวดการคำนวณหลัก" ทั้งใน forward และ adjoint.

ไอเดีย: เทรน NN เลียนแบบเฉพาะ "ตัวแก้เคมี" (chemistry solver) ของ CMAQ → ได้ทั้ง forward (เร็วขึ้น 80–130×) และ adjoint (เร็วขึ้น 45–102×) จากโมเดลเดียว เพราะ NN หาอนุพันธ์ได้.

🎯 ทำไมตรงกับเราเป๊ะ
HazeNet = forward (พยากรณ์ฝุ่น) + adjoint (หาแหล่ง). เปเปอร์นี้คือ proof-of-concept ของ "ครึ่ง adjoint" ในโดเมนเคมีบรรยากาศ — โดเมนเดียวกับเรา, งบเล็กเหมือนเรา. ถ้าเขาทำได้ R²=0.995 → ยืนยันว่าแนวทาง source inversion ของเรา "ไม่ใช่ฝัน".

2. ทำไม "เคมี" คือคอขวด — ทฤษฎี Stiff ODE

เพื่อเข้าใจว่าทำไมเปเปอร์นี้โจมตี "เคมี" โดยเฉพาะ ต้องรู้จักคำว่า Stiff ODE (สมการเชิงอนุพันธ์แบบแข็ง).

เคมีบรรยากาศ = ระบบสมการ ODE ของอัตราการเกิด/สลายสารหลายสิบตัว:

dCi/dt = (สารเกิด) − (สารสลาย)  สำหรับสารทุกตัว i พร้อมกัน

ปัญหา "stiff": ปฏิกิริยาในบรรยากาศมีอัตราต่างกันมหาศาล — บางปฏิกิริยาเกิดในเสี้ยววินาที (เช่น radical OH) บางอันใช้เวลาหลายชั่วโมง. เมื่อมีทั้งเร็วจัดและช้าจัดปนกัน เรียกว่า "stiff".


~μวินาที NO₂ photolysis~นาที O₃ สะสม~ชั่วโมง เร็วช้า
ทำไม stiff = ช้า: ตัวแก้สมการต้องเดินด้วย "ก้าวเวลาจิ๋ว ๆ" เพื่อตามปฏิกิริยาที่เร็วที่สุดให้ทัน (ไม่งั้นคำตอบระเบิด) → ต้องใช้ implicit solver (เช่น Rosenbrock) ที่ต้องคำนวณ Jacobian matrix (เมทริกซ์อนุพันธ์) ทุกก้าว → แพงมหาศาล. นี่คือเหตุผลที่เคมีกิน 50–90% ของเวลารัน CMAQ.
🎯 เชื่อมกับเรา
จำได้ไหมว่า FastCTM โมดูลเคมี (④) ก็เป็นจุดอ่อน (NH₄⁺ R²=0.30)? นี่คือเหตุผลเชิงลึก — เคมีทั้งช้าและซับซ้อนเพราะ stiffness. เปเปอร์นี้โจมตีจุดนี้ตรง ๆ. สำหรับ HazeNet: ถ้าเราเทรน NN ให้ "ข้าม" การแก้ stiff ODE ได้ในก้าวเดียว = ได้ทั้งความเร็วและความ differentiable (ซึ่ง stiff solver ดั้งเดิมไม่มี).

3. ทฤษฎีหัวใจ — Adjoint = Automatic Differentiation = .backward()

นี่คือทฤษฎีที่สำคัญที่สุดของเปเปอร์ และเป็นแกนของบท 3.7 ที่คุณเรียน. ขอปูจากศูนย์:

3.1 "Sensitivity" คืออะไร และทำไมต้องการ

เราอยากรู้: "ถ้าเพิ่ม emission ที่จุด A นิดนึง ฝุ่นที่เชียงใหม่จะเปลี่ยนเท่าไหร่?" = ∂(ฝุ่นเชียงใหม่)/∂(emission ที่ A). นี่คือ "ความไว (sensitivity)".

ถ้ามี emission แค่จุดเดียวก็ง่าย — ลองเพิ่มแล้วดู (finite difference). แต่ถ้ามี 16,384 จุด (กริด 128×128) ต้องลองทีละจุด = รัน 16,384 รอบ = เป็นไปไม่ได้.

3.2 Adjoint method = ได้ความไว "ทุกจุดพร้อมกัน" ในรอบเดียว

เวทมนตร์ของ adjoint: แทนที่จะลองเปลี่ยน input ทีละตัว (forward sensitivity, แพงตามจำนวน input) — adjoint คำนวณย้อนกลับจาก output ทีเดียว แล้วได้ความไวเทียบกับ input ทุกตัวพร้อมกัน. ต้นทุน ≈ รัน forward แค่ 1–2 รอบ ไม่ว่าจะมี input กี่ล้านตัว.

3.3 ความจริงที่เปลี่ยนทุกอย่าง: Adjoint = Backpropagation

Adjoint method  =  Reverse-mode Automatic Differentiation  =  loss.backward()

ทั้งสามคำนี้คือสิ่งเดียวกันทางคณิตศาสตร์ 100%. นักวิทยาศาสตร์บรรยากาศเรียก "adjoint", คน ML เรียก "backprop / autograd", แต่มันคืออัลกอริทึมเดียวกัน (chain rule ย้อนกลับ).

นี่คือ insight ทั้งหมดของเปเปอร์: ถ้าตัวแก้เคมีเป็น NN (ไม่ใช่ Rosenbrock solver) → adjoint ของมันก็คือ .backward() ที่ PyTorch/TF ให้มาฟรี — ไม่ต้องเขียน adjoint code เองเลย. แค่เรียก autodiff. นั่นคือทำไมเขาได้ "ทั้งสองโหมดจากโมเดลเดียว".
🎯 เชื่อมกับเรา — นี่คือบท 3.7 ตัวเป็น ๆ
บท 3.7 เราอ้างว่า "adjoint = .backward()". เปเปอร์นี้คือการพิสูจน์เชิงตัวเลขของคำกล่าวนั้น: เขาเอา sensitivity ที่ได้จาก autodiff ของ NN ไปเทียบกับ sensitivity อ้างอิง → ได้ R²=0.995 (เกือบเป๊ะ). → เราเอาตัวเลขนี้ไปอ้างใน proposal ได้ว่า "วิธี source inversion ของเรามีงานยืนยันแล้วว่าแม่น".

4. ทำไม adjoint แบบดั้งเดิมถึง "นรก" (และทำไมเราถึงได้เปรียบ)

เพื่อเห็นคุณค่าของ "adjoint ฟรีจาก autodiff" ต้องรู้ว่าวิธีเดิมมันทรมานแค่ไหน:

Adjoint ดั้งเดิม (เช่น CMAQ-Adj)Adjoint จาก NN (เปเปอร์นี้ + เรา)
วิธีได้มาเขียน "adjoint code" ด้วยมือ — derive อนุพันธ์ของทุกสมการแล้ว code กลับทิศทีละบรรทัดเรียก .backward()
เวลาพัฒนาหลายปี (งาน "The Adjoint of CMAQ" เป็นงานวิจัยใหญ่ทั้งชิ้น)0 วัน (autodiff ให้ฟรี)
แก้/อัปเดตเปลี่ยนสมการ = เขียน adjoint ใหม่ทั้งหมดเทรนใหม่ → autodiff ตามให้เอง
ความเร็วช้า (stiff Jacobian)เร็ว 45–102×
"The Adjoint of CMAQ" (Hakami et al., ES&T 2007) — งานคลาสสิกที่ใช้ความพยายามมหาศาลเขียน adjoint ของ CMAQ ด้วยมือ. เปเปอร์ที่เราอ่านอยู่นี้แสดงว่า: ถ้าใช้ NN emulator คุณ "ข้าม" ความทรมานทั้งหมดนั้นไปได้เลย. (ดูงาน adjoint ดั้งเดิม)

5. สถาปัตยกรรม — FiLM-ResNet

เปเปอร์ใช้สถาปัตยกรรมชื่อ FiLM-ResNet = ResNet (โครงหลัก) + FiLM layers (ตัวปรับตามเงื่อนไข). มาดูทั้งสองทฤษฎี:

5.1 ResNet — เรียน "ส่วนต่าง" ไม่ใช่ค่าเต็ม

ปัญหาที่ ResNet แก้: network ลึก ๆ เทรนยาก (gradient หาย — vanishing gradient). วิธีของ ResNet (He et al. 2016): ให้แต่ละบล็อกเรียน "ส่วนที่ต้องเปลี่ยน (residual)" แล้วบวกกลับเข้า input ผ่าน "skip connection".

ทำไม ResNet เหมาะกับเคมีพอดี: เคมีคือ "ความเข้มข้นเปลี่ยนไปนิดหน่อยในแต่ละก้าวเวลา" → เรียน ΔC (ส่วนต่าง) ง่ายกว่าเรียน C ใหม่ทั้งก้อน. โครง C_new = C + F(C) จึงเป็น "ภาษาธรรมชาติ" ของการเดินเวลา. เหมือน FastCTM ที่ใช้ C(t+1)=C(t)+ΔC เป๊ะ!

5.2 FiLM — ปรับ network ตาม "เงื่อนไข" (กลางวัน/กลางคืน)

ปัญหาที่ FiLM แก้: เคมีบรรยากาศกลางวันกับกลางคืนคนละโลก — กลางวันมีแสงแดดขับ "photolysis" (เช่น NO₂ + แสง → โอโซน), กลางคืนไม่มี. ถ้าใช้ network เดียวตรง ๆ มันจะงงว่าตอนนี้กลางวันหรือคืน.

วิธีของ FiLM (Feature-wise Linear Modulation, Perez et al. 2018): ให้ "เงื่อนไข" (condition) มาปรับ feature ภายใน network ด้วยการ คูณ (γ) และบวก (β):

FiLM(x) = γ(condition) · x + β(condition)

เมื่อ γ, β ถูกคำนวณจากเงื่อนไข (เช่น มี/ไม่มีแสงแดด, อุณหภูมิ). → network เดียวปรับตัวเองให้เข้ากับสภาวะได้.

💡 การค้นพบสำคัญสำหรับเรา — FiLM คือ "Conditionally-Linear Operator" (บท 3.5) ตัวเป็น ๆ!

ดูสูตร FiLM ให้ดี: γ(met)·C + β(met) — มันคือโอเปอเรเตอร์ที่เป็นเชิงเส้นใน C แต่สัมประสิทธิ์ (γ, β) ขึ้นกับเงื่อนไข (met). นี่คือนิยามของ "conditionally-linear operator" ที่คุณเรียนในบท 3.5 เป๊ะ ๆ!

แปลว่า: FiLM คือวิธี implement บท 3.5 ที่พิสูจน์แล้วในงานจริง — และเพราะมันเชิงเส้นใน C → ปลอดภัยกับ tagged-tracer (3.6): ทุก tracer ใช้ γ, β ชุดเดียวกัน (มาจาก met เดียวกัน) แยกแล้วบวกกลับได้. นี่คือสะพานเชื่อม FastCTM (K=Encoder(met)) → FiLM → บท 3.5 → บท 3.6 ทั้งหมด.

6. สองโหมดจากโมเดลเดียว

เมื่อมี FiLM-ResNet ที่เทรนแล้ว (differentiable ทั้งตัว) ก็ได้ทั้งสองโหมดทันที:

Forward mode: ป้อนความเข้มข้น + เงื่อนไข → network คายความเข้มข้นใหม่หลังเคมีทำงาน 1 ก้าวเวลา

C_next = FiLM_ResNet(C_now, condition)   # เร็ว 80–130×

Adjoint mode: เรียก autodiff → ได้ความไวของ output เทียบ input ทุกตัว

C_next = FiLM_ResNet(C_now, condition)
loss = objective(C_next)
loss.backward()        # ← adjoint! ได้ ∂loss/∂C_now ฟรี เร็ว 45–102×
sensitivity = C_now.grad
จุดที่ต้องซึ้ง: โหมด adjoint ไม่ต้องเขียนโค้ดเพิ่มเลยนอกจาก .backward(). นี่คือความต่างมหาศาลจาก CMAQ-Adj ที่ต้องเขียน adjoint code แยกเป็นปี ๆ. และเพราะใช้โมเดล forward ตัวเดิม → adjoint "สอดคล้อง" กับ forward อัตโนมัติ (consistent).

7.3 วิธี validate adjoint (สำคัญสำหรับเรา)

เขาพิสูจน์ว่า adjoint จาก autodiff "ถูก" โดยเอาความไวที่ได้จาก .backward() ไปเทียบกับความไวอ้างอิง (เช่น finite-difference / ตัวแก้ดั้งเดิม) แล้ววัด R². ได้ 0.995 = เกือบเป๊ะ → ยืนยันว่า "autodiff ให้ adjoint ที่ถูกต้อง".

🎯 เมธอดนี้ = validation plan ของเรา
HazeNet ต้องทำเหมือนกัน: หลังได้ gradient จาก .backward() (สำหรับ source inversion) ต้องเทียบกับ finite-difference ในจุดทดสอบ เพื่อพิสูจน์ว่า sensitivity ของเราถูก. นี่คือ "gradient check" — ขั้นตอนที่กรรมการจะถามหา. เปเปอร์นี้ให้ template ของการ validate ที่อ้างอิงได้.

ตัวเลข speed-up (forward 80–130×, adjoint 45–102×)ตั้งความคาดหวังที่สมจริงสำหรับ HazeNet

9.3 สิ่งที่เรา "ไปไกลกว่า" เปเปอร์นี้

  • เขาทำแค่เคมี (0 มิติ) — เราทำทั้งระบบ: HazeNet รวม transport + diffusion + chemistry + deposition + emission แล้ว adjoint ทั้ง pipeline (ไม่ใช่แค่กล่องเคมี).
  • เขาทำ sensitivity — เราทำ inversion เต็มรูป: เขาแค่คำนวณความไว. เราใช้ความไวนั้นใน optimization loop เพื่อกู้คืน emission Q จริง จากค่าที่วัดได้ (บท 3.7 full inversion).
  • เราเพิ่ม spatial attribution: tagged-tracer (3.6) แยกแหล่งเชิงพื้นที่ (เมียนมา/ลาว/ไทย) — เปเปอร์นี้ไม่มี.
  • เราเพิ่ม uncertainty: quantile/CRPS (3.9).
บทเรียนเตือนใจจากข้อจำกัดเขา: เขา validate แค่ "ก้าวเวลาเดียว". แต่ source inversion ของเราต้อง backprop ผ่าน rollout ยาวหลายสิบชั่วโมง → gradient อาจระเบิด/หาย (vanishing/exploding ตอน backprop ยาว). เราต้องเตรียม gradient checkpointing (บท 3.7) + จับตา rollout stability (Tier 5). อย่าคิดว่า "ก้าวเดียวแม่น = ยาวก็แม่น".

10. สรุป + เช็กลิสต์นำไปใช้

เปเปอร์นี้ในหนึ่งประโยค: "เอา NN (FiLM-ResNet) มาเลียนแบบตัวแก้เคมีของ CMAQ — ได้ทั้ง forward (เร็ว 80–130×, R²>0.999) และ adjoint (เร็ว 45–102×, R²=0.995) จากโมเดลเดียว เพราะ NN หาอนุพันธ์ได้ → autodiff ให้ adjoint ฟรี แทนการเขียน adjoint code เป็นปี ๆ."

✅ เช็กลิสต์ดึงมาใส่ HazeNet:

  • ☐ ยืนยันแนวทาง source inversion ด้วย .backward() (มีงานพิสูจน์แล้ว R²=0.995)
  • ☐ ใช้ FiLM condition โมดูลด้วย met → implement บท 3.5 + ปลอดภัยกับ tagged-tracer
  • ☐ ใช้ ResNet (เรียน ΔC + skip connection) เป็นบล็อกพื้นฐาน
  • ☐ ทำ gradient check (AD vs finite-difference) เป็นหลักฐานความถูกต้อง
  • ☐ adjoint ทั้ง pipeline ไม่ใช่แค่เคมี + ใช้ทำ inversion เต็มรูป (ไปไกลกว่าเขา)
  • ☐ เตรียม gradient checkpointing สำหรับ backprop ผ่าน rollout ยาว
🎯 ประโยคพูดกับกรรมการ
"แนวทาง source inversion ของ HazeNet — ใช้ automatic differentiation แทน adjoint code ดั้งเดิม — มีงานยืนยันแล้วในโดเมนเคมีบรรยากาศ (Atmosphere 2025) ว่าให้ sensitivity ที่แม่นถึง R²=0.995 และเร็วขึ้น ~50–100×. HazeNet ต่อยอดด้วยการ adjoint ทั้ง pipeline (ไม่ใช่แค่เคมี) และใช้มันทำ source inversion เชิงพื้นที่เต็มรูป ผ่าน tagged-tracer — ซึ่งยังไม่มีใครทำกับปัญหาหมอกควันข้ามแดนอาเซียน"
