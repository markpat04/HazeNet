5.3 — Baselines: สิ่งที่ต้องเอาชนะ 🟡 กลาง

Persistence · Climatology · ConvLSTM · PM2.5-GNN · AirPhyNet — แต่ละตัวพิสูจน์ว่า "ความซับซ้อนที่เราเพิ่มเข้าไป จำเป็นจริงไหม"

ต่อจาก 5.1 (skill score ต้องเทียบ baseline) + 5.2 (เทียบบน split เดียวกัน) · ต่างจาก 5.4 (ablation = ตัดชิ้นส่วนของ "เรา" / baseline = "โมเดลอื่น") · โยง 3.5/1.2 (ฟิสิกส์เรา) · 3.6 (attribution = จุดต่าง)

โจทย์: 5.1 บอกว่า "ตัวเลขดิบไร้ความหมายถ้าไม่เทียบ" · บทนี้คือการเลือก "ตัวเทียบ" ให้ถูกและยุติธรรม

หัวใจ — บันได baseline: เรียง baseline จากง่าย→ยาก · การที่เรา "ชนะแต่ละขั้น" ตอบคำถามต่างกัน:
ชนะ persistence → จำเป็นต้องพยากรณ์เลยไหม? · ชนะ ConvLSTM → ฟิสิกส์ช่วยจริงไหม? · เทียบ AirPhyNet → สูตรฟิสิกส์+ความสามารถของเราดีกว่าไหม?

กฎเหล็กของความซื่อสัตย์: เทียบกับ baseline ที่ แข็งแกร่งที่สุด ที่เกี่ยวข้อง ไม่ใช่หุ่นฟาง (strawman) ที่ตั้งมาให้ตัวเองชนะ

1. ทำไมต้องมี baseline — และทำไมต้องแข็ง ★

baseline คือ "เส้นมาตรฐานที่ต้องข้าม" · ถ้าโมเดลซับซ้อนของเราเอาชนะวิธีง่าย ๆ ไม่ได้ → ความซับซ้อนนั้นไม่คุ้ม · กรรมการ/reviewer ทุกคนจะถามคำถามเดียวกัน: "แล้วมันดีกว่าอะไรล่ะ?"

กับดักหุ่นฟาง (strawman): นักวิจัยอ่อนชอบเทียบกับ baseline ที่จงใจทำให้อ่อน (เช่น linear regression ที่ไม่จูน) แล้วเคลมชัยชนะ · reviewer มือฉมังจับได้ทันทีแล้ว reject · การเลือก baseline ที่แข็งแล้วยังชนะ (หรืออธิบายได้ว่าทำไมเสมอแต่มีค่าเพิ่ม) คือสิ่งที่ทำให้งานน่าเชื่อถือ

2. บันได baseline — แต่ละขั้นพิสูจน์อะไร

ตรรกะแบบ ablation ภายนอก: บันไดนี้คือการ "ลอกเปลือกความซับซ้อนทีละชั้น" จากภายนอก · ถ้าชนะ ConvLSTM (DL แต่ไม่มีฟิสิกส์) ได้ = หลักฐานว่า "การใส่ฟิสิกส์ของเรา" คือสิ่งที่สร้างความต่าง ไม่ใช่แค่ "ใช้ deep learning เฉย ๆ"

3. Persistence — บาร์แรกที่ลวงตาว่าง่าย

กฎ: ทำนาย t+h = ค่าที่สังเกตล่าสุด (PM2.5 ตอนนี้) · ไม่ต้องเทรน ฟรี 100%

อย่าดูถูก: สำหรับระยะสั้น (ไม่กี่ชั่วโมง) persistence แข็งมาก — เพราะ PM2.5 เปลี่ยนช้า ค่าตอนนี้ทำนายอีก 3 ชม.ข้างหน้าได้ดีอยู่แล้ว · โมเดลซับซ้อนจำนวนมากในงานวิจัย AQ "แพ้ persistence ที่ระยะสั้น" โดยไม่รู้ตัว · ถ้าชนะ persistence ไม่ได้ที่ t+3 = สัญญาณว่ามีบั๊กหรือโมเดลไม่ได้เรียนรู้อะไรเลย
# persistence: ทำนายเท่าค่าล่าสุด — ใช้เป็นเส้นอ้างอิงทุก lead time
pred_persist = pm25_now.broadcast_to((H,) + pm25_now.shape)

4. Climatology — บาร์ของระยะยาว

กฎ: ทำนายด้วย "ค่าเฉลี่ยในอดีต" ตามฤดูกาล/ชั่วโมงของวัน (เช่น ค่าเฉลี่ย PM2.5 ของ "15 มี.ค. 14:00 น." จากหลายปี)

persistence เก่งระยะสั้นแต่พังระยะยาว · climatology กลับกัน — เสถียรที่ระยะยาว (จับ "หน้าแล้งฝุ่นเยอะ") · ทั้งคู่คือ baseline ที่ ไม่ต้องเทรน → ทำก่อนเลยฟรี ๆ

เป้าที่ชัดเจน: โมเดลเราต้อง error ต่ำกว่าทั้ง persistence และ climatology พร้อมกันทุกระยะ · จุดที่ persistence ตัดกับ climatology คือ "ระยะที่ยากที่สุด" — โชว์ว่าเราชนะตรงนั้นได้คือหลักฐานแกร่ง

5. ConvLSTM — Deep Learning แต่ "ไม่มีฟิสิกส์"

ConvLSTM = รวม CNN (เห็นเชิงพื้นที่) + LSTM (จำเชิงเวลา) · เป็น baseline DL มาตรฐานสำหรับพยากรณ์ spatiotemporal (เดิมใช้พยากรณ์ฝนจากเรดาร์) · มันเรียนจากข้อมูลล้วน ๆ ไม่รู้สมการ ADR

ทำไม ConvLSTM คือ baseline ที่สำคัญเชิงแนวคิด: ถ้าเราชนะมันได้ → พิสูจน์ว่า "ส่วนที่ชนะ" มาจาก การใส่ฟิสิกส์ (1.2, 3.5) ไม่ใช่แค่ "ใช้ neural net ใหญ่ ๆ" · นี่คือคู่เทียบที่ตอบคำถาม "physics-informed คุ้มไหม" โดยตรง · เป็น baseline DL ที่เรา ควรลงมือสร้าง/เทรนเอง (ใช้งบ A100 ส่วนหนึ่ง)

6. PM2.5-GNN — SOTA เฉพาะโดเมนสายกราฟ

PM2.5-GNN มอง สถานีเป็นโหนดในกราฟ เชื่อมกันด้วยระยะทาง/ลม แล้วส่งข้อมูลระหว่างสถานี (message passing) · เป็นแนวทางยอดนิยมและแข็งสำหรับพยากรณ์ PM2.5 แบบ "ราย-สถานี"

จุดที่ต่างจากเราเชิงโครงสร้าง: GNN ทำงานบน "กราฟของสถานี" (จุด) → เก่งทำนาย ที่สถานีที่มีอยู่ · แต่ ผลิต field เต็มกริดตรงที่ไม่มีสถานีได้ยาก (โยง leave-stations-out, 5.2) · ขณะที่ HazeNet ทำงานบนกริด → ผลิตแผนที่เต็มได้ตามธรรมชาติ · นี่เป็นคู่เทียบที่ดีในการโชว์ข้อได้เปรียบ "การให้ field" ของเรา

ถ้ามีโค้ดสาธารณะ → รันเทียบบนข้อมูลเดียวกันได้ · ถ้าไม่มีเวลาพอ → อ้างอิงตัวเลขจากเปเปอร์อย่างระมัดระวัง (คนละ dataset เทียบตรงไม่ได้ ต้องระบุ)

7. AirPhyNet — คู่เทียบที่สำคัญที่สุด ★

AirPhyNet เป็น physics-informed NN สำหรับพยากรณ์อากาศที่ฝัง advection-diffusion (สมการเดียวกับ 1.2!) เข้ากับ neural network · นี่คือ baseline ที่ ใกล้จิตวิญญาณของ HazeNet ที่สุด — ทั้งคู่คือ "ฟิสิกส์ + AI"

ทำไมนี่คือคู่เทียบที่ "ต้องมี": ชนะ persistence/ConvLSTM ได้ ยังไม่พอพิสูจน์ว่างานเรา "ใหม่" — เพราะ AirPhyNet ก็ใส่ฟิสิกส์เหมือนกัน · การวางตำแหน่งของเราเทียบ AirPhyNet คือหัวใจของ contribution · เราต้องตอบให้ได้ว่า HazeNet ต่าง/ดีกว่าตรงไหน:

8. กติกาเทียบให้ "ยุติธรรม" — ไม่งั้นโดนยำ ★

10. Cheat sheet + แบบฝึกหัด

Cheat sheet — แต่ละ baseline พิสูจน์อะไร

  • Persistence ("เหมือนเมื่อวาน") — แข็งระยะสั้น · แพ้ไม่ได้ที่ t+3
  • Climatology ("เฉลี่ยตามฤดู") — แข็งระยะยาว · เป้า: ต่ำกว่าทั้งคู่ทุกระยะ
  • สถิติตื้น — พิสูจน์ "ต้อง deep learning ไหม"
  • ConvLSTM — DL ไร้ฟิสิกส์ → ชนะ = "ฟิสิกส์ช่วยจริง"
  • PM2.5-GNN — SOTA สายกราฟ (เก่งที่สถานี อ่อนเติม field)
  • AirPhyNet — physics+AI ใกล้เราสุด → จุดต่าง = attribution/inversion
  • เสมอ: baseline แข็ง + split/metric เดียวกัน + จูนเป็นธรรม

❓ แบบฝึกหัด

  1. ทำไม persistence ถึงเป็น baseline ที่ "ลวงตาว่าง่าย"?
  2. ชนะ ConvLSTM พิสูจน์อะไรที่ชนะ persistence ไม่ได้พิสูจน์?
  3. AirPhyNet สำคัญกว่าคู่เทียบอื่นยังไง และเราต่างจากมันตรงไหน?
  4. ยกตัวอย่างการเทียบแบบ "ไม่ยุติธรรม" 1 อย่าง
  5. ถ้า forecast skill เราเสมอ AirPhyNet ยังเคลม contribution ได้ไหม?
เฉลย
  1. PM2.5 เปลี่ยนช้า → ค่าตอนนี้ทำนายระยะสั้นได้ดีมาก โมเดลซับซ้อนหลายตัวแพ้มันที่ t สั้นโดยไม่รู้ตัว
  2. ชนะ persistence = "เรียนรู้อะไรบ้าง" · ชนะ ConvLSTM (DL ไร้ฟิสิกส์) = "ฟิสิกส์ที่เราใส่คือสิ่งที่สร้างความต่าง" ไม่ใช่แค่ใช้ DL
  3. มันคือ physics+AI เหมือนเรา → ใกล้สุด · เราต่างที่ ADR เต็ม+conditionally-linear, field เต็มกริด, และ attribution/inversion ที่มันไม่มี
  4. จูนของเราเต็มที่แต่ปล่อย baseline ไม่จูน / ให้ input เราเยอะกว่า / เทียบคนละ split
  5. ได้ — "ทำ attribution+inversion ได้ โดยแม่นไม่แพ้กัน" คือ contribution (ทำสิ่งที่มันทำไม่ได้)

✅ เช็กลิสต์ก่อนไป 5.4

  • ☐ มี persistence + climatology + สถิติตื้น เป็นเส้นอ้างอิง (ฟรี)
  • ☐ เทรน ConvLSTM บน split เดียวกัน
  • ☐ วางแผน AirPhyNet/PM2.5-GNN (รันเอง หรือระบุว่าอ้างอิงเชิงบริบท)
  • ☐ ทุก baseline ใช้ data/split/metric เดียวกัน + จูนเป็นธรรม
  • ☐ เตรียมตารางจุดต่าง (โดยเฉพาะ attribution/inversion vs AirPhyNet)
