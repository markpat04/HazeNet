HazeNet — สิ่งที่ต้องรู้ทั้งหมด (Learning Roadmap)

เช็กลิสต์ความเข้าใจ ก่อนลงมือเขียนโค้ด · เลือกหัวข้อไปให้อธิบายลึกทีละอัน

🟢 ง่าย · 🟡 กลาง · 🔴 ยาก · ⭐ CORE = หัวใจของงานเรา

วิธีใช้: ไล่อ่านชื่อหัวข้อ → เห็นอันไหนยังไม่เข้าใจ บอกเลขมา (เช่น "อธิบาย 2.5") ผมจะอธิบายลึกจนเข้าใจ · ชุดขั้นต่ำเริ่มโค้ด: 0.1→0.2→1.1→1.2→2.1→2.2→2.5→3.5→4.1→4.4 · 3 หัวใจ: 1.2 → 2.5 → 3.5 (ต่อกันคือจิตวิญญาณทั้งโปรเจกต์)

โมดูล 0 — พื้นฐานที่ขาดไม่ได้

0.1
NumPy / tensor thinking🟢
คิดข้อมูลเป็นก้อนหลายมิติ — ทุกอย่างในงานเราคือ tensor
0.2
PyTorch + autograd🟡
สร้าง+เทรนโมเดล+หา gradient อัตโนมัติ — เครื่องมือหลักเขียนทุกอย่าง
0.3
Gradient descent / loss / optimization🟢
โมเดลเรียนรู้ยังไง — หัวใจการเทรน
0.4
Train/Val/Test · overfitting · generalization🟢
แบ่งข้อมูลกันโกง — พลาดตรงนี้ผลเชื่อไม่ได้
0.5
CNN พื้นฐาน (convolution, receptive field)🟢
เน็ตเห็นภาพเชิงพื้นที่ — ฐานของ U-Net

โมดูล 1 — โดเมนปัญหา (วิทยาศาสตร์บรรยากาศ)

1.1
PM2.5 / AOD / FRP คืออะไร🟢
ฝุ่น/ความหนาควันดาวเทียม/ความแรงไฟ — input/output ของเรา
1.2
สมการ Advection-Diffusion-Reaction (ADR)⭐🔴
สมการการเคลื่อนที่ของมลพิษ — แกนฟิสิกส์ของทั้งโปรเจกต์
1.3
ตัวแปรอุตุนิยม (ลม/PBL/ความชื้น) จาก ERA5🟡
สภาพอากาศที่ขับการเคลื่อนของควัน — อินพุตหลัก
1.4
Transboundary smoke + source apportionment🟡
ควันข้ามแดน + การระบุแหล่ง — pain point + จุดขาย
1.5
WRF-Chem / PMF คืออะไร🟡
แบบจำลองฟิสิกส์/รับสาร — "ground-truth" ที่ใช้ validate

โมดูล 2 — คณิต/ฟิสิกส์เชิงตัวเลข

2.1
Discretization + timestep (กริด+เวลา)🟡
แปลงสมการต่อเนื่องเป็นช่องตาราง — พื้นฐานก่อนเขียนโมเดลเชิงตัวเลข
2.2
Semi-Lagrangian advection⭐🔴
วิธีเลื่อนควันตามลม — คือส่วน 𝒜 (ฟิสิกส์) ในโมเดล
2.3
Operator splitting (Strang)🟡
แยกสมการเป็นส่วนแล้วแก้ทีละส่วน — วิธีรวมฟิสิกส์+เรียนรู้
2.4
Mass conservation · non-negativity · monotone scheme⭐🟡
มวลไม่รั่ว/ควันไม่ติดลบ — คุณสมบัติที่เคลมว่า "พิสูจน์ได้"
2.5
Superposition (หลักการบวกซ้อน)⭐🔴
ทำไม C = Σ C⁽ˢ⁾ ได้ — กุญแจที่ทำให้ attribution + counterfactual เป็นจริง

โมดูล 3 — เทคนิค ML / SOTA ที่เราใช้

3.1
Fourier transform / FFT🔴
แปลงสัญญาณเป็นคลื่นความถี่ — ฐานของ AFNO (มีไฟล์แล้ว)
3.2
Neural Operator + AFNO/FourCastNet🔴
เน็ตเรียน "ตัวดำเนินการ" ในโดเมนความถี่ — backbone ของ ℱθ
3.3
U-Net🟡
สถาปัตยกรรม encoder-decoder — ทางเลือก/ส่วนผสมของ backbone
3.4
Autoregressive rollout + curriculum training🟡
พยากรณ์ต่อเนื่องหลาย step — วิธีไปถึง t+48
3.5
Conditionally-Linear Operator (อัลกอริทึมใหม่ของเรา)⭐🔴
เชิงเส้นใน C แต่สัมประสิทธิ์ไม่เชิงเส้น — "สิ่งที่เราประดิษฐ์" สำคัญสุด
3.6
Tagged-tracer (Smoke Fingerprint)⭐🟡
ติดป้ายแหล่งแล้วตามแยก — กลไก attribution
3.7
Differentiable programming → adjoint → source inversion🔴
ใช้ autodiff แก้ย้อนหาแหล่ง — capability เท่ (stretch/A*)
3.8
Diffusion models (พอเข้าใจว่าทำไมไม่ใช้)🟡
โมเดลสร้างภาพ — เพื่อตอบกรรมการว่าทำไมไม่ทำ (มีไฟล์แล้ว)
3.9
Uncertainty: ensemble / quantile / CRPS🟡
บอกความไม่แน่นอน — แทน diffusion ในการให้ uncertainty

โมดูล 4 — ข้อมูล + วิศวกรรม

4.1
แหล่งข้อมูล + วิธีดึง (FIRMS/ERA5/Himawari/Air4Thai/DEM)🟡
วัตถุดิบทั้งหมดของงาน
4.2
รูปแบบ geospatial: NetCDF / Zarr / xarray / GeoTIFF🟡
ไฟล์ข้อมูลเชิงพื้นที่ — เครื่องมืออ่าน/เขียน datacube
4.3
Regridding / CRS / reprojection🟡
รวมข้อมูลคนละความละเอียดให้กริดเดียว — ขั้นสร้าง datacube
4.4
สร้าง Datacube (pipeline)🟡
รวมทุกอย่างเป็นก้อนเดียวพร้อมเทรน — งานสัปดาห์ที่ 1
4.5
Colab/A100 workflow (memory, mixed precision)🟢
รันบน GPU จำกัด — ไม่ให้ OOM/งบหมด

โมดูล 5 — การประเมิน + การทดลอง (rigor)

5.1
Metrics: RMSE/MAE/ACC + CSI/POD/FAR + onset-lead⭐🟡
วัดความแม่น/การจับวันวิกฤต — ตัวเลขที่เราเคลม
5.2
Cross-validation: temporal + spatial blocking + leave-stations-out⭐🟡
ประเมินกันรั่ว — ทำให้ผลเชื่อถือได้
5.3
Baselines (Persistence/ConvLSTM/PM2.5-GNN/AirPhyNet)🟡
สิ่งที่ต้องเอาชนะ — เข้าใจเพื่อเทียบ
5.4
Ablation study🟡
ตัดทีละชิ้นดูผล — พิสูจน์ว่าแต่ละส่วนจำเป็น
5.5
Error decomposition / Oracle substitution⭐🔴
แยก error รวมว่ามาจากส่วนไหน — เสาหลักของ rigor
5.6
Statistical significance (bootstrap CI, paired test)🟡
เคลม "ดีกว่า" อย่างมีหลักฐาน — กันโดนยำ

โมดูล 6 — ทฤษฎีเชิงลึก (สำหรับ A* — ทำทีหลังได้)

6.1
Attribution Error Bound (ทฤษฎีบท)🔴
พิสูจน์ว่า error ของ attribution มีขอบเขต — ยกระดับเป็นงานวิจัย
6.2
Causal framing (do-operator, identifiability)🔴
ทำให้เคลม "ดับไฟ→ลดฝุ่น" มีรากฐาน — ตอบ reviewer สาย causal
6.3
Inverse problem (ill-posed, regularization)🔴
ทำไม inversion ยากและแก้ยังไง — ฐานของ 3.7

ลำดับการเรียนที่แนะนำ

เป้าหมายลำดับ
เริ่มเขียนโค้ดเร็วสุด0.1 → 0.2 → 1.1 → 1.2 → 2.1 → 2.2 → 2.5 → 3.5 → 4.1 → 4.4
เข้าใจ "ทำไมงานใหม่"1.2 → 2.5 → 3.5 (3 หัวใจ)
