1.2 — สมการ Advection-Diffusion-Reaction (ADR)

แกนฟิสิกส์ที่ร้อยทั้งโปรเจกต์เข้าด้วยกัน · หัวใจที่ 1 ใน 3 (1.2 → 2.5 → 3.5)

สอนจากศูนย์ · สร้างสัญชาตญาณก่อนคณิต · แกะทีละพจน์ · ไม่ต้องมีพื้นแคลคูลัสก็เข้าใจได้

อย่าเพิ่งกลัวสมการ! ADR ที่ดูน่ากลัวนี้ จริง ๆ แล้วเป็นแค่ "การทำบัญชีควัน" — มันตอบคำถามเดียว: "ในกล่องอากาศเล็ก ๆ หนึ่งกล่อง ปริมาณฝุ่นเพิ่มหรือลดเพราะอะไรบ้าง?" · คำตอบมี 5 สาเหตุ (ลมพัดเข้า/ออก, กระจาย, ไฟปล่อยเพิ่ม, ตกหาย, เคมีแปลงร่าง) · บทนี้แกะทีละสาเหตุจนคุณอ่านสมการออกทั้งบรรทัด

1. ภาพใหญ่ — "บัญชีควันในกล่องอากาศ"

นึกภาพแบ่งอากาศเหนือภาคเหนือเป็น กล่องเล็ก ๆ เรียงเป็นตาราง (กริด — เหมือน array ใน 0.1!) · ในแต่ละกล่องมีปริมาณฝุ่น C · คำถามคือ "1 ชั่วโมงผ่านไป ฝุ่นในกล่องนี้เปลี่ยนยังไง?"

ฝุ่นในกล่องเปลี่ยนเพราะ 5 สาเหตุ นี้พอดี — และสมการ ADR ก็คือการเขียน "5 สาเหตุนี้" เป็นภาษาคณิตเท่านั้นเอง:

การเปลี่ยนของฝุ่น = (ลมพัดเข้า−ออก) + (กระจาย) + (ไฟปล่อย) − (ตกหาย) + (เคมี)

2. อ่านฝั่งซ้าย: ∂C/∂t

ฝั่งซ้ายของสมการคือ ∂C/∂t อ่านว่า "อัตราการเปลี่ยนของ C ตามเวลา"

  • ∂ (อ่านว่า "partial") = เหมือน d ในอนุพันธ์ แค่บอกว่า "เปลี่ยนเฉพาะตัวแปรนี้ ตัวอื่นคงที่"
  • ∂C/∂t = "ฝุ่นเปลี่ยนเร็วแค่ไหนต่อหนึ่งหน่วยเวลา" — เป็นบวก = ฝุ่นกำลังเพิ่ม, เป็นลบ = กำลังลด
คุณรู้จักมันอยู่แล้ว! ใน 0.2–0.3 คุณเจอ "gradient = อัตราการเปลี่ยน/ความชัน" · ∂C/∂t ก็คือความชันของ C เทียบกับเวลา — แนวคิดเดียวกัน · ฝั่งซ้าย = "ผลลัพธ์" · ฝั่งขวา = "5 สาเหตุที่ทำให้เกิดผลนั้น"

3. เครื่องมือคณิต 2 ตัว: ∇ (gradient) และ ∇· (divergence)

ก่อนแกะพจน์ ต้องรู้จัก 2 สัญลักษณ์นี้แบบสัญชาตญาณ (ไม่ต้องท่องสูตร)

สัญลักษณ์ชื่อสัญชาตญาณ
∇Cgradient"ความชัน + ทิศ" ของ C — ชี้ไปทางที่ฝุ่นเข้มขึ้นเร็วสุด · ฝุ่นเรียบ = ∇C เล็ก · ฝุ่นเปลี่ยนฮวบ = ∇C ใหญ่
∇·(...)divergence"การไหลสุทธิออกจากจุด" — ของไหลออกมากกว่าเข้า = divergence บวก · เข้ามากกว่าออก = ลบ
เปรียบเทียบให้เห็นภาพ:
  • ∇C (gradient) = ยืนบนเนินฝุ่น มันบอก "ทางขึ้นเขาชันสุดอยู่ไหน" (เหมือน gradient ใน 0.2 เป๊ะ)
  • ∇· (divergence) = ดูน้ำที่ก๊อก: น้ำพุ่งออกทุกทิศ = divergence บวก (เป็นแหล่ง) · ท่อระบายดูดเข้า = ลบ (เป็นที่จม)

ทั้งคู่ทำงานบน "สนาม" (field = ค่ากระจายทั่วพื้นที่ เช่นแผนที่ฝุ่น/ลม) — ตรงกับ array 2D ของเราพอดี

4. พจน์ที่ 1: Advection — ลมพัดพา ★ สำคัญสุดสำหรับควันข้ามแดน

− ∇·(uC) u = ความเร็วลม (สนามลมจาก ERA5) · C = ฝุ่น · uC = "ฟลักซ์ฝุ่นที่ลมพาไป"

ความหมาย: ลมหอบฝุ่นจากที่หนึ่งไปอีกที่ — เหมือนสายพานลำเลียง · ถ้าลมพัดฝุ่นเข้ากล่องมากกว่าออก → ฝุ่นในกล่องเพิ่ม

ในกรณีง่าย 1 มิติ (ลมพัดทิศเดียว ความเร็วคงที่) พจน์นี้ย่อเป็น −u·∂C/∂x = "ฝุ่นเลื่อนไปตามลม" · นี่คือพจน์ที่ ครองทุกอย่างสำหรับควันข้ามแดน เพราะควันเดินทางไกลตามลมเป็นหลัก

เชื่อมกับงานเรา: นี่คือพจน์ที่เราจะฝังเป็น "ฟิสิกส์แข็ง" (semi-Lagrangian) ในโมเดล (หัวข้อ 2.2) — ไม่ปล่อยให้เน็ตเดาการเคลื่อนที่ของควันเอง · และมันคือคำตอบของปัญหา receptive field จาก CNN (0.5): ลมขนฝุ่นไกล ๆ ให้เลย ไม่ต้องให้เน็ตเรียน

5. พจน์ที่ 2: Diffusion — การกระจาย

+ ∇·(K∇C) K = สัมประสิทธิ์การกระจายปั่นป่วน · ∇C = ความชันของฝุ่น

ความหมาย: ฝุ่นกระจายจากที่เข้มไปที่จาง (ความปั่นป่วนของอากาศคลุกเคล้า) — เหมือนหยดหมึกในน้ำที่ค่อย ๆ จางและกว้างออก · ทำให้ก้อนควัน "เบลอ" และแผ่กว้าง

diffusion ทำหน้าที่ตรงข้ามกับ "ความชัน" — ที่ไหนฝุ่นเป็นยอดแหลม (∇C เปลี่ยนเร็ว) มันจะถูกปาดให้เรียบ · ในงานจริงพจน์นี้เล็กกว่า advection มาก (การกระจายช้ากว่าการพัด)

เชื่อมกับงานเรา: ส่วนนี้กับ K ที่เปลี่ยนตามสภาพอากาศ เป็นส่วนที่เราให้เน็ตเรียนรู้ (อยู่ในสัมประสิทธิ์ที่ AFNO ทำนาย หัวข้อ 3.5) เพราะการกระจายใต้กริดซับซ้อนเกินจะเขียนตรง ๆ

6. พจน์ที่ 3: Emission — การปล่อย (ไฟ)

+ E E = อัตราการปล่อยฝุ่นเข้ามาในกล่อง (จาก FRP ที่เรียนใน 1.1)

ความหมาย: ที่ไหนมีไฟ ก็มีฝุ่นเกิดใหม่เติมเข้ามา · พจน์นี้ง่ายสุด — แค่ "บวกเพิ่ม" ตามความแรงไฟ · E มาจาก FRP (ไฟ) ที่เราแปลงเป็นอัตราปล่อยควัน

เชื่อมกับงานเรา: นี่คือจุดที่เรา "ติดป้ายแหล่ง" — แทนที่จะมี E เดียว เราแยกเป็น E จากเมียนมา, จากลาว, จากอำเภอต่าง ๆ → เป็นจุดเริ่มของ tagged-tracer (หัวข้อ 3.6) ที่ทำให้ระบุแหล่งได้

7. พจน์ที่ 4: Loss/Sink — การสูญเสีย

− L·C L = อัตราการสูญเสีย (การตกสะสม dry deposition, ฝนชะ wet deposition)

ความหมาย: ฝุ่นค่อย ๆ หายไป — ตกลงพื้น หรือถูกฝนชะล้าง · สังเกตว่าเป็น −L·C = "ยิ่งมีฝุ่นมาก ยิ่งหายมาก" (สัดส่วนกับปริมาณที่มี) → ให้พฤติกรรมแบบ "สลายตัว"

เชื่อมกับงานเรา: L ก็เป็นสัมประสิทธิ์ที่เน็ตเรียนรู้ (ขึ้นกับฝน/ความชื้น) — ในสมการแกนเราเขียนเป็น D_θ·C โดย D_θ คืออัตราสูญเสียที่ AFNO ทำนาย (หัวข้อ 3.5)

8. พจน์ที่ 5: Chemistry — เคมีแปลงร่าง (พจน์ที่ซื่อสัตย์)

+ Rchem การเกิด/สลายฝุ่นจากปฏิกิริยาเคมี (secondary aerosol) — มักไม่เชิงเส้น

ความหมาย: ฝุ่นบางส่วนไม่ได้ถูกปล่อยตรง ๆ แต่เกิดจากปฏิกิริยาเคมีในอากาศ (เช่น ก๊าซแปลงเป็นอนุภาค = secondary aerosol) · พจน์นี้ซับซ้อนและไม่เชิงเส้น (อาจขึ้นกับ C², หรือสารอื่น)

★ จุดที่ต้องซื่อสัตย์ (สำคัญต่อทั้งโปรเจกต์): พจน์ R_chem นี้คือเหตุผลที่ PM2.5 ไม่ใช่ "สารอนุรักษ์" 100% — มันเกิด/หายได้จากเคมี · ดังนั้นเราจะ ไม่บังคับ "อนุรักษ์มวลเคร่งครัด" กับ PM2.5 ทั้งหมด (จะผิดฟิสิกส์) · แต่เราโฟกัส primary smoke (ควันเผาที่ปล่อยตรง ระยะสั้น 24–48 ชม.) ที่ 4 พจน์แรกครอง และให้เน็ตเรียน R_chem แยก · เชื่อมกับวินัยฟิสิกส์ใน research-contribution

10. ★ ทำไมสมการนี้สำคัญกับเรา — 3 ข้อ

① เราจะ "เลียนแบบฟิสิกส์จริง" ไม่ใช่เดาสุ่ม

สมการ ADR นี้คือสิ่งที่ WRF-Chem แก้เชิงตัวเลข (แบบจำลองมาตรฐานระดับโลก) · HazeNet เป็น "neural emulator" ที่เคารพโครงสร้างนี้ → เร็วกว่า WRF-Chem มหาศาลแต่ยังอิงฟิสิกส์จริง · ทำให้น่าเชื่อถือและ generalize ได้

② ★★ มันเป็น "เชิงเส้นใน C" — กุญแจสู่ attribution (เชื่อม 2.5)

ดู 4 พจน์แรกให้ดี: advection, diffusion, emission, loss — ไม่มี C², ไม่มี C×C · ทุกพจน์แค่ "เอา C มาคูณ/ดำเนินการเชิงเส้น" · นี่คือคุณสมบัติที่ทรงพลังที่สุด:

ถ้าสมการเชิงเส้นใน C → C = Σs C⁽ˢ⁾ (แยกตามแหล่งแล้วบวกกลับได้เป๊ะ) เลียนแบบทำไม? เพราะถ้าแยกควันตามแหล่งกำเนิด แต่ละแหล่งวิ่งตามสมการเดียวกัน → บวกกลับได้พอดี = superposition

นี่คือเมล็ดพันธุ์ของหัวใจที่ 2 (2.5): ความเป็นเชิงเส้นนี้คือเหตุผลที่เราระบุแหล่ง (attribution) ได้แม่นแบบ exact และทำ counterfactual ("ดับไฟ s" = ลบ C⁽ˢ⁾) ได้ฟรี · จำพจน์ "เชิงเส้น" นี้ไว้ให้ดี — มันคือทั้งความใหม่ของงานเรา

③ เราจะ "ฝังบางส่วน เรียนบางส่วน" (เชื่อม 3.5)

เราไม่เอาเน็ตเดาทุกพจน์ และไม่เอาฟิสิกส์ล้วน — เราแบ่ง:

  • ฝังเป็นฟิสิกส์แข็ง: advection (พจน์ที่ครองและรู้แน่)
  • ให้เน็ตเรียนสัมประสิทธิ์: K (diffusion), L (loss), การปรับ E — เพราะขึ้นกับสภาพอากาศซับซ้อน

= ได้ทั้งความน่าเชื่อถือของฟิสิกส์ + ความยืดหยุ่นของการเรียนรู้ (นี่คือ operator splitting หัวข้อ 2.3 + conditionally-linear 3.5)
