HazeNet — ภาพรวมทั้งโปรเจกต์

ตั้งแต่ต้นจนจบ มีกี่ส่วน · แต่ละส่วนเอาข้อมูล/งานวิจัยมาจากไหน · SOTA ยังไงเทียบของเดิม

อ่านไฟล์นี้ไฟล์เดียวเห็นทั้งโปรเจกต์ · ลิงก์งานวิจัยตัวเต็มอยู่ใน related-work-sota.html · แผนกลยุทธ์อยู่ใน hazenet-focused-plan.html

ทางลัด: แผนที่ทั้งระบบ ข้อมูลมาจากไหน 8 ส่วน ละเอียด ตาราง SOTA รวม งานวิจัยอ้างอิง
สรุป 1 ย่อหน้า: HazeNet รับข้อมูล open-source (ไฟจาก FIRMS, ควันจากดาวเทียม Himawari, ลมจาก ERA5, ฝุ่นจากสถานี Air4Thai) มารวมเป็น datacube → ป้อนเข้า โมเดล neural ที่เคารพฟิสิกส์การพัดพา (ส่วนพัดพาเป็นฟิสิกส์ + ส่วนที่เหลือเรียนรู้) → ออกมา 3 อย่างพร้อมกัน: ① พยากรณ์ฝุ่น 24–48 ชม. · ② ระบุแหล่งที่มา (เมียนมา/ลาว/อำเภอไหนกี่%) · ③ ตอบ what-if ("ดับไฟตรงนี้ลดกี่%") — ทั้งหมดในการรันเดียว แล้วพิสูจน์ความถูกต้องด้วยการเทียบ WRF-Chem + PMF จริง

🔧 8 ส่วน อธิบายละเอียด (ทำอะไร · อ้างอิงจากไหน · SOTA ยังไง)

0

Datacube — รวมข้อมูลเป็นก้อนเดียวพร้อมใช้

ทำอะไร
ดึงข้อมูล 5–6 แหล่ง มา regrid ให้อยู่กริดเดียว (0.1°) เวลาเดียว เก็บเป็น Zarr (อ่านเร็ว, ขนานได้) + แยกไฟติดป้ายแหล่ง E⁽เมียนมา⁾, E⁽ลาว⁾, E⁽ไทย-อำเภอ i⁾
อ้างอิง
แนวคิด analysis-ready cloud cube: WeatherBench2, pipeline ข้อมูลของ Aurora/ARCO-ERA5 · FIRMS, P-Tree เป็นมาตรฐานวงการ
SOTA ยังไง
งานมลพิษไทยเดิมมักใช้ตารางรายสถานี/CSV — เราทำเป็น spatiotemporal datacube แบบเดียวกับ foundation model สมัยใหม่ → ขยาย/ทำซ้ำได้
1

Emission tagging — แปลงไฟเป็น "การปล่อยที่ติดป้ายแหล่ง"

ทำอะไร
แปลง FRP จาก FIRMS → อัตราการปล่อยควัน แล้วแยกเป็นช่องตามแหล่งกำเนิด (ประเทศ/อำเภอ) เพื่อให้ติดตามแยกได้ภายหลัง
อ้างอิง
ความสัมพันธ์ FRP→ปริมาณปล่อย: FINN / FINNv2.5 (Wiedinmyer et al.) · แนวคิด tagged-tracer จาก GEOS-Chem / WRF-Chem
SOTA ยังไง
WRF-Chem ทำ tagged-tracer ได้แต่ในแบบจำลองฟิสิกส์ที่ช้า — เราย้ายแนวคิดนี้เข้าโมเดล neural ที่ differentiable (เรียนรู้/ไหลย้อนได้)
2

𝒜 · Physics advection — พัดพาควันตามลม (ฟิสิกส์แท้)

ทำอะไร
เลื่อนสนามควันตามลม ERA5 ด้วยวิธี semi-Lagrangian (ลากย้อนรอยอนุภาค) แบบ monotone → รับประกันควันไม่ติดลบ/ไม่ overshoot
อ้างอิง
semi-Lagrangian ใน NWP: Staniforth & Côté 1991 · การฝังฟิสิกส์ในโมเดล ML: สาย physics-informed / CorrDiff
SOTA ยังไง
AI มลพิษส่วนใหญ่ (PM2.5-GNN, AirFormer) ให้เน็ตเดาการเคลื่อนที่เอง — เราใส่การพัดพาเป็น hard constraint → generalize ไป pattern ไฟใหม่ได้ดีกว่า + ประหยัดข้อมูล
3

ℱθ · Conditionally-linear operator — ส่วนเรียนรู้ (หัวใจความใหม่)

ทำอะไร
ให้ AFNO (โครงเดียวกับ FourCastNet ทำงานในโดเมนความถี่) ทำนายสัมประสิทธิ์ gθ (ปล่อย), Dθ (สูญเสีย), K (กระจาย) จาก Met/Static — แต่การกระทำต่อ C เป็นเชิงเส้น
อ้างอิง
FourCastNet (Pathak 2022) + AFNO (Guibas 2021) · physics-guided pollution: AirPhyNet (ICLR 2024, Neural ODE advection-diffusion), PM2.5-GNN (SIGSPATIAL 2020)
SOTA ยังไง
ของใหม่: AirPhyNet มี physics แต่ไม่มี attribution; AFNO เป็น nonlinear ทั้งตัว — เรารวมเป็น "conditionally-linear" ที่ได้ทั้งพลังเรียนรู้ + เชิงเส้นใน C (เพื่อ attribution exact)
4

Rollout — พยากรณ์ต่อเนื่องถึง t+48

ทำอะไร
ป้อนผลลัพธ์กลับเข้าโมเดลทีละชั่วโมง (autoregressive) จนถึง 24–48 ชม. · เทรนแบบ multi-step เพื่อกัน error สะสม
อ้างอิง
curriculum/rollout training: GraphCast (Science 2023), FourCastNet · เทียบ horizon ตรงกับ pain การเตือนภัย
SOTA ยังไง
ใช้กลยุทธ์ rollout แบบโมเดลพยากรณ์อากาศชั้นนำ แต่ปรับมากับมลพิษ + คงคุณสมบัติเชิงเส้นตลอด rollout
5

Smoke Fingerprint · Attribution — ระบุแหล่ง (ฟรีจากเชิงเส้น)

ทำอะไร
เพราะ C = Σ C⁽ˢ⁾ ติดตามแต่ละ tracer แยกกัน → รู้ทันทีว่าฝุ่นที่เชียงใหม่ ณ ชม.ใด มาจากเมียนมา/ลาว/อำเภอไหนกี่% โดยไม่ต้องรันเพิ่ม
อ้างอิง
แนวคิด source apportionment: HYSPLIT (NOAA back-trajectory), receptor model PMF · tagged-tracer จาก CTM
SOTA ยังไง
ของใหม่: HYSPLIT/PMF ทำหลังเหตุการณ์ + ทีละจุด/ช้า — เราทำ inline + exact + ทุก pixel ทุก ชม. ในโมเดลพยากรณ์ตัวเดียว ยังไม่มี AI มลพิษตัวไหนทำ
6

What-if · Counterfactual — "ดับไฟตรงนี้ลดกี่%" (ฟรี)

ทำอะไร
เพราะเชิงเส้น "ดับไฟแหล่ง s" = ลบ C⁽ˢ⁾ ออก → ได้ฝุ่นเมืองที่ลดลงทันทีในวินาที · ทำหลายสถานการณ์เพื่อหาว่าจัดการตรงไหนคุ้มสุด
อ้างอิง
เทียบมาตรฐาน source-off ของ WRF-Chem (Case1−Case2) · validate: เชียงราย 45% TB ฯลฯ
SOTA ยังไง
ของใหม่: WRF-Chem ทำ source-off ได้แต่ 1 สถานการณ์ใช้คลัสเตอร์รันเป็นวัน — เราทำ interactive ระดับวินาที = สิ่งที่ "มีอยู่ไม่ได้" ถ้าไม่มี emulator เร็ว
7

Validation — พิสูจน์ทุกข้อด้วยข้อมูลจริง

ทำอะไร
V1 skill (CSI/POD/FAR, onset-lead) · V2 งบมวลปิด <2% · V3 สัดส่วนแหล่งตรง WRF-Chem+PMF · V4 counterfactual ตรง source-off · ablation
อ้างอิง
metric/CV: WeatherBench2 · ground-truth ไทย: WRF-Chem Toxics 2024, PMF Chiang Mai AtmEn 2024, WRF-CMAQ ISAM 2026
SOTA ยังไง
ของใหม่: ส่วนใหญ่ validate แค่ skill — เราทำ dual-validation (ฟิสิกส์ WRF-Chem + receptor PMF อิสระ) ยืนยัน attribution คนละแกน หาได้ยากในงาน AI มลพิษ
