HazeCast v2 — ระบบ "ตัดสินใจ" ไม่ใช่แค่ "พยากรณ์"

แผนฉบับสมบูรณ์: โปรเจกต์ควรมีอะไรบ้าง เพื่อให้แตกต่าง · สร้างสรรค์ · มี impact จริง · ใช้ SOTA · ทำได้ใน 2 เดือน / A100 ตัวเดียว / open data เท่านั้น

แก่นความคิดเดียวที่ต้องจำ: งานเหรียญทอง SS5 คือ "โมเดลพยากรณ์" — ของเราต้องเป็น "ระบบตัดสินใจที่มีโมเดลเป็นเครื่องยนต์" ความแตกต่างไม่ได้อยู่ที่ "เปลี่ยนหัวข้อจากอากาศเป็นควัน" แต่อยู่ที่เราต่อ 4 ชั้น ที่แต่ละชั้นกินผลผลิตของชั้นก่อนหน้า และมีชั้นที่ มีอยู่ได้เพราะ AI-emulator เร็วเท่านั้น (กายภาพ/WRF-Chem ทำไม่ได้) — นั่นคือความต่างเชิงโครงสร้าง ไม่ใช่เชิงหัวข้อ

เกณฑ์คัดของเข้าโปรเจกต์ (ใช้ตัดสินทุกฟีเจอร์)

ทุกอย่างที่จะใส่เข้าโปรเจกต์ต้องผ่าน 4 ข้อนี้ ไม่ผ่าน = ตัดทิ้ง ไม่ว่าจะเท่แค่ไหน

  1. ใช้แกนโมเดลเดิม — ต้องต่อยอดเครื่องยนต์ ไม่ใช่โปรเจกต์แปะข้าง
  2. คุณค่าที่เป็นไปไม่ได้ถ้าไม่มีแกนเรา — ถ้าทำได้ด้วย Excel/dashboard ธรรมดา = ไม่นับ
  3. วัดผลของตัวเองได้ — มี metric เฉพาะ ไม่ใช่ "ดูดี"
  4. ต้นทุน ≤ ~1 สัปดาห์ — ไม่งั้นกินเวลาแกนวิทยาศาสตร์

ภาพรวม: สถาปัตยกรรม 4 ชั้น

รายละเอียดแต่ละชั้น — ทำอะไร / SOTA ตรงไหน / ทำให้ของเดิมดีขึ้นยังไง

ชั้น 1 — FORECAST (แกนวิทยาศาสตร์ · ห้ามตาย)

พยากรณ์ PM2.5 เชิงพื้นที่-เวลา

เทคโนโลยี: U-Net + AFNO hybrid Semi-Lagrangian advection (hard constraint) softplus non-negativity

SOTA ตรงไหน: Neural operator ในโดเมนความถี่ (AFNO = แกน FourCastNet) + การฝังกายภาพการพัดพาเป็นข้อจำกัดแข็ง ไม่ใช่ปล่อยให้เน็ตเดาเอง — แนวเดียวกับงาน NeurIPS/ICLR สาย physics-informed neural operator

ทำให้ของเดิมดีขึ้น: ระบบ PCD/GISTDA ที่มีอยู่ใช้สถิติ/WRF-Chem ที่หยาบเชิงพื้นที่และช้า — ของเราให้กริดละเอียด 0.1° เร็วระดับวินาที

คงไว้ 100% — นี่คือหัวใจ ถ้าชั้นนี้ไม่ชนะ baseline ทุกอย่างข้างบนพัง

ชั้น 2 — FINGERPRINT (จากแผนเดิม)

Source Attribution — "ควันก้อนนี้มาจากใคร"

เทคโนโลยี: Tagged-tracer / "Smoke Fingerprint" — ติดป้ายแหล่งกำเนิด (เมียนมา/ลาว/ไทยรายอำเภอ) แล้วลากตามสายลม

SOTA ตรงไหน: แทน HYSPLIT back-trajectory ที่ช้าและรันทีละจุด ด้วยการพ่วง attribution เข้าไปใน forward model ครั้งเดียว

ทำให้ของเดิมดีขึ้น: เถียงเรื่อง "ฝุ่นข้ามแดน" ด้วยตัวเลข ไม่ใช่ความรู้สึก — มีน้ำหนักเชิงนโยบาย/การทูตจริง

คงไว้ — เป็นสะพานบังคับไปสู่ชั้น 3 (ต้องรู้ "ใคร" ก่อนถึงจะ "ลบ" ได้)

ชั้น 3 — WHAT-IF (ไอเดีย Digital Twin ของคุณ · ปรับสโคปใหม่)

What-if Engine + Burn-Window Optimizer

เทคโนโลยี: Counterfactual simulation Scenario search (greedy)

ทำอะไร: เพราะชั้น 1 รับ "ไฟ" เป็นอินพุตอยู่แล้ว เราลบ/ลด/เลื่อนไฟบางพื้นที่แล้วรันใหม่ในไม่กี่วินาที ตอบคำถามที่ WRF-Chem ต้องใช้คลัสเตอร์รันเป็นวัน:

  • "ถ้าห้ามเผา อ.X วันนี้ ฝุ่นเชียงใหม่พรุ่งนี้ลดกี่ %"
  • Burn-Window Optimizer: ค้นหลายพันสถานการณ์ → จัดตาราง "เผาวันไหน-ที่ไหน คนเมืองดมควันน้อยสุด" ต่อเข้าระบบขออนุญาตเผาที่มีจริง (เช่น FireD ของ มช.)

★ ทำไมถึงต่างเชิงโครงสร้าง: ฟีเจอร์นี้มีอยู่ไม่ได้ถ้าไม่มี AI-emulator ที่รันเป็นวินาที — กายภาพล้วนทำ search หลายพันรอบไม่ไหว นี่คือคุณค่าที่ผ่านเกณฑ์ข้อ 2 แบบสมบูรณ์

วิจารณ์ตรง ๆ (ต้องประกาศข้อจำกัด): การลบไฟ = ถามโมเดลนอก distribution ที่เคยเห็น (causal validity จำกัด) — advection ช่วยเพราะเชิงเส้นต่อแหล่งกำเนิด แต่ต้องเสนอเป็น "first-order intervention estimate" ไม่ใช่คำพยากรณ์ศักดิ์สิทธิ์ · ใช้ greedy search ไม่ใช่ RL หรู (eval ไม่ทันใน 2 เดือน)

รับ — แต่ ทิ้งคำว่า "Digital Twin" (เฟ้อ โดนยำง่าย เทียบ Earth-2/DestinE ไม่ได้) เรียก "What-if Engine" ตรงกว่า · ต้นทุน ~3–4 วัน

ชั้น 4 — COMMUNICATE (ไอเดีย GeoLLM ของคุณ · ปรับสโคปใหม่)

Grounded Thai-LLM + Health-Impact Layer

เทคโนโลยี: Typhoon (LLM ไทย open ของ SCB10X) Tool-calling / RAG-grounded Dose-response health model

เวอร์ชันที่รอด (ไม่ใช่ GPT อ่าน JSON):

  • Grounded bulletin generator: ทุกเช้า สร้างรายงานไทยรายจังหวัดอัตโนมัติ — LLM ทำหน้าที่เรียบเรียงภาษาอย่างเดียว ทุกตัวเลขมาจาก datacube ผ่าน template คำแนะนำสุขภาพเป็น rule ตายตัวตามเกณฑ์ สธ.
  • Q&A แบบ tool-calling: ถาม "พรุ่งนี้แม่สายเป็นไง" → LLM เรียกฟังก์ชันคืนตัวเลขจริงจากโมเดล แล้วเรียบเรียง — ห้ามมีตัวเลขจากความจำ LLM เด็ดขาด
  • Health Layer: µg/m³ × WorldPop × dose-response (เปเปอร์เชียงใหม่) → "คาดการณ์ภาระ ER พรุ่งนี้" — เปลี่ยน "ไมโครกรัม" เป็น "จำนวนคนไข้" ที่กรรมการสาย impact ฟังออก

metric ของตัวเอง: grounding accuracy = % ประโยคที่ trace กลับไปหาข้อมูลจริงได้ (ไม่ใช่วัด "ฉลาด")

อันตรายที่สุด — ต้องระวัง: นี่คือโดเมนสุขภาพ LLM hallucinate คำแนะนำผิดครั้งเดียวใน live demo = พังทั้งงาน · ต้องมี canned fallback สำหรับวันเดโม · ซื่อสัตย์ในเอกสารว่าชั้นนี้คือ "last mile of impact" ไม่ใช่วิทยาศาสตร์ใหม่

รับ 50% — ชั้นบาง 1 สัปดาห์ ห้ามให้กินเวลาแกน

ไอเดีย SOTA เพิ่มเติม (ของผมเสนอ + วิจารณ์ตัวเอง)

ไอเดียทำอะไร / SOTAวิจารณ์ตัวเองคำตัดสิน
AlphaEarth fire-forecast
stretch		 ใช้ AlphaEarth embeddings (ฟรีบน GEE) + ประวัติเผา + ปฏิทินเก็บเกี่ยว ทำนาย "ความน่าจะเป็นเผารายอำเภอสัปดาห์หน้า" → ป้อนเป็น fire scenario โจมตีเพดานใหญ่สุด (ไฟอนาคตไม่รู้) + ตอบ "foundation model 2025 อยู่ไหนใน stack" การทำนายไฟยากเป็นโปรเจกต์ย่อยเต็มตัว ถ้าแกนไม่แน่นพอ ตัดทิ้งก่อน W7 stretch เท่านั้น
Health-Impact Layer (อยู่ในชั้น 4) ถูกสุด คุ้มสุด — แค่คูณเลข แต่เปลี่ยนภาษา impact เสี่ยง overclaim ระบาดวิทยา — ต้องใส่ CI + อ้างสัมประสิทธิ์จากเปเปอร์ ไม่ประดิษฐ์เอง รับ

สิ่งที่ผม "ปฏิเสธจะเพิ่ม" (วินัยคือจุดขาย)

3D twin visualization สวยแต่ skill = 0 · Multi-agent orchestration โรงละคร · RL เต็มรูป eval ไม่ทัน 2 เดือน · Real-time streaming infra เดโมไม่ต้องการ · Diffusion ensemble เกินงบ GPU

การ "ไม่ใส่" คือสิ่งที่คุณบอกเองว่าต้องการ — งานเหรียญทองชนะด้วย pipeline เดียวที่ทำเป็นเลิศ ไม่ใช่บุฟเฟต์ฟีเจอร์ที่ละ 60%

Scope Ladder — ลำดับการตัดเมื่อเวลาไม่พอ

  1. MUST (W1–W5): ชั้น 1 + ชั้น 2 ชนะ baseline ทุกตัว (CSI/POD/FAR/onset-lead) — ถ้าตรงนี้ไม่เสร็จ ทุกอย่างข้างล่างหด
  2. SHOULD (W5–W6): ชั้น 3 What-if Engine (greedy) + Health Layer
  3. NICE (W6–W7): ชั้น 4 Grounded Thai-LLM (มี canned fallback)
  4. STRETCH (W7): Burn-Window Optimizer เต็มรูป + AlphaEarth fire-forecast
  5. W8: ablation 3 ชุด, เขียน report, ซ้อมเดโม

กฎเหล็ก: ถ้า W5 ชั้น 1 ยังไม่ชนะ baseline → ชั้น 3–4 หดเป็นเดโมแนวคิดทันที แกนวิทยาศาสตร์ห้ามตายเพื่อฟีเจอร์

ราคาที่ต้องจ่าย (พูดตรง)

  • 8 สัปดาห์เดิมตึงอยู่แล้ว — เพิ่มชั้น 3+4 = ต้องตัด diffusion stretch ตาย
  • ลด ablation เหลือ 3 ชุด
  • ชั้น 3 ใช้ greedy ไม่ใช่ optimizer หรู · ชั้น 4 มี template fallback
  • งบ GPU จริง Colab Pro ≈ 15 A100-hr → ต้องวางแผน budget แน่น ไม่ใช่ 60–70 hr ที่เคยเคลม
  • AlphaEarth อยู่ท้ายคิวสุด

สรุป: ทำไมโครงนี้ "ดีกว่า" ไม่ใช่ "เหมือน"
