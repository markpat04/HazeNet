① กลุ่มที่ 1 — "AI แทนโมเดลฟิสิกส์" (รากฐานความเชื่อทั้งโปรเจกต์)

คำถามแรกที่กรรมการจะถาม: "AI จะจำลองการเคลื่อนของสารในบรรยากาศแทนสมการเคมี-ฟิสิกส์ได้จริงเหรอ?" กลุ่มนี้คือหลักฐานว่า ได้ และทำไปแล้วสำเร็จระดับ Nature

1. Aurora — Foundation Model ที่พยากรณ์ "มลพิษอากาศทั้งโลก" ได้แม่นกว่าระบบยุโรป ★สำคัญที่สุดสำหรับเรา
Microsoft Research · Nature 2025 · Nature: s41586-025-09005-y · arXiv:2405.13063 · โค้ด/โมเดลเปิด
3D Swin TransformerFoundation modelCAMS air quality1.3B params
คืออะไร: โมเดลฐานของบรรยากาศที่ pretrain จากข้อมูลธรณีฟิสิกส์ >1 ล้านชั่วโมง แล้ว fine-tune ไปทำงานอะไรก็ได้ — การทดลองที่เกี่ยวกับเราที่สุดคือ fine-tune ให้พยากรณ์ มลพิษอากาศ (PM2.5, PM10, NO₂, O₃, SO₂, CO) แทนระบบ CAMS ของยุโรป

ทำงานยังไง (ทีละขั้น)

  1. Pretrain: ป้อนข้อมูลบรรยากาศหลายแหล่ง (ERA5, GFS, IFS, แบบจำลองภูมิอากาศ) ให้เรียน "พลวัตทั่วไปของบรรยากาศ" — เหมือนเด็กที่ดูเมฆ ลม ฝนมาทั้งชีวิตจนรู้สึกได้ว่าอากาศ "ไหลยังไง"
  2. สถาปัตยกรรม: encoder ยืดหยุ่น — รับตัวแปรกี่ตัวก็ได้ (ผิวพื้น + หลายชั้นความสูง) แปลงเป็นโทเคน 3 มิติ → ตัวประมวลกลางเป็น 3D Swin Transformer U-Net → decoder ถอดกลับเป็นตัวแปรชุดใดก็ได้
  3. Fine-tune กับ CAMS: เอาโมเดลที่ "เข้าใจอากาศ" แล้ว มาสอนต่อด้วยข้อมูลวิเคราะห์มลพิษของ CAMS (0.4°) แค่ช่วงสั้น ๆ — จุดเด็ดคือไม่ป้อนข้อมูลการปล่อยมลพิษ (emission) เลย
  4. พยากรณ์: วนแบบ autoregressive ได้หลายวัน — ผลคือเทียบเท่า/ชนะ CAMS ใน 74% ของเป้าหมาย ที่ต้นทุนถูกกว่า ~5,000 เท่า
  5. สิ่งที่น่าทึ่งสุด: โมเดลเรียนรู้ "รูปแบบการปล่อยจากกิจกรรมมนุษย์" เองโดยปริยาย — เช่น NO₂ ลดลงวันหยุดสุดสัปดาห์ วัฏจักรกลางวัน-กลางคืน ทั้งที่ไม่มีใครบอก
ใช้กับ HazeCast ยังไง: นี่คือ "หลักฐานชั้น Nature" ว่าแนวคิดแกนของเราเป็นไปได้ — AI เรียนการเคลื่อนของมลพิษโดยไม่ต้องรู้ emission สมบูรณ์ แต่ Aurora หยาบ (0.4° ≈ 40 กม. ทั้งแอ่งเชียงใหม่ = พิกเซลเดียว) ไม่มีไฟเป็นเงื่อนไข ไม่มี attribution — ช่องว่างของเราคือทำสิ่งเดียวกันแบบ regional 0.1° + fire-conditioned + อธิบายแหล่งที่มา
2. GraphCast — ผู้พิสูจน์สูตร "เทรนทำนายทีละก้าว แล้ววนให้ไกล"
Google DeepMind · Science 2023 · arXiv:2212.12794
GNN message passingMulti-meshAutoregressive
คืออะไร: โมเดลพยากรณ์อากาศโลก 10 วันที่ชนะระบบ HRES ของ ECMWF ใน 90% ของตัวชี้วัด — ตัวกำหนดมาตรฐานของวงการก่อนยุค foundation model

ทำงานยังไง

  1. แปลงโลกเป็น กราฟ icosahedral หลายระดับ (ตาข่ายหยาบ→ละเอียดซ้อนกัน) — เส้นเชื่อมระยะไกลช่วยให้ข้อมูลข้ามทวีปถึงกันเร็ว
  2. Encoder: ดึงข้อมูลจากกริด lat-lon เข้าโหนดของตาข่าย → Processor: message passing 16 รอบ (โหนดแลกข้อมูลกับเพื่อนบ้าน) → Decoder: คายกลับเป็นกริด
  3. ทำนายทีละ 6 ชม. แล้ว วนป้อนตัวเอง 40 รอบ = 10 วัน
  4. เคล็ดลับการเทรนที่เราต้องลอก: ค่อย ๆ เพิ่มจำนวนก้าวตอน fine-tune (เทรน 1 ก้าวก่อน → 2 → ... → 12) เพื่อให้โมเดลทนต่อ error สะสมของตัวเอง
ใช้กับ HazeCast ยังไง: สูตร "autoregressive + curriculum เพิ่มก้าว" คือสิ่งที่ Stage B ต้องทำตอน rollout fine-tune สัปดาห์ที่ 5 — ปัญหา drift ที่เรากลัว GraphCast เจอก่อนและแก้ด้วยวิธีนี้
3–4. FourCastNet (AFNO) + StormCast — พิมพ์เขียวจากงานเหรียญทองเดิม
NVIDIA · FourCastNet arXiv:2202.11214 · AFNO arXiv:2111.13587 · StormCast arXiv:2408.10958
คืออะไร: สองโมเดลที่งานเหรียญทอง SS5 ใช้ — FourCastNet ผสมข้อมูลทั้งภาพผ่านโลกความถี่ (FFT→ฟิลเตอร์→iFFT, O(N log N)) ส่วน StormCast คือสูตร "Regression ทายค่าเฉลี่ย + Diffusion เติมรายละเอียด"
ใช้กับ HazeCast ยังไง: AFNO block = ตัวเลือก bottleneck ของ Stage B (จับ transport ระยะไกลข้ามแดน) · สูตร mean+residual ของ StormCast = แบบแผนถ้าเราเพิ่ม diffusion sharpening เป็น stretch goal — อธิบายละเอียดครบแล้วในเอกสารก่อนหน้า (concepts-deep-dive.html, math-deepdive-diffusion-fourier.html) จึงไม่ทวนที่นี่

② กลุ่มที่ 2 — พยากรณ์คุณภาพอากาศด้วย Deep Learning (งานสายตรงของเรา)

สามงานนี้คือวิวัฒนาการของการพยากรณ์ PM2.5: กราฟ+ความรู้โดเมน → Transformer ระดับประเทศ → ใส่สมการฟิสิกส์เข้าไปในโครงข่าย

5. PM2.5-GNN — คนแรกที่ "เขียนความรู้เรื่องลมและภูเขา" ลงในกราฟ
SIGSPATIAL 2020 · arXiv:2002.12898 · GitHub (โค้ดเปิด) · ACM DL
GNN + GRUDomain knowledgeเคยใช้งานออนไลน์จริง
คืออะไร: โมเดลกราฟพยากรณ์ PM2.5 ของจีนถึง 72 ชม. ที่ฉลาดตรง "วิธีสร้างกราฟ" — ไม่ได้เชื่อมทุกเมืองมั่ว ๆ แต่เชื่อมตามความเป็นไปได้จริงที่มลพิษจะลอยถึงกัน

ทำงานยังไง

  1. สร้างกราฟด้วยความรู้โดเมน: เมือง = โหนด · ลากเส้นเชื่อมเฉพาะคู่เมืองที่ (ก) ใกล้กันพอ และ (ข) ไม่มีภูเขาขวาง (เช็คความสูงภูมิประเทศตามแนวเส้น!) — ฝุ่นข้ามเทือกเขาไม่ได้ กราฟก็ไม่ควรมีเส้นนั้น
  2. ใส่ลมลงบนเส้นเชื่อม: attribute ของแต่ละเส้น = ความเร็วลม×ทิศลมฉายลงแนวเส้น + ระยะทาง → เส้นไหน "ลมพัดตามแนว" ส่งอิทธิพลแรง เส้นไหนลมสวนทาง = อิทธิพลต่ำ — นี่คือ advection เวอร์ชันกราฟ
  3. เดินเวลา: GNN ส่งข้อความระหว่างโหนด (ฝุ่นไหลตามเส้น) ทำงานคู่กับ GRU (ความจำเชิงเวลาของแต่ละเมือง) ทำนายทีละชั่วโมงแบบ autoregressive
  4. ผล: ชนะ baseline ทุกตัวในยุคนั้น โดยเฉพาะ การทำนายระยะไกล (48–72 ชม.) ที่ตัวไร้โครงสร้าง transport พังหมด — และถูก deploy เป็นบริการจริง
ใช้กับ HazeCast ยังไง: ยืนยันสองดีไซน์ของเรา: (1) ภูมิประเทศต้องอยู่ในโครงสร้างโมเดล (แอ่งเชียงใหม่!) (2) ลมต้องเป็น "ตัวนำทางการไหล" ไม่ใช่แค่ฟีเจอร์แบน ๆ — ต่างกันที่เราทำบน กริดเต็มแผ่น (ครอบพื้นที่ไร้สถานี+ฝั่งเมียนมา) ด้วย advection operator จริง ไม่ใช่กราฟจุดสถานี
6. AirFormer — Transformer ระดับประเทศ + กลไกความไม่แน่นอนในตัว
AAAI 2023 · AAAI Proceedings · PDF เต็ม
Dartboard attentionStochastic latents1,085 สถานี
คืออะไร: Transformer พยากรณ์คุณภาพอากาศ ทั้งประเทศจีน 1,085 สถานีพร้อมกัน ถึง 72 ชม. — แก้สองปัญหา: attention ปกติแพงเกิน (N² ของพันสถานี) และพยากรณ์เดี่ยวไม่บอกความมั่นใจ

ทำงานยังไง

  1. Stage ล่าง (deterministic): ออกแบบ "Dartboard Spatial Attention" — แต่ละสถานีไม่มองทุกสถานี แต่มองเพื่อนบ้านเป็น "วงแหวน+เสี้ยวทิศ" รอบตัว (เหมือนเป้าปาลูกดอก) → ต้นทุนลดมหาศาล และตรงฟิสิกส์: อิทธิพลฝุ่นมาตามระยะ+ทิศ
  2. คู่กับ causal temporal attention เรียนลำดับเวลาของแต่ละสถานี
  3. Stage บน (stochastic): ตัวแปรแฝงแบบ VAE ซ้อนหลายชั้น จับ "ความไม่แน่นอนแท้จริง" ของอากาศ → สุ่มอนาคตได้หลายแบบ ไม่ใช่เส้นเดียว
  4. ผล: ลด error 5–8% จาก SOTA เดิมที่ 72 ชม. โดยเฉพาะช่วง "เปลี่ยนกะทันหัน" (sudden change) ที่โมเดลธรรมดาตามไม่ทัน
ใช้กับ HazeCast ยังไง: (1) dartboard = หลักฐานว่า attention เชิงพื้นที่ที่ "เคารพระยะ+ทิศ" ดีกว่า attention ดิบ — สนับสนุนการเลือก U-Net+AFNO แทน ViT เต็มรูป (2) stochastic latents คือทางเลือก uncertainty ที่ถูกกว่า diffusion — เทียบกับ quantile heads ที่เราวางไว้ใน MVP
7. AirPhyNet — ใส่ "สมการ advection–diffusion" เข้าไปในโครงข่ายตรง ๆ ★ญาติใกล้ชิดที่สุดของ Stage B
ICLR 2024 · arXiv:2402.03784 · OpenReview · ICLR Poster
Physics-guidedNeural ODEGraph Laplacian diffusionWind-directed advection
คืออะไร: งานที่ใกล้แนวคิด HazeCast ที่สุดในโลกวิชาการตอนนี้ — แทนที่จะให้ NN เดาล้วน ๆ มันบังคับให้ "แกนพลวัต" ของโมเดลเป็น สมการเชิงอนุพันธ์ของการแพร่ (diffusion) และการพา (advection) ของอนุภาคจริง ๆ

ทำงานยังไง

  1. Encoder: GRU+GNN อ่านประวัติ PM2.5 และอุตุฯ ของเครือข่ายสถานี → บีบเป็น "สถานะความเข้มข้นแฝง" (latent concentration)
  2. หัวใจ — Physics Core: ให้สถานะแฝงวิวัฒน์ตาม Neural ODE ที่ขวามือของสมการคือ: dC/dt = k_diff · (Graph Laplacian)·C (การแพร่จากที่เข้มไปที่จาง) + advection ตามทิศลมบนเส้นกราฟ (การพาโดยลม) — สัมประสิทธิ์เรียนรู้จากข้อมูล แต่รูปสมการล็อกตามฟิสิกส์
  3. Solver: แก้ ODE เดินหน้าในเวลาต่อเนื่อง (ไม่ใช่ก้าวกระโดดแบบ RNN) ไปถึง horizon ที่ต้องการ
  4. Decoder: ถอดสถานะแฝงกลับเป็นค่า PM2.5 รายสถานี
  5. ผล: ลด error สูงสุด ~10% ที่ 24/48/72 ชม. และเด่นชัดสุดในสองสถานการณ์ที่เราแคร์ที่สุด: ข้อมูลเบาบาง (sparse) และเหตุการณ์เปลี่ยนกะทันหัน + พลวัตภายในตีความได้เชิงฟิสิกส์
ใช้กับ HazeCast ยังไง: นี่คือหลักฐานเด็ดที่สุดว่า "physics-guided ชนะ data-driven ล้วนตรง onset และที่ข้อมูลน้อย" — ตรงคำวิจารณ์ที่เรากลัวที่สุดพอดี แต่ AirPhyNet ยังอยู่บนกราฟสถานี ไม่มีไฟ ไม่มีดาวเทียม ไม่มีสนามเต็มแผ่น ไม่มี attribution → ช่องของเรา = ยกแนวคิดนี้ขึ้นกริด 2 มิติ + เติมไฟ/AOD + Smoke Fingerprint (และต้องอ้างงานนี้ในโปสเตอร์อย่างให้เกียรติ — กรรมการสาย ML จะรู้จักมัน)
8. TabNet + TROPOMI (ประเทศไทย) — prior art ไทยที่ใกล้เราที่สุด (แต่เป็น "ประมาณค่า" ไม่ใช่ "พยากรณ์")
Remote Sensing Applications 2023 · ScienceDirect
คืออะไร: ใช้ TabNet (deep learning ตีความได้สำหรับข้อมูลตาราง) ประมาณ PM2.5 ผิวพื้นทั่วไทยจากก๊าซที่ดาวเทียม TROPOMI วัด (CO, NO₂, aerosol index) ช่วง 2018–2020 — พบว่า CO คือตัวแปรชี้นำสำคัญสุด ซึ่งเชื่อมตรงกับการเผาชีวมวล
ใช้กับ HazeCast ยังไง: (1) เป็น prior art ไทยที่ต้องอ้างใน gap table — มัน estimate ปัจจุบัน เราพยากรณ์อนาคต (2) บอกใบ้ฟีเจอร์: TROPOMI CO อาจเป็นช่องอินพุตเสริมที่แรงสำหรับ Stage A เพราะผูกกับควันไฟตรง ๆ

③ กลุ่มที่ 3 — ทางเลือกสถาปัตยกรรมที่ต้อง "รู้ก่อนเลือก" (ไม่ใช่เลือกตามความเคยชิน)

9. MetNet-3 — ขั้วตรงข้ามของ autoregressive: ทำนาย "ตรงไปยังเวลาเป้าหมาย" + เรียนจากสถานีเบาบางได้

คืออะไร: โมเดลพยากรณ์ 24 ชม. ที่ไม่วนป้อนตัวเองเลย — บอก lead time ที่ต้องการเป็นอินพุต แล้วทำนายชั่วโมงนั้นตรง ๆ (error ไม่สะสม) และมีเทคนิคที่ตรงปัญหาเราเป๊ะ: เรียน "สนามหนาแน่น" จาก "จุดสังเกตเบาบาง"

  1. อินพุต: เรดาร์+ดาวเทียม (หนาแน่น) + รายงานสถานีอุตุฯ ที่เบาบาง (แปะลงกริดเป็นจุด ๆ) + สถานะ NWP + ภูมิประเทศ
  2. Densification (เคล็ดลับทอง): ตอนเทรน สุ่มปิดบังสถานีบางส่วน แล้วบังคับให้โมเดลทำนายค่าที่สถานีถูกปิด → โมเดลเรียน "การประมาณข้ามจุด" (data assimilation โดยปริยาย) จนคายสนามเต็มแผ่นได้ทั้งที่ truth เป็นแค่จุด
  3. โครงข่าย UNet/MaxVit รับ lead time เป็นเงื่อนไข — ถามกี่ชั่วโมงข้างหน้าก็ตอบช็อตเดียว ไม่ง้อการวน
  4. ผล: ชนะ HRRR (โมเดลฟิสิกส์ภูมิภาคที่ดีสุดของสหรัฐฯ) ถึง 24 ชม. และรันจริงใน Google
ใช้กับ HazeCast ยังไง (สองจุดใหญ่): (1) เทคนิค densification = วิธี supervise Stage A จากสถานี ~40 จุดที่ถูกต้องตามตำราที่สุด — มีคนพิสูจน์แล้วระดับ production (2) คำวิจารณ์แผนเรา (รอบล่าสุด) บอกว่าเลือก autoregressive เพราะ "ลอกพิมพ์เขียว" — MetNet-3 คือฝั่งตรงข้ามที่ต้องลองเทียบใน W4: AR (สะสม error แต่ consistent เชิงพลวัต) vs Direct (ไม่สะสมแต่แต่ละ lead แยกกัน) ให้ตัวเลขตัดสิน ไม่ใช่ศรัทธา
10. CorrDiff — สูตรสำเร็จ "UNet ทายค่าเฉลี่ย + Diffusion แก้ส่วนต่าง" สำหรับ downscaling 25→2 กม.
NVIDIA · Communications Earth & Environment 2025 · arXiv:2309.15214 · วารสาร · โค้ดใน PhysicsNeMo
Residual diffusionEDMไต้หวัน 2 กม.
คืออะไร: เจเนอเรทีฟ downscaling — รับสภาพหยาบ 25 กม. แล้วสร้างสนามละเอียด 2 กม. ที่คมและสถิติถูก (สเปกตรัม/การกระจายตรงกับความจริง) ทดสอบกับไต้ฝุ่นไต้หวัน

ทำงานยังไง

  1. ขั้น 1 UNet regression: เรียน mapping หยาบ→ละเอียดแบบ deterministic — ได้ "ค่าเฉลี่ยที่แม่นแต่เบลอ" (ปัญหา MSE เฉลี่ยทุกความเป็นไปได้)
  2. ขั้น 2 Diffusion บน residual: เทรน EDM diffusion ให้สร้างเฉพาะ ความจริง − ค่าเฉลี่ย = รายละเอียดความถี่สูงที่หายไป
  3. Inference: mean + สุ่ม residual หลายครั้ง → ได้ ensemble สนามคมจริง · 4.1 วินาที/ตัวอย่าง บน GPU เดียว
  4. เคสไต้ฝุ่น: ความชันลม/อุณหภูมิคมขึ้น แถบฝนรอบตาไต้ฝุ่นถูกสังเคราะห์กลับมา — ของที่ regression ล้วนทำไม่ได้
ใช้กับ HazeCast ยังไง: ถ้า stretch goal "sharpening/ensemble" ถูกปลดล็อก นี่คือสูตรที่พิสูจน์แล้วบน GPU เดียว (เร็วพอ Colab) — และเชิงแนวคิด มันยืนยันบทเรียนจาก StormCast: อย่าให้โมเดลเดียวแบกทั้ง "ค่าเฉลี่ย" และ "รายละเอียด" — แยกงานแล้วบวกกัน
11. GenCast — พยากรณ์แบบ "ความน่าจะเป็น" ด้วย diffusion ensemble
Google DeepMind · Nature 2025 · arXiv:2312.15796
Conditional diffusionEnsembleCRPS
คืออะไร: แทนที่จะทำนายอนาคตเส้นเดียว GenCast ใช้ diffusion สุ่มอนาคตที่เป็นไปได้หลายสิบแบบ (ensemble) — ชนะระบบ ensemble ปฏิบัติการของยุโรป (ENS) ใน 97.2% ของเป้าหมาย

ทำงานยังไง

  1. มอง "การพยากรณ์" เป็นการสุ่มจากการแจกแจง p(อนาคต | ปัจจุบัน) — เทรน conditional diffusion ที่เริ่มจาก noise แล้ว denoise เป็นสถานะอากาศถัดไป (ก้าว 12 ชม.)
  2. สุ่มหลายครั้ง = สมาชิก ensemble หลายตัว → คำนวณ "ความน่าจะเป็นที่ฝน/ลมจะเกิน threshold"
  3. วัดด้วย CRPS (คะแนนความน่าจะเป็น) ไม่ใช่แค่ RMSE — มาตรฐานใหม่ของการประเมินพยากรณ์
ใช้กับ HazeCast ยังไง: ปลายทางของผลิตภัณฑ์เตือนภัยต้องเป็นความน่าจะเป็น ("โอกาส 80% ที่แม่สายจะแดงพรุ่งนี้") — MVP เราใช้ quantile heads (ถูกกว่า) แต่ GenCast คือเป้าระยะยาวและเป็นเหตุผลที่เราเก็บ CRPS ไว้ในชุดตัวชี้วัดตั้งแต่แรก

④ กลุ่มที่ 4 — จากดาวเทียมสู่ PM2.5 ภาคพื้น (วิชาของ Stage A)

12. van Donkelaar / ACAG — มาตรฐานโลก 25 ปีของการแปลง AOD → PM2.5 (และบทเรียน "อย่าให้ ML มั่วในที่ไร้สถานี")
Washington University ACAG · ACS EST Air 2024 · เปเปอร์ · ชุดข้อมูลเปิด (ทั้งโลก รายเดือน) · ใช้ใน GEE ได้
Geophysical priorCNN bias correctionGEOS-Chem
คืออะไร: ชุดข้อมูล PM2.5 ทั้งโลกที่ถูกอ้างอิงมากที่สุด (ใช้ใน Global Burden of Disease) — หัวใจคือวิธี 3 ชั้นที่แก้ปัญหา "AOD ไม่ใช่ PM2.5 และสถานีมีไม่ทั่วโลก"

ทำงานยังไง

  1. Geophysical prior: ใช้โมเดลเคมี GEOS-Chem คำนวณ "อัตราส่วน AOD→PM2.5" (η) รายพื้นที่รายเดือน — เพราะ AOD คือฝุ่นทั้งคอลัมน์อากาศ แต่ PM2.5 คือชั้นที่จมูกคน η ขึ้นกับความสูงชั้นผสม ความชื้น ชนิดฝุ่น
  2. รวม AOD หลายดาวเทียม (MODIS×2, MISR, VIIRS×2, SeaWiFS — ถ่วงน้ำหนักตามความไม่แน่นอนเทียบ AERONET) × η = PM2.5 เชิงธรณีฟิสิกส์
  3. CNN เรียนเฉพาะ "bias": โครงข่ายไม่ทาย PM2.5 ตรง ๆ แต่ทายส่วนต่างระหว่างค่าธรณีฟิสิกส์กับสถานีจริง — และ loss ถูกออกแบบให้ยึดกับ prior ในพื้นที่ไร้สถานี (กันโมเดลเพ้อเจ้อนอกเขตที่ตรวจสอบได้ — ปัญหาที่ MSE เปล่า ๆ ทำพัง)
  4. ผลลัพธ์: แผนที่รายเดือน 0.01° ทั้งโลก 1998–ปัจจุบัน พร้อมความไม่แน่นอน — เปิดฟรี
ใช้กับ HazeCast ยังไง: ปรัชญา Stage A ของเราคือเวอร์ชันรายชั่วโมงของสูตรนี้: prior จากโมเดล (CAMS) + ML แก้ bias เฉพาะที่ตรวจสอบได้ + ยึด prior ตรงช่องว่างเมียนมา — และ loss แบบ "anchored" ของเขาคือคำตอบตรงสำหรับคำวิจารณ์เรื่อง "9,000 เซลล์ / 40 สถานี" ของเรา
13. PM2.5 รายชั่วโมงจากดาวเทียมค้างฟ้า Himawari — ทำไม "ค้างฟ้า" เปลี่ยนเกม
arXiv 2018 (งานบุกเบิกแนวนี้) · arXiv:1803.03409 · ข้อมูลจริง: JAXA P-Tree (AOD 5 กม./รายชั่วโมง เปิดฟรี)
คืออะไร: สายงานที่ใช้ AOD จากดาวเทียมค้างฟ้า (ถ่ายทุก 10 นาที ไม่ใช่วันละ 1–2 รอบแบบดาวเทียมโคจรต่ำ) + ML → PM2.5 ผิวพื้น รายชั่วโมงแบบไร้รอยต่อ — จับวัฏจักรรายวันของฝุ่น (เช้าหนัก-บ่ายคลาย) ที่ MODIS มองไม่เห็น
ใช้กับ HazeCast ยังไง: Himawari L3 AOD คืออินพุตหลักของ Stage A — สายงานนี้พิสูจน์ว่ามันแปลงเป็น PM2.5 รายชั่วโมงได้จริง พร้อมเตือนข้อจำกัดที่เรารับมาแล้ว: bias สูงใน SEA, ใช้ไม่ได้กลางคืน/ใต้เมฆ/ควันหนาจัด → ต้องมี mask channel เสมอ

⑤ กลุ่มที่ 5 — ฟิสิกส์อ้างอิง + บริบทไทย (ครู ตัววัด และบรรพบุรุษของ Smoke Fingerprint)

14. WRF-Chem ศึกษาหมอกควันข้ามแดนภาคเหนือ — "ครูฟิสิกส์" ที่เราใช้ตรวจคำตอบ attribution
2024 · PMC11280843 (เปิดอ่านฟรี)
คืออะไร: รันโมเดลเคมี-อุตุนิยมวิทยาเต็มรูป (WRF-Chem + emission จากไฟ FINN) เหนือภาคเหนือ แล้วทำการทดลองแยกแหล่ง (เปิด/ปิด emission รายภูมิภาค) — ข้อสรุปหลัก: PM2.5 ข้ามแดนเริ่มไหลเข้าตั้งแต่ ม.ค. แรงสุดเดือน มี.ค. และบ่อยครั้งมากกว่าแหล่งในประเทศ

ทำงานยังไง (แบบย่อ)

  1. ตั้งโดเมนจำลองครอบไทย-เมียนมา-ลาว ป้อนสภาพอุตุฯ ขอบเขต + emission จากจุดไฟดาวเทียม
  2. แก้สมการครบ: การพา การแพร่ เคมี การตกสะสม — กิน CPU คลัสเตอร์หลายชั่วโมงต่อ 1 เคส
  3. รันซ้ำแบบ "ปิดแหล่งทีละอัน" → ส่วนต่างคือ contribution ของแหล่งนั้น (วิธี attribution มาตรฐานทอง)
ใช้กับ HazeCast ยังไง: (1) นี่คือสิ่งที่เรา "emulate" — ความช้าของมันคือเหตุผลการมีอยู่ของเรา (2) ตัวเลขสัดส่วนข้ามแดนของมันคือ ข้อสอบเฉลย ที่ Smoke Fingerprint ต้องสอบให้ทิศทางตรง (3) วิธี "ปิดแหล่งทีละอัน" = แม่แบบของ occlusion attribution ที่เราเพิ่มเข้าแผนหลังการวิจารณ์
15. FINNv2.5 NRT สำหรับภาคเหนือ — สะพาน "จุดไฟ → ปริมาณการปล่อย" แบบใกล้เรียลไทม์
2025 · PMC12846012
คืออะไร: ระบบที่แปลงจุดความร้อน VIIRS เป็น "ปริมาณการปล่อย PM2.5 รายวัน" สำหรับภาคเหนือโดยเฉพาะ: จุดไฟ → ชนิดเชื้อเพลิง (จาก land cover) → มวลที่เผา → คูณ emission factor ต่อชนิดป่า/พืช
ใช้กับ HazeCast ยังไง: ดีกว่าที่เราคิดไว้เดิม — แทนที่จะใช้ FRP ดิบเป็นพร็อกซี อาจใช้สูตร FINN (หรือ feed ของมัน) เป็นช่อง "emission โดยประมาณ" ที่มีหน่วยฟิสิกส์จริง → ลดภาระให้ NN เรียนเอง และอ้างเป็น prior art ไทยที่เราต่อยอด (เขาหยุดที่ emission เราพาไปถึง "ความเข้มข้นที่จมูกคน + พยากรณ์ + แหล่งที่มา")
16. HYSPLIT — เครื่องมือ back-trajectory อายุ 40 ปี: บรรพบุรุษทางความคิดของ Smoke Fingerprint
NOAA ARL · เว็บทางการ (ใช้ฟรีออนไลน์)
คืออะไร: โมเดล Lagrangian มาตรฐานโลก: ปล่อย "อนุภาคสมมติ" แล้วตามรอยมัน ย้อนเวลาตามสนามลม → ตอบว่า "อากาศที่จุดนี้ เมื่อ 72 ชม. ก่อนอยู่ที่ไหน" — นักวิจัยหมอกควันทั่วเอเชียใช้พิสูจน์เส้นทางควันข้ามแดนมาหลายสิบปี
ใช้กับ HazeCast ยังไง: Smoke Fingerprint ของเราคือญาติ "เดินหน้า + ทั้งกริด + อัตโนมัติรายชั่วโมง" ของ HYSPLIT — ใช้ HYSPLIT รันเคสตรวจสอบไขว้ราคาถูก (ฟรี ออนไลน์) เพื่อ validate ทิศทาง attribution ของเราในเหตุการณ์เด่น ๆ — เป็น sanity check ที่กรรมการสายบรรยากาศเชื่อถือ
