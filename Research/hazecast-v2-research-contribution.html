HazeCast v2 — Research Contribution · Physics · Validation

ร่างละเอียด: เปลี่ยนจาก "ระบบที่ดี" ให้เป็น "งานวิจัยที่ยากพอจะได้เหรียญทอง" ด้วยการกระจุกความยากที่จุดเดียวที่ลึกจริง + ฝังสมการฟิสิกส์เฉพาะที่มันถูกต้อง

คำตัดสินเรื่อง "ใช้ฟิสิกส์ดีมั้ย": ดีมาก — เพิ่มความน่าเชื่อถือ, generalize ได้, ประหยัดข้อมูล (สำคัญมากเพราะเรามี GPU เดียว), และที่สำคัญที่สุด ความสามารถ attribution + counterfactual ของเรา "มีอยู่ไม่ได้" ถ้าไม่มีโครงสร้างฟิสิกส์เชิงเส้น — แต่ต้องมีวินัย: อย่าบังคับฟิสิกส์ตรงที่มันไม่จริง (PM2.5 ไม่ใช่สารอนุรักษ์ — มี secondary aerosol + การตกสะสม) บังคับเกิน = ผิดกายภาพ + แม่นยำลดลง

0. แกนคิดเดียว: สมการ Advection-Diffusion-Reaction (ADR)

ฟิสิกส์การเคลื่อนที่ของฝุ่นในอากาศคือสมการนี้ (เป็นสมการที่ WRF-Chem แก้เชิงตัวเลข):

∂C/∂t = −∇·(uC) + ∇·(K∇C) + E − L·C + Rchem C = ความเข้มข้น PM2.5 · u = ลม · K = สัมประสิทธิ์การกระจายปั่นป่วน · E = การปล่อย (ไฟ) · L = อัตราการสูญเสีย (ตกสะสม/ชะล้าง) · Rchem = เคมี (เกิด secondary aerosol)
★ ข้อสังเกตทางคณิตที่เป็นกุญแจของทั้งโปรเจกต์:
4 พจน์แรก (advection, diffusion, emission, linear sink) เป็นเชิงเส้นใน C → ใช้หลัก superposition ได้
C = Σs C⁽ˢ⁾     โดยแต่ละ tracer C⁽ˢ⁾ วิ่งตามสมการตัวขนส่งตัวเดียวกัน แต่ปล่อยจาก E⁽ˢ⁾ ของตัวเองเท่านั้น s = แหล่งกำเนิด (เมียนมา / ลาว / ไทยรายอำเภอ / ป่า / เกษตร)
→ attribution แม่นแบบ exact และ counterfactual ("ดับไฟ s") = C − C⁽ˢ⁾ ตรง ๆ
พจน์เดียวที่ทำลายเชิงเส้นคือ Rchem — จุดที่เราต้องประกาศขอบเขตซื่อสัตย์ (ดูข้อ 4)

1. สถาปัตยกรรม: Operator Splitting (ฟิสิกส์ + เรียนรู้)

เราไม่เอาเน็ตเดาทุกอย่าง และไม่เอาฟิสิกส์ล้วน — เราแยกหนึ่งสเต็ปเวลาออกเป็น 2 ส่วน (เทคนิค Strang/operator splitting จากการแก้ PDE เชิงตัวเลข):

Ct+1 = 𝒜(Ct, ut) ⊕ ℱθ(Ct, Firet, Mett, AODt, Static) 𝒜 = ตัวขนส่ง semi-Lagrangian (ฟิสิกส์ แม่นตรง) · ℱ_θ = ส่วนเรียนรู้ (diffusion ใต้กริด + ปล่อย + sink + แก้ error ที่ฟิสิกส์ง่ายพลาด)

1.1 ส่วนฟิสิกส์ 𝒜 — Semi-Lagrangian Advection

ลากย้อนรอยอนุภาคตามเส้น characteristic แล้ว interpolate ค่าจากตำแหน่งต้นทาง — นี่คือวิธีมาตรฐานใน NWP (ตอบโจทย์ "ลมพัดควันไปไหน" ตรงตามกายภาพ ไม่ปล่อยให้เน็ตเดา)

ต้องซื่อสัตย์: semi-Lagrangian แบบ interpolation มาตรฐาน ไม่อนุรักษ์มวลอย่างเคร่งครัด → ต้องใช้ flux-form semi-Lagrangian (FFSL) หรือใส่ mass-fixer ภายหลัง และต้องเลือก interpolation ที่ monotone (น้ำหนัก ≥ 0) เพื่อรับประกันว่าควันไม่ติดลบและไม่เกิด overshoot

1.2 ส่วนเรียนรู้ ℱ_θ — "Conditionally-Linear Neural Operator" (หัวใจความใหม่)

นี่คือจุดที่ผมออกแบบให้ ความใหม่เชิงวิจัย เกิดขึ้น และเป็นเหตุผลที่ attribution ของเรา exact ได้ — เราออกแบบ ℱ_θ ให้ เป็นเชิงเส้นในสนามความเข้มข้น C แต่สัมประสิทธิ์ขึ้นกับสภาพอากาศแบบไม่เชิงเส้น:

Ct+1⁽ˢ⁾ = 𝒜mono Ct⁽ˢ⁾  +  gθ(Met,Static) · Et⁽ˢ⁾  −  Dθ(Met,Static) ⊙ Ct⁽ˢ⁾ 𝒜mono = ตัวขนส่ง+กระจาย ที่ไม่เป็นลบ · gθ ≥ 0 = ตัวคูณการปล่อยที่เรียนรู้ · 0 ≤ Dθ < 1 = อัตราการสูญเสียที่เรียนรู้ (ตกสะสม/ชะล้าง)
ทำไมการออกแบบนี้ถึง "ฉลาด":
  • เชิงเส้นใน C → ผลรวม tracer เท่าเดิม: Σs C⁽ˢ⁾ = C เป๊ะ (superposition exact) → attribution ไม่มี error เชิงโครงสร้าง
  • ไม่เป็นลบโดยโครงสร้าง (ทุกพจน์ ≥ 0, D_θ < 1) → ไม่ต้องใช้ softplus ปิดท้าย ซึ่งจะทำลายเชิงเส้น (เพราะ softplus(a+b) ≠ softplus(a)+softplus(b)) — non-negativity กลายเป็น "คุณสมบัติของ scheme" ไม่ใช่การ clamp เอาท์พุต
  • ความไม่เชิงเส้น/การเรียนรู้ทั้งหมดอยู่ใน "สัมประสิทธิ์" (g_θ, D_θ, K ใน 𝒜) ที่ขึ้นกับ Met/Static → ยังจับ physics ที่ซับซ้อน (ลมเปลี่ยน, ความชื้นเร่งการตก) ได้ครบ

สัมประสิทธิ์ g_θ, D_θ, K สามารถให้ AFNO/U-Net เป็นตัวทำนายจาก Met/Static ได้ — AFNO ทำงานในโดเมนความถี่ จับ pattern เชิงพื้นที่ระยะไกล (แนวเทือกเขา, ลมมรสุม) ได้ดี

2. Research Contribution — 1 ประโยคที่ต้อง defend ได้

"เราเสนอ physics-constrained, conditionally-linear neural transport emulator สำหรับ PM2.5 ที่ฝัง differentiable source-attribution ไว้ในตัวด้วยหลัก superposition ทำให้ตอบ counterfactual intervention ('ถ้าดับไฟตรงนี้ ฝุ่นเมืองลดเท่าไร') ได้ในการ inference ครั้งเดียวระดับวินาที พร้อม validity bound ที่ calibrate เทียบ WRF-Chem — สิ่งที่แบบจำลองเคมี-ขนส่งเชิงกายภาพต้องใช้คลัสเตอร์รันเป็นวันต่อหนึ่งสถานการณ์"

แตกเป็น 3 contribution ย่อยที่ "ยากและใหม่จริง" แต่ละอันมีของพิสูจน์:

#Contributionใหม่/ยากตรงไหนเทียบงานเดิม
C1 Conditionally-linear neural operator ที่อนุรักษ์มวล (เชิงสัมพัทธ์) + ไม่เป็นลบ โดยโครงสร้าง คุณสมบัติเชิงคณิตพิสูจน์ได้ ไม่ใช่ปล่อยเน็ตเดา; non-negativity จาก scheme ไม่ใช่ clamp PM2.5-GNN/AirFormer = nonlinear ล้วน ไม่มี guarantee; AirPhyNet มี physics แต่ไม่มี attribution

3. แผนการ Validate (ทำให้ "ยากพอ" เป็นจริง ไม่ใช่แค่พูด)

contribution จะ "นับ" ก็ต่อเมื่อมีการทดลองพิสูจน์ ต่อไปนี้คือชุดการทดลองที่ต้องรันจริง:

V1 · Forecast Skill (พิสูจน์ว่าแกนแม่นกว่าของเดิม)

เทียบ baseline (ต้องแข็ง ไม่ใช่ persistence อย่างเดียว): PersistenceClimatologyPCD operational forecastPM2.5-GNNAirPhyNetConvLSTM
Metric: RMSE/MAE · CSI/POD/FAR ที่เกณฑ์ 37.5 & 75 µg/m³ · onset-lead (จับวันเริ่มวิกฤตล่วงหน้ากี่ชม.) · ประเมินแบบ leave-stations-out + spatially-blocked CV (กันรั่ว)

V2 · Mass-Budget Closure (พิสูจน์ C1)

ตรวจว่า งบมวลปิด: Σ E (ปล่อย) − Σ L·C (สูญเสีย) = Δ(มวลสะสม) + flux ออกขอบเขต ภายใน ε เล็ก ๆ ต่อทุก timestep
ตัวเลขเป้า: relative mass residual < ~1–2% (ส่วนเกินคือ Rchem ที่เราไม่จับ — ต้องรายงานตรง ๆ)

V3 · Attribution Validity (พิสูจน์ C2)

  • Superposition error: วัด ‖C − ΣsC⁽ˢ⁾‖ — ถ้าออกแบบ conditionally-linear ถูก ควร ≈ 0 (machine precision); ถ้าเพิ่มพจน์เคมีไม่เชิงเส้น รายงาน error ที่โตขึ้น
  • เทียบสัดส่วนแหล่ง กับ (ก) WRF-Chem tagged-tracer จากวรรณกรรม (ข) PMF receptor modeling ที่สถานีไทย (ค) สัดส่วน hotspot ข้ามแดนจาก FIRMS — ดูว่า "เมียนมา X%" ของเราสอดคล้องหลักฐานอิสระไหม

V4 · Counterfactual Validity (พิสูจน์ C3 — ยากและสำคัญสุด)

  • เทียบ WRF-Chem source-off: หาผลการทดลอง "ปิดแหล่ง s" จากเปเปอร์ → เทียบ ΔC ของเรากับของ WRF-Chem → รายงาน calibration / validity bound
  • Natural experiment (ฉลาดและฟรี): หาเหตุการณ์ที่ ฝนดับไฟจริงในบางพื้นที่ (hotspot หายใน FIRMS) = "source-off ธรรมชาติ" → ดูว่าโมเดลทำนายการลดของฝุ่นปลายทางถูกไหม
  • ประกาศขอบเขต: เสนอเป็น "first-order intervention estimate" มี CI เสมอ ไม่ใช่คำพยากรณ์ศักดิ์สิทธิ์ (เพราะการลบไฟ = ถามนอก distribution บางส่วน)

4. วินัยเรื่องฟิสิกส์ — "ใช้ดีมั้ย" ตอบแบบมีเงื่อนไข

ฟิสิกส์ช่วยตรงไหน (ใส่เต็มที่):
  • Advection/transport — แข็ง ถูกต้อง ใส่เป็น hard constraint ได้เลย
  • Non-negativity + linear deposition — ถูกต้องเชิงกายภาพ ใส่เป็นโครงสร้างได้
  • Generalization + data efficiency — physics prior ช่วยให้เรียนจากข้อมูลน้อย/GPU เดียวได้ (เหตุผลสำคัญที่สุดสำหรับข้อจำกัดเรา)
  • ปลดล็อก attribution/counterfactual — โครงสร้างเชิงเส้นคือสิ่งที่ทำให้ฟีเจอร์เรือธงเป็นไปได้
ฟิสิกส์ทำร้ายเราตรงไหน (ห้ามบังคับ):
  • PM2.5 ไม่ใช่สารอนุรักษ์! — มี secondary organic aerosol (Rchem) + hygroscopic growth + การตกสะสม → บังคับ "อนุรักษ์มวลเคร่งครัด" = ผิดกายภาพ และทำ skill แย่ลง
  • ทางออก: อนุรักษ์มวลของ primary smoke tracer (ควันชีวมวลที่ปล่อยตรง) + ให้ ℱ_θ เรียนรู้ sink/chemistry เป็นพจน์แยก → จับขอบเขตที่ linearity จริง (ควัน primary, ระยะสั้น 24–48 ชม. ที่ transport ครอง) ซึ่งตรงกับ use-case เราพอดี (ควันเผาข้ามแดน = primary PM เป็นหลัก)
  • หลักการ: hard physics เฉพาะที่ถูก, learn ส่วนที่เหลือ, รายงานที่ที่ฟิสิกส์เป็นแค่ approximation
